# 二、Spring 模块化设计

Spring 大概分为了二十多个模块，部分常见模块如下：

  • spring-aop：面向切面编程的模块
  • spring-aspects：对 Aspect 的支持
  • spring-context-indexer
  • spring-context-support
  • spring-context：spring-context、spring-beans 模块就是 Ioc 容器的实现。并且，spring-context、spring-beans 都需要依赖 spring-core 模块。
  • spring-core：包含 Java 语法特性的支持，以及核心处理。
  • spring-expression：spring 表达式的实现，从 spring 3.x 开始支持。
  • spring-instrument：对 Java Agent 的支持。
  • spring-jcl：对 Jakarta Commons Logging (JCL) 的支持，JCL 是一个日志框架，Spring 使用它作为默认的日志框架。spring-jcl 模块允许 Spring 应用程序使用 JCL 进行日志记录，用于统一日志管理，同时也支持其他日志框架，比如 Log4j、Logback 等。从 spring 5.x 开始支持。
  • spring-jdbc：对 JDBC 的支持。
  • spring-jms：对 Java Message Service (JMS，即 Java 消息服务) 的支持。
  • spring-messaging：用于对消息服务的统一管理，统一 JMS、Kafka、RocketMQ、RabbitMQ 等消息服务进行统一实现。
  • spring-orm：对 Hibernate、JPA 的整合。
  • spring-oxm：对“对象-XML 映射”（XML 的序列化与反序列化）的支持
  • spring-test：spring 的测试。
  • spring-tx：对事务的抽象，统一封装事务。
  • spring-web：对 web 服务支持。
  • spring-webflux
  • spring-webmvc
  • spring-websocket

我们可以访问 https://github.com/spring-projects/spring-framework 查看目录的相关模块。

在使用 spring-framework 时，我们不需要将所有的模块全部引入到项目中，可以根据实际情况按需引入。

Spring Framework 的核心模块：

  • spring-core：Spring 基础 API 模块，如资源管理、泛型处理等。
  • spring-beans：Spring Bean 相关，如依赖查找、依赖注入。
  • spring-aop：Spring AOP 处理，如动态代理、AOP 字节码提升。
  • spring-context：事件驱动（ApplicationEvent）、注解驱动（@Component、@ComponentScan）、模块驱动（@EnableCaching）等。
  • spring-expression：Spring 表达式语言模块。

# 五、Spring 编程模型

Spring 提供了以下的编程模型：

1.面向对象编程

  • 契约接口：Aware、ApplicationContextAware、BeanFactoryAware、BeanPostProcessor 等
  • 设计模式：观察者模式（如 ApplicationEvent）、组合模式（如 CompositeCacheManager）、模板模式（如 RestTemplate、JdbcTemplate）等
  • 继承：Abstract*

2.面向切面编程

  • 动态代理：JdkDynamicAopProxy、CglibAopProxy
  • 字节码提升：ASM、Cglib、AspectJ 等

3.面向元编程

  • 注解：模式注解（@Component、@Service、@Repository 等）
  • 配置：Environment 抽象、PropertySources、BeanDefinition 等
  • 泛型：GenericTypeResolver、ResolvableType 等

4.函数驱动

  • 函数式接口：ApplicationEventPublisher、ApplicationListener
  • Reactive、Spring WebFlux

5.模块驱动

  • Maven Artifacts
  • OSGI Bundles
  • Java 9 Automatic Modules
  • @EnableXxxx：@EnableCaching、@EnableAsync、@EnableTransactionManagement、@EnableWebMvc 等

# 六、专属设计模式

前缀模式：

  • Enable 模式
  • Configurable 模式

后缀模式：

  • 处理器模式：Processor、Resoler、Handler
  • 意识模式：Aware
  • 配置器模式：Configuror
  • 选择器模式：org.springframework.context.annotation.ImportSelector
