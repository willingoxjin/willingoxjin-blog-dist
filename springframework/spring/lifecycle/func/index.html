99.Spring常用生命周期函数

# 一、Spring 常用接口

获取接口

  • Aware

    • BeanNameAware 获得到容器中Bean的名称
    • BeanFactoryAware 获得当前bean Factory，从而调用容器的服务
    • ApplicationContextAware 当前的 applicationContext 从而调用容器的服务
    • MessageSourceAware 得到 message source 从而得到文本信息
    • ApplicationEventPublisherAware 应用时间发布器，用于发布事件
    • ResourceLoaderAware 为我们提供了一个统一的getResource()方法来通过资源路径检索外部资源。从而将资源或文件(例如文本文件、XML文件、属性文件或图像文件)加载到Spring应用程序上下文中的不同实现
    • EnvironmentAware 用于获取 application.properties 的配置文件配置的属性值

  • ApplicationRunner / CommandLineRunner

  • 事件发布订阅

    • ApplicationListener 监听ApplicationEvent类型的事件
      • ApplicationReadyEvent 所有的Bean都准备完毕的事件
    • ApplicationEventPublisher 用于推送事件
      • ApplicationEventPublisherAware 用于注入 ApplicationEventPublisher（注入 ApplicationEventPublisher也可以使用属性注入或使用构造方法注入）

# 一、Aware

# 1.ApplicationContextAware

通过注入 ApplicationContext ，使用静态方法获取 Spring 上下文环境

@Component
public class SpringContextUtils implements ApplicationContextAware {

	/**
	 * 上下文对象实例
	 */
	private static ApplicationContext applicationContext;

	@Override
	public void setApplicationContext(ApplicationContext applicationContext) throws BeansException {
		this.applicationContext = applicationContext;
	}

	/**
	 * 获取applicationContext
	 */
	public static ApplicationContext getApplicationContext() {
		return applicationContext;
	}
	
	/**
	 * 通过name获取Bean.
	 */
	public static Object getBean(String name) {
		return getApplicationContext().getBean(name);
	}

	/**
	 * 通过class获取Bean.
	 */
	public static <T> T getBean(Class<T> clazz) {
		return getApplicationContext().getBean(clazz);
	}

	/**
	 * 通过name以及Clazz返回指定的Bean
	 */
	public static <T> T getBean(String name, Class<T> clazz) {
		return getApplicationContext().getBean(name, clazz);
	}
}

示例：扫描加了某个注解的类及方法，然后缓存对象

@Component
public class ProcessBeanScanner implements BeanPostProcessor {

   @Override
   public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
       return bean;
   }

   @Override
   public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
       // 扫描加了注解@Module的类（方法或类中加了此注解）
       
       // 获取bena类型
       Class<?> clazz = bean.getClass();
       // 是否被注解@Modulen修饰
		boolean isPresent = clazz.isAnnotationPresent(Module.class);
       if (isPresent) {
           // 获取类中被@Execute修饰的方法
           Method[] methods = clazz.getMethods();
           if(methods.length > 0) {
               Module module = clazz.getAnnotation(Module.class);
               String moduleValue = module.value();
               if(ObjectUtils.isEmpty(moduleValue) {
                   moduleValue = clazz.getSimpleName();
               }
               for(Method m : methods) {
                   Execute execute = m.getAnnotation(Execute.class);
                   if(execute == null) {
                       continue;
                   }

                   String executeValue = execute.value();
                   if(ObjectUtils.isEmpty(executeValue)) {
						continue;
                   }
                   
                   // 缓存起来（具体业务可忽略）
                   // Invoker.createInvoker 会创建Invoker对象
                   // InvokerTable.addInvoker 会添加到缓存中
                   InvokerTable.addInvoker(moduleValue, executeValue, Invoker.createInvoker(m, bean)); 
               }
           }
       }
       
       return bean;
   }
    
}

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Execute {
	String value();
}

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Module {
	String value();
}

# 3.CommandLineRunner / ApplicationRunner

@FunctionalInterface
public interface ApplicationRunner {

	void run(ApplicationArguments args) throws Exception;

}

@FunctionalInterface
public interface CommandLineRunner {

	void run(String... args) throws Exception;

}

SpringBoot 项目启动时，若想在启动之后直接执行某一段代码，就可以用 ApplicationRunner 这个接口，并实现 run() 方法。可用于读取配置文件，数据库连接。

其中，CommandLineRunner 和 ApplicationRunner 的作用是相同的，他们只有其 run 方法的参数不同：

  • CommandLineRunner 接口的 run() 的参数宇 main 方法的参数是一样的。
  • ApplicationRunner 接口的 run() 方法接收ApplicationArguments对象作为参数，是对原始参数做了进一步的封装。

# 4.@PostConstruct

如果想在生成对象时候完成某些初始化操作，而偏偏这些初始化操作又依赖于依赖注入，那么就无法在构造函数中实现，就可以使用 @PostConstruct 注解一个方法来完成初始化。

@PostConstruct 注解的方法将会在依赖注入完成后被自动调用。

优先级： Constructor > @Autowired > @PostConstruct

执行的顺序：@PostConstruct > InitializingBean > ApplicationRunner > CommandLineRunner

# 三、销毁相关

# 1.DisposableBean

DisposableBean 接口定义了一个方法 destroy() ，该方法在 Bean 被销毁时调用。这个接口可以被用来执行一些清理工作，例如释放资源或关闭连接。当一个 Spring 容器被关闭时，所有实现了 DisposableBean 接口的 Bean 都会被销毁，这样可以确保应用程序的资源被正确地释放

# 2.@PreDestroy

我们也可以使用 @PreDestroy 注解来代替 DisposableBean 接口，因为它更简单，更易于使用。

执行顺序（排在前面的先执行）：DisposableBean > @PreDestroy

# 五、指定类加载顺序、执行顺序

# 1.依赖于某个bean

使用 @DependsOn 来和 @Component 或 @Bean 一起使用，表示当前初始化的Bean依赖于某个bean @DependsOn 中的 value 属性接收一个字符型数组，该数组为所依赖的bean的名称列表

如：以下表示在 springContextUtils 对象初始化完成之后，spring 才会对 FastDFSUtil 进行实例化

	@Bean
	@DependsOn("springContextUtils")
	FastDFSUtil fastDFSUtil() {
		return new FastDFSUtil();
	}
