04.Spring Beans

# 一、BeanDefinition

# 1.BeanDefinition

BeanDefinition 是 Spring Framework 中定义 Bean 的配置元信息接口，该接口允许我们设置和获取相关的元信息，包含：

  1. Bean 的类型（类名）。
  2. Bean 的配置元信息。如作用域、自动绑定的模式、生命周期回调等。
  3. 对其他的 Bean 引用。又称为合作者（Collaborators）或者依赖（Dependencies）
  4. 配置设置。如 Bean 属性（Properties）
  5. ……

BeanDefinition 的元信息如下：

属性 说明
Class Bean 全类名，必须是具体类，不能用抽象类或接口
Name Bean 的名称，必须唯一
Scope Bean 的作用域（如 singleton，prototype）
Constructor arguments（构造函数参数） Bean 的构造器参数（用于依赖注入）
Properties Bean 属性设置（用于依赖注入）
Autowiring mode（自动装配模式） Bean 自动绑定模式（如通过名称 byName）
Lazy initialization mode（延迟初始化模式） Bean 延迟初始化模式（延迟和非延迟）
Initialization method Bean 初始化回调方法名称
Destruction method Bean 销毁方法名称

# 2.构建 BeanDefinition

构建一个 BeanDefinition 实例有以下方式：

  1. 通过 BeanDefinitionBuilder
  2. 通过 AbstractBeanDefinition 以及派生类

下面我们演示如何构建 BeanDefinition：

public class BeanDefinitionCreatIionDemo {

    public static void main(String[] args) {
        // 1.通过 BeanDefinitionBuilder 构建
        BeanDefinition beanDefinition = BeanDefinitionBuilder.genericBeanDefinition(User.class)
                .addPropertyValue("id", 3L)
                .addPropertyValue("username", "wangwu")
                .getBeanDefinition();
        // BeanDefinition 并非 Bean 的最终状态，还可以进行自定义修改，如下
        beanDefinition.setBeanClassName("org.willingxojin.ioc.spring.overrive.domain.User");

        // 2.通过 AbstractBeanDefinition 以及派生类构建
        GenericBeanDefinition genericBeanDefinition = new GenericBeanDefinition();
        genericBeanDefinition.setBeanClass(User.class);
        MutablePropertyValues propertyValues = new MutablePropertyValues();
        propertyValues.addPropertyValue("id", 4L);
        propertyValues.addPropertyValue("username", "zhaosi");
        genericBeanDefinition.setPropertyValues(propertyValues);
    }

}

# 二、Bean 的命名

# 1.命名 Bean

我们在上面的 BeanDefinition 元信息中提到 Name 属性是 Bean 的名称，必须唯一。

在 Spring Bean 中，每个 Bean 都有一个或多个标识符（identifiers），这些标识符在 Bean 所在的容器必须是唯一的。通常，一个 Bean 仅有一个标识符，如果需要额外的标识符，可以考虑使用别名（Alias）扩展。

在基于 XML 文件的配置元信息中，可以使用 id 或者 name 属性来规定 Bean 的标识符。通常 Bean 的标识符由字母组成，允许出现特殊字符。并且，可以在 name 属性中使用半角逗号（,）或分号（;）间隔多个标识符（即配置别名）。

Bean 的 id 或 name 属性缺省时，容器会自动为 Bean 生成一个唯一的名称。Bean 的命名虽然没有限制，但是 Spring 官方建议使用驼峰式命名，这样更符合 Java 的命名约定。

Bean 名称生成器（BeanNameGenerator）

有 Spring Framework 2.0.3 引入 BeanNameGenerator，框架内建如下实现：

  • DefaultBeanNameGenerator：默认通用 BeanNameGenerator 实现。

  • AnnotationBeanNameGenerator：基于注解（如 @Component）扫描的 BeanNameGenerator 实现，起始于 Spring Framework 2.5。如果默认没有使用注解，则使用 BeanClassName 的短名（首字母小写）作为 Bean 名称。

  • FullyQualifiedAnnotationBeanNameGenerator：基于注解（如 @Component）扫描的 BeanNameGenerator 实现，起始于 Spring Framework 5.2.3，继承自 AnnotationBeanNameGenerator，如果默认没有使用注解，则使用 BeanClassName 作为 Bean 名称。

# 三、注册 Bean

注册 Spring Bean 有如下方式：

  1. 通过 XML 配置元信息：<bean name="..." ...>
  2. 通过 Java 注解配置元信息
    • @Bean
    • @Component
    • @Import
  3. 通过 Java API 配置元信息
    • 命名方式：BeanDefinitionRegistry#registerBeanDefinition(String,BeanDefinition)
    • 非命名方式：BeanDefinitionReaderUtils#registerWithGeneratedName(AbstractBeanDefinition,BeanDefinitionRegistry)
    • 配置类方式：AnnotatedBeanDefinitionReader#register(Class<?> ...)

# 1.通过 XML 配置元信息

在之前的使用中，我们都是使用 xml 文件的方式配置元信息，这里我们就不在示例了。

# 3.通过 Java API 配置元信息

如下是通过 Java API 配置元信息的示例：

public class JavaApiBeanDefinitionDemo {

    public static void main(String[] args) {
        AnnotationConfigApplicationContext applicationContext = new AnnotationConfigApplicationContext();
        // 通过 BeanDefinition 注册 API 实现
        // 1. 命名 Bean 的注册方式
        registerUserBeanDefinition(applicationContext, "willingxojin-user", User.class);
        // 2. 非命名 Bean 的注册方式
        registerUserBeanDefinition(applicationContext, User.class);
        // 启动 Spring 应用上下文
        applicationContext.refresh();
        // 按照类型依赖查找
        Map<String, User> userBeans = applicationContext.getBeansOfType(User.class);
        System.out.println(userBeans);
        // 显式地关闭 Spring 应用上下文
        applicationContext.close();
    }

    /**
     * 命名 Bean 的注册方式
     */
    private static void registerUserBeanDefinition(BeanDefinitionRegistry registry, @Nullable String beanName, Class<?> beanClass) {
        BeanDefinitionBuilder beanDefinitionBuilder = BeanDefinitionBuilder.genericBeanDefinition(beanClass);
        beanDefinitionBuilder
                .addPropertyValue("id", 1)
                .addPropertyValue("username", "willingxojin");
        BeanDefinition beanDefinition = beanDefinitionBuilder.getBeanDefinition();
        if (StringUtils.hasText(beanName)) {
            // 命名 Bean 的注册方式
            registry.registerBeanDefinition(beanName, beanDefinition);
        } else {
            // 非命名 Bean 的注册方式
            beanName = BeanDefinitionReaderUtils.generateBeanName(beanDefinition, registry);
            registry.registerBeanDefinition(beanName, beanDefinition);
        }
    }

    /**
     * 非命名 Bean 的注册方式
     */
    private static void registerUserBeanDefinition(BeanDefinitionRegistry registry, Class<?> beanClass) {
        registerUserBeanDefinition(registry, null, beanClass);
    }

}

# 4.将外部单体对象注册 Bean

通过外部单体对象注册 Spring Bean 代码如下：

public class SingletonBeanRegistrationDemo {

    public static void main(String[] args) {
        // 创建 BeanFactory 容器
        AnnotationConfigApplicationContext applicationContext = new AnnotationConfigApplicationContext();
        // 注册外部单例对象
        String beanName = "userFactory";
        UserFactory userFactory = new DefaultUserFactory();
        ConfigurableListableBeanFactory beanFactory = applicationContext.getBeanFactory();
        beanFactory.registerSingleton(beanName, userFactory);
        // 启动 Spring 应用上下文
        applicationContext.refresh();
        // 通过依赖查找获取 UserFactory
        UserFactory userFactoryByLookup = applicationContext.getBean(beanName, UserFactory.class);
        System.out.println("userFactory == userFactoryByLookup: " + (userFactory == userFactoryByLookup));
        // 关闭 Spring 应用上下文
        applicationContext.close();
    }

}

# 四、实例化 Bean

实例化 Spring Bean 可分为常规方式和特殊方式，具体如下：

  • 常规方式：
    1. 通过构造器（配置元信息：XML、Java 注解和 Java API）
    2. 通过静态工厂方法（配置元信息：XML 和 Java API）
    3. 通过 Bean 工厂方法（配置元信息：XML 和 Java API）
    4. 通过 FactoryBean （配置元信息：XML、Java 注解和 Java API）
  • 特殊方式：
    1. 通过 ServiceLoaderFactoryBean（配置元信息：XML、Java 注解和 Java API）
    2. 通过 AutowireCapableBeanFactory#createBean(Class<?> beanClass, int autowireMode, boolean dependencyCheck)
    3. 通过 BeanDefinitionRegistry#registerBeanDefinition(String beanName, BeanDefinition beanDefinition)

# 2.通过 ServiceLoaderFactoryBean 实例化

首先，我们来了解一下什么是 ServiceLoader。

ServiceLoader 是 Java 的内置机制，用于在运行时动态加载和实例化服务提供者。ServiceLoader 通常用于实现服务提供者接口（Service Provider Interface，SPI）的机制。通过定义一个接口和相关的文件（通常是 META-INF/services/ 接口全限定名），服务提供者可以实现该接口并将其实现类的全限定名写入该文件。然后，在运行时可以使用 ServiceLoader 来加载并获取这些实现类的实例。

下面使用 ServiceLoader，示例如下：

新建resources/META-INF/services/org.willingxojin.spring.beans.factory.UserFactory 文件如下：

org.willingxojin.spring.beans.factory.DefaultUserFactory
1

如下代码将使用 ServiceLoader 创建 UserFactory：

public class SpecialBeanInstantiationDemo {

    public static void main(String[] args) throws Exception {
        serviceLoaderDemo();
    }

	private static void serviceLoaderDemo() {
        // Thread.currentThread().getContextClassLoader() 为当前线程的上下文类加载器
        ServiceLoader<UserFactory> serviceLoader = ServiceLoader.load(UserFactory.class,
                Thread.currentThread().getContextClassLoader());
        Iterator<UserFactory> iterator = serviceLoader.iterator();
        while (iterator.hasNext()) {
            UserFactory userFactory = iterator.next();
            System.out.println(userFactory.createUser());
        }
    }

}

# 五、初始化 Bean

Spring Bean 的初始化时，暴露给用户的可配置方式如下：

  1. 使用 @PostConstruct 注解修饰方法
  2. 实现 InitializingBean 接口的 afterPropertiesSet() 方法
  3. 自定义初始化方法：
    • XML 配置：<bean init-method="initXxxx" ... >
    • Java 注解配置：@Bean(initMethod="initXxxx")
    • Java API 配置：使用 AbstractBeanDefinition#setInitMethodName(String initMethodName) 方法（实际上，自定义初始化方法中的 XML 配置和 Java 注解配置最终都是通过 AbstractBeanDefinition#setInitMethodName 方法实现的）

[!IMPORTANT]

假设某个 Bean 中，同时定义了上面的三种方式，他们的执行顺序为：@PostConstruc > InitializingBean > 自定义初始化方法。

优先级： Constructor > @Autowired > @PostConstruct

  • @PostConstruct 注解的方法将会在依赖注入完成后被调用。
  • InitializingBean 的 afterPropertiesSet 方法将在属性被设置之后调用。
  • 自定义初始化方法会在

首先，我们在 DefaultUserFactory 中定义 init、initUserFactory、afterPropertiesSet 方法，如下：

public interface UserFactory {

    default User createUser() {
        return User.createUser();
    }

}

public class DefaultUserFactory implements UserFactory , InitializingBean {

    @PostConstruct
    public void init() {
        System.out.println("@PostConstruct : UserFactory 初始化中...");
    }

    @Override
    public User createUser() {
        return UserFactory.super.createUser();
    }

    public void initUserFactory() {
        System.out.println("自定义初始化方法：UserFactory 初始化中...");
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("InitializingBean#afterPropertiesSet : UserFactory 初始化中...");
    }
}

# 六、延迟初始化 Bean

Bean 延迟初始化（Lazy Initialization）的配置方式如下：

  • XML 方式：<bean lazy-init="true" ... />
  • Java 注解：@Lazy(value = true)（@Lazy 注解的默认属性是 true）

下面我们使用 Java 注解的方式演示延迟加载。

我们在 ApplicationContext.refresh() 方法执行后打印一句：”Spring 应用上下文已启动..“。代码如下

@Configuration // Configuration Class
public class LazyBeanDemo {

    public static void main(String[] args) {
        // 创建 BeanFactory 容器
        AnnotationConfigApplicationContext applicationContext = new AnnotationConfigApplicationContext();
        // 注册 Configuration Class
        applicationContext.register(LazyBeanDemo.class);
        // 启动 Spring 应用上下文
        applicationContext.refresh();
        System.out.println("Spring 应用上下文已启动...");

        // 非延迟加载在 Spring 应用上下文启动完成后，被初始化
        // 进行依赖查找
        UserFactory userFactory = applicationContext.getBean(UserFactory.class);
        System.out.println(userFactory);

        // 关闭 Spring 应用上下文
        applicationContext.close();
    }

    @Lazy
    @Bean(initMethod = "initUserFactory")
    public UserFactory userFactory() {
        return new DefaultUserFactory();
    }

}

# 七、销毁 Bean

Spring Bean 对象销毁（Destroy）的回调方式如下：

  1. 使用 @PreDestroy 注解修饰方法
  2. 实现 DisposableBean 接口的 destroy() 方法
  3. 自定义销毁方法
    • XML 配置：<bean destroy="destroyXxxx" ... />
    • Java 注解配置：@Bean(destroy="destroyXxxx")
    • Java API 配置：使用 AbstractBeanDefinition#setDestroyMethodNames(String... destroyMethodNames) 方法（实际上，自定义初始化方法中的 XML 配置和 Java 注解配置最终都是通过 AbstractBeanDefinition#setDestroyMethodNames 方法实现的）

可以看出，Spring Bean 对象销毁和初始化时对应的。

public interface UserFactory {

    default User createUser() {
        return User.createUser();
    }

}

public class DefaultUserFactory implements UserFactory, InitializingBean, DisposableBean {

    @Override
    public User createUser() {
        return UserFactory.super.createUser();
    }

    @PreDestroy
    public void preDestroy() {
        System.out.println("@PreDestroy : UserFactory 销毁中...");
    }

    public void doDestroy() {
        System.out.println("自定义销毁方法：UserFactory 销毁中...");
    }

    @Override
    public void destroy() throws Exception {
        System.out.println("DisposableBean#destroy : UserFactory 销毁中...");
    }

}

# 八、回收 Bean

Spring Bean 的垃圾回收（GC）：

  1. 关闭 Spring 容器（调用 Spring 应用上下文的 close() 方法）
  2. 执行 GC
  3. Spring Bean 覆盖的 finalize() 方法被回调

实际上，我们不需要手动显式的进行垃圾回收，JVM 会自动回收销毁的 Bean。
