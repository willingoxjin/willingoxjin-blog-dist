Spring Boot Actuator

Spring Boot Actuator 官方文档：

  • https://spring.io/guides/gs/actuator-service/ (opens new window)
  • https://docs.spring.io/spring-boot/docs/2.7.5/reference/htmlsingle/#actuator.enabling (opens new window)

# 一、介绍

Spring Boot Actuator 用来对 Spring Boot 应用进行监控、指标采集、管理，提供一些 endpoint （端点）。

Spring Boot Actuator 提供的 endpoint （端点）能够监控应用程序并与之交互。Spring Boot 有许多内置 endpoint，并允许自定义 endpoint 。例如，health 端点提供基本应用程序运行状况信息。

# 二、快速使用

新建一个 maven 项目，在 pom.xml 引入依赖：

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-actuator</artifactId>
    </dependency>

# 三、Endpoints

出于安全考虑，默认情况下只有 /health 端点通过 HTTP 公开。

以下是 Spring Boot Actuator 内置的 Endpoints：

ID HTTP方法 描述
auditevents GET /actuator/auditevent 显示当前应用程序的审计事件信息，需要搭配 Spring Security 使用
beans GET /actuator/beans 显示应用 Spring Bean 的完整列表
caches GET /actuator/caches 显示可用缓存信息
conditions GET /actuator/conditions 显示自动装配类的状态及及应用信息
configprops GET /actuator/configprops 显示所有 @ConfigurationProperties 信息列表
env GET /actuator/env 显示 ConfigurableEnvironment 对象的属性
flyway GET /actuator/flyway 显示 Flyway 数据库信息
health GET /actuator/health 显示应用的健康信息（未认证只显示status，认证显示全部信息详情）
httptrace GET /actuator/httptrace 显示 HTTP 跟踪信息（默认显示最后100个HTTP请求 - 响应交换）
info GET /actuator/info 显示当前应用的信息
integrationgraph GET /actuator/integrationgraph 显示 Spring 集成图。需要依赖 spring-integration-core
loggers GET /actuator/loggers 显示和修改应用程序中loggers的配置
liquibase GET /actuator/liquibase 显示 Liquibase 数据库信息
metrics GET /actuator/metrics 展示当前应用的 metric 信息
mappings GET /actuator/mappings 显示所有 @RequestMapping 路径列表
quartz GET /actuator/quartz 查看定时任务（quartz）的的信息
scheduledtasks GET /actuator/scheduledtasks 查看定时任务（scheduled）的的信息
sessions GET /actuator/sessions 查看从 Spring 会话支持的会话存储中检索和删除用户会话
shutdown POST /actuator/shutdown 优雅地关闭 Spring Boot 程序
startup GET /actuator/startup 显示由 ApplicationStartup 收集的启动步骤数据。要求 SpringApplication 配置一个 BufferingApplicationStartup。
threaddump GET /actuator/threaddump 执行线程转储。

如果应用使用 web 服务（Spring MVC, Spring WebFlux, or Jersey），还包括如下 Endpoints：

ID HTTP方法 描述
heapdump GET /actuator/heapdump 返回堆转储文件。在 HotSpot JVM 上，返回一个 hprof 格式的文件。在 OpenJ9 JVM 上，返回一个 phd 格式的文件。
jolokia GET /actuator/jolokia 当 Jolokia 位于类路径上时(对 WebFlux 不可用)，通过 HTTP 公开 JMX bean。需要依赖 jolokia-core。
logfile GET /actuator/logfile 返回日志文件的内容(如果已经设置了 logging.file.name 或 logging.file.path 属性)。支持使用 HTTP Range 头来检索部分日志文件的内容。
prometheus GET /actuator/prometheus Prometheus 服务器抓取的格式显示 metrics 信息

可以为 actuator 进行一些配置，暴露 Endpoint，配置 info 信息等。如下：

# actuator的端口
management.server.port=8081
# actuator的ip（仅允许127.0.0.1访问）
management.server.address=127.0.0.1

# actuator的访问根路径
management.endpoints.web.base-path=/actuator
# actuator暴露的endpoints,多个使用逗号隔开,全部使用*
management.endpoints.web.exposure.include=*
# 禁止某些endpoints暴露出来
#management.endpoints.web.exposure.exclude=

# 显示 health 的详细信息
management.endpoint.health.show-details=always

# 启用info的变量
management.info.env.enabled=true
info.app-name=actuator-test
info.owner=willingoxjin
info.email=abc.@willingoxjin.cn

# 1.DataSourceHealthIndicator 源码

下面以 DataSourceHealthIndicator 为例，看看其源码是如何实现的：

源码基于 spring-boot-starter-actuator 2.7.5

doHealthCheck 方法是其核心方法，

builder.up().withDetail() 用来构建健康状态的详情信息

源码如下：

@Override
protected void doHealthCheck(Health.Builder builder) throws Exception {
    if (this.dataSource == null) {
        // 没有数据源的情况
        builder.up().withDetail("database", "unknown");
    }
    else {
        // 存在数据源的情况
        doDataSourceHealthCheck(builder);
    }
}

private void doDataSourceHealthCheck(Health.Builder builder) throws Exception {
    // getProduct() 方法用于获取当前所连接数据库的类型（mysql、oracle等）
    builder.up().withDetail("database", getProduct());
    // 验证查询
    String validationQuery = this.query;
    // 是否定义数据库的检查sql
    if (StringUtils.hasText(validationQuery)) {
        // validationQuery-验证查询语句
        builder.withDetail("validationQuery", validationQuery);
        // Avoid calling getObject as it breaks MySQL on Java 7 and later
        List<Object> results = this.jdbcTemplate.query(validationQuery, new SingleColumnRowMapper());
        // result-验证查询语句执行的结果
        Object result = DataAccessUtils.requiredSingleResult(results);
        builder.withDetail("result", result);
    }
    else {
        builder.withDetail("validationQuery", "isValid()");
        boolean valid = isConnectionValid();
        builder.status((valid) ? Status.UP : Status.DOWN);
    }
}

private String getProduct() {
    return this.jdbcTemplate.execute((ConnectionCallback<String>) this::getProduct);
}

如果上述环节抛出异常，就会构建健康状态为 down 的状态，该处理是在父类 AbstractHealthIndicator 的 health() 方法中进行的 。

源码如下：

public abstract class AbstractHealthIndicator implements HealthIndicator {
	@Override
	public final Health health() {
		Health.Builder builder = new Health.Builder();
		try {
			doHealthCheck(builder);
		}
		catch (Exception ex) {
			builder.down(ex);
		}
		logExceptionIfPresent(builder.getException());
		return builder.build();
	}
    
    // ...
    
    protected abstract void doHealthCheck(Health.Builder builder) throws Exception;
}

# 2.自定义健康检查

如果我们想要使用 Spring Boot Actuator 对使用的某个组件（如文件存储、自研内存组件）进行健康检查，而 Actuator 中没有对此组件的内置支持，我们就可以自定义健康检查。

自定义健康检查，只需要继承 AbstractHealthIndicator 类，并重写 doHealthCheck() 方法。

@Component
public class CustomHealthIndicator extends AbstractHealthIndicator implements InitializingBean {

    // ...
    
    @Override
	protected void doHealthCheck(Health.Builder builder) throws Exception {
		builder.up().withDetail("status", "up");
       builder.up().withDetail("message", "hello");
	}

}

# 四、自定义 Endpoint

# 1.相关概念

Spring Boot 2.x 开始，Actuator 支持 CRUD 模型，而不是旧的 RW（读/写）模型。

# Endpoint 相关注解

自定义一个监控端点主要有如下常用注解：

  • @Endpoint 定义一个端点，同时支持 HTTP 和 JMX 两种方式。
  • @JmxEndpoint 定义一个端点，只支持 JMX 技术
  • @WebEndpoint 定义一个端点，只支持 http

扩展相关

  • @EndpointWebExtension 扩展 WebEndpoint
  • @EndpointJmxExtension扩展 JmxEndpoint

web 服务相关

  • @ServletEndpoint 适用于 Servlet 的 Endpoint
  • @RestControllerEndpoint Endpoint 不支持复杂的参数，如果想要传递对象或者数组等复杂参数，可以使用 @RestControllerEndpoint，使用 @RestControllerEndpoint 的类中的方法对应的路由路径，直接使用 @RequestMapping 就可以，就像写 web 接口一样。
  • @ControllerEndpoint 同 @RestControllerEndpoint

参数相关

  • @Selector 作用在参数上，用来定位一个端点的具体指标路由。
  • @Nullable 作用在参数上，表示该参数可以为空
  • @EndpointConverter 输入参数的类型转换

# HTTP 方法

通过在一个端点类（必须是Spring Bean）上添加上面其中一个来表明该类是一个端点类。 在类的方法使用 @ReadOperation，@WriteOperation 或 @DeleteOperation。这分别会映射到Http 中的方法，如下：

Operation HTTP method
@ReadOperation GET
@WriteOperation POST
@DeleteOperation DELETE

带有 @WriteOperation 注解的方法可以用来修改指标，方法需要用 post 进行访问，访问的参数可以直接使用字符串传参，也可以直接使用 json 进行传参，修改之后再次查看就可以发现指标已经被动态修改：

# 参数

  • @Selector 作用在参数上，用来定位一个端点的具体指标路由。
  • @Nullable 作用在参数上，表示参数可以为空

# 2.自定义 Endpoint 示例

@Component
@Endpoint(id = "custom")
public class CustomEndpoint {

    private static final Map<String, String> infos = new ConcurrentHashMap<>();

    static {
        infos.put("a", "abc");
        infos.put("b", "def");
        infos.put("c", "ggg");
        infos.put("d", "123");
    }

    @ReadOperation
    public Map<String, String> infos() {
        return infos;
    }

    // 使用 @Selector，请求参数则使用 /actuator/xxx/{name}
    @ReadOperation
    public String info(@Selector String name) {
        return infos.get(name);
    }

    @WriteOperation
    public void writeInfo(@Selector String name, @Nullable String value) {
        if (value == null) {
            value = name;
        }
        infos.put(name, value);
    }

    // 不使用 @Selector，请求参数则使用 /actuator/xxx?name={name}
    @DeleteOperation
    public void deleteInfo(String name) {
        infos.remove(name);
    }

}

# 4.使用 JMX 方式调用

JMX（Java Management Extensions，即 Java 管理扩展）是一个为应用程序植入管理功能的框架。

JMX 最常用到的就是对于 JVM 的监测和管理，比如 JVM 内存、CPU 使用率、线程数、垃圾收集情况等等。另外，还可以用作日志级别的动态修改，比如 log4j 就支持 JMX 方式动态修改线上服务的日志级别。其最主要的还是被用来做各种监控工具。
