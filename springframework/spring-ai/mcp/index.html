05.Spring AI 集成 MCP

# 一、MCP 介绍

# 1.MCP 是什么？

MCP中文文档：https://mcpcn.com/docs/

Model Context Protocol (MCP) 是一个标准化协议，它为大模型与外部数据源、工具间的交互提供了一套统一的、标准化的方式，被称为 AI 系统的“USB-C接口”。

MCP 通过标准化协议层实现和外部工具的交互：

MCP 遵循客户端-服务器架构：

MCP 定义了三方通信角色：

  • MCP Host（主机 / AI 应用）：发起连接的 LLM/AI 主体，负责用户交互、权限管控、上下文管理。如 Claude Desktop、Cursor、GitHub Copilot。
  • MCP Client（客户端 / 协议连接器）：嵌入 Host 内部的协议处理层，责建立与 Server 的连接、通信。
  • MCP Server（服务端 / 能力提供方）：对外暴露标准化能力的服务。Server 可以是本地进程（文件系统、SQLite）、远程服务（数据库、API、云服务）

# 2.MCP 通信机制

MCP 的通信建立在一个层次分明的模型之上：

  • 协议层 (Protocol Layer)：定义了“讲什么”，即通信的语言和规则，包括消息格式、生命周期管理等。
  • 传输层 (Transport Layer)：定义了“怎么讲”，传输层定义了消息是如何通过网络或进程发送的。
  • JSON-RPC 2.0 (消息格式)：协议层与传输层之间的“通用语言”。MCP中的所有通信都必须遵循 JSON-RPC 2.0 规范

MCP 提供了两种官方标准的传输方式：

  1. 本地模式 （stdio：标准输入/输出）
    • 用于本地进程间通信。客户端将 MCP 服务器作为子进程启动，通过标准输入（stdin）发送消息，从标准输出（stdout）读取消息。
    • 适用于文件、操作 IDE、打开命令行工具等需要在本地主机操作的场景。
  2. 远程模式（SSE/streamableHttp）
    • MCP 服务器是一个独立进程，与客户端独立，可处理多个客户端的远程连接，支持分布式部署。
    • 适用于需要远程通讯的服务，如实时搜索、天气查询、地图、定位查询等。

# 2.远程 MCP 工具（SSE）

远程 MCP 工具的配置和本地的一样，除了参数不同而已。

例如，我们增加一个高德地图的 MCP 远程服务（参考文档：https://lbs.amap.com/api/mcp-server/application-case/tourism-planning）。

我们修改配置，增加 amap-amap-sse 的高德 MCP 服务配置，如下：

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/willingoxjin/Codes/cursor/test"
      ]
    },
    "amap-amap-sse": {
      "url": "https://mcp.amap.com/sse?key=您在高德官网上申请的key",
    }
  }
}

# 三、Spring AI 集成第三方 MCP 服务

# 1.添加依赖及配置

在 pom.xml 中增加依赖：

<dependency>
  <groupId>org.springframework.ai</groupId>
  <artifactId>spring-ai-starter-mcp-client</artifactId>
</dependency>

在 application.yml 中增加 mcp 的配置：

spring:
  ai:
	# mcp
    mcp:
      client:
        enabled: true
        name: spring-ai-study-demo-mcp-client
        type: async
        stdio:
          servers-configuration: classpath:mcp-server.json
        sse:
          connections:
            amap-sse-server:
              url: https://mcp.amap.com
              sse-endpoint: /sse?key=${AMAP_KEY}  ## $AMAP_KEY 替换为你在高德开发平台应用申请的key

# 2.大模型集成 MCP 服务

在 ChatClient 的 Bean 配置中，添加 ToolCallback 相关配置，内容如下：

@Configuration
public class SpringAIChatClientConfig {

    @Bean
    public ChatClient defaultChatClient(ChatClient.Builder chatClientBuilder, ToolCallbackProvider toolCallbackProvider) {
        Builder builder = chatClientBuilder
            .defaultSystem(PromptConst.DEFAULT_SYSTEM_PROMPT)
            // 增加 Tool 配置 
            .defaultToolCallbacks(toolCallbackProvider);
        return builder.build();
    }

}
