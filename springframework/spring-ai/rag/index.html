03.Spring AI 集成 RAG 知识库

# 一、RAG 如何实现

Retrieval-Augmented Generation（检索增强生成），是让大语言模型（LLM）在生成回答前，先从外部知识库检索相关事实，再结合检索结果生成答案的混合架构。

我们这里就来实现一下，步骤如下：

  1. 使用 Spring AI 解析文档内容。
  2. 将文档内容存储到向量库 redis-stack 中。
  3. 在用户提问问题时，先查询向量库匹配的数据。
  4. 将向量库匹配的数据和用户问题，一起发给大模型，并且要求大模型只输出向量库匹配到的数据。

# 二、实现 RAG 知识库

# 1.安装 redis-stack

Redis Stack 是 Redis 官方推出的增强版发行包，整合了 Redis 核心、官方扩展模块与可视化工具，让 Redis 从单纯缓存升级为一站式实时数据平台。

Redis Stack 分为两个版本：

  • redis-stack：包含 Redis Stack Server 和 Redis Insight（Redis 的 Web 可视化界面），该版本最适合本地开发。

  • redis-stack-server：仅提供 Redis Stack server，该版本最适合生产部署。

  • Github：https://github.com/redis-stack/redis-stack/tree/master

  • Docker：https://hub.docker.com/r/redis/redis-stack/tags

由于我本机有 Redis 连接的客户端，所以我安装 server 版本，Docker 安装命令：

docker run -d \
--name redis-stack \
-p 9379:6379 \
-v /opt/docker/redis-stack/data:/data \
-v /opt/docker/redis-stack/conf/redis-stack.conf:/etc/redis-stack.conf \
--restart unless-stopped \
redis/redis-stack-server:7.4.0-v8

@Slf4j
@Service
public class DocumentServiceImpl implements DocumentService {

    @jakarta.annotation.Resource
    private RedisVectorStore redisVectorStore;

    @Override
    public List<Document> loadDocText(Resource resource, Map<String, Object> metadata) {
        // 读取文件内容
        TextReader reader = new TextReader(resource);
        Map<String, Object> customMetadata = reader.getCustomMetadata();
        if (MapUtils.isNotEmpty(metadata)) {
            customMetadata.putAll(metadata);
        }
        List<Document> documents = reader.get();

        // 切分文档内容
        TextSplitter textSplitter = new CustomTokenTextSplitter();
        List<Document> splitDocs = textSplitter.apply(documents);

        if (log.isDebugEnabled()) {
            for (Document splitDoc : splitDocs) {
                log.debug("splitDoc: {} \n", splitDoc);
            }
        }

        return splitDocs;

    }

    @Override
    public void saveDocument(List<Document> documents) {
        // 添加到向量存储中
        redisVectorStore.add(documents);
    }


    static class CustomTokenTextSplitter extends TextSplitter {

        // 使用 markdown 标题切分
        private static final Pattern CHAPTER_PATTERN = Pattern.compile("(?=^(#|##) )", Pattern.MULTILINE);

        @Override
        protected List<String> splitText(String text) {
            return List.of(CHAPTER_PATTERN.split(text.trim()));
        }
    }

}

# 5.对接大模型

定义的 RAG 的提示词：

public class PromptConst {

    public static final String RAG_PROMPT = """
            基于上下文的知识库内容回答问题：
            【上下文】
            {context}
            
            【问题】
            {question}
            
            【输出】
            如果没有查到，请返回“您正在使用知识库查询，知识库中暂无此问题答案。”。
            如果查到，请返回具体的内容，不相关的近似内容不必输出。
            """;

    public static String buildRagPrompt(String context, String question) {
        return RAG_PROMPT.replace("{context}", context).replace("{question}", question);
    }

}

@Slf4j
@Service
public class ChatServiceImpl implements ChatService {  

    @Resource
    private ChatClient chatClient;

	@Override
    public Flux<String> chatStreamResponse(String promptText, List<Document> ragDocContext) {
        Prompt prompt;
        if (CollectionUtils.isNotEmpty(ragDocContext)) {
            // RAG Prompt
            String ragContext = ragDocContext.stream().map(Document::getText).collect(Collectors.joining("\n"));
            String docPromptText = PromptConst.buildRagPrompt(ragContext, promptText);
            prompt = new Prompt(docPromptText);
        } else {
            // Default Prompt
            prompt = new Prompt(promptText);
        }

        return chatClient.prompt(prompt).stream().content();
    }
    
}

增加 ChatService#chatStreamResponse 方法，若向量库中匹配到数据，则让 AI 自己输出向量库返回的，和用户问题最匹配的数据。

@RestController
@RequestMapping("/chat")
public class ChatController {

    @Resource
    private ChatService chatService;

    @Resource
    private DocumentService documentService;

    @GetMapping(value = "/stream/doChatByRag", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<ChatMessageContent>> chatByRag(ChatMessageRequest request) {
        List<Document> ragDocContext = null;
        if (request.getKb()) {
            ragDocContext = documentService.searchDocument(request.getPrompt());
        }

        Flux<String> responseFlux = chatService.chatStreamResponse(request.getPrompt(), ragDocContext);

        return responseFlux.map(content -> ServerSentEvent.builder(ChatMessageContent.Builder.builder().content(content).build())
                        .event(SseEventType.CHUNK.getValue())
                        .build()
                ).concatWith(Flux.just(ServerSentEvent.builder(ChatMessageContent.Builder.builder().content("").build())
                        .event(SseEventType.CLOSE.getValue())
                        .build()
                ));
    }
    
}

启动服务后，我们询问一个文档中相关的问题，大模型回答如下：

如果需要将不同的 app 区分存储的向量，可以使用 RedisVectorStore#builder 处理，示例：

public class RedisVectorStoreFactory {
    
    private final RedisVectorStore redisVectorStore;

    private final EmbeddingModel embeddingModel;

    @Value("${spring.ai.vectorstore.redis.prefix}")
    private String commonPrefix;

    public RedisVectorStoreFactory(RedisVectorStore redisVectorStore, EmbeddingModel embeddingModel) {
        this.redisVectorStore = redisVectorStore;
        this.embeddingModel = embeddingModel;
    }

    /**
     * 根据前缀获取 RedisVectorStore
     *
     * @param prefix 前缀
     * @return RedisVectorStore
     */
    public RedisVectorStore getRedisVectorStoreByPrefix(String prefix) {
        String defaultPrefix = DEFAULT_PREFIX;
        if (this.commonPrefix != null) {
            defaultPrefix = this.commonPrefix;
        }

        return RedisVectorStore.builder(redisVectorStore.getJedis(), embeddingModel)
                .prefix(defaultPrefix + SEPARATOR + prefix + SEPARATOR)
                .build();
    }
    
}
