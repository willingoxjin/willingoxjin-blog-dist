04.CAS 服务端自定义登录页

自定义登录页或主题样式文档：CAS 动态主题 (opens new window)

# 一、自定义 CAS Server 主题

在使用 CAS Server 作为单点登录的服务器时，我们根据自己公司开发的系统，自定义系统的登录页，而不是使用 CAS Server 默认的登录页面。

下面，我们就来自定义一个 CAS Server 的主题，以下是 demo 的代码。

# 1.自定义主题文件

创建目录 src/main/resources/static/themes/myTheme/css，在目录下新建 cas.css 文件，内容如下：

在 src/main/resources/static/themes/[theme_name] 目录中，除了可以配置 css 文件，还可以添加 js 、字体、图片等其他静态资源文件。

# 2.主题配置

在 src/main/resources 目录下新增 [theme_name].properties 文件，这个文件就是用来配置我们的主题对应的资源路径、以及主题名称、主题描述、是都显示相关的菜单等等配置。

这里我们新建 myTheme.properties 文件，内容如下：

myTheme.theme.name=myTheme Theme
myTheme.theme.description=My Custom Theme

myTheme.standard.css.file=/themes/myTheme/css/cas.css
1
2
3
4

创建目录 src/main/resources/templates/myTheme/login 目录，并在 myTheme 目录下配置我们的登录页面的 html。

需要注意的是，从 CAS 6.0 开始，CAS 自定义主题的登录页面casLoginView.html 文件，需要配置在 resources/templates/[theme_name]/login 目录下。在之前的版本中，该文件是存放在 resources/templates/[theme_name] 下的。

如下，在 src/main/resources/templates/myTheme/login 目录下添加 casLoginView.html 文件内容如下：

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title th:text="#{cas.login.pagetitle}">CAS Login Page</title>
  <link rel="stylesheet" th:href="@{${#themes.code('myTheme.standard.css.file')}}">
</head>
<body>
<div class="container">
  <h1>CAS Sign in</h1>
  <form method="post" th:object="${credential}" accept-charset="UTF-8">
    <label for="username">Username</label>
    <input type="text" id="username" name="username" required>
    <label for="password">Password</label>
    <input type="password" id="password" name="password" required>
    <input type="hidden" name="execution" th:value="${flowExecutionKey}"/>
    <input type="hidden" name="_eventId" value="submit"/>
    <input type="hidden" name="geolocation"/>
    <div id="err-msg" th:if="${#fields.hasErrors('*')}">
      <span th:each="err : ${#fields.errors('*')}" th:utext="${err}"/>
    </div>
    <button type="submit">Sign in</button>
    <a href="#" class="forgot-password">Forgot password?</a>
    <div class="or-separator">
      <hr>
      <span>or</span>
      <hr>
    </div>
    <div class="social-login">
      <a href="#">Sign up</a>
    </div>
  </form>
</div>
</body>
</html>

这里，我们可以使用 thymeleaf 的语法，获取我们在 myTheme.properties 主题配置文件中配置内容。如上面的 casLoginView.html 文件中的 th:href="@{${#themes.code('myTheme.standard.css.file')}}"。

# 3.使用自定义的主题

在 CAS Server 的 application.yml 文件中，添加如下配置：

cas:
  theme:
    default-theme-name: myTheme

# 4.测试

启动 CAS Server，访问登录界面，不出意外的话，就可以看到我们自己配置的登录界面了。如下图：

需要注意的是，如果我们的配置有问题，或者 login/casLoginView.html 文件的 thymeleaf 配置存在问题，页面则会使用默认的登录页。

这里，我们仅仅配置了登录页面，其他的页面当然也可以配置，比如登陆成功的页面、注销登录的页面等等，可定义的页面见：https://apereo.github.io/cas/6.6.x/ux/User-Interface-Customization-Views.html#templates (opens new window)。

# 二、为 CAS Client 配置主题

上面在 CAS Server 的 application.yml 文件中配置的 cas.theme.default-theme-name 属性属于全局配置，如果想要针对某一个 CAS Client 配置主题，可以在 src/main/resources/services/casclient-100.json 配置中设置 theme 属性。如下：

{
  "@class": "org.apereo.cas.services.RegexRegisteredService",
  "serviceId": "^(https|http)://client.cas.example.com.*",
  "name": "cas-client",
  "id": 100,
  "description": "cas client based on spring security",
  "evaluationOrder": 1,
  "theme": "myTheme"
}

# 三、使用 CAS 提供的其他主题

CAS Server 还提供了内置的 example 和 twbs 两种主题可以使用。

如下，在 build.gradle 文件中，添加 cas-server-support-themes-collection 依赖：

dependencies {
		// ...
		implementation "org.apereo.cas:cas-server-support-themes-collection:${project.'cas.version'}"
}

然后，在 CAS Server 的 application.yml 文件中，添加如下配置：

  theme:
    default-theme-name: twbs

# 四、国际化支持

# 1.使用国际化

CAS Server 默认支持国际化，在页面中使用 locale 参数，可以切换对应的国际化语言，如下：

https://server.cas.example.com:18443/cas/login?locale=ca
https://server.cas.example.com:18443/cas/login?locale=zh_CN
1
2

国际化的语言包都存放在 src/main/resources 的 messages_xx.properties 文件下。系统默认使用英语，默认的语言包文件为 messages.properties。

如果我们想要自定义内容显示到视图中，也可以在 custom_messages.properties 文件下定义。

如下，我们在 src/main/resources 目录下配置不同的语言包内容：

  • messages.properties 默认语言包内容：
login.page.title=CAS SIGN IN
1
  • messages_zh_CN.properties 中文语言包内容：
login.page.title=统一登录认证平台
1
  • custom_messages.properties 自定义内容：
welcome.msg=欢迎使用CAS系统
1

然后，在我们自定义的 casLoginView.html 文件中，使用语言包：

<div class="container">
	<h1 th:utext="#{login.page.title}">CAS Sign in</h1>
  <h1 th:utext="#{welcome.msg}"></h1>
	<!--  ... --->

</div>
1
2
3
4
5
6

其实，这和我们在其他系统使用 Thymeleaf 和国际化是一样的。

配置好之后，我们重启服务，访问 https://server.cas.example.com:18443/cas/login?locale=ca 看到的是英文，访问 https://server.cas.example.com:18443/cas/login?locale=zh_CN 看到的是中文内容。

# 2.为 CAS Client 配置国际化

如果我们想要为不同的客户端服务配置不同的默认语言，可以在 src/main/resources/services/casclient-100.json 中配置 locale 属性。如下：

{
  "@class": "org.apereo.cas.services.RegexRegisteredService",
  "serviceId": "^(https|http)://client.cas.example.com.*",
  "name": "cas-client",
  "id": 100,
  "description": "cas client based on spring security",
  "evaluationOrder": 1,
  "locale": "zh_CN"
}
