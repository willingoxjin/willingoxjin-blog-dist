# 二、CAS 服务端配置

CAS 服务端基于数据库认证的文档：Database Authentication (opens new window)

CAS 服务端认证时，查询用户信息数据来源支持数据库、Cassandra、Redis、MongoDB 等等。

这里，我们继续上篇文章继续配置 CAS Server。

dependencies {

# 2. 配置密码编码器

如果不配置密码编码器，CAS Server 默认读取明文密码。不过，在实际开发中，我们都会保存加密后的密码。

这里，我们新建一个类，实现 PasswordEncoder 接口，具体的方法实现，我们就直接借助 DelegatingPasswordEncoder 实现了。代码如下：

public class CustomPasswordEncoder implements PasswordEncoder {

    private DelegatingPasswordEncoder delegate;

    public CustomPasswordEncoder() {
        Map<String, PasswordEncoder> encoders = new HashMap<>();
        String encodingId = "noop";
        PasswordEncoder bcrypt = new BCryptPasswordEncoder(10);
        encoders.put("bcrypt", bcrypt);
        PasswordEncoder noop = NoOpPasswordEncoder.getInstance();
        encoders.put(encodingId, noop);

        delegate = new DelegatingPasswordEncoder(encodingId, encoders);
    }

    @Override
    public String encode(CharSequence rawPassword) {
        return delegate.encode(rawPassword);
    }

    @Override
    public boolean matches(CharSequence rawPassword, String encodedPassword) {
        return delegate.matches(rawPassword, encodedPassword);
    }

    @Override
    public boolean upgradeEncoding(String encodedPassword) {
        return delegate.upgradeEncoding(encodedPassword);
    }
}

# 3 配置数据源和密码编码

配置 application.yml 文件：

cas:
  authn:
    jdbc:
      query:
      	# 数据源配置
        - driver-class: com.mysql.cj.jdbc.Driver
          url: jdbc:mysql://127.0.0.1:3306/security_demo?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
          user: root
          password: 123456
          # 健康检查的sql
          health-query: SELECT 1
          # 查询用户信息的sql
          sql: SELECT * FROM sys_user WHERE username=?
          # 用户密码的字段名
          field-password: password
          # 数据库连接池配置
          pool:
            min-size: 10
            max-size: 20
          # 配置密码编码器
          passwordEncoder:
          	# 密码编码器类型。默认不使用密码编码器
            type: org.apereo.cas.config.CustomPasswordEncoder

# 2.UserDetailsService 的实现

自定义 MybatisUserDetailsServiceImpl 类，实现 UserDetailsService 接口。代码如下：

public class MybatisUserDetailsServiceImpl implements UserDetailsService {

    @Autowired
    private SysUserMapper sysUserMapper;

    @Autowired
    private SysUserRoleMapper sysUserRoleMapper;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // 查询用户信息
        SysUser user = sysUserMapper.selectOne(Wrappers.<SysUser>lambdaQuery().eq(SysUser::getUsername, username));
        if (user == null) {
            throw new UsernameNotFoundException(username);
        }

        UserBuilder userBuilder = User.withUsername(user.getUsername())
                .password(user.getPassword())
                .disabled(user.getEnable().equals(IS_DISABLED));
      	String[] roles = sysRoleMenuMapper.selectUserRoles(user.getUserId());
        if (roles != null) {
            userBuilder.authorities(roles);
        }

        return userBuilder.build();
    }

}

# 3.Spring Security 配置

配置 MybatisUserDetailsServiceImpl 和 AuthenticationManager ：

@Configuration
public class AuthenticationManagerConfig {

    @Autowired
    private PasswordEncoder passwordEncoder;

    @Bean
    public UserDetailsService mybatisUserDetailsService() {
        return new MybatisUserDetailsServiceImpl();
    }

    @Bean
    public AuthenticationManager authenticationManager(CasAuthenticationProvider provider) {
        return new ProviderManager(provider);
    }
  
}

# 4.application.properties 配置

application.properties 文件中的数据源、mybatis-plus、cas 配置如下：

spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/security_demo?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=123456

mybatis-plus.mapper-locations=classpath:mapper/*.xml
# 自动转换驼峰命名
mybatis-plus.configuration.map-underscore-to-camel-case=true
# 逻辑删除配置
mybatis-plus.global-config.db-config.logic-delete-field=delFlag
mybatis-plus.global-config.db-config.logic-delete-value=1
mybatis-plus.global-config.db-config.logic-not-delete-value=0
# sql日志打印
mybatis-plus.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl

# 是否开启cas
security.cas.enabled=true
# cas服务端配置
## cas服务端地址
security.cas.server.prefix=https://server.cas.example.com:18443/cas
## cas服务端登录地址
security.cas.server.login=${security.cas.server.prefix}/login
## cas服务端登出地址
security.cas.server.logout=${security.cas.server.prefix}/logout

# cas客户端配置
security.cas.client.prefix=http://client.cas.example.com:${server.port}
## cas客户端登录地址
security.cas.client.login=${security.cas.client.prefix}/login/cas
## cas客户端登出地址-相对路径
security.cas.client.logout-relative=/logout/cas
## cas客户端登出地址
security.cas.client.logout=${security.cas.client.prefix}${security.cas.client.relative}
