12.Spring Security 实现 CAS 单点登录

# 一、CAS 服务端搭建

# 1.证书生成与导入

CAS Server 从 4.0 版本开始，需要使用 HTTPS 访问服务，所以我们得提前准备 SSL 证书。我们可以购买 SSL 证书，也可以在云服务厂商（阿里云、腾讯云等）申请免费的 SSL 证书，也可以自己使用 JDK 自带的 keytool 证书管理工具，生成一个 SSL 证书即可。

这里，我们就直接使用 JDK 中自带的 keytool 生成证书了。

# 1.1 keytool 介绍

keytool 是 JDK 中自带的数据证书管理工具，这个工具一般存放在 $$JAVA_HOME 的 \jre\bin\目录下。利用 keytool 可以生成我们需要的密钥和证书信息。

这些生成的所有数字证书都是以一条一条（采用别名来区别）的形式存入证书库中，证书库中的每个证书都包含该条证书的私钥，公钥和对应的数字证书的信息。证书库中的每条证书都可以导出为数字证书文件，数字证书文件只包括主体信息和对应的公钥信息。

Keytool 会把密钥（key） 和证书（certificates） 存在于一个名为为 keystore 的文件中，在 keystore 文件中，包含两种数据：

  • 密钥实体（Key entity） — 私钥（Secret key）;
  • 可信任的证书实体（Trusted certificate entries） — 公钥。

keytool 常用命令如下（Unix平台）：

# 生成证书
keytool -genkey -alias myserver -keyalg RSA -keysize 1024 -validity 365 \
-keystore ./myserver1.keystore -dname "CN=myserver,OU=danwei,O=zuzhi,L=shi,ST=sheng,C=CN"

# 生成证书
keytool -genkey -alias myserver -keyalg RSA -keysize 1024 -validity 365 \
-keystore ./myserver3.keystore -keypass 123456 -storepass 123456\
-dname "CN=myserver,OU=danwei,O=zuzhi,L=shi,ST=sheng,C=CN"

# 导出证书到文件
keytool -export -alias myserver -keystore ./myserver.keystore \
-file ./myserver.cer -storepass 123456

# 导入证书到jre密钥库的命令
keytool -import -trustcacerts -alias myserver \
-keystore "$JAVA_HOME/jre/lib/security/cacerts" -file ./myserver.cer -storepass 123456

# 列出jre密钥库名为myserver的证书
keytool -list -alias myserver -keystore "$JAVA_HOME/jre/lib/security/cacerts" \
-storepass 123456

# 查看证书列表
keytool -list -keystore ./myserver.keystore
# 查看证书列表,并显示密钥库中证书的详细信息
keytool -list -v -keystore ./myserver.keystore -storepass 123456
# 查看证书列表,并以 RFC 样式输出
keytool -list -rfc -keystore ./myserver.keystore -storepass 123456

# 删除证书库里面的某个证
keytool -delete -alias myserver -keystore "$JAVA_HOME/jre/lib/security/cacerts" \
-storepass 123456 

# 修改证书密码
keytool -keypasswd -alias myserver -keypass 123456 -new 666666 \
-keystore myserver.keystore -storepass 123456

# 修改密钥库存储密码
keytool -storepasswd -new 666666 -keystore myserver.keystore -storepass 123456

# 修改密钥库别名,myserver修改为new-myserver
keytool -changealias -keystore myserver.keystore -alias myserver -destalias new-myserver

部分参数说明：

  • -genkey 生成秘钥库，默认会在用户的主目录中创建一个 .keystore 文件，还会产生一个 myserver 的别名，myserver 中包含用户的公钥、私钥和证书信息；

    • -dname 设置拥有者信息。CN: 姓名；OU: 组织单位名称；O: 组织名称；L: 省/市/自治区名称；C: 国家/地区代码；
    • -alias 别名，每个 keystore 都关联一个独一无二的 alias，alias 通常不区分大小写，自己随便定义即可；
    • -keyalg 密钥算法，可选值为 RSA 和 DSA（若不指定默认采用DSA）；
    • -keysize 设置秘钥长度；
    • -validity 设置密钥的有效期是多少天，默认为 90 天；
    • -keystore 设置生成的秘钥库文件存储路径和名称；
    • -keypass 设置秘钥口令；

  • -storepass 设置秘钥库口令；

  • -delete 删除密钥库中某条目；

  • -import 将已签名数字证书导入密钥库。

  • -list 列出秘钥库中的条目，显示密钥库中的证书信息；

    • -v 显示密钥库中证书的详细信息；
    • -rfc 以 RFC 样式输出。

# 2.2 配置证书

首先，将上面生成的证书文件 cas-server.keystore，放入到源码的 src/resources 目录下。

然后配置证书，修改 src/resources 目录下的 application.yml 文件：

server:
	# 服务启动的端口
  port: 18443
  servlet:
  	# 服务访问的根路径
    context-path: /cas
  ssl:
  	# 秘钥库文件位置
    key-store: classpath:server_cas_example_com.keystore
    # 密钥库口令
    key-store-password: 123456
    # 密钥口令,和密钥库口令相同
    key-password: 123456

cas:
  authn:
    accept:
    	# 基于内存的用户密码配置,使用::分割用户名密码
      users: "willingoxjin::123456"

# 3.配置 CAS Client 的 services 信息

上面，我们已经把 CAS Server 服务端搭建好，并集成了数据源。不过，我们还需要把 CAS 客户端接入到 CAS 服务端，如果我们没有定义服务记录，客户端在访问时会提示：“未认证授权的服务：不允许使用CAS来认证您访问的目标应用”。

在 CAS Server 支持多种注册服务的方式，包括 InMemory、 JSON、YAML、Git、MongoDB、LDAP、JPA、Redis、Cassandra、REST、自定义等等。具体支持和配置可见 CAS 服务注册文档：Service Management (opens new window)

这里作为学习，我们采用 JSON 声明文件的方式注册。不过，如果你在生产环境，就不建议你这么做了。

# 3.1 注册服务的 JSON 文件

首先，我们来配置 JSON 文件，这个文件是用来进行服务注册的，其对应客户端，

JSON 文件的命名规范为：

JSON filename=servicename + "-" + serviceNumericId +".json"
1

我们命名为 casclient-10000.json，并存放在 CAS Server 的 src/main/resources/services 目录下

这个 JSON 文件内容如下：

{
  "@class": "org.apereo.cas.services.RegexRegisteredService",
  "serviceId": "^(https|http)://client.cas.example.com*",
  "name": "casclient",
  "id": 100,
  "description": "cas client based on spring security",
  "evaluationOrder": 1
}
1
2
3
4
5
6
7
8
  • @class：必须配置为 org.apereo.cas.services.RegexRegisteredService；
  • serviceId：可以通过正则表达式来匹配每一个 CAS Client 的地址；
  • name：指定 CAS Client 的名称，可以随意命名；
  • id：全局唯一的编号，也就是 JSON 文件的命名中的 serviceNumericId 字段；
  • description：对 CAS Client 的描述；
  • evaluationOrder：指明 CAS Client 服务的执行顺序。

# 3.2 配置 services 信息

修改 src/main/resources 下的 application.yml 文件，内容如下：

cas:
  serviceRegistry:
  json:
    # 加载json资源
    location: classpath:/services
  core:
    # 是否开启json初始化识别功能,默认为false
    init-from-json: true
  tgc:
    # 忽略https安全协议，使用 HTTP 协议
    secure: false

# 2.CAS 的配置

添加 application.properties 中，cas 的配置：

security.cas.enabled=true

# cas服务端配置
## cas服务端地址
security.cas.server.prefix=https://server.cas.example.com:18443/cas
## cas服务端登录地址
security.cas.server.login=${security.cas.server.prefix}/login
## cas服务端登出地址
security.cas.server.logout=${security.cas.server.prefix}/logout

# cas客户端配置(即我们当前服务的地址)
security.cas.client.prefix=http://client.cas.example.com:${server.port}
## cas客户端登录地址
security.cas.client.login=${security.cas.client.prefix}/login/cas
## cas客户端登出地址-相对路径
security.cas.client.relative=/logout/cas
## cas客户端登出地址
security.cas.client.logout=${security.cas.client.prefix}${security.cas.client.relative}

密码编码器配置：

@Configuration
public class PasswordEncoderConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        Map<String, PasswordEncoder> encoders = new HashMap<>();
        String encodingId = "noop";
        PasswordEncoder bcrypt = new BCryptPasswordEncoder(10);
        encoders.put("bcrypt", bcrypt);
        PasswordEncoder noop = NoOpPasswordEncoder.getInstance();
        encoders.put(encodingId, noop);

        return new DelegatingPasswordEncoder(encodingId, encoders);
    }
  
}

UserDetailsService 和 AuthenticationManager 的配置：

@Configuration
public class AuthenticationManagerConfig {

    @Autowired
    private PasswordEncoder passwordEncoder;

  	@Bean
    public InMemoryUserDetailsManager userDetailsService(SecurityProperties properties) {
        List<UserDetails> users = new ArrayList<>();
        // 获取application.properties配置文件中的用户信息
        SecurityProperties.User user = properties.getUser();
        if (user != null) {
            UserDetails userDetails = User.withUsername(user.getName())
                    .password(passwordEncoder.encode(user.getPassword()))
                    .roles(StringUtils.toStringArray(user.getRoles()))
                    .build();
            users.add(userDetails);
        }

        UserDetails userDetails2 = User.withUsername("zhangsan")
                .password(passwordEncoder.encode("123456"))
                .roles("USER")
                .build();
        users.add(userDetails2);
        return new InMemoryUserDetailsManager(users);
    }

    @Bean
    public AuthenticationManager authenticationManager(CasAuthenticationProvider provider) {
        return new ProviderManager(provider);
    }

}

@Configuration
@ConditionalOnProperty(prefix = "security.cas", name = "enabled", havingValue = "true")
@EnableConfigurationProperties(CasSecurityProperties.class)
public class CasSecurityConfiguration {

    @Autowired
    private CasSecurityProperties casSecurityProperties;

    /**
     * 配置CAS Client的属性
     */
    @Bean
    public ServiceProperties serviceProperties() {
        ServiceProperties serviceProperties = new ServiceProperties();
        // 与CasAuthenticationFilter监视的URL一致
        serviceProperties.setService(casSecurityProperties.getClient().getLogin());
        // 是否关闭单点登录，默认为false，所以也可以不设置
        serviceProperties.setSendRenew(false);
        return serviceProperties;
    }

    @Bean
    @Primary
    public AuthenticationEntryPoint authenticationEntryPoint(ServiceProperties serviceProperties) {
        CasAuthenticationEntryPoint entryPoint = new CasAuthenticationEntryPoint();
        // CAS Server认证的登录地址
        entryPoint.setLoginUrl(casSecurityProperties.getServer().getLogin());
        entryPoint.setServiceProperties(serviceProperties);
        return entryPoint;
    }

    /**
     * 配置ticket校验功能，需要提供CAS Server校验ticket的地址
     */
    @Bean
    public TicketValidator ticketValidator() {
        // 默认情况下使用Cas20ProxyTicketValidator，验证入口是${casServerPrefix}/proxyValidate
        return new Cas20ProxyTicketValidator(casSecurityProperties.getServer().getPrefix());
    }

    @Bean
    public CasAuthenticationProvider casAuthenticationProvider(ServiceProperties serviceProperties,
                                                                TicketValidator ticketValidator,
                                                                UserDetailsService userDetailsService) {
        CasAuthenticationProvider provider = new CasAuthenticationProvider();
        provider.setServiceProperties(serviceProperties);
        provider.setTicketValidator(ticketValidator);
        provider.setUserDetailsService(userDetailsService);
        provider.setKey("CAS-AUTH");
        return provider;
    }

    @Bean
    public CasAuthenticationFilter casAuthenticationFilter(ServiceProperties serviceProperties,
            AuthenticationManager authenticationManager) {
        CasAuthenticationFilter filter = new CasAuthenticationFilter();
        filter.setServiceProperties(serviceProperties);
        filter.setAuthenticationManager(authenticationManager);
        return filter;
    }

    @Bean
    public SingleSignOutFilter singleSignOutFilter() {
        SingleSignOutFilter singleSignOutFilter = new SingleSignOutFilter();
        singleSignOutFilter.setIgnoreInitConfiguration(true);
        return singleSignOutFilter;
    }

    @Bean
    public LogoutFilter logoutFilter() {
        SecurityContextLogoutHandler logoutHandler = new SecurityContextLogoutHandler();
        LogoutFilter logoutFilter = new LogoutFilter(casSecurityProperties.getServer().getLogout(), logoutHandler);
        // 设置客户端注销请求的路径
        logoutFilter.setFilterProcessesUrl(casSecurityProperties.getClient().getLogoutRelative());
        return logoutFilter;
    }

}

配置相关的对象、过滤器到 Spring Security 中：

@Configuration
public class SecurityConfig {

    @Autowired
    private AuthenticationManager authenticationManager;

    @Autowired
    private AuthenticationEntryPoint entryPoint;

    @Autowired
    private CasAuthenticationFilter casAuthenticationFilter;

    @Autowired
    private SingleSignOutFilter singleSignOutFilter;

    @Autowired
    private LogoutFilter requestSingleLogoutFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                    .requestMatchers(HttpMethod.OPTIONS).permitAll()
                    .requestMatchers("/login/cas", "/favicon.ico", "/error").permitAll()
                    .anyRequest().authenticated()
            )
            .exceptionHandling(exceptionHandling ->
                    exceptionHandling.authenticationEntryPoint(entryPoint)
            )
          	.authenticationManager(authenticationManager)
          	// 添加CAS认证过滤器
            .addFilter(casAuthenticationFilter)
          	// 单点注销处理
            .addFilterBefore(singleSignOutFilter, CasAuthenticationFilter.class)
          	// 处理退出登录
            .addFilterBefore(requestSingleLogoutFilter, LogoutFilter.class)
    }
}
