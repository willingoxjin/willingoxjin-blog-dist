06.Spring Security 的密码

# 一、密码该如何加密？

# 1.明文存储

一开始，密码以明文形式存储。密码被认为是安全的，因为数据存储密码被保存在访问它所需的凭据中。

然而，恶意用户能够通过使用 SQL 注入等攻击找到获取用户名和密码的大量“数据转储”的方法。

因此，明文存储非常不安全。

# 2.MD5 哈希

后来，开发者对密码单向哈希（例如 SHA-256）后，再进行存储。当用户尝试进行身份验证时，将用户输入的密码的哈希值与存储的哈希密码比较。

如果发生泄露，只会暴露密码的单向哈希值。由于哈希值是单向的，并且在计算上很难根据哈希值猜测密码，因此不值得花费精力找出系统中的每个密码。

# 3.哈希 + 盐

但是，某些恶意用户为了破解密码，创建了彩虹表 (opens new window)。他们没有每次都猜测每个密码，而是计算一次密码并将其存储在查找表中。

为了降低彩虹表的有效性，开发人员开始使用加盐密码。在对密码进行哈希计算时，为每个用户的密码生成随机字节（称为盐），盐和用户的密码通过哈希函数计算后，生成唯一的哈希值。并且盐将以明文形式与用户密码一起存储。然后，当用户尝试进行身份验证时，哈希密码将与存储的盐的哈希值和用户输入的密码进行比较。

这样一来，每个盐和密码组合的哈希值都不同，就大大降低了彩虹表的有效性。

# 4.自适应单向函数

但是，就算加盐的哈希（如 SHA-256）也不再安全。因为利用现代硬件，我们每秒可以执行数十亿次哈希计算。这意味着每个密码可以被轻松破解。不过又有了自适应单向函数来解决这个问题。

在密码学中，单向函数是一种只能单向计算的函数，即从输入计算出输出很容易，但从输出反推输入几乎是不可能的。而自适应单向函数则是在单向函数的基础上，加入了自适应性，使得其更加难以被破解。

自适应单向函数（Adaptive One-Way Function，简称 AOWF）与普通的单向函数不同的是，自适应单向函数允许配置“工作因素”（work factor），该因素可以随着硬件的改进而增长。

自适应函数的工作因素主要包括输入数据的复杂度和计算资源的可用性：

  • 输入数据的复杂度是指输入数据的难度和复杂程度，例如密码的长度、字符集、组合方式等等。自适应函数会根据输入数据的复杂度，自适应地调整输出结果的难度，以增强密码和敏感数据的安全性。

  • 计算资源的可用性是指计算机的处理能力和存储空间。自适应函数会根据计算资源的可用性，自适应地调整计算强度和计算时间，以保证计算结果的正确性和安全性。

自适应函数的工作因素决定了自适应函数的计算强度和计算时间，以保证输出结果的安全性和正确性。

常用的自适应单向函数如下：

  1. BCrypt：BCrypt 是一种密码哈希函数，采用了 Blowfish 加密算法的一部分，并加入了自适应性和盐值机制，可以根据输入密码的长度和复杂度，自适应地调整计算强度，增强密码的安全性。

  2. PBKDF2：PBKDF2 是一种基于口令的密钥派生函数，可以将一个简单的密码转换为一个更加安全的密钥。它采用了迭代哈希算法，可以根据输入密码和盐值，自适应地调整计算强度，增强密码的安全性。

  3. Argon2：Argon2 是一种密码哈希函数，可以根据输入密码和盐值，自适应地调整计算强度，增强密码的安全性。它采用了内存硬化技术，可以抵御侧信道攻击和缓存攻击。

  4. Scrypt：Scrypt 是一种基于密码的密钥派生函数，可以将一个简单的密码转换为一个更加安全的密钥。它采用了内存硬化技术和随机访问算法，可以根据输入密码和盐值，自适应地调整计算强度，增强密码的安全性。

但是，自适应单向函数的计算成本比较高，因为它需要进行多次迭代计算，才能生成最终的输出结果。这会增加服务器的负担，降低系统的性能。因此，建议将自适应函数的“工作因素”调整为大约需要一秒钟来验证系统上的密码。这种权衡是为了让攻击者很难破解密码，但成本又不会太高。

# 二、PasswordEncoder

Spring Security 的 PasswordEncoder 接口用于对密码进行单向转换，以安全地存储密码，并且对用户认证时提供的密码和系统存储的密码进行匹配。

PasswordEncoder 接口中只有三个方法，如下：

public interface PasswordEncoder {

	String encode(CharSequence rawPassword);

	boolean matches(CharSequence rawPassword, String encodedPassword);

	default boolean upgradeEncoding(String encodedPassword) {
		return false;
	}

}

在 Spring Security 中，常用的 PasswordEncoder 实现类有 BCryptPasswordEncoder、NoOpPasswordEncoder、Pbkdf2PasswordEncoder、SCryptPasswordEncoder 等

# 三、DelegatingPasswordEncoder

Spring Security 引入了DelegatingPasswordEncoder ，能够灵活的处理密码的加密：

  • 使用新的密码加密存储策略对密码进行编码，并且无需修改之前使用其他编码格式存储的密码。
  • 允许验证不同格式的密码（如密码编码哈希和 BCrypt 可以共存）。
  • 允许将来升级编码。

# 1.创建 DelegatingPasswordEncoder 实例

使用 PasswordEncoderFactories 的 createDelegatingPasswordEncoder() 方法可以构建一个 DelegatingPasswordEncoder 实例。如下：

PasswordEncoder passwordEncoder =
    PasswordEncoderFactories.createDelegatingPasswordEncoder();

PasswordEncoderFactories.createDelegatingPasswordEncoder() 方法最终会返回一个 DelegatingPasswordEncoder 对象。PasswordEncoderFactories 源码如下：

public final class PasswordEncoderFactories {

	private PasswordEncoderFactories() {
	}

	@SuppressWarnings("deprecation")
	public static PasswordEncoder createDelegatingPasswordEncoder() {
		String encodingId = "bcrypt";
		Map<String, PasswordEncoder> encoders = new HashMap<>();
		encoders.put(encodingId, new BCryptPasswordEncoder());
		encoders.put("ldap", new org.springframework.security.crypto.password.LdapShaPasswordEncoder());
		encoders.put("MD4", new org.springframework.security.crypto.password.Md4PasswordEncoder());
		encoders.put("MD5", new org.springframework.security.crypto.password.MessageDigestPasswordEncoder("MD5"));
		encoders.put("noop", org.springframework.security.crypto.password.NoOpPasswordEncoder.getInstance());
		encoders.put("pbkdf2", Pbkdf2PasswordEncoder.defaultsForSpringSecurity_v5_5());
		encoders.put("pbkdf2@SpringSecurity_v5_8", Pbkdf2PasswordEncoder.defaultsForSpringSecurity_v5_8());
		encoders.put("scrypt", SCryptPasswordEncoder.defaultsForSpringSecurity_v4_1());
		encoders.put("scrypt@SpringSecurity_v5_8", SCryptPasswordEncoder.defaultsForSpringSecurity_v5_8());
		encoders.put("SHA-1", new org.springframework.security.crypto.password.MessageDigestPasswordEncoder("SHA-1"));
		encoders.put("SHA-256",
				new org.springframework.security.crypto.password.MessageDigestPasswordEncoder("SHA-256"));
		encoders.put("sha256", new org.springframework.security.crypto.password.StandardPasswordEncoder());
		encoders.put("argon2", Argon2PasswordEncoder.defaultsForSpringSecurity_v5_2());
		encoders.put("argon2@SpringSecurity_v5_8", Argon2PasswordEncoder.defaultsForSpringSecurity_v5_8());
		return new DelegatingPasswordEncoder(encodingId, encoders);
	}

}

这里的使用了静态工厂方法的设计模式。

静态工厂方法模式是一种创建型设计模式，它与简单工厂模式类似，但是不需要创建工厂类的实例。它通过在工厂类中提供静态方法来创建对象，这些静态方法通常称为工厂方法。

静态工厂方法模式适用于需要创建的对象较少并且不需要频繁添加新的产品类的情况。与简单工厂模式相比，静态工厂方法模式具有更好的灵活性和可扩展性，因为它可以通过增加新的静态方法来创建新的产品对象，而不需要修改现有的代码。

结合 DelegatingPasswordEncoder 的构造方法可以发现，默认情况下使用的密码编码器是 BCryptPasswordEncoder。

当然，我们也可以创建自己的自定义 DelegatingPasswordEncoder 实例，只需要稍微改造一下上面的 createDelegatingPasswordEncoder 方法即可（根据业务的需求，添加不同的 PasswordEncoder）。

默认情况下，若我们没有在 Spring 容器中自定义 PasswordEncoder 实例，就会调用 PasswordEncoderFactories.createDelegatingPasswordEncoder() 方法，使用 DelegatingPasswordEncoder 的实例作为密码编码器。

# 2.DelegatingPasswordEncoder 源码

DelegatingPasswordEncoder 是 PasswordEncoder 接口的实现类，它将密码编码的工作委托给其他 PasswordEncoder 实现类。

DelegatingPasswordEncoder 委托（Delegate）处理的内容如下：

  • String idForEncode：用于查找应该使用哪个密码编码器进行编码的 id。
  • PasswordEncoder passwordEncoderForEncode：通过上面的 idForEncode 匹配到的 PasswordEncoder 实例。该实例会对密码进行编码（即加密）。
  • Map<String, PasswordEncoder> idToPasswordEncoder：这是一个 Map 类型的属性，它存储了所有被委托的 PasswordEncoder 实现类。Map 的 key 表示 PasswordEncoder 的 ID，value 是一个 PasswordEncoder 实例。
  • PasswordEncoder defaultPasswordEncoderForMatches：默认的密码编码匹配器。当从 idToPasswordEncoder 中没有匹配到 PasswordEncoder 实例时，就会调用它的 matches 方法进行密码匹配。

DelegatingPasswordEncoder 的源码如下：

public class DelegatingPasswordEncoder implements PasswordEncoder {

	private static final String DEFAULT_ID_PREFIX = "{";

	private static final String DEFAULT_ID_SUFFIX = "}";

	private final String idPrefix;

	private final String idSuffix;

	private final String idForEncode;

	private final PasswordEncoder passwordEncoderForEncode;

	private final Map<String, PasswordEncoder> idToPasswordEncoder;

	private PasswordEncoder defaultPasswordEncoderForMatches = new UnmappedIdPasswordEncoder();

  public DelegatingPasswordEncoder(String idForEncode, Map<String, PasswordEncoder> idToPasswordEncoder) {
		this(idForEncode, idToPasswordEncoder, DEFAULT_ID_PREFIX, DEFAULT_ID_SUFFIX);
	}

	// 构造方法
	public DelegatingPasswordEncoder(String idForEncode, Map<String, PasswordEncoder> idToPasswordEncoder,
			String idPrefix, String idSuffix) {
    // 各种参数判空
		if (idForEncode == null) {
			throw new IllegalArgumentException("idForEncode cannot be null");
		}
		if (idPrefix == null) {
			throw new IllegalArgumentException("prefix cannot be null");
		}
		if (idSuffix == null || idSuffix.isEmpty()) {
			throw new IllegalArgumentException("suffix cannot be empty");
		}
		if (idPrefix.contains(idSuffix)) {
      // 前缀和后缀不能相互包含字符
			throw new IllegalArgumentException("idPrefix " + idPrefix + " cannot contain idSuffix " + idSuffix);
		}

    // 编码器映射idToPasswordEncoder中必须有idForEncode映射的对象
		if (!idToPasswordEncoder.containsKey(idForEncode)) {
			throw new IllegalArgumentException(
					"idForEncode " + idForEncode + "is not found in idToPasswordEncoder " + idToPasswordEncoder);
		}
    // 编码器映射idToPasswordEncoder中的key不能包含id前缀和id后缀
		for (String id : idToPasswordEncoder.keySet()) {
			if (id == null) {
				continue;
			}
			if (!idPrefix.isEmpty() && id.contains(idPrefix)) {
				throw new IllegalArgumentException("id " + id + " cannot contain " + idPrefix);
			}
			if (id.contains(idSuffix)) {
				throw new IllegalArgumentException("id " + id + " cannot contain " + idSuffix);
			}
		}
		this.idForEncode = idForEncode;
    // 设置用于密码编码（即加密）的PasswordEncoder
		this.passwordEncoderForEncode = idToPasswordEncoder.get(idForEncode);
		this.idToPasswordEncoder = new HashMap<>(idToPasswordEncoder);
		this.idPrefix = idPrefix;
		this.idSuffix = idSuffix;
	}


	@Override
	public String encode(CharSequence rawPassword) {
    // 对密码进行加密
    // rawPassword -> {bcrypt}encodePassword 
		return this.idPrefix + this.idForEncode + this.idSuffix + this.passwordEncoderForEncode.encode(rawPassword);
	}


  @Override
	public boolean matches(CharSequence rawPassword, String prefixEncodedPassword) {
		// 原密码和加密后的密码都为空,则直接return ture
    if (rawPassword == null && prefixEncodedPassword == null) {
			return true;
		}
    // 从加密后的密码中获取idForEncode
		String id = extractId(prefixEncodedPassword);
    // 根据idForEncode匹配对应的PasswordEncoder实例
		PasswordEncoder delegate = this.idToPasswordEncoder.get(id);
    // 若没有匹配到,就使用默认的密码编码器执行处理（默认是UnmappedIdPasswordEncoder,会直接抛出异常）
		if (delegate == null) {
			return this.defaultPasswordEncoderForMatches.matches(rawPassword, prefixEncodedPassword);
		}
    // 若匹配到了PasswordEncoder实例,则进行密码匹配处理
    // extractEncodedPassword方法会去除加密密码的的前缀,
    // 		如{bcrypt}$2a$10$X5RFdTsKE43FSd8VgLp10.nY/3YjL/mcRTryLAQLmvipdO9dAkcEK -> $2a$10$X5RFdTsKE43FSd8VgLp10.nY/3YjL/mcRTryLAQLmvipdO9dAkcEK
		String encodedPassword = extractEncodedPassword(prefixEncodedPassword);
		return delegate.matches(rawPassword, encodedPassword);
	}

  @Override
	public boolean upgradeEncoding(String prefixEncodedPassword) {
    // 若当前密码器使用的加密函数与加密后的密码不一致，
    // 则将旧的密码重新编码成新的密码格式。
		String id = extractId(prefixEncodedPassword);
		if (!this.idForEncode.equalsIgnoreCase(id)) {
			return true;
		}
		else {
			String encodedPassword = extractEncodedPassword(prefixEncodedPassword);
			return this.idToPasswordEncoder.get(id).upgradeEncoding(encodedPassword);
		}
	}
  
  // ......
}

# 3.密码升级

DelegatingPasswordEncoder 还支持升级密码加密算法：在用户登录时，如果密码匹配成功，则将用户的密码通过新的加密算法加密后（密码是用户在认证请求时传过来的），得到新算法加密的密码，然后将”新算法加密的密码“更新到数据库中，以此完成升级。

加密算法的具体逻辑，见源码 DaoAuthenticationProvider#createSuccessAuthentication。

# 四、Spring Security 中实现的编码器

# 1.BCryptPasswordEncoder

# 1.1 BCrypt 算法

BCrypt 是一种密码哈希函数，其使用 Blowfish 加密算法的一部分，BCrypt 算法将 salt （盐）随机混入到最终加密的密码中，验证时，无需单独提供之前的 salt ，因此，无需单独处理 salt 的问题。

BCrypt 可以将密码散列化为一个固定长度的字符串，且哈希后的字符串不可逆。

BCrypt 加密后的格式如下：

$2a$10$YpcBGj7GAizKQUYQzBW.pu0pCdPms73JHRuvL5SWVsfJRl4HUnfHm
1
  • $ 是分隔符。仅用来分割，没有其他意义。上面的密码被分割成了 2a、04、YpcBGj7GAizKQUYQzBW.pu0pCdPms73JHRuvL5SWVsfJRl4HUnfHm；
  • $2a$04$ 表示使用的是 bcrypt 算法;
    • 2a 是 bcrypt 函数加密的版本号；
    • 10 是 const 的值，表示迭代了 2 的 10 次幂 次；
  • 最后的 YpcBGj7GAizKQUYQzBW.pu0pCdPms73JHRuvL5SWVsfJRl4HUnfHm 则可以被分为两部分
    • 前 22 位：YpcBGj7GAizKQUYQzBW.pu 是随机生成的 salt（盐）；
    • 最后的字符：0pCdPms73JHRuvL5SWVsfJRl4HUnfHm 是哈希后的密文；

# 1.2 BCryptPasswordEncoder 的构造参数

  • int strength：用于指定哈希计算时的迭代次数。

    • strength 的值必须在 4~31 之间，默认为 10。
    • 迭代次数越多，哈希计算的复杂度就越高，破解密码的难度也就越大。通常情况下，strength 的值应该在10 到 31之间，其中 10 表示迭代次数为 1024 次，31 表示迭代次数为 2147483648 次。一般来说，建议使用10到12之间的值，这样可以保证哈希计算的安全性和速度之间的平衡。

  • BCryptVersion version：BCrypt 算法使用的版本，默认 $2a。可选值见 BCryptPasswordEncoder.BCryptVersion

  • SecureRandom random：随机实例，用于在生成盐的时候使用。

Spring 官方推荐使用 BCryptPasswordEncoder 验证系统上的密码的速度为 1 秒（即 matches 方法函数执行的时长 ）。不过这一秒在不同的系统上的 strength 参数是不一样的，这需要在服务器上自己去测试。

# 2.Argon2PasswordEncoder

Argon2PasswordEncoder 使用 Argon2 算法对密码进行哈希处理。为了阻止定制硬件上的密码破解，Argon2 是一种故意缓慢的算法，需要大量内存。与其他自适应单向功能一样，应将其调整为大约需要 1 秒来验证系统上的密码。

使用 Argon2PasswordEncoder ，需要引入 BouncyCastle 依赖。

在 pom.xml 中添加依赖如下：

<dependency>
  <groupId>org.bouncycastle</groupId>
  <artifactId>bcprov-jdk18on</artifactId>
  <version>1.75</version>
</dependency>

# 3.Pbkdf2PasswordEncoder

Pbkdf2PasswordEncoder 使用 PBKDF2 算法对密码进行哈希处理。为了阻止密码破解，PBKDF2 是一种故意缓慢的算法。与其他自适应单向功能一样，应将其调整为大约需要 1 秒来验证系统上的密码。当需要 FIPS 认证时，该算法是一个不错的选择。

FIPS（Federal Information Processing Standards）是美国联邦政府颁布的一系列计算机安全标准，旨在确保政府机构使用的计算机系统和软件能够满足一定的安全性要求。FIPS 标准由美国国家标准技术研究所（NIST）颁布，适用于美国联邦政府和其承包商使用的计算机系统和软件。

FIPS 认证是指通过 NIST 的认证程序，验证产品是否符合 FIPS 标准的要求。FIPS 认证通常包括对产品的安全性能进行评估和测试，以确保其符合 FIPS 标准的要求。FIPS 认证通常需要通过一系列的测试，包括功能测试、性能测试、安全性测试和漏洞测试等。

FIPS 认证通常被视为一项重要的安全认证，因为它确保了产品的安全性能符合国际标准，并且在政府机构和其他安全性要求较高的组织中得到广泛的应用。一些领域，如金融、医疗和军事等，也对 FIPS 认证有着较高的要求。

# 5.编码器的适用场景

Spring Security 提供了多种密码编码器，不同的编码器适用于不同的场景。以下是一些常用的密码编码器及其适用场景：

  1. BCryptPasswordEncoder：适用于大多数场景，特别是存储在数据库中的密码。它使用 salt 和逐渐增加的计算成本来防止彩虹表攻击。
  2. Argon2PasswordEncoder：适用于需要更高安全性的场景。它使用内存硬化来防止定制硬件攻击。但是，由于 Argon2 算法相对较新，可能不适用于所有的环境。
  3. Pbkdf2PasswordEncoder：适用于需要与旧系统兼容的场景。它使用 salt 和可配置的迭代次数来提高安全性。
  4. SCryptPasswordEncoder：适用于需要更高安全性的场景。它使用内存硬化来防止定制硬件攻击。
  5. NoOpPasswordEncoder：不推荐使用，因为它不提供任何安全性。
  6. 其他：其他编码器均已经废弃，因此不推荐使用。

综上，我们在一般的业务场景中，使用 BCryptPasswordEncoder 即可。
