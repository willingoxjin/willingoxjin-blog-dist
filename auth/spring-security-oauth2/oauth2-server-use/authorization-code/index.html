# 一、授权服务器介绍

Spring Authorization Server 是一个框架，提供 OAuth 2.1 和 OpenID Connect 1.0 规范和其他相关规范的实现，它基于 Spring Security，提供了一组易于使用的 API 和默认实现，以便开发人员可以快速创建和管理 OAuth2 授权服务器，也能方便的进行扩展。

如果你使用的是 Spring Boot、Spring Security 等 Spring 系列的框架，那么，它能快速的集成到你的系统中。

官网相关介绍见：Spring Authorization Server (opens new window)

# 二、自定义授权服务器—授权码模式示例

# 1.自定义授权服务器

新建 maven 项目，名为 auth-server

在 pom.xml 中添加相关依赖如下：

<dependency>

  <artifactId>spring-boot-starter-security</artifactId>

<!-- spring security oauth2 授权服务 -->

  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-oauth2-authorization-server</artifactId>

server:
  port: 21000

logging:
  level:
    org.springframework.security: trace

---
# spring security user config
spring:
  security:
    user:
      name: willingoxjin
      password: 123456
      roles: admin

---
# spring security oath2 authorization server
spring:
  security:
    oauth2:
      authorizationserver:
        client:
          mall:
            registration:
              client-name: mall
              client-id: mall-client-id
              client-secret: mall-client-secret
              client-authentication-methods:
                - client_secret_basic
                - client_secret_post
              authorization-grant-types:
                - authorization_code
                - refresh_token
                - client_credentials
              redirect-uris:
              	# 注意，由于这里是本地测试，授权服务器和客户端均使用localhost，
              	# 会导致sessionId在另外的服务中获取session时，由于为空导致sessionId会被刷新
              	# 因此需要使用ip或域名访问
                - http://oauth2-client.local:22000/login/oauth2/code/mall
              scopes:
                - openid
                - profile
                - phone
            require-authorization-consent: true

OAuth2 授权服务器配置

@EnableWebSecurity(debug = true)
@Configuration
public class OAuth2ServerConfig {

    /**
     * 授权服务器配置
     * 授权服务器的过滤器链的执行顺序级别需要时最优先的
     */
    @Bean
    @Order(Ordered.HIGHEST_PRECEDENCE) // 优先级需要高于默认的
    public SecurityFilterChain authorizationServerSecurityFilterChain(HttpSecurity http) throws Exception {
        OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http);
        http.getConfigurer(OAuth2AuthorizationServerConfigurer.class)
                // Enable OpenID Connect 1.0
                .oidc(withDefaults());

        http.exceptionHandling(exceptions -> exceptions
              		.defaultAuthenticationEntryPointFor(
                      new LoginUrlAuthenticationEntryPoint("/login"),
                      new MediaTypeRequestMatcher(MediaType.TEXT_HTML)
                  )
              )
              .oauth2ResourceServer(resourceServer -> resourceServer
              		.jwt(withDefaults())
       			 	);
        return http.build();
    }

}

授权服务器的登录配置：

@Configuration
public class WebSecurityConfig {

    @Bean
    @Order(2)
    public SecurityFilterChain defaultSecurityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                    .anyRequest().authenticated()
            )
            .formLogin(withDefaults());

        return http.build();
    }

}

用于测试的接口：

@RestController
public class IndexController {
  
    @Autowired
    private OAuth2AuthorizedClientRepository authorizedClientRepository;

    @GetMapping("/client")
    public String client() {
        return "Hello, client!";
    }

    @RequestMapping("/index")
    public String index(HttpServletRequest request) {
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
        OAuth2AuthorizedClient authorizedClient =
                authorizedClientRepository.loadAuthorizedClient("mall", authentication, request);
        OAuth2AccessToken accessToken = authorizedClient.getAccessToken();
        System.out.println(accessToken);
        return "index";
    }

    @RequestMapping("/index2")
    public String index2(@RegisteredOAuth2AuthorizedClient("mall") OAuth2AuthorizedClient authorizedClient) {
        OAuth2AccessToken accessToken = authorizedClient.getAccessToken();
        System.out.println(accessToken);
        return "index2";
    }

}

我们可以使用如下方式获取 OAuth2AuthorizedClient 实例：

  1. 使用 OAuth2AuthorizedClientRepository @Bena 实例的 loadAuthorizedClient 方法；
  2. 使用 @RegisteredOAuth2AuthorizedClient 注解自动注入。

OAuth2AuthorizedClient 是已经授权成功的客户端信息。

OAuth2AuthorizedClientRepository 中存储了已经授权的客户端（即 OAuth2AuthorizedClient）。

OAuth2AccessToken 是授权服务器办法的令牌，可用来请求资源服务器的数据。

# 4.测试

先启动授权服务，再启动客户端的服务。

由于客户端在启动时，会请求授权服务器的 issuer-uri 接口，所以我们需要先启动授权服务器，然后启动服务端。否则客户端在启动的时候会报错。

不过，我们也可以不配置 issuer-uri，直接指定具体的 provider 配置。

启动完成后，我们根据如下步骤测试，如下步骤也是 OAuth2 授权码模式的流程：

  1. 在浏览器中先打开开发者工具，并开启保留日志的选项（Preseve log），然后访问客户端的接口：http://oauth2-client.local:22000/client

在开发者工具中，可以看到请求被进行重定向了多次：

备注—重定向到 /login 页面较慢时的问题：

由于 Spring Security 中会引用 https://maxcdn.bootstrapcdn.com/bootstrap/4.0.0-beta/css/bootstrap.min.css，该地址需要开启外网代理才能访问，如果你没有开启代理，最后，所以会存在慢的情况。

最后请求被重定向到了 ``/login` 页面，如下：

这个页面就是授权服务器提供的登录页面。然后我们输入账号密码：willingoxjin/123456。

登录成功后，会进行如下重定向：

最终，会重定向到提示资源所有者授权的页面，如下：

我们勾选 profile 选项，然后点击 Submit Consent 按钮。同意授权后，经过请求的重定向，最终会回到一开始请求的 /client 接口。

最后，整个授权流程完成。我们也可以成功访问客户端的接口了，如下：

需要注意的是，当访问客户端接口或资源，会先将请求重定向到客户端的登录页。

但是，我们的示例代码中，只配置了一种 OAuth2 客户端，所以系统会默认使用 mall 客户端的 OAuth2 登录，即直接被重定向到 http://oauth2-client.local:22000/oauth2/authorization/mall 。

  • A：用户请求客户端服务的 /client 接口
  • B：客户端检测到用户未登录，将请求重定向到客户端服务的登录页
  • C：用户选择 OAuth2 授权登录，点击按钮，跳转到客户端服务的 /oauth2/authorization/mall
  • D：客户端将请求重定向到授权服务器端的 /oauth2/authorize（该请求包含参数：response_type=code、client_id、scope、state、redirect_uri、nonce）
  • E：授权服务器检测到用户未登录，将用户重定向到授权服务器的登录页面
  • F：用户进行登录操作
  • G：授权服务器检测到用户未对客户端进行授权，将请求重定向到授权服务器的授权页面
  • H：用户进行授权操作
  • I：授权服务器将请求重定向到一开始（D步骤）的请求的 /oauth2/authorize
  • J：授权服务器校检用户登录状态、授权状态、客户端信息，携带 code 等参数，将重定向到 redirect_uri （即/login/oauth2/code/mall）
  • K：客户端完成授权，向授权服务成功获取 token 后，返回用户一开始访问的资源（即 /client 接口）
