# 一、客户端配置与存储

# 1.OAuth2ClientProperties

OAuth2ClientProperties 用于读取 application.yml 中的关于 Spring Security OAuth2 中，客户端相关的配置。

OAuth2ClientProperties 的源码如下：

public class OAuth2ClientProperties implements InitializingBean {

  /**
	 * OAuth provider details.
	 */
	private final Map<String, Provider> provider = new HashMap<>();

	/**
	 * OAuth client registrations.
	 */
	private final Map<String, Registration> registration = new HashMap<>();

  // other methods ......
  
}

这两个属性对应的类如下：

  • Provider 用于配置 OAuth2 提供者的属性

# 1.1 Provider

Provider 中的属性都是 OAuth2 授权服务提供者配置。OAuth2 提供者的配置 是指实现 OAuth2 协议的授权服务器。

其相关属性如下：

（1）issuerUri：OpenID Connect 发现端点 URI。

issuerUri 默认的地址如下：

  • /.well-known/oauth-authorization-server 用于获取 OAuth2 授权服务器的配置信息。
    • 它的作用是提供一种标准化的方式，让客户端能够动态地获取授权服务器的配置信息，而无需硬编码在客户端代码中。
    • 我们也可以直接在浏览器中访问该地址，如
  • /.well-known/openid-configuration 用于获取 OpenID Connect 认证服务器的配置信息。

如果你配置了 issuerUri ，客户端会自动获取授权服务器的授权地址、获取令牌地址、用户信息地址等等信息。

（2）authorizationUri：授权服务器的授权端点 URI。

该端点用于重定向用户到进行身份验证和授权的认证服务器。用户将在该页面上提供其凭据，并授权客户端访问其受保护的资源。

（3）tokenUri：授权服务器的令牌端点 URI。

该端点用于向认证服务器请求访问令牌。客户端将向此URI发送身份验证凭据，并获取包含访问令牌的响应

（4）userInfoUri：授权服务器的用户信息的端点 URI。

该端点用于向认证服务器请求用户的详细信息。客户端可以使用访问令牌向此URI发送请求，以获取关于用户的额外信息，如姓名、电子邮件地址等

（5）userInfoAuthenticationMethod：用户信息认证的方法。

  • 该属性用于指定客户端向 userInfoUri 请求用户信息时使用的认证方法。
  • 默认情况下，该值为空，表示不进行认证。常见的认证方法包括header 和 query 方法。

（6）userNameAttribute：用户名属性

该属性用于指定从用户信息响应中提取用户名的属性名称。当客户端请求用户信息时，认证服务器将返回包含用户详细信息的响应。通过指定 userNameAttribute，客户端可以从响应中提取用户名，并在需要时使用该用户名

（7）jwkSetUri：获取授权服务器 JSON Web Key Set（JWKS）的公钥 URI。

该端点用于获取授权服务器的 JSON Web Key 的集合，公钥用于验证和解析访问令牌的签名。通过提供 jwkSetUri，客户端可以获取用于验证令牌的公钥，以确保令牌的完整性和安全性。

# 1.2 Registration

OAuth2 客户端配置

（1）provider：提供商

该属性用于指定OAuth2提供商的名称或标识符。它可以是预定义的提供商（如Google、Facebook等），也可以是自定义的提供商

（2）clientId：客户端 ID

该属性是客户端的唯一标识符，用于向认证服务器标识自己。在注册客户端时，认证服务器会为每个客户端分配一个唯一的 clientId。可以理解为客户端相对于授权服务器的唯一用户名

（3）clientSecret：客户端秘钥

该属性用于与认证服务器进行安全通信。只有客户端和认证服务器知道该秘钥，用于验证客户端的身份。可以理解为客户端相对于授权服务器的唯一用户名对应的密码。

（4）clientAuthenticationMethod：客户端认证方

该属性用于指定客户端向认证服务器进行身份验证的方法。

常见的可选值有：

  • client_secret_basic：使用 HTTP Basic 认证，在认证请求中，将 clientId 和 clientSecret 通过 : 号拼接，并通过 Base64 编码得到一个字符串。在请求时授权服务器时，将此编码字符串放到 Authorization 请求头中；
  • client_secret_post：将 clientId 和 clientSecret 放到认证请求的请求体（表单数据）中；
  • client_secret_jwt：
    • 客户端通过 HMAC 算法（对称加密），使用客户端秘钥（clientSecret）签名一个 JWT，并将该 JWT 发送作为认证凭据放到认证请求中；
    • 授权服务器也通过 clientSecret 校检 JWT 的签名；
  • private_key_jwt：
    • 客户端通过 RSA 算法（非对称加密），使用私钥签名一个 JWT，并将该 JWT 作为认证凭据放到认证请求中。
    • 授权服务器则通过公钥（公钥是客户端提供的）校检 JWT 的签名；
  • none：不提供客户端认证。当客户端是公共客户端时认证服务器不会对客户端进行验证，PKCE 流程要求客户端为公共客户端。

（5）authorizationGrantType：授权类型

该属性用于指定客户端使用的授权类型。常见的可选值有：

  • authorization_code 授权码模式
  • refresh_token 刷新 token 模式
  • client_credentials 客户端模式
  • password 密码模式（该模式已废弃）
  • urn:ietf:params:oauth:grant-type:jwt-beare JWT承载授权模式。这是一种基于 JWT 的授权类型。在这种模式下，客户端使用自己的私钥对 JWT 进行签名，并将其作为请求参数发送到认证服务器。认证服务器通过验证 JWT 的签名来验证客户端的身份，并颁发访问令牌。
  • urn:ietf:params:oauth:grant-type:device_code 设备授权模式

（6）redirectUri：重定向地址

该属性用于指定认证服务器在授权成功后将用户重定向到客户端的地址。客户端需要在注册时提供有效的重定向地址，并在授权请求中使用相应的重定向 URI。

（7）scope：授权范围

该属性用于指定客户端请求的访问范围。范围定义了客户端可以访问的资源类型或权限。

（8）clientName：客户端名称

该属性用于指定客户端的名称或标识符。它可以是用于显示给用户的友好名称。

# 2.ClientRegistration

ClientRegistration 是用于存储”OAuth2客户端的注册信息“的实体类，这里的存储指的是数据结构容器，并非在内存中保存。

ClientRegistration 类的常用属性包括：

  • registrationId
  • clientId
  • clientSecret
  • clientAuthenticationMethod
  • authorizationGrantType
  • redirectUri
  • scope
  • providerDetails
  • clientName

可以看到，这些属性和上面的 OAuth2ClientProperties 中 Provider 和 Registration 的属性重复。这里要注意区分这两个类的作用：

  • OAuth2ClientProperties
    • 用于读取 Spring Boot 的 application.yml 文件中，”OAuth2 客户端注册信息“的配置。
    • 并且，在 Spring 初始化时，会将 OAuth2ClientProperties 转为 ClientRegistration 类型的对象，具体转换的代码见 OAuth2ClientPropertiesMapper 类的源码。
  • ClientRegistration 用于存储”OAuth2 客户端注册信息“的实体类，在读取、修改、保存客户端注册信息时，需要使用到该类。

在 spring-boot-starter-oauth2-client 中，内置了 CommonOAuth2Provider 类，该类为 google、github、facebook、okta 等服务商，提供了相应的 ClientRegistration 的默认配置。

# 3.2 InMemoryClientRegistrationRepository

ClientRegistrationRepository 有一个基于内存的实现：InMemoryClientRegistrationRepository。

InMemoryClientRegistrationRepository 使用 ConcurrentHashMap 存储配置的所有的 ClientRegistration 。

Spring Boot Security 默认使用该实现作为客户端存储，在 OAuth2ClientRegistrationRepositoryConfiguration 配置类的 clientRegistrationRepository 方法中，将 OAuth2ClientProperties 中的 客户端配置转换为 ClientRegistration，并实例化一个 InMemoryClientRegistrationRepository。源码如下：

@Bean
@ConditionalOnMissingBean(ClientRegistrationRepository.class)
InMemoryClientRegistrationRepository clientRegistrationRepository(OAuth2ClientProperties properties) {
    List<ClientRegistration> registrations = new ArrayList<>(
        new OAuth2ClientPropertiesMapper(properties).asClientRegistrations().values());
    return new InMemoryClientRegistrationRepository(registrations);
}

# 二、已授权客户端的存储

# 1.OAuth2AuthorizedClient

OAuth2AuthorizedClient 用于存储已经授权的客户端信息的容器，其内部包括属性及对应的类如下：

  • ClientRegistration clientRegistration 已授权的客户端信息；
  • String principalName 授权客户端的主体名称（即用户的唯一标识符）；
  • OAuth2AccessToken accessToken 已授权的 token；
  • OAuth2RefreshToken refreshToken 已授权的 refresh token；

# 2.OAuth2AuthorizedClient 的存储

# 2.1 OAuth2AuthorizedClientService

该接口作用如下：

  • 该接口在应用程序级别管理（查询、持久化存储、删除） 授权客户端（OAuth2AuthorizedClient）。

  • 这里存储的客户端，将会访问令牌凭据关联到客户端和资源所有者（最初授予授权的主体）。

该类有如下实现：

  1. InMemoryOAuth2AuthorizedClientService 该实现将已授权的客户端存储在内存中，使用 ConcurrentHashMap 作为存储容器；
  2. JdbcOAuth2AuthorizedClientService 该实现使用 spriing-data-jdbc 的 JdbcOperations 持久化存储已授权的客户端。
  • 如果想要使用该依赖，需要添加 spring-boot-starter-data-jdbc 的依赖。

OAuth2AuthorizedClientService 的实现将通过 ClientRegistrationRepository 查询客户端信息。这里需要注意区分 ClientRegistrationRepository 和我们下面介绍的 OAuth2AuthorizedClientRepository，一个是用于管理所有配置的客户端、另外一个是用于管理已经授权的客户端。

# 2.2 OAuth2AuthorizedClientRepository

该接口负责授权客户端在请求之间的持久化管理（查询、持久化存储、删除）。

该接口有如下实现：

  • HttpSessionOAuth2AuthorizedClientRepository 该实现将 OAuth2AuthorizedClient 存储在 HttpSession 中。
    • 该类为 OAuth2AuthorizedClientRepository 的默认实现。
  • AuthenticatedPrincipalOAuth2AuthorizedClientRepository
    • 如果当前主体经过身份验证，则委托 OAuth2AuthorizedClientService 执行具体的查询、存储、删除等操作。
    • 如果当前请求未经身份验证（或匿名），则委托 HttpSessionOAuth2AuthorizedClientRepository 执行具体的操作。

# 3.OAuth2AuthorizedClientManager

OAuth2AuthorizedClientManager 是一个函数式接口，其只有一个方法，如下：

@Nullable
OAuth2AuthorizedClient authorize(OAuth2AuthorizeRequest authorizeRequest);
1
2

这个方法的作用是在用户进行身份验证后，为客户端应用程序获取有效的访问令牌，以便后续的资源访问请求可以使用该令牌进行授权。

其作用为：

  1. 通过利用 OAuth2AuthorizedClientProvider（s） 授权（或重新授权）OAuth 2.0 客户端。
  2. 委派 OAuth2AuthorizedClient 的持久性，通常使用 OAuth2AuthorizedClientService 或 OAuth2AuthorizedClientRepository。

其实现如下：

  • DefaultOAuth2AuthorizedClientManager：默认实现，用于在 HttpServletRequest 的上下文中使用。
  • AuthorizedClientServiceOAuth2AuthorizedClientManager：能够在 HttpServletRequest 的上下文之外运行，例如在调度后台线程和服务层中。

# 4.OAuth2AuthorizedClientProvider

OAuth2AuthorizedClientProvider 用于具体进行授权逻辑，如：

  • 在授权码模式中，重定向服务器的授权页面，以获取用户的授权。
  • 在客户端模式中，将根据客户端配置，发送 HTTP 请求，以获取客户端授权。

OAuth2AuthorizedClientProvider 的实现类如下：

  • DelegatingOAuth2AuthorizedClientProvider：这是一个委托类，用于委托其他的 OAuth2AuthorizedClientProvider 执行。

  • AuthorizationCodeOAuth2AuthorizedClientProvider：授权码模式的授权请求实现。

    • 如果客户端配置中，不支持授权码模式或授权成功，会返回 null。
    • 如果未授权，将抛出 ClientAuthorizationRequiredException 异常，由 OAuth2AuthorizationRequestRedirectFilter 过滤器重定向请求到指定的授权页面。

  • 下面 OAuth2AuthorizedClientProvider 的实现，都会根据 OAuth2 协议，向授权服务器发送请求，以获取授权，得到 accessToken ，最后组装 OAuth2AuthorizedClient 并返回。

    • ClientCredentialsOAuth2AuthorizedClientProvider：客户端模式的授权请求实现。

    • JwtBearerOAuth2AuthorizedClientProvider：jwt 授权模式的授权请求实现。

    • PasswordOAuth2AuthorizedClientProvider（废弃）：密码模式的授权请求实现。

    • RefreshTokenOAuth2AuthorizedClientProvider：刷新 token 模式的授权请求实现。

你可以将 Spring Security OAuth2 Client 中的接口与 Spring Security 中的类进行类比，由于他们的设计风格一致，所以很多接口在架构的功能与命名都是类似的，以下的接口对应的功能是类似的。

  • OAuth2AuthorizationCodeGrantFilter——Spring Security 中的 UsernamePasswordAuthenticationFilter。
  • OAuth2AuthorizedClientManager —— Spring Security 中的 AuthenticationManager 。
  • OAuth2AuthorizedClientProvider —— Spring Security 中的 AuthenticationProvider 。
  • ……

# 1.OAuth2AuthorizationRequestRedirectFilter

OAuth2 Client 用于认证的过滤器如下：

OAuth2AuthorizationRequestRedirectFilter 过滤器，看名字大概就可以猜到，这个过滤器是用来将 OAuth2 授权请求中重定向的。

我们上面也提到了该过滤器，在 AuthorizationCodeOAuth2AuthorizedClientProvider 中，如果是授权码模式。

public final class AuthorizationCodeOAuth2AuthorizedClientProvider implements OAuth2AuthorizedClientProvider {

	@Override
	@Nullable
	public OAuth2AuthorizedClient authorize(OAuth2AuthorizationContext context) {
      Assert.notNull(context, "context cannot be null");
      if (AuthorizationGrantType.AUTHORIZATION_CODE.equals(
          	context.getClientRegistration().getAuthorizationGrantType()) && context.getAuthorizedClient() == null) {
          // ClientAuthorizationRequiredException is caught by
          // OAuth2AuthorizationRequestRedirectFilter which initiates authorization
          throw new ClientAuthorizationRequiredException(context.getClientRegistration().getRegistrationId());
      }
			return null;
	}

OAuth2AuthorizationRequestRedirectFilter 过滤器的源码如下：

public class OAuth2AuthorizationRequestRedirectFilter extends OncePerRequestFilter {
  
  	// 默认处理接口
  	public static final String DEFAULT_AUTHORIZATION_REQUEST_BASE_URI = "/oauth2/authorization";
  
  	// ...
  
		@Override
		protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain)
					throws ServletException, IOException {
        try {
          	// 提取请求中的OAuth2相关的参数，转换为OAuth2AuthorizationRequest对象
          	OAuth2AuthorizationRequest authorizationRequest = this.authorizationRequestResolver.resolve(request);
            // 若为OAuth2授权请求，则直接重定向
          	if (authorizationRequest != null) {
                this.sendRedirectForAuthorization(request, response, authorizationRequest);
                return;
            }
        }
        catch (Exception ex) {
            this.unsuccessfulRedirectForAuthorization(request, response, ex);
            return;
        }
      	// 执行其他过滤器
        try {
         	 filterChain.doFilter(request, response);
        }
        catch (IOException ex) {
          	throw ex;
        }
        catch (Exception ex) {
            // 检查是否需要处理ClientAuthorizationRequiredException异常
          	// 获取第一个类型为ClientAuthorizationRequiredException的异常
            Throwable[] causeChain = this.throwableAnalyzer.determineCauseChain(ex);
            ClientAuthorizationRequiredException authzEx = (ClientAuthorizationRequiredException) this.throwableAnalyzer
                	.getFirstThrowableOfType(ClientAuthorizationRequiredException.class, causeChain);
            
          	// 若获取到ClientAuthorizationRequiredException异常，则提取异常中的clientRegistrationId
          	// 构造OAuth2AuthorizationRequest，进行重定向操作
          	if (authzEx != null) {
              try {
                  OAuth2AuthorizationRequest authorizationRequest = this.authorizationRequestResolver.resolve(request,
                      	authzEx.getClientRegistrationId());
                  if (authorizationRequest == null) {
                    	throw authzEx;
                  }
                  this.requestCache.saveRequest(request, response);
                  this.sendRedirectForAuthorization(request, response, authorizationRequest);
              }
              catch (Exception failed) {
                	this.unsuccessfulRedirectForAuthorization(request, response, failed);
              }
              return;
          }
          // 其他异常直接转换一下后抛出
          if (ex instanceof ServletException) {
            	throw (ServletException) ex;
          }
          if (ex instanceof RuntimeException) {
            	throw (RuntimeException) ex;
          }
          throw new RuntimeException(ex);
        }
		}
  
}

# 2.OAuth2LoginAuthenticationFilter

OAuth2LoginAuthenticationFilter 主要用于处理基于 OAuth2 的登录认证流程，它通常用于处理用户登录请求，获取用户的授权并获取用户信息。它适用于将 OAuth2 作为身份提供者的情况，例如使用第三方身份提供者（如 Google、Facebook）进行登录

默认情况下，该过滤器会匹配 URI 为 /login/oauth2/code/{clientRegistrationId} 的请求，当然我们也可以自定义配置匹配的 URI。

其部分源码如下：

public class OAuth2LoginAuthenticationFilter extends AbstractAuthenticationProcessingFilter {

    @Override
    public Authentication attemptAuthentication(HttpServletRequest request, HttpServletResponse response)
        throws AuthenticationException {
      	// 将请求的参数转为MultiValueMap
        MultiValueMap<String, String> params = OAuth2AuthorizationResponseUtils.toMultiMap(request.getParameterMap());
      	// 若没有授权回调的参数，则抛异常
        if (!OAuth2AuthorizationResponseUtils.isAuthorizationResponse(params)) {
          OAuth2Error oauth2Error = new OAuth2Error(OAuth2ErrorCodes.INVALID_REQUEST);
          throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString());
        }
     		// 从缓存中删除并返回与提供的 HttpServletRequest 和 HttpServletResponse 关联的 OAuth2AuthorizationRequest，
      	// 如果不可用，则返回 null。
        OAuth2AuthorizationRequest authorizationRequest = this.authorizationRequestRepository
            	.removeAuthorizationRequest(request, response);
      	// 为null(不可用)时，抛出异常
        if (authorizationRequest == null) {
            OAuth2Error oauth2Error = new OAuth2Error(AUTHORIZATION_REQUEST_NOT_FOUND_ERROR_CODE);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString());
        }
      	// 获取请求路径中的registrationId参数
        String registrationId = authorizationRequest.getAttribute(OAuth2ParameterNames.REGISTRATION_ID);
      	// 根据registrationId查询对应的客户端注册信息
        ClientRegistration clientRegistration = this.clientRegistrationRepository.findByRegistrationId(registrationId);
        // 客户端注册信息不存在，则抛出异常
      	if (clientRegistration == null) {
            OAuth2Error oauth2Error = new OAuth2Error(CLIENT_REGISTRATION_NOT_FOUND_ERROR_CODE,
                	"Client Registration not found with Id: " + registrationId, null);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString());
        }
      	// 获取请求参数中的redirectUri
        // @formatter:off
        String redirectUri = UriComponentsBuilder.fromHttpUrl(UrlUtils.buildFullRequestUrl(request))
              .replaceQuery(null)
              .build()
              .toUriString();
        // @formatter:on
      	// 组装OAuth2LoginAuthenticationToken对象
        OAuth2AuthorizationResponse authorizationResponse = OAuth2AuthorizationResponseUtils.convert(params,
            	redirectUri);
        Object authenticationDetails = this.authenticationDetailsSource.buildDetails(request);
        OAuth2LoginAuthenticationToken authenticationRequest = new OAuth2LoginAuthenticationToken(clientRegistration,
            	new OAuth2AuthorizationExchange(authorizationRequest, authorizationResponse));
        authenticationRequest.setDetails(authenticationDetails); // 设置Authentication的detail信息
      	// 调用AuthenticationManager校检，获取到已认证的OAuth2LoginAuthenticationToken对象
        OAuth2LoginAuthenticationToken authenticationResult = (OAuth2LoginAuthenticationToken) this
            	.getAuthenticationManager().authenticate(authenticationRequest);
      	// 将OAuth2LoginAuthenticationToken对象转换为OAuth2AuthenticationToken
        OAuth2AuthenticationToken oauth2Authentication = this.authenticationResultConverter
            	.convert(authenticationResult);
        Assert.notNull(oauth2Authentication, "authentication result cannot be null");
        oauth2Authentication.setDetails(authenticationDetails); // 设置Authentication的detail信息

      	// 保存OAuth2AuthorizedClient到OAuth2AuthorizedClientRepository中，持久化存储
        OAuth2AuthorizedClient authorizedClient = new OAuth2AuthorizedClient(
              authenticationResult.getClientRegistration(), oauth2Authentication.getName(),
              authenticationResult.getAccessToken(), authenticationResult.getRefreshToken());

        this.authorizedClientRepository.saveAuthorizedClient(authorizedClient, oauth2Authentication, request, response);
        return oauth2Authentication;
    }

}

# 3.OAuth2AuthorizationCodeGrantFilter

OAuth2AuthorizationCodeGrantFilter 过滤器，用于处理 OAuth2 中授权码模式的授权回调请求。其内部的处理逻辑和上面的 OAuth2LoginAuthenticationFilter 过滤器很相似。

其部分源码如下：

public class OAuth2AuthorizationCodeGrantFilter extends OncePerRequestFilter {
 
    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain)
        	throws ServletException, IOException {
      	// 匹配请求中的参数，若为回调的授权的，则处理
        if (matchesAuthorizationResponse(request)) {
            processAuthorizationResponse(request, response);
            return;
        }
      	filterChain.doFilter(request, response);
    }

  
    private boolean matchesAuthorizationResponse(HttpServletRequest request) {
      	// 将请求参数转为MultiValueMap
        MultiValueMap<String, String> params = OAuth2AuthorizationResponseUtils.toMultiMap(request.getParameterMap());
        if (!OAuth2AuthorizationResponseUtils.isAuthorizationResponse(params)) {
          	return false;
        }
      	// 查询缓存的请求，若为查询到，直接返回false，代表未匹配成功
        OAuth2AuthorizationRequest authorizationRequest = this.authorizationRequestRepository
            	.loadAuthorizationRequest(request);
        if (authorizationRequest == null) {
          	return false;
        }
      	// 获取回调请求中的redirect_uri参数，和之前缓存请求的redirect_uri
        UriComponents requestUri = UriComponentsBuilder.fromUriString(UrlUtils.buildFullRequestUrl(request)).build();
        UriComponents redirectUri = UriComponentsBuilder.fromUriString(authorizationRequest.getRedirectUri()).build();
        Set<Map.Entry<String, List<String>>> requestUriParameters = new LinkedHashSet<>(
            	requestUri.getQueryParams().entrySet());
        Set<Map.Entry<String, List<String>>> redirectUriParameters = new LinkedHashSet<>(
            	redirectUri.getQueryParams().entrySet());
        // 从授权响应(请求)中删除额外的请求参数(如果有的话)
        requestUriParameters.retainAll(redirectUriParameters);
      	// 将两个redirect_uri的参数(如有)进行精确比较，若全部一致，代表匹配成功
        if (Objects.equals(requestUri.getScheme(), redirectUri.getScheme())
              && Objects.equals(requestUri.getUserInfo(), redirectUri.getUserInfo())
              && Objects.equals(requestUri.getHost(), redirectUri.getHost())
              && Objects.equals(requestUri.getPort(), redirectUri.getPort())
              && Objects.equals(requestUri.getPath(), redirectUri.getPath())
              && Objects.equals(requestUriParameters.toString(), redirectUriParameters.toString())) {
          	return true;
        }
        return false;
    }

    private void processAuthorizationResponse(HttpServletRequest request, HttpServletResponse response)
        throws IOException {
      	// 从缓存中删除及获取之前的请求
        OAuth2AuthorizationRequest authorizationRequest = this.authorizationRequestRepository
            	.removeAuthorizationRequest(request, response);
      	// 从回调中获取registrationId参数，并查询对应的客户端注册信息
        String registrationId = authorizationRequest.getAttribute(OAuth2ParameterNames.REGISTRATION_ID);
        ClientRegistration clientRegistration = this.clientRegistrationRepository.findByRegistrationId(registrationId);
      	// 将请求参数转为MultiValueMap
        MultiValueMap<String, String> params = OAuth2AuthorizationResponseUtils.toMultiMap(request.getParameterMap());
        // 获取redirect_uri完整的地址
      	String redirectUri = UrlUtils.buildFullRequestUrl(request);
      	// 构建未认证的OAuth2AuthorizationCodeAuthenticationToken
        OAuth2AuthorizationResponse authorizationResponse = OAuth2AuthorizationResponseUtils.convert(params,
            	redirectUri);
        OAuth2AuthorizationCodeAuthenticationToken authenticationRequest = new OAuth2AuthorizationCodeAuthenticationToken(
            	clientRegistration, new OAuth2AuthorizationExchange(authorizationRequest, authorizationResponse));
        authenticationRequest.setDetails(this.authenticationDetailsSource.buildDetails(request));
        OAuth2AuthorizationCodeAuthenticationToken authenticationResult;
        try {
          	// 认证处理，认证成功后，可获取到已认证的OAuth2AuthorizationCodeAuthenticationToken
            authenticationResult = (OAuth2AuthorizationCodeAuthenticationToken) this.authenticationManager
                .authenticate(authenticationRequest);
        }
        catch (OAuth2AuthorizationException ex) {
          	// OAuth2授权异常时，重定向到redirect_uri，并携带具体的错误参数
            OAuth2Error error = ex.getError();
            UriComponentsBuilder uriBuilder = UriComponentsBuilder.fromUriString(authorizationRequest.getRedirectUri())
                	.queryParam(OAuth2ParameterNames.ERROR, error.getErrorCode());
            if (!StringUtils.isEmpty(error.getDescription())) {
              	uriBuilder.queryParam(OAuth2ParameterNames.ERROR_DESCRIPTION, error.getDescription());
            }
            if (!StringUtils.isEmpty(error.getUri())) {
              	uriBuilder.queryParam(OAuth2ParameterNames.ERROR_URI, error.getUri());
            }
            this.redirectStrategy.sendRedirect(request, response, uriBuilder.build().encode().toString());
            return;
        }
  
      	// 保存OAuth2AuthorizedClient到OAuth2AuthorizedClientRepository中，持久化存储
        Authentication currentAuthentication = this.securityContextHolderStrategy.getContext().getAuthentication();
        String principalName = (currentAuthentication != null) ? currentAuthentication.getName() : "anonymousUser";
        OAuth2AuthorizedClient authorizedClient = new OAuth2AuthorizedClient(
              authenticationResult.getClientRegistration(), principalName, authenticationResult.getAccessToken(),
              authenticationResult.getRefreshToken());
        this.authorizedClientRepository.saveAuthorizedClient(authorizedClient, currentAuthentication, request,
            	response);
        String redirectUrl = authorizationRequest.getRedirectUri();
      	// 清除请求缓存
        SavedRequest savedRequest = this.requestCache.getRequest(request, response);
        if (savedRequest != null) {
            redirectUrl = savedRequest.getRedirectUrl();
            this.requestCache.removeRequest(request, response);
        }
      	// OAuth2授权成功，重定向到redirect_uri
        this.redirectStrategy.sendRedirect(request, response, redirectUrl);
    }
  
}

我们发现，既然 OAuth2LoginAuthenticationFilter 过滤器用于 OAuth2 登录，为什么还要单独定义一个 OAuth2AuthorizationCodeGrantFilter 来处理授权码模式呢？

经过使用搜索引擎，我发现两者区别如下：

  • OAuth2LoginAuthenticationFilter：适用于 OpenId Connect 协议的授权服务器
  • OAuth2AuthorizationCodeGrantFilter：适用于普通的 OAuth2 协议的授权服务器

在 HttpSecurity 中的 oauth2Login() 和 oauth2Client() 方法的注释中也可以发现两者的区别：

  • oauth2Login()：Configures authentication support using an OAuth 2.0 and/or OpenID Connect 1.0 Provider.
  • oauth2Client()：Configures OAuth 2.0 Client support.

OAuth2LoginAuthenticationFilter 与 OAuth2AuthorizationCodeGrantFilter 的区别参考自：https://stackoverflow.com/questions/54900696/oauth2authorizationcodegrantfilter-vs-oauth2loginauthenticationfilter-in-spring
