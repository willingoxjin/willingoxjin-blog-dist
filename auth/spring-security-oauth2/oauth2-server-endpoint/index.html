07.Spring Security OAuth2 授权服务器的 Endpoint

官方文档：Protocol Endpoints: OAuth2 and OpenID Connect 1.0 protocol endpoint implementations (opens new window)

# 一、授权服务器的 endpoint

我们知道，授权服务器需要对客户端或用户提供授权、获取 token、注销等 API 接口，而 Spring Security OAuth2 Authorization Server 中也基于 OAuth2 协议和 OpenID Connect 1.0 协议提供了对应的 endpoint。

我们知道，在 Spring Boot 中，可以使用 spring-web 的 @RestController + @RequestMapping 相关注解来定义接口。也可以使用 spring-boot-actuator 中的 @Endpoint 来定义端点。

不过，Spring Security OAuth2 Authorization Server 并没有这么做，而是通过过滤器来实现的 endpoint。

为什么要使用过滤器呢？我个人觉得，首先，过滤器拦截的 URI 可以通过配置动态的修改。其次，使用过滤器处理请求的方式与 Spring Security 编码风格一致。

# 二、OAuth2 授权相关的 endpoint

# 1.OAuth2 授权码获取

OAuth2AuthorizationEndpointConfigurer 类用于配置 OAuth2AuthorizationEndpointFilter 对象。我们可以使用该配置类自定义 OAuth2 授权端点的配置，如 endpoint 的 URI、用户同意授权的页面、授权成功的处理、等等。

其对应的配置类为 OAuth2AuthorizationEndpointConfigurer。

OAuth2AuthorizationEndpointFilter：用于处理 OAuth2 授权请求（和同意）的 endpoint。其相关的属性的类如下：

public final class OAuth2AuthorizationEndpointFilter extends OncePerRequestFilter {
    /**
     * 默认的授权endpoint URI路径
     */
    private static final String DEFAULT_AUTHORIZATION_ENDPOINT_URI = "/oauth2/authorize";
    /**
     * 用于处理认证凭证的AuthenticationManager
     */
    private final AuthenticationManager authenticationManager;
    /**
     * 用于匹配授权请求
     * 默认情况下，会拦截如下请求：
     * - 授权请求：
     * ---- `GET /oauth2/authorize`;
     * ---- `POST /oauth2/authorize?response_type=xxx&scope=openid` (response_type不能为空，scope参数必须为openid)
     * - 同意授权请求：
     * ---- `POST /oauth2/authorize` (没有response_type参数)
     */
    private final RequestMatcher authorizationEndpointMatcher;
    /**
     * 用于重定向当前请求。当客户端身份验证成功后，将会重定向该请求到用户同意授权页面
     */
    private final RedirectStrategy redirectStrategy = new DefaultRedirectStrategy();
    /**
     * 用于从请求中读取相关的信息
     */
    private AuthenticationDetailsSource<HttpServletRequest, ?> authenticationDetailsSource = new WebAuthenticationDetailsSource();
    /**
     * 用于将请求中携带的参数转换为Authentication对象
     * 默认为OAuth2AuthorizationCodeRequestAuthenticationConverter和OAuth2AuthorizationConsentAuthenticationConverter
     */
    private AuthenticationConverter authenticationConverter;
    /**
     * 授权成功的处理器：当授权成功后，将会重定向到redirect_uri
     *（将携带code、state（若存在）参数）
     */
    private AuthenticationSuccessHandler authenticationSuccessHandler = this::sendAuthorizationResponse;
    /**
     * 用于从请求中读取相关的信息：当授权失败后，将会重定向到redirect_uri
     *（若存在state、error、error_uri、error_description等参数，则重定向地址将会携带这些参数）
     */
    private AuthenticationFailureHandler authenticationFailureHandler = this::sendErrorResponse;
    /**
     * session授权策略，默认为空函数
     */
    private SessionAuthenticationStrategy sessionAuthenticationStrategy = (authentication, request, response) -> {};
   	/**
     * 用户同意授权的页面，若没有设置，spring security oauth2 srever则会使用内置的授权页面
     * 默认生成授权页面的类见DefaultConsentPage
     */
  	private String consentPage;
  
  	// other methods ...

}

OAuth2AuthorizationEndpointFilter 中有两条主线。

第一条主线，用于处理客户端重定向用户 OAuth2 授权请求，请求地址示例： GET /oauth2/authorize?response_type=code&client_id=xxx&client_secret=xxx&redirect_uri=xxx&scope=xxx&state=xxx&... 。

代码逻辑如下：

  1. 请求转换：OAuth2AuthorizationCodeRequestAuthenticationConverter：该类主要用于尝试从 HttpServletRequest 中提取 OAuth2 授权请求的参数，并转换为 OAuth2AuthorizationCodeRequestAuthenticationToken 对象。
  2. 认证处理：：OAuth2AuthorizationCodeRequestAuthenticationToken 对应的 Authentication 类型为 OAuth2AuthorizationCodeRequestAuthenticationProvider 。其用于验证授权码请求是否合法，如校检客户端信息、请求参数是否合法等等，不过对于是否登录、是否授权，会进行不同的处理，如下：
    • 未登录：返回未认证的 OAuth2AuthorizationCodeRequestAuthenticationToken 对象，会直接返回参数传过来的未授信凭证。
    • 已登录：
      • 资源所有者未授权：返回已认证的 OAuth2AuthorizationConsentAuthenticationToken 对象。
      • 资源所有者已授权：返回已认证的 OAuth2AuthorizationCodeRequestAuthenticationToken 对象。
    • 相关校检的方法：
      • 校检资源所有者是否登录：#isPrincipalAuthenticated
      • 校检资源所有者是否授权：#requireAuthorizationConsent
  3. 资源所有者（用户）登录校检：
    • 若用户未登录，则会继续执行过滤器，直到抛出未认证异常，由 AuthenticationEntryPoint 捕捉到后，将请求重定向到授权服务器的登录页。并且在登录成功后，请求将重定向一开始请求的地址，再次由 OAuth2AuthorizationEndpointFilter 处理；
    • 若用户登录成功，继续执行后面的逻辑；
  4. 资源所有者（用户）授权校检：若上面认证中得到的是 OAuth2AuthorizationCodeRequestAuthenticationToken 类型的对象，就代表需要资源所有者授权。
    • 若用户从未授权过，或授权已过期：将请求重定向到用户授权页面，提示用户是否进行授权。
    • 若用户之前已经同意授权：不处理。
  5. 授权请求结果处理：
    • 授权成功：根据请求参数中的 redirect_uri，携带 code、state 等参数拼接 URI，并重定向。
    • 授权失败：若整个流程出现异常，将会捕捉异常，并根据请求参数中的 redirect_uri，携带 error（错误信息）、error_description、state 等参数拼接 URI，并重定向。

第二条主线，用户在浏览器授权页面同意授权，发送请求：POST /oauth2/authorize?response_type=code&client_id=xxx&client_secret=xxx&redirect_uri=xxx&scope=xxx&state=xxx&...。

代码逻辑如下：

  1. 请求转换：OAuth2AuthorizationConsentAuthenticationConverter：该类主要用于尝试从 HttpServletRequest 中提取 OAuth2 授权请求的参数，并转换为 OAuth2AuthorizationConsentAuthenticationToken 对象。

  2. 认证处理：OAuth2AuthorizationConsentAuthenticationToken 对应的 Authentication 类型为 OAuth2AuthorizationConsentAuthenticationProvider 用于验证”用户的同意授权“的处理。

    • 若认证成功：返回已认证的 OAuth2AuthorizationCodeRequestAuthenticationToken 对象。
    • 若认证失败，将会抛出 OAuth2AuthorizationCodeRequestAuthenticationException 异常

# 三、OAuth2 Token 相关的 Endpoint

# 1.OAuth2 Token 的获取

OAuth2TokenEndpointFilter 用于客户端向授权服务器获取 OAuth2 Token 请求的处理。

相关类：

  • 配置类：OAuth2TokenEndpointConfigurer

  • 授权码模式：

    • OAuth2AuthorizationCodeAuthenticationConverter
    • OAuth2AuthorizationCodeAuthenticationToken
    • OAuth2AuthorizationCodeAuthenticationProvider

  • 刷新token模式：

    • OAuth2RefreshTokenAuthenticationConverter
    • OAuth2RefreshTokenAuthenticationToken
    • OAuth2RefreshTokenAuthenticationProvider

  • 客户端模式：

    • OAuth2ClientCredentialsAuthenticationConverter
    • OAuth2ClientCredentialsAuthenticationToken
    • OAuth2ClientCredentialsAuthenticationProvider

  • 设备码授权模式：

    • OAuth2DeviceCodeAuthenticationConverter
    • OAuth2DeviceCodeAuthenticationConverter
    • OAuth2DeviceCodeAuthenticationProvider

注：上面所有的的 XxxAuthenticationProvider 对应的 Authentication 类型为：OAuth2AccessTokenAuthenticationToken。

# 2.OAuth2 Token 内省

OAuth2TokenIntrospectionEndpointFilter 用于验证和解析 OAuth 2.0 令牌的有效性，即处理令牌内省，它会返回令牌的元数据信息，如令牌的有效期、授权范围、用户身份等。

相关类：

  • OAuth2TokenIntrospectionEndpointConfigurer
  • OAuth2TokenIntrospectionAuthenticationConverter
  • OAuth2TokenIntrospectionAuthenticationToken
  • OAuth2TokenIntrospectionAuthenticationProvider

# 3.OAuth2 Token 撤回

OAuth2TokenRevocationEndpointFilter 用于处理 OAuth 2.0 令牌撤销请求，其能够撤销已颁发的访问令牌或已颁发的刷新令牌。

当用户注销或撤销授权，或令牌被泄露时，客户端可能需要撤销已颁发的令牌，以使其失效。

相关类：

  • OAuth2TokenRevocationEndpointConfigurer
  • OAuth2TokenRevocationAuthenticationConverter
  • OAuth2TokenRevocationAuthenticationToken
  • OAuth2TokenRevocationAuthenticationProvider

# 四、OAuth2 授权服务器信息的 endpoint

OAuth2AuthorizationServerMetadataEndpointFilter 用于处理 OAuth 2.0 认证服务器元数据（Metadata）请求。它返回提供授权服务器的元数据信息。

在 OAuth 2.0 授权流程中，客户端可能需要获取认证服务器的元数据信息，以了解认证服务器的功能和配置。元数据信息包括认证服务器的授权端点、令牌端点、支持的授权方式、支持的范围等。

授权服务器通过向客户端提供认证服务器的元数据信息，使客户端能够了解授权服务器的功能和配置。这有助于客户端正确配置和使用 OAuth 2.0 认证流程，并与授权服务器进行正确的交互。
