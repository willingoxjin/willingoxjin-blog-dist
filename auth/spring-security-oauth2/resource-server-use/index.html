10.Spring Security OAuth2 资源服务器的使用

# 一、授权服务器中的配置

# 1.开启资源服务器配置

# 1.1 客户端服务

在整个 OAuth2 的授权码登录或其他授权登录流程中，当客户端获取到 OAuth2AccessToken 之后，客户端会向用户信息端点请求用户信息数据。

默认没有配置用户信息端点的情况下，会使用授权服务器的 /userinfo 端点获取用户信息。

在客户端服务中，我们可以通过以下方式自定义访问用户信息的配置：

# 用户信息接口地址
spring.security.oauth2.client.provider.[client-name].user-info-uri=http://oauth2-server.local:21000/userinfo
# 用户信息接口请求的方法：get、post、......
spring.security.oauth2.client.provider.[client-name].user-info-authentication-method=get
# 用户信息接口返回json内容中，用户名的key
spring.security.oauth2.client.provider.[client-name].user-name-attribute=sub

# 1.2 授权服务器

可以通过以下配置，在授权服务器中启用 OpenID Connect 1.0 ，以及启用资源服务器：

@Bean
@Order(Ordered.HIGHEST_PRECEDENCE) // 优先级需要高于默认的
public SecurityFilterChain authorizationServerSecurityFilterChain(HttpSecurity http) throws Exception {
    OAuth2AuthorizationServerConfiguration.applyDefaultSecurity(http);

    http.getConfigurer(OAuth2AuthorizationServerConfigurer.class)
          // Enable OpenID Connect 1.0
          .oidc(withDefaults()
    );

    // 资源服务器配置
    http.oauth2ResourceServer(resourceServer ->
          resourceServer.jwt(withDefaults())
    );
  
  	// ...

    return http.build();
}

# 2.BearerTokenAuthenticationFilter

当我们在授权服务器启用资源服务器配置后，过滤器 BearerTokenAuthenticationFilter 就会被自动注册到 Spring Security 的过滤器链中。

BearerTokenAuthenticationFilter 是用来解析、校检 Bearer 的过滤器，其是资源服务器 spring-security-oauth2-resource-server 提供的过滤器。

  1. 如果我们的请求头中包含 Authorization 属性，且前缀为 Bearer 或 bearer，则会在被 JwtAuthenticationProvider 校检 Token 的有效性。
  2. JwtAuthenticationProvider 会根据我们在 spring.security.oauth2.resourceserver.jwt 中 JWT 相关配置，根据规则校检 Token 是否有效。
  3. 最后，如果 Token 校检通过，则可以被允许访问资源。

BearerTokenAuthenticationFilter 相关类：

  • BearerTokenAuthenticationToken
  • JwtAuthenticationProvider

# 3.OidcUserInfoEndpointFilter

当我们在授权服务器启用 OpenID Connect 1.0 后，过滤器 OidcUserInfoEndpointFilter 就会自动注册到 Spring Security 的过滤器链中。

OidcUserInfoEndpointFilter 是授权服务器提供的端点，用于获取 OAuth2 用户的信息，该端点最终会返回 OidcUserInfo 对象。

关于 OpenID Connect 1.0 的用户信息见 https://openid.net/specs/openid-connect-core-1_0.html#UserInfo (opens new window)

我在请求测试 http://oauth2-server.local:21000/userinfo 接口（即用户信息端点），返回的内容如下：

{"sub":"willingoxjin"}

Spring Security OAuth2 资源服务器的配置如下：

@EnableWebSecurity(debug = true)
@Configuration(proxyBeanMethods = false)
public class ResourceServerConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
                .securityMatcher("/**")
                .authorizeHttpRequests(authorizeRequests -> authorizeRequests
                        .anyRequest().authenticated()
                )
                .oauth2ResourceServer(resourceServer -> resourceServer
                        .jwt(withDefaults())
                );
        return http.build();
    }

}

# 2.客户端配置

客户端配置授权码模式，具体代码略。

客户端用于测试的接口：

@RestController
@RequestMapping("/resource")
public class ResourceSvrTestController {

    private final RestTemplate restTemplate = new RestTemplate();

    @GetMapping("/userinfo")
    public String userinfo(@RegisteredOAuth2AuthorizedClient("mall") OAuth2AuthorizedClient authorizedClient)
            throws URISyntaxException {
        String token = authorizedClient.getAccessToken().getTokenValue();
        HttpHeaders headers = new HttpHeaders();
        headers.setBearerAuth(token);
        URI uri = new URI("http://resource01.local:23000/api/resource01/userinfo");
        RequestEntity<Object> requestEntity = new RequestEntity<>(headers, HttpMethod.GET, uri);
        try {
            ResponseEntity<String> response = restTemplate.exchange(requestEntity, String.class);
            return response.getBody();
        } catch (RestClientException e) {
            return e.getMessage() != null ? e.getMessage() : "Unknown error";
        }
    }

}

# 三、自定义 Token 解析

在 JwtAuthenticationProvider 的 jwtAuthenticationConverter 属性用于解析 JWT 内容。其默认为 JwtGrantedAuthoritiesConverter，因此我们只需要自定义 JwtGrantedAuthoritiesConverter 即可实现解析 Token的目的。

下面我们就来解析 jwt 中 claims 存储的 authorities 属性，并将属性对应的权限内容，设置到当前用户的去耐心拿范围中。

# 1.授权服务器配置

我们在授权服务器自定义 Token 的学习中知道，可以通过定义 OAuth2TokenCustomizer @Bean 来自定义返回的 OAuth2AccessToken 内容。

这里咱直接把之前的代码贴上来：

@Bean
public OAuth2TokenCustomizer<JwtEncodingContext> oauth2TokenCustomizer() {
    return (JwtEncodingContext context) -> {
        if (context.getPrincipal().getPrincipal() instanceof UserDetails user) {
            // 权限并转为字符串
            Collection<? extends GrantedAuthority> authorities = user.getAuthorities();
            Set<String> authoritySet = Optional.ofNullable(authorities)
                    .orElse(Collections.emptyList())
                    .stream()
                    .map(GrantedAuthority::getAuthority)
                    .collect(Collectors.toSet());
            // 获取申请的scopes
            authoritySet.addAll(context.getAuthorizedScopes());

            JwtClaimsSet.Builder claims = context.getClaims();
            // 将权限信息放入jwt的claims中（也可以生成一个以指定字符分割的字符串放入）
            claims.claim("authorities", authoritySet);

            // 其它自定内容...
        }
    };
}

  1. 定义 JwtAuthenticationConverter @Bean：
@Bean
public JwtAuthenticationConverter jwtAuthenticationConverter() {
    JwtGrantedAuthoritiesConverter grantedAuthoritiesConverter = new JwtGrantedAuthoritiesConverter();
    // 设置解析权限信息的前缀，设置为空是去掉前缀
    grantedAuthoritiesConverter.setAuthorityPrefix("");
    // 设置权限信息在claims中的key
    grantedAuthoritiesConverter.setAuthoritiesClaimName("authorities");

    JwtAuthenticationConverter jwtAuthenticationConverter = new JwtAuthenticationConverter();
    jwtAuthenticationConverter.setJwtGrantedAuthoritiesConverter(grantedAuthoritiesConverter);
    return jwtAuthenticationConverter;
}
