08.Spring Security OAuth2 授权服务器自定义 Token

# 一、OAuth2Token 相关 API

# 1.OAuth2TokenContext

OAuth2TokenContext 用于保存 OAuth2Token 到请求上下文中。

OAuth2TokenContext 提供了如下方法：

  1. getRegisteredClient()：获取授权的客户端。
  2. getPrincipal()：获取资源所有者（或客户端）的 Authentication 实例。
  3. getAuthorizationServerContext()：获取授权服务器运行环境的 AuthorizationServerContext 对象。
  4. getAuthorization()：获取授权相关的 OAuth2Authorization。
  5. getAuthorizedScopes()：获取客户端的授权范围。
  6. getTokenType()：获取 OAuth2TokenType。支持的枚举有 code、access_token、refresh_token 和 id_token。
  7. getAuthorizationGrantType()：获取与授权类型 AuthorizationGrantType。
  8. getAuthorizationGrant()：获取处理授权授予的 AuthenticationProvider 使用的 Authentication 实例。

# 2.OAuth2TokenGenerator

OAuth2TokenGenerator 根据从 OAuth2TokenContext 获取到的配置信息，生成 OAuth2Token 实例。

不过，生成 OAuth2Token 实例类型，由 OAuth2TokenContext 中指定的 OAuth2TokenType 决定。如下：

  • code：生成 OAuth2AuthorizationCode 实例。
  • access_token：生成 OAuth2AccessToken 实例。
  • refresh_token：生成 OAuth2RefreshToken 实例。
  • id_token：生成 OidcIdToken 实例。
  • user_code 生成 OAuth2UserCode 实例
  • device_code 生成 OAuth2DeviceCode 实例

其次，生成 OAuth2AccessToken 的格式由 TokenSettings#getAccessTokenFormat() 方法的返回值决定：

  • OAuth2TokenFormat.SELF_CONTAINED：生成 Jwt 令牌
  • OAuth2TokenFormat.REFERENCE：生成 Opaque （不透明）令牌

我们可以在 application.yml 中的 spring.security.oauth2.authorizationserver.client.<client-name>.token.access-token-format 配置，默认为 self-contained。

最后，如果生成的 OAuth2Token ，设置了 claims，并且其实现了 ClaimAccessor 接口，则可以从OAuth2Authorization.Token.getClaims() (opens new window) 访问 claims

OAuth2TokenGenerator 主要被使用在实现授权处理的组件中，例如，authorization_code、client_credentials和refresh_token。

OAuth2TokenGenerator 的默认实现如下：

  • OAuth2AccessTokenGenerator：生成一个“不透明”（OAuth2TokenFormat.REFERENCE）访问令牌
  • OAuth2RefreshTokenGenerator：
  • JwtGenerator：生成一个Jwt（OAuth2TokenFormat.SELF_CONTAINED）
  • DelegatingOAuth2TokenGenerator：OAuth2TokenGenerator 的代理类
  • OAuth2AuthorizationCodeGenerator：用于授权码生成
  • 设备码授权相关：
    • OAuth2UserCodeGenerator：用于用户码生成
    • OAuth2DeviceCodeGenerator：用于设备码生成

在 OAuth2ConfigurerUtils#getTokenGenerator 中进行了 OAuth2TokenGenerator 的配置：

  • 默认情况下，会使用 OAuth2AccessTokenGenerator 和 OAuth2RefreshTokenGenerator 组成的 DelegatingOAuth2TokenGenerator。
  • 如果注册了JwtEncoder @Bean 或 JWKSource<SecurityContext> @Bean，那么，默认在DelegatingOAuth2TokenGenerator 中，会另外增加 JwtGenerator。

可以通过如下方式配置 OAuth2TokenGenerator：

  1. @Bean 的方式
@Bean
public OAuth2TokenGenerator<?> tokenGenerator() {
    JwtEncoder jwtEncoder = ...
    JwtGenerator jwtGenerator = new JwtGenerator(jwtEncoder);
    OAuth2AccessTokenGenerator accessTokenGenerator = new OAuth2AccessTokenGenerator();
    OAuth2RefreshTokenGenerator refreshTokenGenerator = new OAuth2RefreshTokenGenerator();
    return new DelegatingOAuth2TokenGenerator(
        jwtGenerator, accessTokenGenerator, refreshTokenGenerator);
}

# 3.OAuth2TokenCustomizer

使用 OAuth2TokenCustomizer ，可以自定义 OAuth2Token 的内容。

如下是使用 OAuth2TokenClaimsContext 作为上下文的配置示例：

@Bean
public OAuth2TokenGenerator<?> tokenGenerator() {
    JwtEncoder jwtEncoder = ...
    JwtGenerator jwtGenerator = new JwtGenerator(jwtEncoder);
    OAuth2AccessTokenGenerator accessTokenGenerator = new OAuth2AccessTokenGenerator();
    accessTokenGenerator.setAccessTokenCustomizer(accessTokenCustomizer());
    OAuth2RefreshTokenGenerator refreshTokenGenerator = new OAuth2RefreshTokenGenerator();
    return new DelegatingOAuth2TokenGenerator(
        	jwtGenerator, accessTokenGenerator, refreshTokenGenerator);
}

@Bean
public OAuth2TokenCustomizer<OAuth2TokenClaimsContext> accessTokenCustomizer() {
    return context -> {
      OAuth2TokenClaimsSet.Builder claims = context.getClaims();
      // Customize claims

    };
}

@Bean
public OAuth2TokenGenerator<?> tokenGenerator() {
    JwtEncoder jwtEncoder = ...

    jwtGenerator.setJwtCustomizer(jwtCustomizer());
    OAuth2AccessTokenGenerator accessTokenGenerator = new OAuth2AccessTokenGenerator();
    OAuth2RefreshTokenGenerator refreshTokenGenerator = new OAuth2RefreshTokenGenerator();

				jwtGenerator, accessTokenGenerator, refreshTokenGenerator);

public OAuth2TokenCustomizer<JwtEncodingContext> jwtCustomizer() {

        JwsHeader.Builder headers = context.getJwsHeader();
        JwtClaimsSet.Builder claims = context.getClaims();

          // Customize headers/claims for access_token

        } else if (context.getTokenType().getValue().equals(OidcParameterNames.ID_TOKEN)) {
          // Customize headers/claims for id_token

        }
    };

# 二、自定义授权服务器返回内容

# 1.授权服务器配置

如果我们想自定义 Token 的返回内容，只需要直接自定义一个 OAuth2TokenCustomizer @Bean 即可。如下：

@Bean
public OAuth2TokenCustomizer<JwtEncodingContext> oauth2TokenCustomizer() {
    return (JwtEncodingContext context) -> {
        if (context.getPrincipal().getPrincipal() instanceof UserDetails user) {
            // 权限并转为字符串
            Collection<? extends GrantedAuthority> authorities = user.getAuthorities();
            Set<String> authoritySet = Optional.ofNullable(authorities)
                    .orElse(Collections.emptyList())
                    .stream()
                    .map(GrantedAuthority::getAuthority)
                    .collect(Collectors.toSet());
            // 获取申请的scopes
            authoritySet.addAll(context.getAuthorizedScopes());

            JwtClaimsSet.Builder claims = context.getClaims();
            // 将权限信息放入jwt的claims中（也可以生成一个以指定字符分割的字符串放入）
            claims.claim("authorities", authoritySet);

            // 其它自定内容...
        }
    };
}
