# 1.1 issuer

issuer 属性用于指定授权服务器的唯一标识符，以便客户端可以验证令牌的来源和合法性。

issuer 必须是一个 http 的 URI 地址，如果不设置，默认为请求访问授权服务器时的请求 host。

我们之前在 Spring Security OAuth2 Client 的 spring.security.oauth2.client.provider.xxx.issuer-uri 的配置，对应的就是授权服务器的 issuer。

Spring Security OAuth2 Authorization Server 中会自动生成 issuer 的请求地址，逻辑如下：

  1. 在请求时，会先在 AuthorizationServerContextFilter 过滤器中，往 AuthorizationServerContextHolder 中设置 AuthorizationServerContext 类型的上下文内容。
    • AuthorizationServerContextHolder 中通过 ThreadLocal 存储一个 AuthorizationServerContext 类型的对象，
    • AuthorizationServerContext 用于保存授权服务器请求时环境的上下文。
  2. 当请求到达 OAuth2AuthorizationServerMetadataEndpointFilter 过滤器中，则会读取第一步中存储的 AuthorizationServerContext 信息，构建授权服务器的 metadata 信息，最后写入 JSON 格式的数据到 Response 中。

# 1.2 client

client 属性是一个 Map。key 为客户端的 registrationId；value 为客户端对应的配置内容，其类型为 Client。

Client 中的属性如下：

  • registration：客户端注册信息。其中包括 clentId，clientSecret、允许的授权方法、允许的授权类型、重定向地址、授权范围等等。
  • requireProofKey：在授权码授权流程中，是否要求客户端在授权码流程中使用 Proof Key for Code Exchange (PKCE)。
  • requireAuthorizationConsent ：是否要求用户在授权流程中显式同意授权请求。
  • jwkSetUri ：JSON Web Key (JWK) 集合的 URI。
  • tokenEndpointAuthenticationSigningAlgorithm ：对授权服务器的令牌端点进行身份验证的签名算法。
  • token：已授权客户端的令牌设置。其中包括授权码的失效时间、access token 的失效时间、access token 的格式、设备码的失效时间、返回 refresh token 的失效时间等等。

# 1.3 endpoint

endpoint 属性对应的类型为 Endpoint，主要用于配置授权服务器对外提供的 endpoint（端点）。

Endpoint 类中有如下属性，以下属性均为授权服务器用于授权而暴露给其他服务的 URI 相关配置：

  • authorizationUri：授权请求 URI 地址。
  • deviceAuthorizationUri：设备授权请求 URI 地址。
  • deviceVerificationUri：设备验证请求 URI 地址。
  • tokenUri：获取 Token 请求的 URI 地址。
  • jwkSetUri：获取 JWK 公钥集合的 URI 地址。
  • tokenRevocationUri：令牌吊销 URI 地址。
  • tokenIntrospectionUri：令牌侦测 URI 地址。
  • oidc：OpenID Connect 1.0 相关的 endpoints。
    • logoutUri：OpenID Connect 1.0 的注销端点。
    • clientRegistrationUri：OpenID Connect 1.0 的客户端注册端点。
    • userInfoUri：OpenID Connect 1.0 的用户信息端点。

# 2.AuthorizationServerSettings

AuthorizationServerSettings 是用于授权服务器配置设置的工具，该类继承自 AbstractSettings。

AbstractSettings 类中维护一个 Map 集合，默认为 HashMap。AuthorizationServerSettings 中的属性配置都被存储在 Map 集合中。

你可以使用其对应的构建类 AbstractBuilder 创建 AbstractSettings 类型的对象。AbstractSettings 类还有两个其他子类，为：ClientSettings 和 TokenSettings 。

与 AuthorizationServerSettings 相关的配置见：

  • OAuth2AuthorizationServerPropertiesMapper 用于将 OAuth2AuthorizationServerProperties 映射为 OAuth2AuthorizationServerProperties 对象或 RegisteredClient 类型的集合。
  • OAuth2AuthorizationServerConfiguration 是 Spring Boot 中的自动配置类，该类用于初始化 RegisteredClientRepository 和 AuthorizationServerSettings 对象到 IOC 容器中。

# 4.RegisteredClientRepository

RegisteredClientRepository 用于管理（保存、查询）客户端的注册信息（即 RegisteredClient）。

RegisteredClientRepository 接口的默认实现如下：

  1. InMemoryRegisteredClientRepository 基于内存的实现。

    • 使用 ConcurrentHashMap 存储客户端的信息。

    • 其相关配置在 OAuth2AuthorizationServerConfiguration 的 registeredClientRepository() 方法中，该方法将 applucation.yml 中配置的客户端信息，存储到 InMemoryRegisteredClientRepository 实例对象中。

  2. JdbcRegisteredClientRepository 基于 JdbcOperations 的实现。

    • 该实现的数据表必须使用与 JdbcRegisteredClientRepository 类同级的 oauth2-registered-client-schema.sql 文件内的表结构。

    • 使用该实现，需要添加 spring-boot-starter-jdbc 的依赖，并手动实例化 JdbcRegisteredClientRepository ，将该对象配置到 Spring Security OAuth2 中。

我们可以自定义实现 RegisteredClientRepository 接口，这样就可以使用自己业务系统的 ORM 框架，也可以自定义存储客户端信息的表结构。如使用 Mybatis 或 Spring Data JPA 向 MySQL 数据库中存储、查询客户端信息。

RegisteredClientRepository 被使用在与 OAuth2 相关的 AuthenticationProvider 中，你可以在 IDEA 中使用 Find Usage 功能查看该类在哪里被引用了。如下为相关的引用类：

  • JwtClientAssertionAuthenticationProvider
  • ClientSecretAuthenticationProvider
  • OidcClientConfigurationAuthenticationProvider
  • …..

# 二、用户的授权存储

# 1.OAuth2Authorization

保存与资源所有者（即用户）授予客户端的授权相关的状态的数据容器对象

其相关属性如下：

  1. id 客户端唯一标识
  2. registeredClientId 客户端 id。
  3. principalName 授权主体的名称，通常是用户的标识。如用户名。
  4. authorizationGrantType 授权类型，例如授权码、密码等。
  5. authorizedScopes 允许客户端访问资源的授权范围。
  6. tokens 已经颁发给客户端的访问令牌。
    • tokens 属性的类型为 Map<Class<? extends OAuth2Token>, Token<?>>。key 为 OAuth2Token 的类型，value 为 Token 的信息（包括 Token 及其元数据）。
  7. attributes 授权信息的附加属性，可以包含一些自定义的授权属性。

# 2.OAuth2AuthorizationService

OAuth2AuthorizationService 是一个接口，其用于管理（包括保存、删除、查询等方法） 授权服务器已授权给客户端的信息，即保存 OAuth2Authorization 相关的信息。

其有两个默认实现：

  1. InMemoryOAuth2AuthorizationService：基于内存的实现。

该类通过两个 Map 存储 OAuth2 授权信息，用于存储信息的部分代码如下：

public final class InMemoryOAuth2AuthorizationService implements OAuth2AuthorizationService {
		private int maxInitializedAuthorizations = 100;
  
  	/**
  	 * 存储尚未授权的 OAuth2Authorization 对象。
  	 * 此时状态为 "initialized" (uncompleted)
     * 当用户在授权页面，且未点击同意授权时，OAuth2Authorization 对象则为此状态 。
  	 */
  	private Map<String, OAuth2Authorization> initializedAuthorizations =
				Collections.synchronizedMap(new MaxSizeHashMap<>(this.maxInitializedAuthorizations));

    /**
  	 * 存储已经授权的 OAuth2Authorization 对象。
  	 * 此时状态为 "completed"
  	 */
    private final Map<String, OAuth2Authorization> authorizations = new ConcurrentHashMap<>();

  	// ...
}

# 3.OAuth2AuthorizationConsent

保存用户对客户端允许访问数据授权范围的数据容器对象

OAuth2AuthorizationConsent 只有三个属性：

  1. registeredClientId：客户端 id。
  2. principalName：授权主体的名称，通常是用户的标识。如用户名。
  3. authorities：用户同意授权的范围（或权限）
    • 见 OAuth2UserAuthority、OidcUserAuthority

# 4.OAuth2AuthorizationConsentService

此接口的实现负责管理（保存、删除、查询） 资源所有者（即用户）已同意授权的信息（即 OAuth2AuthorizationConsentService）。

OAuth2AuthorizationConsentService 也有两个默认实现：

  1. InMemoryOAuth2AuthorizationConsentService：基于内存的实现
  2. JdbcOAuth2AuthorizationConsentService ：基于 JdbcOperations 的实现。
    • 该实现的数据表必须使用与 JdbcOAuth2AuthorizationConsentService 类同级的 oauth2-authorization-consent-schema.sql 文件内的表结构。
    • 使用该实现，需要添加 spring-boot-starter-jdbc 的依赖，并需要将 JdbcOAuth2AuthorizationService 对象配置到 Spring Security OAuth2 中。

# 三、用户授权处理

# 1.OAuth2TokenContext

OAuth2TokenContext是一个接口，用于在 OAuth 2.0 流程中管理令牌及相关信息的上下文。它提供了获取和设置当前令牌的方法，并可以在整个应用程序中传递和共享令牌信息。

其被 OAuth2TokenGenerator 和 OAuth2TokenCustomizer 使用

# 2.OAuth2Token

OAuth2Token 是表示 OAuth 2.0 令牌的接口。它是一个抽象的令牌表示，可以表示不同类型的令牌，如访问令牌、刷新令牌等。OAuth2Token 接口定义了三个通用的方法：获取令牌值、获取令牌发布时间、获取令牌过期时间。

其源码如下：

public interface OAuth2Token {
  	
  	String getTokenValue();

    @Nullable
    default Instant getIssuedAt() {
      return null;
    }

    @Nullable
    default Instant getExpiresAt() {
      return null;
    }

}

# 3.OAuth2TokenGenerator

OAuth2TokenGenerator 是一个接口，看名字大概就知道，其是 OAuth2 Token 的生成器，用于生成 OAuth 2.0 令牌（Token）。

该接口是一个函数式接口，其只有一个 generate 方法，用于将 OAuth2TokenContext 对象生成（或转换成） OAuth2Token 对象。

其相关实现如下：

  • DelegatingOAuth2TokenGenerator： OAuth2TokenGenerator 的委托类。
  • JwtGenerator：JWT 令牌生成器，对应 Jwt。
  • OAuth2AccessTokenGenerator：访问令牌（access token）生成器，对应 OAuth2AccessToken。
  • OAuth2AuthorizationCodeGenerator：授权码生成器，对应 OAuth2AuthorizationCode。
  • OAuth2DeviceCodeGenerator：设备码生成器，对应 OAuth2DeviceCode。
  • OAuth2RefreshTokenGenerator：刷新令牌（refresh token）生成器，对应 OAuth2RefreshToken。
  • OAuth2UserCodeGenerator：用户码生成器，对应 OAuth2UserCode。

# 4.授权相关的 AuthenticationProvider

相信大家对 AuthenticationProvider 接口并不陌生，其是 Spring Security 中定义的，用于身份验证处理。

OAuth2 相关的 AuthenticationProvider：

  • ClientSecretAuthenticationProvider 使用客户端密钥方法对客户端进行身份验证
  • JwtClientAssertionAuthenticationProvider 处理使用 JSON Web Token (JWT) 客户端断言对客户端进行身份验证
  • 授权码相关
    • OAuth2AuthorizationCodeRequestAuthenticationProvider 用于处理客户端的授权吗请求，最终会在此生成授权码
    • OAuth2AuthorizationConsentAuthenticationProvider 用于用户同意授权的处理
    • OAuth2AuthorizationCodeAuthenticationProvider 用于验证授权码，并生成 OAuth2Token
  • OAuth2ClientCredentialsAuthenticationProvider 使用客户端凭证授权类型对客户端进行身份验证
  • 设备码相关
    • OAuth2DeviceAuthorizationRequestAuthenticationProvider 负责对客户端的设备授权请求进行身份验证
    • OAuth2DeviceCodeAuthenticationProvider 处理使用设备代码授权类型对客户端进行身份验证
    • OAuth2DeviceAuthorizationConsentAuthenticationProvider 处理对客户端的设备授权同意进行身份验证
    • OAuth2DeviceVerificationAuthenticationProvider 用于对客户端的设备验证进行身份验证
  • OAuth2RefreshTokenAuthenticationProvider 处理对客户端的刷新令牌进行身份验证
  • OAuth2TokenIntrospectionAuthenticationProvider 负责对客户端的令牌内省请求进行身份验证
  • OAuth2TokenRevocationAuthenticationProvider 处理对客户端的令牌撤销请求进行身份验证
  • PublicClientAuthenticationProvider 用于公共客户端的身份验证

OpenID Connect (OIDC) 相关的 AuthenticationProvider：

  • OidcClientConfigurationAuthenticationProvider 用于对 OIDC 客户端配置进行身份验证
  • OidcClientRegistrationAuthenticationProvider 用于对 OIDC 客户端注册的身份验证
  • OidcLogoutAuthenticationProvider 用于 OIDC 客户端注销登录的身份验证
  • OidcUserInfoAuthenticationProvider 用于 OIDC 客户端的用户信息身份验证

# 5.OAuth2TokenCustomizer

OAuth2TokenCustomizer 是一个接口，用于自定义 OAuth 2.0 令牌的生成和配置。它提供了一些方法，如自定义令牌的有效期、添加自定义属性等。你可以实现 OAuth2TokenCustomizer 接口，并将其应用于令牌生成器，以根据需要对令牌进行自定义操作。

具体配置可见官方文档：OAuth2TokenCustomizer (opens new window)

# 四、过滤器

授权服务器中的过滤器如下：

  • OAuth2ClientAuthenticationFilter 用于处理 OAuth 客户端的请求，在其他过滤器前校检 OAuth 客户端的认证信息。
  • Endpoint 相关的过滤器
    • OAuth2AuthorizationEndpointFilter 用于处理 OAuth 2.0 客户端的授权请求。
    • OAuth2AuthorizationServerMetadataEndpointFilter 于处理 OAuth 2.0 授权服务器元数据请求。
    • OAuth2TokenEndpointFilter 令牌端点：客户端用于将授权授予交换为访问令牌，通常使用客户端身份验证。
    • ……

# 1.OAuth2ClientAuthenticationFilter

这里，我们先只看 OAuth2ClientAuthenticationFilter。由于端点（Endpoint）相关的过滤器有些多，因此，专门留出一篇文章来学习。

OAuth2ClientAuthenticationFilter 用于处理 OAuth 客户端的请求，在其他过滤器前校检 OAuth 客户端的认证信息。

该过滤的部分源码如下：

public final class OAuth2ClientAuthenticationFilter extends OncePerRequestFilter {
  
    private final AuthenticationManager authenticationManager;
    private final RequestMatcher requestMatcher;
  	// ...
    private final AuthenticationDetailsSource<HttpServletRequest, ?> authenticationDetailsSource =
        	new WebAuthenticationDetailsSource();
    private AuthenticationConverter authenticationConverter;
    private AuthenticationSuccessHandler authenticationSuccessHandler = this::onAuthenticationSuccess;
    private AuthenticationFailureHandler authenticationFailureHandler = this::onAuthenticationFailure;

    // ...
  
    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain)
        	throws ServletException, IOException {
      // 匹配请求
      if (!this.requestMatcher.matches(request)) {
          filterChain.doFilter(request, response);
          return;
      }

      try {
          Authentication authenticationRequest = this.authenticationConverter.convert(request);
          if (authenticationRequest instanceof AbstractAuthenticationToken) {
              ((AbstractAuthenticationToken) authenticationRequest).setDetails(
                  this.authenticationDetailsSource.buildDetails(request));
          }
          if (authenticationRequest != null) {
              // 校检客户端id是否为可见字符
              validateClientIdentifier(authenticationRequest);
              // 由AuthenticationManager校检请求参数（客户端请求认证）
              Authentication authenticationResult = this.authenticationManager.authenticate(authenticationRequest);
              // 认证成功的处理：设置SecurityContext上下文
              this.authenticationSuccessHandler.onAuthenticationSuccess(request, response, authenticationResult);
          }
          filterChain.doFilter(request, response);

        } catch (OAuth2AuthenticationException ex) {
            if (this.logger.isTraceEnabled()) {
              	this.logger.trace(LogMessage.format("Client authentication failed: %s", ex.getError()), ex);
            }
        		// 认证失败的处理：返回json格式的错误信息
            this.authenticationFailureHandler.onAuthenticationFailure(request, response, ex);
        }
    }
  
    // ...
  
}

# 1.1 匹配哪些请求？

在 OAuth2ClientAuthenticationConfigurer 中，定义了授权服务器 OAuth2 客户端的配置：

  • 在 init 方法中，初始化了 requestMatcher 。

  • 在 configure 方法中，配置了 OAuth2ClientAuthenticationFilter 过滤器。

部分源码如下：

public final class OAuth2ClientAuthenticationConfigurer extends AbstractOAuth2Configurer {
  
    private RequestMatcher requestMatcher;
    private final List<AuthenticationConverter> authenticationConverters = new ArrayList<>();
    private Consumer<List<AuthenticationConverter>> authenticationConvertersConsumer = (authenticationConverters) -> {};
    private final List<AuthenticationProvider> authenticationProviders = new ArrayList<>();
    private Consumer<List<AuthenticationProvider>> authenticationProvidersConsumer = (authenticationProviders) -> {};
  
  	// ...

    @Override
    void init(HttpSecurity httpSecurity) {
        AuthorizationServerSettings authorizationServerSettings = OAuth2ConfigurerUtils.getAuthorizationServerSettings(httpSecurity);
        this.requestMatcher = new OrRequestMatcher(
              new AntPathRequestMatcher(
                  authorizationServerSettings.getTokenEndpoint(),
                  HttpMethod.POST.name()),
              new AntPathRequestMatcher(
                  authorizationServerSettings.getTokenIntrospectionEndpoint(),
                  HttpMethod.POST.name()),
              new AntPathRequestMatcher(
                  authorizationServerSettings.getTokenRevocationEndpoint(),
                  HttpMethod.POST.name()),
              new AntPathRequestMatcher(
                  authorizationServerSettings.getDeviceAuthorizationEndpoint(),
                  HttpMethod.POST.name()));
      
      	// 处理AuthenticationProviders
        List<AuthenticationProvider> authenticationProviders = createDefaultAuthenticationProviders(httpSecurity);
        if (!this.authenticationProviders.isEmpty()) {
          authenticationProviders.addAll(0, this.authenticationProviders);
        }
        this.authenticationProvidersConsumer.accept(authenticationProviders);
        authenticationProviders.forEach(authenticationProvider ->
				httpSecurity.authenticationProvider(postProcess(authenticationProvider)));
    }

    @Override
    void configure(HttpSecurity httpSecurity) {
      	// 获取AuthenticationManager
        AuthenticationManager authenticationManager = httpSecurity.getSharedObject(AuthenticationManager.class);
      	// 初始化OAuth2ClientAuthenticationFilter过滤器
        OAuth2ClientAuthenticationFilter clientAuthenticationFilter = new OAuth2ClientAuthenticationFilter(
              authenticationManager, this.requestMatcher);
      	// 创建默认的AuthenticationConverter
        List<AuthenticationConverter> authenticationConverters = createDefaultAuthenticationConverters();
        // 合并自定义的AuthenticationConverter到同一个List
      	if (!this.authenticationConverters.isEmpty()) {
            authenticationConverters.addAll(0, this.authenticationConverters);
        }
      	// 处理AuthenticationConverter
        this.authenticationConvertersConsumer.accept(authenticationConverters);
      	// 设置AuthenticationConverter到过滤器中
        clientAuthenticationFilter.setAuthenticationConverter(
              new DelegatingAuthenticationConverter(authenticationConverters));
      	// 设置认证成功的处理器
        if (this.authenticationSuccessHandler != null) {
            clientAuthenticationFilter.setAuthenticationSuccessHandler(this.authenticationSuccessHandler);
        }
     		 // 设置认证失败的处理器
        if (this.errorResponseHandler != null) {
            clientAuthenticationFilter.setAuthenticationFailureHandler(this.errorResponseHandler);
        }
        httpSecurity.addFilterAfter(postProcess(clientAuthenticationFilter), AbstractPreAuthenticatedProcessingFilter.class);
    }
 
  	// ...
}

# 1.2 过滤器中的 AuthenticationProvider 有哪些？

在 OAuth2ClientAuthenticationConfigurer#createDefaultAuthenticationConverters 方法创建了默认的 AuthenticationConverter。源码如下：

private static List<AuthenticationConverter> createDefaultAuthenticationConverters() {
    List<AuthenticationConverter> authenticationConverters = new ArrayList<>();

    authenticationConverters.add(new JwtClientAssertionAuthenticationConverter());
    authenticationConverters.add(new ClientSecretBasicAuthenticationConverter());
    authenticationConverters.add(new ClientSecretPostAuthenticationConverter());
    authenticationConverters.add(new PublicClientAuthenticationConverter());

    return authenticationConverters;
}

在 OAuth2ClientAuthenticationConfigurer#createDefaultAuthenticationProviders 方法中创建了默认的 AuthenticationProvider。源码如下：

private static List<AuthenticationProvider> createDefaultAuthenticationProviders(HttpSecurity httpSecurity) {
		List<AuthenticationProvider> authenticationProviders = new ArrayList<>();

		RegisteredClientRepository registeredClientRepository = OAuth2ConfigurerUtils.getRegisteredClientRepository(httpSecurity);
		OAuth2AuthorizationService authorizationService = OAuth2ConfigurerUtils.getAuthorizationService(httpSecurity);

		JwtClientAssertionAuthenticationProvider jwtClientAssertionAuthenticationProvider =
					new JwtClientAssertionAuthenticationProvider(registeredClientRepository, authorizationService);
		authenticationProviders.add(jwtClientAssertionAuthenticationProvider);

		ClientSecretAuthenticationProvider clientSecretAuthenticationProvider =
					new ClientSecretAuthenticationProvider(registeredClientRepository, authorizationService);
		PasswordEncoder passwordEncoder = OAuth2ConfigurerUtils.getOptionalBean(httpSecurity, PasswordEncoder.class);
		if (passwordEncoder != null) {
				clientSecretAuthenticationProvider.setPasswordEncoder(passwordEncoder);
		}
		authenticationProviders.add(clientSecretAuthenticationProvider);

		PublicClientAuthenticationProvider publicClientAuthenticationProvider =
					new PublicClientAuthenticationProvider(registeredClientRepository, authorizationService);
		authenticationProviders.add(publicClientAuthenticationProvider);

		return authenticationProviders;
}

在上面的代码中可以看到，默认情况下，OAuth2ClientAuthenticationToken 对应的 AuthenticationProvider 有三个：

  • JwtClientAssertionAuthenticationProvider：仅处理 clientAuthenticationMethod 为 urn:ietf:params:oauth:client-assertion-type:jwt-bearer 的请求。
  • ClientSecretAuthenticationProvider：仅处理 clientAuthenticationMethod 为 client_secret_basic 或 client_secret_post 的请求。
  • PublicClientAuthenticationProvider：仅处理 clientAuthenticationMethod 为 none 的请求。

在上面的 AuthenticationProvider 中，会验证客户端请求的参数，如是否存在对应的客户端、 clientSecret 是否正确。若为授权码授权请求，则校检授权码。
