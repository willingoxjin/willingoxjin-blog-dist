02.Spring Security OAuth2 Client 的使用

参考文档：

  • Spring Security OAuth2 官方文档 (opens new window)

# 一、OAuth2 Client 介绍

Spring Security 提供了 spring-security-oauth2-client ，来对 OpenID Connect 1.0 (opens new window) 规范进行实现，见：Spring Security—OAuth2 Login (opens new window)。

并且，其还有对应的 Spring Boot Starter，即 spring-boot-starter-oauth2-client 。这样就可以方便在 Spring Boot 应用中集成 OAuth2 客户端功能。其简化了 OAuth2 客户端的配置和使用，提供了一套统一的 API 和自动化的功能，使得在 Spring Boot 应用中实现 OAuth2 认证变得更加容易。

# 二、OAuth2 Client 的使用

下面，我们使用 Github 和 Google 作为授权服务器，测试下整个 OAuth2 授权流程。

# 3.Google 作为授权服务器

使用 Google API 作为授权服务器，和 Github 的流程是一致的。

# 1.1 配置 授权的 App

首先，进入 https://console.cloud.google.com/apis/dashboard (opens new window) 。

在这里新建项目 willingoxjin-project 后，并选择该项目：

然后，选中左侧的“凭据”，点击“创建凭据—OAuth 客户端 ID”。如下：

此时，如果你没有创建“OAuth 同意屏幕”，则会让你先创建，我们输入应用名和邮箱创建即可。这里的 OAuth 同意屏幕 的意思是同意用户授权访问的意思。

在“创建 OAuth 客户端 ID”页面中，填入应用类型、名称、重定向的 URL。如下：

点击创建，即可看到对应的客户端 ID 和客户端密钥。
