02.Spring Security 快速使用

# 一、搭建 Spring Boot 父项目

这里，我们创建一个 创建 Spring Boot 项目，名为 spring-security-demo，并将 Spring Security 集成进来。

pom.xml 文件部分内容如下：

<groupId>org.willingoxjin</groupId>
<artifactId>spring-security-demo</artifactId>
<version>1.0-SNAPSHOT</version>
<packaging>pom</packaging>

<properties>
  <java.version>17</java.version>
  <maven.compiler.source>17</maven.compiler.source>
  <maven.compiler.target>17</maven.compiler.target>
  <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
  <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
</properties>

<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-parent</artifactId>
      <version>3.1.0</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>

# 三、配置基于内存的密码

# 1. 基于配置文件

如下，在 application.properties 中配置用户名和密码：

spring.security.user.name=willingoxjin
spring.security.user.password=123456
1
2

配置后，重启服务，输入我们配置的账号密码，访问成功，就说明我们的配置生效了。

如果我们这里配置了密码，控制台就不再打印密码了。

# 2.基于 Java 代码的配置

我们也可以使用 Java 代码配置，创建一个名为 SecurityConfig 的配置类，并且继承自 WebSecurityConfigurerAdapter 父类。代码如下：

@Configuration
public class SecurityConfig {

    /**
     * 不对密码进行密码
     * 注意：方式已过期，请勿在生产环境中使用
     */
    @Bean
    public PasswordEncoder passwordEncoder() {
        return NoOpPasswordEncoder.getInstance();
    }

    /**
     * 配置鉴权管理
     */
    @Bean
    public UserDetailsService userDetailsService()  {
        List<UserDetails> userList = new ArrayList<>();
        userList.add(User.builder().username("willingoxjin")
                     .password("123456")
                     .roles("admin")
                     .build()
        );
        return new InMemoryUserDetailsManager(userList);
    }

}

# 四、密码相关源码分析

当我们没有配置密码的时候，在 SecurityProperties$User 类中的 password 属性默认等于 UUID.randomUUID().toString() 方法生成的随机 UUID 字符串。

如下图为 SecurityProperties$User 的代码片段：

控制台上打印的密码日志，是在 UserDetailsServiceAutoConfiguration 类的 getOrDeducePassword() 方法中输出的：

在 SecurityProperties$User#setPassword() 被调用时（即我们在配置中设置了密码），则会将 passwordGenerated 属性设置为 false。而 UserDetailsServiceAutoConfiguration#getOrDeducePassword() 方法会根据 passwordGenerated 判断是否打印日志。所以，我们后面配置了密码后，控制台的日志就不会打印密码了。
