04.Spring Security 实现基于 JWT 的认证

# 一、JWT 的前置知识

# 1.有状态服务和无状态服务

前后端分离的模式，要求我们对用户会话状态要进行一个无状态处理。那什么是无状态？什么是有状态呢？

# 1.1 有状态的服务

有状态的服务指的是在服务运行期间会保留客户端的状态信息，以便后续的请求可以使用这些信息。这种服务需要在不同的请求之间保留上下文信息，以便将来的请求可以使用

在 Web 应用中，我们通常使用 Session 管理用户会话：

  • 浏览器第一次与服务器建立连接的时候，服务器会生成一个 sessionid 返回浏览器，浏览器把这个 sessionid 存储到 Cookie 当中，以后每次发起请求都会在请求头 cookie 中带上这个 sessionid 信息，服务器就可以根据 sessionid 作为索引获取到具体 session。
  • Session 是保存在服务端的，服务器通过 Session 辨别用户，进行权限认证等处理。

但是，这样做有一定的缺点：

  • 服务端保存 session，在线用户过多会导致服务器压力变大。
  • 前后端分离模式下，后端服务集群部署，由于 session 是存储在内存的，所以不能在后端服务集群中共享（当然也可以让同一个用户总是访问某一个机器）。

# 1.2 无状态的服务

前后端分离的模式下，后端只提供接口开发，并且接口采用 RESTful 规范 。RESTful 规范是一种无状态的架构风格。这意味着在 RESTful 架构中，客户端和服务器之间的通信是相互独立的，每个请求都必须包含所有必要的信息。服务器不会保留客户端的状态信息，每个请求都是相互独立的。

无状态服务登录的流程：

  1. 客户端：调用接口，将用户名和密码发送到服务端进行认证；
  2. 服务端：认证成功后，将用户不敏感的信息加密编码成一个 token，返回客户端；
  3. 客户端：后续的每次请求都会携带 token；
  4. 服务端：每次客户端请求时，服务端都会对 token 解密，判断 token 是否有效，并获取用户信息。

无状态的好处：

  • 客户端请求不依赖服务端的信息，多次请求不需要必须访问到同一台机器。
  • 服务端的集群和状态对客户端透明。
  • 服务端可以扩缩容，方便集群部署。
  • 服务端无需存储用户状态，可以减少存储压力。

所以有状态和无状态的区别就是：有状态的服务会在服务端保存 Session ，无状态的服务则不会。

# 2.JWT

# 1.1 什么是 JWT？

JWT（JSON Web Token）是一种用于身份验证和授权的开放标准（RFC 7519），是一种 JSON 风格的轻量级的授权和身份认证规范，可以实现无状态分布式的 Web 应用授权。JWT 可以携带一些不敏感的用户信息。这样服务器不用每次都去查询用户信息。开箱即用，方便服务器处理鉴权逻辑。

JWT 是一种轻量级的安全令牌，由三部分组成：

  • 头部（Header）：包含了令牌元数据的信息（JOSN 格式），如令牌类型和签名算法等。
  • 载荷（Payload）：包含了一些声明信息（JOSN 格式），如用户身份、权限等。
  • 签名（Signature）：用于验证令牌的完整性和真实性，确保令牌没有被篡改过。

# 2.2 JWT 的生成规则

JWT 的生成规则如下：

  • 头部（Header）：令牌的元数据信息 Base64 后的字符。
  • 载荷（Payload）：声明信息 Base64 后的字符。
  • 签名（Signature）：通过 Header 中配置的加密算法，对 Header、Payload 和秘钥加密后的结果。

最后，会吧这三部分的数据通过 . 来连接：<header>.<payload>.<signature>。

注意

JWT 中的秘钥（secret）是用来校检 token 是否被篡改的，因此，秘钥是非常重要的，千万不要泄露。

千万不要在 Payload 中存储敏感信息，因为这里的数据时 Base64 编码后的数据，并未加密。

我们可以在 https://jwt.io/ (opens new window) 中校检 JWT：

Header 中的参数解释：

  • alg（Algorithm）：指定 JWT 使用的加密算法，常见的有 HS256、HS384、HS512、RS256、RS384、RS512、ES256、ES384、ES512 等。其中，HS 表示 HMAC-SHA 算法，RS 表示 RSA 算法，ES 表示 ECDSA 算法。
  • typ（Type）：指定 JWT 的类型，通常为 "JWT"。
  • kid（Key ID）：指定用于签名或加密 JWT 的密钥的 ID。如果 JWT 使用多个密钥进行签名或加密，可以使用 kid 参数来指定使用哪个密钥。
  • jku（JWK Set URL）：指定包含公钥信息的 JWK Set 的 URL 地址。如果 JWT 使用公钥进行验证或解密，可以使用 jku 参数来指定公钥信息的来源。
  • jwk（JSON Web Key）：指定包含公钥信息的 JWK 对象。如果 JWT 使用公钥进行验证或解密，可以使用 jwk 参数来指定公钥信息。
  • x5u（X.509 Certificate URL）：指定包含公钥信息的 X.509 证书的 URL 地址。如果 JWT 使用公钥进行验证或解密，可以使用 x5u 参数来指定公钥信息的来源。
  • x5c（X.509 Certificate Chain）：指定包含公钥信息的 X.509 证书链。如果 JWT 使用公钥进行验证或解密，可以使用 x5c 参数来指定公钥信息。
  • x5t（X.509 Certificate SHA-1 Thumbprint）：指定包含公钥信息的 X.509 证书的 SHA-1 摘要。如果 JWT 使用公钥进行验证或解密，可以使用 x5t 参数来指定公钥信息的摘要。
  • x5t#S256（X.509 Certificate SHA-256 Thumbprint）：指定包含公钥信息的 X.509 证书的 SHA-256 摘要。如果 JWT 使用公钥进行验证或解密，可以使用 x5t#S256 参数来指定公钥信息的摘要。

Payload 中的参数解释：

  • iss（Issuer）：指定 JWT 的签发者。
  • sub（Subject）：指定 JWT 的主题，即 JWT 所代表的实体。
  • aud（Audience）：指定 JWT 的受众，即 JWT 所要传递给的一方或多方。
  • exp（Expiration Time）：指定 JWT 的过期时间，以 UNIX 时间戳表示。
  • nbf（Not Before）：指定 JWT 的生效时间，即在此时间之前 JWT 无效，以 UNIX 时间戳表示。
  • iat（Issued At）：指定 JWT 的签发时间，以 UNIX 时间戳表示。
  • jti（JWT ID）：指定 JWT 的唯一标识符，用于防止 JWT 被重复使用。

Payload 中的参数不是必须的，具体使用哪些参数取决于具体的使用场景和需求。在实际使用中，可以根据需要自定义 Payload 中的参数，并在验证 JWT 时根据这些参数进行验证。

# 2.3 JWT 的缺点

JWT 并不完美，其有以下缺点：

  1. 一个用户同时出现两个或多个可用 token 的情况。
  2. 如果 token 过期失效，需要续期的问题不好处理。
    • 当然可以在 JWT 中添加 refreshToken，并引入 Redis 缓存 JWT，以此续期，但是这让 JWT 变得不伦不类。
  3. 会出现 token 被盗用的问题。
  4. 用户注销时，如何让 token 失效的问题。
    • 这种方式可以为每个用户生成秘钥（secret），并和用户信息一起存储在数据库中，用户注销时，就修改秘钥。
  5. 用户信息修改时，token 中信息同步的问题。

# 3.JOSE

# 3.1 JOSE 介绍

JOSE (opens new window) (Javascript Object Signing and Encryption) 是一种旨在提供在各方之间安全传递声明（claims）的方法的规范集。我们常用的 JWT 就包含了允许客户端访问特定应用下特定资源的声明。JOSE 制定了一系列的规范来达到此目的。目前该规范还在不断的发展，我们常用的包含以下几个 RFC :

  • JWS（RFC 7515）：JSON Web 签名，描述生成和处理签名消息。
  • JWE（RFC 7516）：JSON Web 加密，描述了保护和处理加密消息。
  • JWK（RFC 7517）：JSON Web 密钥，描述 Javascript 对象签名和加密中加密密钥的格式和处理。
  • JWA（RFC 7518）：JSON Web 算法，描述了 Javascript 对象签名和加密中使用的加密算法。
  • JWT（RFC 7519 (opens new window)）：JSON Web 令牌，描述以 JSON 编码并由 JWS 或 JWE 保护的声明的表示形式。

# 3.2 重新理解 JWT

上面 JWT 的描述是：以 JSON 编码并由 JWS 或 JWE 保护的声明的表示形式，这啥意思？乍一看感觉很懵，别急，咱们慢慢往下看。

官方对 JWT 的定义：JSON Web Token (JWT) is a compact URL-safe means of representing claims to be transferred between two parties。翻译：JSON Web令牌（JWT）是一种紧凑的URL安全方法，用于表示要在两方之间转移的声明。

也就是说我们通常说的 JWT 实际上是一个对声明进行 JOSE 处理方式的统称。我们之前用的应该叫 JWS（JSON Web Signature）。实际上，JWS 只是对声明（claims）进行加密生成签名，保证其不被篡改，是 JWT 的一种实现。

除了 JWS、 JWT 还有另一种实现 JWE（JSON Web Encryption）。我们上面知道，JWS 仅仅能保证数据的完整性而不能保证数据不被泄露，其中的负载（Payload）就只是进行了 Base64，相当于明文展示，而 JWE 就可以解决这个问题。

# 3.4 JWK

JWT 本身要使用私钥进行签名防止信息被篡改，公钥则用来发给下游消费方来验证 JWT 的可靠性。

通常情况下，公钥的配置方式为静态文件集成，这有一个弊端，当上游公私钥进行了改动，下游就无法动态进行公钥适配。这就是 JWK 要解决的问题，它对密码算法和标识符进行了规范设计，它紧凑的 JSON 数据结构非常方便在上下游之间传输。

JWK 是表示加密密钥的 JSON 对象。该对象包含的 key 名称必须是唯一的，在此基础上 JWK 可以包含一些自定义字段。下面是一个P-256 EC（椭圆曲线离散密码）秘钥的 JWK 表示：

# 3.5 JWA

JWA 规范规定了哪些算法可以作为 JWS 和 JWE 的密码算法。还规定了这些算法对应的 JWK 中的 alg 属性，以及特定算法在 JWK 包含的属性，例如前面 EC 算法中的 crv、x、y，这些属性并不是一成不变的，它们会根据算法的迭代进行调整。

JWA的规范参阅 RFC7518 (opens new window)。

常用的 JWT 工具，如下：

  • io.jsonwebtoken:jjwt (opens new window)：一个简单的JWT实现，提供了创建、解析和验证 JWT 的 API。它支持多种算法和密钥类型，使用起来非常方便。
  • com.auth0:java-jwt (opens new window)： 一个简单的 JWT 实现，提供了创建、解析和验证 JWT 的 API。它支持多种算法和密钥类型，并提供了一些有用的功能，例如 JWT 的过期和不可撤销。
  • com.nimbusds:nimbus-jose-jwt (opens new window)：一个完整的 JOSE 和 JWT 实现，提供了许多有用的功能，例如加密、压缩和自定义声明。它支持多种算法和密钥类型，并提供了 Java 和 JavaScript 的实现。

在生成 JWT Token 时，需要选择合适的加密算法。常见的加密算法有 HMAC-SHA 算法和 RSA 算法，其中 HMAC-SHA 算法又分为 HS256、HS384、HS512 等几种。不同的算法有不同的特点和适用场景，需要根据具体的需求来选择。

  • 如果要求 JWT Token 的生成速度较快，可以选择 HMAC-SHA 算法，因为 HMAC-SHA 算法的计算速度比 RSA 算法快（RSA 属于非对称加密，）。同时，HMAC-SHA 算法的安全性也比较高，可以保证 JWT Token 的安全性。
  • 如果要求 JWT Token 的安全性较高，可以选择 RSA 算法，因为 RSA 算法具有较高的安全性。同时，RSA 算法也可以实现 JWT Token 的数字签名和加密，可以更好地保护 JWT Token 的安全性。

在实际使用中，可以根据具体的需求来选择合适的加密算法。

当然，如果涉及支付、用户敏感信息等需要高度安全数据传输的场景，我们也可以同时使用两种算法，比如对“秘钥+时间戳+其他相关参数”生成签名，以防止篡改，再使用 RSA 算法对传输的数据加密，以提高安全性。

@Data
@Accessors(chain = true)
@Component
public class JwtProperties implements InitializingBean {

    @Value("${jwt.base64-secret}")
    private String base64Secret;

    @Value("${jwt.issuer}")
    private String issuer = "";

    @Value("${jwt.expired-millis}")
    private long expiredMillis;

    private Key key;

    @Override
    public void afterPropertiesSet() throws Exception {
        initKey();
    }

    public void initKey() {
        Assert.hasLength(base64Secret, "base64Secret must be not null");
        byte[] secretKeyBytes = Decoders.BASE64.decode(base64Secret);
        this.key = Keys.hmacShaKeyFor(secretKeyBytes);
    }

}

最后，我们编写一个 Token 生成处理类，用来生成、校检 JWT 及获取 JWT 中的信息。代码如下：

// token生成、处理、验证接口
public interface TokenProvider {

    String AUTHORITIES_KEY = "authorities";

    String createToken(String subject, String authorities);

    boolean verifyToken(String authToken);

    Map<String, Object> decodeVerifyToken(String authToken);

    String getUsername(String authToken);
}


// JWT Token实现类
@Slf4j
@Component
public class JwtTokenProvider implements TokenProvider {

    @Autowired
    private JwtProperties jwtProperties;

    @Override
    public String createToken(String subject, String authorities) {
        Date now = new Date();
        Date validity = new Date(now.getTime() + jwtProperties.getExpiredMillis() * 1000L);
        JwtBuilder jwtBuilder = Jwts.builder()
                .setSubject(subject)
          			// 这里存储了用户的角色,实际使用中,可不存储（否则当用户角色发生变化,则需要修改用户JWT的秘钥,使旧的JWT过期）
                .claim(AUTHORITIES_KEY, authorities)
                .signWith(jwtProperties.getKey(), SignatureAlgorithm.HS512)
                .setExpiration(validity);
        if (StringUtils.hasText(jwtProperties.getIssuer())) {
            jwtBuilder.setIssuer(jwtProperties.getIssuer());
        }

        return jwtBuilder.compact();
    }

    @Override
    public boolean verifyToken(String authToken) {
        try {
            Jwts.parserBuilder()
                    .setSigningKey(jwtProperties.getKey())
                    .build()
                    .parseClaimsJws(authToken);
            return true;
        } catch (io.jsonwebtoken.security.SecurityException | MalformedJwtException e) {
            log.info("Invalid JWT signature.");
            log.trace("Invalid JWT signature trace: ", e);
        } catch (ExpiredJwtException e) {
            log.info("Expired JWT token.");
            log.trace("Expired JWT token trace: ", e);
        } catch (UnsupportedJwtException e) {
            log.info("Unsupported JWT token.");
            log.trace("Unsupported JWT token trace: ", e);
        } catch (IllegalArgumentException e) {
            log.info("JWT token compact of handler are invalid.");
            log.trace("JWT token compact of handler are invalid trace: ", e);
        }
        return false;
    }

    // TODO: 该方法的返回内容可以封装成一个对象,该对象包含校检结果及payload中的内容
    @Override
    public Map<String, Object> decodeVerifyToken(String authToken) {
        Jws<Claims> claimsJws = Jwts.parserBuilder()
                .setSigningKey(jwtProperties.getKey())
                .build()
                .parseClaimsJws(authToken);
        return claimsJws.getBody();
    }

    @Override
    public String getUsername(String authToken) {
        Jws<Claims> claimsJws = Jwts.parserBuilder()
                .setSigningKey(jwtProperties.getKey())
                .build()
                .parseClaimsJws(authToken);
        Claims body = claimsJws.getBody();
        Assert.notEmpty(body, "claims body must not be empty");
        return body.getSubject();
    }

}

JSON 认证请求过滤器，用来处理用户登录时的认证：

public class JsonLoginFilter extends UsernamePasswordAuthenticationFilter {

    @Override
    public Authentication attemptAuthentication(HttpServletRequest request, HttpServletResponse response)
            throws AuthenticationException {
        // 获取请求头，据此判断请求参数类型
        String contentType = request.getContentType();
        if (MediaType.APPLICATION_JSON_VALUE.equalsIgnoreCase(contentType) || MediaType.APPLICATION_JSON_UTF8_VALUE.equalsIgnoreCase(contentType)) {
            // 仅支持POST请求
            if (!request.getMethod().equals("POST")) {
                throw new AuthenticationServiceException("Authentication method not supported: " + request.getMethod());
            }
            String username = null;
            String password = null;
            try {
                // 解析请求体中的 JSON 参数
                Map<String, Object> map = JsonUtils.toMap(request.getInputStream());
                username = (String) map.get(this.getUsernameParameter());
                username = (username != null) ? username.trim() : "";
                password = (String) map.get(this.getPasswordParameter());
                password = (password != null) ? password : "";
            } catch (IOException e) {
                throw new RuntimeException(e);
            }
            // 构建登录令牌
            UsernamePasswordAuthenticationToken authRequest =
                    UsernamePasswordAuthenticationToken.unauthenticated(username, password);
            // Allow subclasses to set the "details" property
            setDetails(request, authRequest);
            // 执行真正的登录操作
            return this.getAuthenticationManager().authenticate(authRequest);
        }

        return super.attemptAuthentication(request, response);
    }

}

public class JwtAuthenticationSuccessHandler extends SavedRequestAwareAuthenticationSuccessHandler {

    private final TokenProvider tokenProvider;

    public JwtAuthenticationSuccessHandler(TokenProvider tokenProvider) {
        this.tokenProvider = tokenProvider;
    }

    @Override
    public void onAuthenticationSuccess(HttpServletRequest request, HttpServletResponse response,
            Authentication authentication) throws ServletException, IOException {
        Collection<? extends GrantedAuthority> authorities = authentication.getAuthorities();
        StringBuilder as = new StringBuilder();
        for (GrantedAuthority authority : authorities) {
            as.append(authority.getAuthority()).append(",");
        }
        // 生成jwt token
        String token = tokenProvider.createToken(authentication.getName(), as.toString());
        Response<Object> data = ResponseFactory.FACTORY.build(HttpStatus.OK.value())
                .setMsg("登录成功")
                .setData(token);

        response.setContentType("application/json;charset=utf-8");
        PrintWriter out = response.getWriter();
        out.write(JsonUtils.toJsonStr(data));
        out.flush();
        out.close();
    }

}

# 3.过滤器校检 Token

用来校检请求携带的 Token 是否有效的过滤器：

@Slf4j
public class JwtAuthFilter extends GenericFilterBean {

    private static final String TOKEN_HEADER = "Authorization";
    private static final String AUTHORIZATION_PREFIX = "Bearer";

    private TokenProvider tokenProvider;

    public JwtAuthFilter(TokenProvider tokenProvider) {
        this.tokenProvider = tokenProvider;
    }

    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) servletRequest;
        String requestURI = request.getRequestURI();
        String token = request.getHeader(TOKEN_HEADER);
        if (StringUtils.hasText(token)) {
            String jwt = token.replace(AUTHORIZATION_PREFIX, "").trim();
            boolean valid = tokenProvider.verifyToken(jwt);
            if (!valid) {
                throw new CredentialsExpiredException("token已过期");
            }

            Map<String, Object> cla = tokenProvider.decodeVerifyToken(jwt);
            String username = (String) cla.get(Claims.SUBJECT);
            String auth = (String) cla.get(TokenProvider.AUTHORITIES_KEY);
            List<GrantedAuthority> authorities = AuthorityUtils.commaSeparatedStringToAuthorityList(auth);
            Authentication authentication = new UsernamePasswordAuthenticationToken(username, null, authorities);
            SecurityContextHolder.getContext().setAuthentication(authentication);
            log.debug("set Authentication to security context for '{}', uri: {}", authentication.getName(), requestURI);
        } else {
            log.debug("no valid JWT token found, uri: {}", requestURI);
        }

        filterChain.doFilter(request, servletResponse);
    }

}

public class SecurityAuthenticationFailureHandler extends ExceptionMappingAuthenticationFailureHandler {

    @Override
    public void onAuthenticationFailure(HttpServletRequest request, HttpServletResponse response,
            AuthenticationException e) throws IOException, ServletException {
        // TODO: 这里可以自己定义返回的对象
        Response<Void> result = ResponseFactory.FACTORY.build(HttpStatus.BAD_REQUEST.value(), e.getMessage());

        if (e instanceof BadCredentialsException) {
            result.setMsg("用户名或者密码输入错误，登录失败");
        } else if (e instanceof DisabledException) {
            result.setMsg("账户被禁用，登录失败");
        } else if (e instanceof CredentialsExpiredException) {
            result.setMsg("密码过期，登录失败");
        } else if (e instanceof AccountExpiredException) {
            result.setMsg("账户过期，登录失败");
        } else if (e instanceof LockedException) {
            result.setMsg("账户被锁定，登录失败");
        }

        response.setContentType(MediaType.APPLICATION_JSON_UTF8_VALUE);
        PrintWriter out = response.getWriter();
        out.write(JsonUtils.toJsonStr(result));
        out.flush();
        out.close();
    }

}

@Configuration
public class AuthenticationManagerConfig {

    @Autowired
    private PasswordEncoder passwordEncoder;

    @Bean
    public InMemoryUserDetailsManager inMemoryUserDetailsManager(SecurityProperties properties) {
        SecurityProperties.User user = properties.getUser();
        return new InMemoryUserDetailsManager(User.withUsername(user.getName())
                .password(passwordEncoder.encode(user.getPassword()))
                .roles(StringUtils.toStringArray(user.getRoles()))
                .build());
    }

    @Bean
    public AuthenticationManager authenticationManager(InMemoryUserDetailsManager userDetailsService) {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
        provider.setUserDetailsService(userDetailsService);
        provider.setPasswordEncoder(passwordEncoder);
        // 强制使用principal.toString()作为principal
        provider.setForcePrincipalAsString(true);
        // 隐藏用户不存在异常
        provider.setHideUserNotFoundExceptions(true);
        return new ProviderManager(provider);
    }

}


@Configuration
public class SecurityConfig {

    @Autowired
    private AuthenticationManager authenticationManager;

    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http.authenticationManager(authenticationManager);
                http.authorizeHttpRequests(auth ->  auth
                        .requestMatchers("/hello").permitAll()
                        .requestMatchers("/admin/**").hasRole("ADMIN")
                        .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")           
                        .anyRequest().authenticated()
                )
                .exceptionHandling(exceptionHandling -> exceptionHandling
                        .authenticationEntryPoint(new SecurityAuthenticationEntryPoint())
                )
                .logout(logout -> logout
                        .logoutSuccessHandler(new SecurityLogoutSuccessHandler())
                        // 使session失效
                        .invalidateHttpSession(true)
                        // 清除认证信息
                        .clearAuthentication(true)
                        // 注销时,删除自定义的cookie
                        .deleteCookies("JSESSIONID")
                )
                .sessionManagement(sessionManagement -> sessionManagement
                        // 禁用Session管理(可选操作)
                        // https://docs.spring.io/spring-security/reference/servlet/authentication/session-management.html#stateless-authentication
                        .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
                )
                .csrf(AbstractHttpConfigurer::disable);

        http.addFilterBefore(jsonLoginFilter(), UsernamePasswordAuthenticationFilter.class)
 						.addFilterBefore(new JwtAuthFilter(jwtTokenProvider), UsernamePasswordAuthenticationFilter.class);

      	return http.build();
    }

    private JsonLoginFilter jsonLoginFilter() {
        JsonLoginFilter filter = new JsonLoginFilter();
        filter.setAuthenticationManager(authenticationManager);
        filter.setFilterProcessesUrl("/login");
        filter.setAuthenticationSuccessHandler(new JwtAuthenticationSuccessHandler(jwtTokenProvider));
        filter.setAuthenticationFailureHandler(new SecurityAuthenticationFailureHandler());

        return filter;
    }

}
