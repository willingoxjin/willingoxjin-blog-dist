08.Spring Security 的 rember-me

# 一、自动登录（rember-me）介绍

# 1.什么是自动登录？

自动登录（rember-me）允许用户在一段时间内免登录访问应用程序。当用户选择“记住我”选项并成功登录后，系统会在用户的浏览器中设置一个持久化的令牌（通常是一个加密的 Cookie ），以便在下次访问时自动登录。

# 2.为什么需要自动登录

我们在访问网站或者使用各种 App （后面统称为应用）时，这些应用都会要求登录，这些应用也并非同一家公司开发维护，这就导致我们要在每一个应用都要注册一个账户，且注册时密码还有强度要求。可是，这么多账户的密码，我们很容易忘记。用于一旦忘记密码就会选择找回密码，然而，下一次，又会忘记，最后进入“忘记密码—找回密码”的循环。

为了解决这个问题，提高用户的体验，可以提供自动登录（RemberMe）的功能，方便用户。

不过，现在很多的应用都选择 OAuth2 认证、手机验证码认证、邮箱认证等方式来优化忘记密码的问题。

# 3.Spring Security 中的自动登录实现方案

官方文档：Remember-Me Authentication (opens new window)

Spring Security 为 Rember-Me 提供了扩展点： RememberMeServices 接口。并且提供两个实现：

  1. 基于散列加密算法机制的实现：通过加密用户必要的登录信息，并生成令牌来实现自动登录，实现类为 TokenBasedRememberMeServices。

  2. 基于持久化数据存储机制（如数据库）实现：通过生成持久化令牌实现自动登录，实现类为 PersistentTokenBasedRememberMeServices。

这两种实现均继承于 AbstractRememberMeServices 类，AbstractRememberMeServices 进行了部分的实现，并留出了抽象方法和扩展方法以供子类扩展，如 processAutoLoginCookie、 onLoginFail、onLoginSuccess。

上面两种实现都需要设置 UserDetailsService，或者在 Spring 容器中有一个类型为 UserDetailsService 的 bean 对象，否则它将不起作用。

Spring Security 中默认会自动初始化一个 InMemoryUserDetailsManager 类型的对象。InMemoryUserDetailsManager 实现的接口 UserDetailsManager ，就继承自接口 UserDetailsService。

所以，默认情况下不配置 UserDetailsService ，就会使用 InMemoryUserDetailsManager。不过使用 InMemoryUserDetailsManager 会重启后，会导致 Rember-Me 失效。

TokenBasedRememberMeServices 和 PersistentTokenBasedRememberMeServices 的实现比较简单，大家可以自行阅读源码，查看其实现的逻辑。

# 二、基于散列加密算法令牌实现

基于散列加密算法令牌实现原理，是通过散列加密算法生成一个令牌，并将该令牌存储到 Cookie 中。当 Session 失效时，就会获取 Cookie 中的 “remeber-me” 的数据内容，并校检该数据，若校检成功，则直接免密码登录，否则将要求用户登录。

# 1.散列加密算法介绍

散列加密算法生成令牌内容的组成如下:

base64(username + ":" + expirationTime + ":" + algorithmName + ":"
		algorithmHex(username + ":" + expirationTime + ":" password + ":" + key))
1
2
  • username：用户名
  • password：用户的密码
  • expirationTime：remember-me 标记到期的日期和时间，以毫秒表示
  • key：防止篡改 remember-me 令牌的私钥
  • algorithmName：用于生成和验证 remember-me 令牌签名的算法，默认为 SHA256。
  • algorithmHex() ：对令牌内容加密的算法。
  • base64()：将生成的令牌内容转为 Base64。

示例如下：

原内容（username:expirationTime:algorithmName:algorithmHex）：

# 2.散列加密算法令牌的问题

remeber-me 令牌仅在指定的时间内有效，并且仅当用户名、密码和密钥未更改时有效。

需要注意的是，使用此实现会有一个潜在的安全问题。因为捕获的 remember-me 令牌在令牌到期之前，对任何用户都是可用的，这和摘要身份验证（HTTP Digest Authentication）的问题相同。

如果用户意识到一个令牌已经泄露，可以通过修改密码，使所有的 remember-me 令牌失效。

如果需要更高的安全性，我们应该使用持久化存储令牌的实现。或者完全不应该使用“emember-me”服务。

# 3.使用示例

配置密码编码器：

@Configuration
public class PasswordEncoderConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        Map<String, PasswordEncoder> encoders = new HashMap<>();
        PasswordEncoder bcrypt = new BCryptPasswordEncoder(10);
        encoders.put("bcrypt", bcrypt);
        PasswordEncoder noop = NoOpPasswordEncoder.getInstance();
        encoders.put("noop", noop);

        return new DelegatingPasswordEncoder("noop", encoders);
    }
  
}

需要注意的是，如果不配置密码编码器，则会使用 PasswordEncoderFactories.createDelegatingPasswordEncoder() 来创建一个 DelegatingPasswordEncoder 代理的密码编码器。但是 createDelegatingPasswordEncoder 方法中创建的代理密码编码器默认使用 bcrypt 加密算法。

如果说，你的密码使用的是 bcrypt 加密算法存储，这并不会影响，若你配置的是 noop 加密算法（即不加密密码），在登录成功后就会修改你的密码。不过这在数据库存储中不会有问题，因为密码已经发生了改变，而你使用的是基于内存的用户存储（InMemoryUserDetailsManager），当你重启服务器后，密码就会被重置为非加密的状态，而 remeber-me 令牌中却是 bcrypt 加密算法加密后的密码，由于密码不同，最后生成的 token 在校检时也会不同。最终，会造成每次重启后 remeber-me 令牌都失效。

因此，配置的 PasswordEncoder 和 application.properties 的密码必须匹配。

我们上面使用的是 noop 为默认的加密算法，在配置 application.properties 时也要为 {noop} 的未加密密码：

spring.security.user.name=willingoxjin
spring.security.user.password={noop}123456
spring.security.user.roles=USER

然后，我们在 HttpSecurity 中配置 rememberMe：

@Configuration
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception { 
  			http.authorizeHttpRequests(auth -> auth
                    .requestMatchers("/hello").permitAll()
                    .requestMatchers("/myLogin.html**", "/captcha.jpg**", "/captchaMath.jpg**").permitAll()
                    .requestMatchers("/admin/**").hasRole("ADMIN")
                    .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
                    .anyRequest().authenticated()
            )
        		.formLogin(formLogin -> formLogin.permitAll()
           	.rememberMe(rememberMe -> rememberMe
                    // 用于加密的key
                    .key("ZsWnhVVzFHVDJKSGR6SldSekV3WVRKS1VsQlVNRDA9")
                    .tokenValiditySeconds(60 * 60 * 24 * 7)
                    .rememberMeParameter("remember-me")
                    .rememberMeCookieName("remember-me")
                    // 这里必须配置userDetailsService，除非在Spring上下文有一个类型为UserDetailsService的bean
                    // 若使用数据库存储,则需要配置userDetailsService
                    // .userDetailsService(jdbcUserDetailsManager)
            )
           	.csrf(AbstractHttpConfigurer::disable);

				return http.build();
    }
}

@Configuration
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http, DataSource dataSource) throws Exception { 
        JdbcTokenRepositoryImpl tokenRepository = new JdbcTokenRepositoryImpl();
        tokenRepository.setDataSource(dataSource);

  			http.authorizeHttpRequests(auth -> auth
                    .requestMatchers("/hello").permitAll()
                    .requestMatchers("/myLogin.html**", "/captcha.jpg**", "/captchaMath.jpg**").permitAll()
                    .requestMatchers("/admin/**").hasRole("ADMIN")
                    .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
                    .anyRequest().authenticated()
            )
        		.formLogin(formLogin -> formLogin.permitAll()
           	.rememberMe(rememberMe -> rememberMe
                    // 用于加密的key
                    .key("ZsWnhVVzFHVDJKSGR6SldSekV3WVRKS1VsQlVNRDA9")
                    .tokenValiditySeconds(60 * 60 * 24 * 7)
                    .rememberMeParameter("remember-me")
                    .rememberMeCookieName("remember-me")
                    // 这里必须配置userDetailsService，除非在Spring上下文有一个类型为UserDetailsService的bean
                    // 若使用数据库存储,则需要配置userDetailsService
                    // .userDetailsService(jdbcUserDetailsManager)
                    .tokenRepository(tokenRepository)
            )
           	.csrf(AbstractHttpConfigurer::disable);

				return http.build();
    }
}
