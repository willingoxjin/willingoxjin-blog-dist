1.Scala 基础

# 一、Scala 介绍

Scala 相关文档：

  • 官网：http://www.scala-lang.org/ (opens new window)
  • 文档：http://docs.scala-lang.org/index.html (opens new window)
  • API：http://www.scala-lang.org/api/current/#package (opens new window)

Scala 是一门多范式的编程语言，它设计的初衷是为了实现可伸缩的语言 、并集成面向对象编程和函数式编程的各种特性

Scala 最初由 Martin Odersky 于2004年发布，由于它在大数据处理框架如 Apache Spark 和 Apache Kafka 中的应用，近年来越来越受欢迎。它也被一些公司，如 Twitter、LinkedIn 和 Airbnb 用于构建高性能、可扩展的系统。

Scala 特点如下：

  1. 面向对象编程：Scala 具有类、继承、多态等面向对象编程的特性，并且支持面向对象的设计模式。
  2. 函数式编程：Scala 允许使用高阶函数和闭包等函数式编程的概念。
  3. 静态类型系统：Scala 拥有一个严格的静态类型系统，可以在编译时捕获许多类型错误，提高代码的稳定性和可维护性。
  4. 可扩展性：Scala 的设计目标之一是可扩展性，它允许开发人员轻松地定义新的数据类型、语言结构和库。
  5. 描述性语言：Scala 使用简洁、表达力强的语法，可以更好地描述问题和解决方案，提高代码的可读性和可理解性。
  6. 互操作性：Scala 运行在 Java虚拟机上，可以与 Java 互操作，可以利用 Java 生态系统中的库和工具。

# 1.变量

Scala 中的变量分为两种：可变 var 和 不可变 val

  • 可变 var：var 声明的变量可以修改其值
  • 不可变 val：val声明的变量，值不能被修改，否则会出现编译错误：error: reassignment to val（相当于 Java 中对变量添加 final 关键字）

var 使用示例：

object Variable {
	def main(args: Array[String]): Unit = { 
    // 变量var可赋值多次
    var a = 1
    a = 2
    println("a.getClass = " + a.getClass) // a.getClass = int

    // 自动赋值类型为第一次赋值的类型
    var b = '1'
    b = 1
    b = 'b'
    println("b.getClass = " + b.getClass) // b.getClass = char
    println("b = " + b) // b = b
  }
}

# 四、Scala 集合

# 1.集合体系

Scala 的集合体系如下

集合的顶层特征（Trait）是 Iterable，Iterable 下还有一些子特征：Set、Seq、Map 等

这几个子特征常用的实现类如下

  • Set
    • HashSet、LinkedHashSet、SortedSet 等
  • Seq
    • List、Buffer、Range 等
  • Map
    • HashMap、SortedMap、LinkedHashMap 等
  • Buffer
    • ArrayBuffer、ListBuffer 等

Trait 相当于 Java 的接口，与接口不同的是，它还可以定义属性和方法的实现。

一般情况下 Scala 的类只能够继承单一父类，但是如果是 Trait（特征） 的话就可以继承多个，从结果来看就是实现了多重继承。

Trait（特征） 定义的方式与 Class 类类似，但它使用的关键字是 trait。

Trait 使用示例：

trait Equal {
  def isEqual(x: Any): Boolean
  def isNotEqual(x: Any): Boolean = !isEqual(x)
}

# 2.集合

Scala 中的集合是分成可变和不可变两类集合：

  • 可变集合：
    • 可变集合的元素可以动态修改
    • 其相关类都在 scala.collection.mutable 包下
  • 不可变集合：
    • 集合的元素在初始化之后，无法修改（无法增加、删除元素）
    • 其相关类都在 scala.collection.immutable 包下

在创建集合的时候，如果不指定具体的包名，默认会使用不可变集合

val setVal: Set[Int] = Set(1, 2, 3, 4)

我们可以通过 debug 查看变量，来判断是否生成了新的对象，如下：

// 不可变集合在赋值时会生成新的集合
var immutable: Set[Int] = Set(1, 2, 3, 4, 5) // HashSet$HashTrieSet@1056
immutable += 6 // HashSet$HashTrieSet@1077
immutable = immutable + 7 // HashSet$HashTrieSet@1065
println(immutable) // Set(5, 1, 6, 2, 7, 3, 4)

// 可变集合
val mutableSet = scala.collection.mutable.Set(1, 2, 3, 4, 5) // HashSet$HashTrieSet@1080
mutableSet += 6 // HashSet$HashTrieSet@1080
println(mutableSet) // Set(1, 5, 2, 6, 3, 4)

Scala 中的集合元素操作可以使用 += 、-= 、++= 、--= 、add 、remove

question：为什么 scala.collection.mutable.Set 需要写包的全路径？

由于 immutable 包是默认导入的，所以不需要导包，因此使用 scala.collection.immutable.Set 时，无需写签名的包路径。但是也会有个别类虽然在 immutable 包下，但是不写全路径还是报错

而使用 scala.collection.mutable.Set 则需要导入包。

Set 常用实现类如下：

  • HashSet：这个集合的特点是：集合中的元素不重复、无序
  • LinkedHashSet：这个集合的特点是：集合中的元素不重复、有序。它会用一个链表维护插入顺序，可以保证集合中元素是有序的
  • SortedSet：这个集合的特点是：集合中的元素不重复、有序。它会自动根据元素来进行排序

# 3.1 HashSet

HashSet 集合分为可变和不可变之分，immutable包下面的是不可变的，后期无法新增元素 在这里可以使用 new 关键字，也可以不使用，因为 HashSet 既是 class，又是 object，但是包名需要指定，否则无法识别。

使用示例：

# 2.2 辅助 constructor

在 Scala 中，可以给类定义多个辅助 constructor，类似于 Java中的构造函数重载。

辅助 constructor 第一行必须调用主 constructor。

辅助 constructor 之间可以互相调用

使用如下：

class Student {

  // `_`表示默认值
  var id: Int = _

  var name: String = _

  // 辅助constructor
  def this(name: String) {
    this() // 必须先调用主constructor
    this.name = name
  }

  // 辅助constructor
  def this(id: Int, name: String) {
    this() // 必须先调用主constructor
    this.id = id
    this.name = name
  }

  // 重写toString函数
  override def toString: String = s"Student(id=$id, name=$name)"

}

object Student {

  def main(args: Array[String]): Unit = {
    val student = new Student
    println(student)

    val student2 = new Student("zhangsan")
    println(student2)

    val student3 = new Student(1, "lisi")
    println(student3)

  }

}

# 3.对象 object

在 Scala 中，object 关键字用于定义单例对象。

object 可以包含类的静态成员，例如静态字段、静态方法和静态块等。与类不同，object 在首次被访问时被创建，因此它们也被称为惰性实例。

object 不能定义带参数的 constructor，只有空参的 constructor，因此首次访问被创建时调用的是无参构造函数。

Scala 的 object 还具有一些独特的用途和特性：

  1. 作为应用程序入口点：通过定义一个包含 main 方法的 object，可以将其作为应用程序的入口点来执行。
  2. 用于工具方法：object 中的方法可以像静态方法一样被调用，因此可以用来定义工具方法和辅助函数等。
  3. 用于模拟静态类：Scala 不支持静态类，但可以使用 object 来模拟静态类的功能，例如定义一个包含只读常量的 object。
  4. 用于定义类型别名和隐式转换：通过在 object 中定义类型别名和隐式转换，可以提高代码的简洁性和可读性。

object 的使用如下：

object StringUtils {

  val EMPTY_STR = ""

  def isEmpty(str: String): Boolean = str == null || str.isEmpty

  def isNotEmpty(str: String): Boolean = !isEmpty(str)

  def isBlank(str: String): Boolean = str == null || str.trim.isEmpty || str.trim == EMPTY_STR

  def isNotBlank(str: String): Boolean = !isBlank(str)

}

# 4.伴生对象

如果有一个 class，还有一个与 class 同名的 object，那么就称这个 object 是 class 的伴生对象（companion object）。object 与类共享同一个名称和同一个 .scala 源文件。

在 object 中，有一个特殊函数就是 apply。通常在伴生对象中实现 apply 函数，并在其中实现构造伴生类对象的功能。

实现了 apply 函数后，就不需要使用 new Class 的方式创建对象的时候，而是使用 Class 或 Class() 的方式创建对象。这样就会隐式调用伴生对象的 apply 方法，使得对象创建更加简洁。

Array 的伴生对象的 apply 方法就实现了接收可变数量的参数，以及会创建一个Array对象：

val arr = Array(1, 2, 3, 4, 5)

定义 class 和 object，并实现 apply 函数。如下：

class User (id: Int, username: String) {

  def sayHello(): Unit = {
    println(s"Hello, $username, id=$id")
  }

}

object User {

  def apply(id: Int, username: String): User = new User(id, username)

}

# 5.trait

Scala 中的接口称为 trait。trait 类似于Java 中的 interface。

  • 在 triat 中可以定义方法、抽象方法、字段

  • 类可以使用 extends 关键字继承 trait

    • 类继承 trait 后，必须实现 trait 中的抽象方法，实现时不需要使用 override 关键字

  • scala 不支持对类进行多继承，但是支持对 trait 进行多重继承。使用 with 关键字即可进行多重继承

triat 的定义与继承示例：

trait Greeting {
  def greet(name: String): Unit = println(s"Hello, $name!")

  def sayHi(name: String): Unit

}

class Student(val stuName: String) extends Greeting {
  override def sayHi(name: String): Unit = println(s"Hi, $name teacher!")
}

class Teacher(val name: String)  extends Greeting {
  override def sayHi(name: String): Unit = greet(name)
}

triat 使用 with 实现多继承：

trait Logger {
  def log(msg: String): Unit
}

trait ConsoleLogger extends Logger {
  def log(msg: String): Unit = println(msg)
}

trait TimestampLogger extends Logger {
  abstract override def log(msg: String): Unit = {
    super.log(s"${java.time.Instant.now()} $msg")
  }
}

class MyLogger() extends ConsoleLogger with TimestampLogger {

  def info(msg: String): Unit = log(s"[INFO] $msg")

  def error(msg: String): Unit = log(s"[ERROR] $msg")

}
