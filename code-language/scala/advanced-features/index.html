3.Scala 高级特性

# 一、模式匹配

模式匹配（match case），类似于 Java 中的 switch case 语法。对一个值进行条件判断，然后针对不同的条件，进行不同的处理。

Scala 的 match case可以匹配各种情况，比如：变量的类型、集合的元素，是否有值。

在 match case中，只要一个 case 分支满足并处理了，就不会继续判断下一个 case 分支。

# 1.变量值的模式匹配

语法格式：

变量 match { 
  case 值1 => 代码 
  case 值2 => 代码 
  case 值3 => 代码 
  case _ => 代码 // 其他情况 
}

使用示例：

object MatchCase1 extends App {

  def matchDemo1(day: Int) {
    day match {
      case 1 => println("Monday")
      case 2 => println("Tuesday")
      case 3 => println("Wednesday")
      case _ => println("none")
    }
  }

  def matchDemo2(day: Int) {
    // 返回变量的值
    val result = day match {
      case 1 => "Monday"
      case 2 => "Tuesday"
      case 3 => "Wednesday"
      case _ => "none"
    }
    println(result)
  }
  
  
  matchDemo1(2) // Tuesday
  matchDemo1(5) // none
  matchDemo2(3) // Wednesday
}

# 2.变量类型的模式匹配

语法格式：

变量 match { 
  case 变量: 类型1 => 代码 
  case 变量: 类型2 => 代码 
  case 变量: 类型 => 代码 
  case _ => 代码 // 其他情况 
}

使用示例：

object MatchCase1 extends App {

  def processorException(e: Exception) {
    e match {
      case e: IllegalArgumentException => println("IllegalArgumentException")
      case e: NullPointerException => println("NullPointerException")
      case e: FileNotFoundException => println("FileNotFoundException")
      case e: IOException => println("IOException")
      case _: Exception => println("Exception")
    }
  }
  processorException(new IllegalArgumentException) // IllegalArgumentException

}

# 3.case class 与模式匹配

Scala 中提供了一种特殊的类，用 case class 进行声明，中文可以称为样例类。

case class 其实有点类似于 Java 中的 JavaBean 的概念。即只定义 field，会由 Scala 在编译时自动提供 get 和 set 方法，但是没有其它的 method。

case class 的主构造函数接收的参数通常不需要使用 var 或 val 修饰，Scala 自动会使用 val 修饰（但是如果手动使用 var 修饰，那么就会按照 var）。

Scala 自动为 case class 定义了伴生对象，也就是 object，并且定义了 apply() 函数，该函数接收主构造函数中相同的参数，并返回 case class 对象。

case class 定义：

class Person
case class Teacher(var name: String, var subject: String) extends Person
case class Student(var name: String, var clazz: String) extends Person

case class 的使用：

  val zhangsan: Teacher = Teacher("zhangsan", "math")
  println(zhangsan.name) // zhangsan
  println(zhangsan.subject) // math

case class 与模式匹配：

object CaseClassDemo extends App {

   /**
   * 校检人员角色
   */
  def checkPersonRole(person: Person): Unit = {
    person match {
      case Teacher(name, subject) => println(s"Teacher: name=$name, subject=$subject")
      case Student(name, clazz) => println(s"Student: name=$name, clazz=$clazz")
      case _ => println("Unknown")
    }
  }

  checkPersonRole(Teacher("zhangsan", "math")) // Teacher: name=zhangsan, subject=math
  checkPersonRole(Student("lisi", "class-2023-01")) // Student: name=lisi, clazz=class-2023-01
  checkPersonRole(new Person) // Unknown
}

# 4.Option 与模式匹配

Scala 有一种特殊的数据类型：Option。

Option 有两种值：

  • 一种是 Some，表示有值

一种是None，表示没有值

Option 通常会用于模式匹配中，用于判断某个变量是有值还是没有值，这比null来的更加简洁明了

Option 模式匹配使用示例：

object OptionDemo extends App {

  val userAge = Map("Alice" -> 18, "Bob" -> 30, "Cindy" -> 27)

  def getAge(name: String) {
    val age = userAge.get(name)
    age match {
      case Some(age) => println(s"age=$age")
      case None => println("none")
    }
  }

  getAge("Alice") // age=18
  getAge("Bob") // age=30
  getAge("Davy") // none
}

# 二、隐式转换

Scala 的隐式转换，允许手动指定将某种类型的对象转换成其它类型的对象。

隐式转换最核心的就是定义隐式转换函数，即 implicit conversion function。隐式转换函数与普通函数唯一的语法区别是要以 implicit 开头而且最好要定义函数返回类型。

隐式转换非常强大的一个功能，就是可以在不知不觉中加强现有类型的功能。也就是说，我们可以为某个普通类定义一个加强类，并定义对应的隐式转换函数。这样我们在使用加强类里面的方法的时候，Scala 会自动进行隐式转换，把普通类转换为加强类，然后再调用加强类中的方法。

Scala 默认会自动使用下面隐式转换

  1. 源类型，或者目标类型的伴生对象里面的隐式转换函数
  2. 当前程序作用域内可以用唯一标识符表示的隐式转换函数

如果隐式转换函数不在上述两种情况下的话，那么就必须手动使用 import 引入对应的隐式转换函数。

一般建议仅在需要进行隐式转换的地方，比如某个函数内，用 import 导入隐式转换函数，这样可以缩小隐式转换函数的作用域，避免不需要的隐式转换。

# 2.隐式参数

可以将参数标记为隐式参数，在调用方法时不显式地传递这个参数。编译器会在当前作用域内查找与隐式参数类型匹配的隐式值，并自动将其传递给方法。

例如，下面的代码定义了一个需要隐式参数的方法：

object ImplicitParameters extends App {

  def greet(name: String) (implicit greeting: String): Unit = {
    println(s"$greeting, $name!")
  }

  implicit val defaultGreeting: String = "Hello"

  greet("Alice") // "Hello, Alice!"

}
