可变参数

  • 使用...便是可变的参数
  • 固定参数和可变参数同时存在时，固定参数必须放在变长参数之前
  • 使用ipairs{...}来遍历可变参数
  • 使用select(n, …) 来返回从起点 n 开始到结束位置的所有参数列表
  • 使用select("#",...)获取可变参数的数量
#!/usr/local/bin/lua

-- 定义一个可变参数
function max(...)  
	local maxNum 
  	for i, v in ipairs{...} do   --> {...} 表示一个由所有变长参数构成的数组  
      	if maxNum==nil or maxNum<v then
            maxNum = v
        end   
  	end
  	return maxNum 
end  
-- 调用可变参数
print(max(3,4,5,6,7))  -- 7

-- 固定参数和可变参数同时使用时：固定参数必须放在变长参数之前
function add(num1, num2, ...)
    print("可变参数的个数："..select("#",...)) -- 可变参数的个数：3
    print(select(n,...)) -- 3	4	5
	local result = num1+num2
    local args={...} -- 使用{...} 来将所有的可选参数转换为table到变量args
    print(args) -- table: 0x7f98e5409db0
    for k,v in ipairs(args) do
    	result = result + v
    end
    print(result) -- 15
end

add(1,2,3,4,5)

# thread

在 Lua 里，最主要的线程是协同程序（coroutine）。它跟线程（thread）差不多，拥有自己独立的栈、局部变量和指令指针，可以跟其他协同程序共享全局变量和其他大部分东西。

线程跟协程的区别：线程可以同时多个运行，而协程任意时刻只能运行一个，并且处于运行状态的协程只有被挂起（suspend）时才会暂停。

# for

for循环的条件表达式有三个 ,拿下面第一条循环举例

  • i=1 初始化的索引值
  • 5 目标值，当i>5时结束循环
  • 1 i每次增加的步长
  • i会在循环体中的代码执行完毕之后被修改
for i=1,5,1 do
    print(i) -- 1 2 3 4 5
    -- i = i + 1
    -- i 会在循环体中的代码执行完毕之后被修改
end
 
-- 当 i<1 时结束循环
for i=10,1,-2 do
    print(i) -- 10  8  6  4  2 
    -- i = i - 2
end

提示

ipairs的实现原理：

当 Lua 调用 ipairs(a) 开始循环时，他获取三个值：迭代函数 iter、状态常量 a、控制变量初始值 0； 然后 Lua 调用 iter(a,0) 返回 1, a[1]（除非 a[1]=nil）；第二次迭代调用 iter(a,1) 返回 2, a[2]……直到第一个 nil 元素。

function iter (a, i)
    i = i + 1
    local v = a[i]
    if v then
       return i, v
    end
end
 
function ipairs (a)
    return iter, a, 0
end

# 无状态的迭代器

无状态的迭代器是指不保留任何状态的迭代器，因此在循环中我们可以利用无状态迭代器避免创建闭包花费额外的代价。

每一次迭代，迭代函数都是用两个变量（状态常量和控制变量）的值作为参数被调用，一个无状态的迭代器只利用这两个值可以获取下一个元素。

这种无状态迭代器的典型的简单的例子是 ipairs，它遍历数组的每一个元素，元素的索引需要是数值。

function square(iteratorMaxCount,currentNumber)
   print("max="..iteratorMaxCount..",current="..currentNumber) 
   if currentNumber<iteratorMaxCount then
      	currentNumber = currentNumber+1
   		return currentNumber, currentNumber
   end
end

-- square-执行的函数,3-iteratorMaxCount,0-currentNumber
for i,n in square,3,0

   print(i,n)

# 多状态的迭代器

很多情况下，迭代器需要保存多个状态信息而不是简单的状态常量和控制变量，最简单的方法是使用闭包。

还有一种方法就是将所有的状态信息封装到 table 内，将 table 作为迭代器的状态常量，因为这种情况下可以将所有的信息存放在 table 内，所以迭代函数通常不需要第二个参数

array = {"Redis","Java"}

function elementIterator (collection)
   local index = 0
   local count = #collection
   -- 闭包函数
   return function ()
      index = index + 1
      if index <= count then
         --  返回迭代器的当前元素
         return collection[index]
      end
   end
end

for element in elementIterator(array) do
   print(element)
end

# 元方法：__index

这是 metatable 最常用的键。

lua查找表的规则：

  • 当通过键来访问 table 的时候，若存在该键则直接返回，否则继续下一步

  • 如果不存在这个键，那么Lua就会寻找该table的metatable（假定有metatable）中的__index 键。__

  • 如果__index包含一个table，Lua会在table中查找相应的键。

  • 如果__index包含一个函数，该函数接收两个参数（table表名，key键名），需要返回key对应的结果。Lua会将访问table和key作为参数赋值给这个参数的函数，并运行该函数

示例：__index为表

mymetatable = { b = 2 }

t_table = setmetatable(mytable, { __index = mymetatable })

# 元方法：__newindex

__newindex元方法用来对表更新

当给一个table的索引赋值，但是该索引不存在该table中，解释器就会查找__newindex 元方法：如果存在则调用这个函数而不进行赋值操作。

  • 为已存在的索引赋值，会直接修改

  • 为不存在的索引，会调用元方法__newindex

    • 若元方法为table
      • 该table中存在该索引，则更改索引的值
      • 该table中不存在该索引，则新增索引值
    • 若元方法为function，该函数有三个参数为table、 key、value，无返回结果

示例：__newindex为表

mymetatable = {}
mytable = setmetatable({a = 1}, { __newindex = mymetatable })

print(mytable.a)		-- 1

-- 增加新值
mytable.b = "I am b"
print(mytable.b,mymetatable.b)	-- nil	 I am b

mytable.a = 2
print(mytable.a,mymetatable.a)	-- 2   nil

    local str

# 模块加载机制

自定义的模块若想在任意的路径都能访问到该模块，是需要进行配置的，

lua加载的顺序如下

  1. 当 Lua 启动后，会以环境变量 LUA_PATH 的值来初始这个环境变量，没有找到该环境变量，则使用一个编译时定义的默认路径来初始化

  2. 若没有配置 LUA_PATH 这个环境变量的话，用户可以在~/.bash_profile、~/.profile等等相关环境变量配置文件中设置

  3. 若lua中找不到该模块就会去c的程序模块中搜索

  4. 最后实在找不到就报错
