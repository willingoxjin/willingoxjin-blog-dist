05.CDH 部署

# 一、CDH 介绍

# 1.CDH

CDH 是 Cloudera Distribution including Apache Hadoop 的缩写，是一种企业级 Hadoop 发行版，由 Cloudera 公司开发和维护。

CDH 包括了一系列的 Apache 开源组件，包括 Hadoop、Hive、HBase、Impala、Spark、Flume、Oozie、ZooKeeper 等。Cloudera 还对这些组件进行了增强，提供了额外的管理、监控、安全和数据访问等功能。

CDH 的主要特点包括：

  1. 容易安装和配置：Cloudera 提供了一套简单易用的安装和配置工具，使得用户可以快速搭建一个 Hadoop 环境。
  2. 高可靠性和可扩展性：CDH 对 Hadoop 生态系统进行了全面的测试和优化，保证了系统的高可靠性和可扩展性。
  3. 多租户支持：CDH 提供了安全的数据隔离和资源隔离机制，可以支持多个租户共享同一个 Hadoop 集群。
  4. 强大的管理和监控功能：CDH 提供了一系列的管理和监控工具，帮助管理员对 Hadoop 集群进行监控和管理，以确保系统的稳定运行。

# 2.CDP

Cloudera 公司合并 Hortonworks 公司之后（2018年10月份合并），会将 CDH 和 HDP 这两个产品进行合并，推出一款统一旗舰产品 CDP（Cloudera Data Platform），它也是 Cloudera 新兴的“企业数据云”战略的核心。

Cloudera 宣布他们将继续支持现有的 CDH 和 HDP 平台一直到 2022年1月，同时在这3年对现有的产品还会进行交叉组合。

目前（2022） CDP 还不是特别成熟，所以目前在企业中使用的时候还是优先使用 CDH 或者 HDP。

# 二、CDH 服务器、软件准备

# 1.服务器

实际工作中搭建 CDH 集群，理论上集群机器数量是越多越好，如果资源有限的话，前期建议至少4台机器。

如果是搭建测试环境，单机至少分配 8G 内存，否则 CDH 集群可能无法启动，因为 CDH 集群本身是会占用一部分内存资源的。

# 3.设置所有服务器之间免密登录

命令如下：

# 生成本机公钥和秘钥
	# 生成秘钥文件 一直回车
	ssh-keygen -t rsa
    
# 上传公钥到服务器    
	# 本地：将公钥放入authorized_keys文件中 注意：authorized_keys 文件名不能写错
	cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
	# 服务器：上传公钥到服务器
	ssh-copy-id -i ~/.ssh/id_rsa.pub root@cdh01
	ssh-copy-id -i ~/.ssh/id_rsa.pub root@cdh02
	ssh-copy-id -i ~/.ssh/id_rsa.pub root@cdh03
