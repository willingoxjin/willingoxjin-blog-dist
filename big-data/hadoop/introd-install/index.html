01.Hadoop 介绍与安装

官方文档：https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-common/SingleCluster.html

# 一、Hadoop介绍

Hadoop是有Apache开源的软件实用程序的集合，主要用来进行分布式存储和分布式计算

# 1.Hadoop的发行版

  • Apache Hadoop 官方版本
    • 开源
    • 技术支持差，搭建集群麻烦
  • Cloudera Hadoop（CDH） 商业版本
    • 对官方版本进行了优化，提供收费技术支持，提供界面操作，方便集群运维管理
    • 基础功能不收费
  • HortonWorks（HDP） 开源
    • 提供界面操作，方便集群运维管理，多数互联网公司使用此发行版
    • HDP 已经被 CDH 收购，都是属于一个公司的产品

建议实际工作使用 CDH 或 HDP，方便运维管理，学习使用 Apache Hadoop

# 2.Hadoop版本演变历史

  • Hadoop 1.x

    • MapReduce 分布式计算，资源管理
    • HDFS 分布式文件存储

  • Hadoop 2.x

    • MapReduce 分布式计算
    • HDFS 文件存储
    • YARN 资源管理
    • Others

Hadoop 2.x中MapRedure中的资源管理功能被分离出来，单独为一个模块，即YARN

  • Hadoop 3.x
    • MapReduce
    • HDFS
    • YARN
    • Others

Hadoop 3.x新特性：

  • Java版本支持改为支持8及以上；
  • HDFS 支持纠删码；
  • HDFS支持多个 NameNode（之前只支持一个NameNode）;
  • MapReduce 任务级本地优化；​
  • 多重服务默认端口变更；

纠删码：比副本存储更节省存储空间的一个数据数据持久化存储方案，使用此种方案，相同容错的情况下，可以比之前的副本机制节省一半左右的内存空间

# 3.Hadoop的三大核心组件

  • HDFS 负责海量数据的分布式存存储
  • MapReduce MapReduce是一个计算模型，负责海量数据的分布式计算
  • YARN 主要负责集群资源的管理和调度

# 设置 ssh 免密码登录

​ 由于hadoop在集群中需要ssh连接，设置ssh免密之后会比较方便，在伪分布式集群虽然只有一台机器，但是机器自己ssh连接自己，也是需要输入密码的

# 生成本机公钥和秘钥
	# 生成秘钥文件 一直回车
	ssh-keygen -t rsa
	# 查看秘钥文件 
	# .id_rsa.pub (公钥) id_rsa （私钥）
	ll ~/.ssh
    
# 上传公钥到服务器    
	# 本地：将公钥放入authorized_keys文件中 注意：authorized_keys 文件名不能写错
	cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
	# 服务器：上传公钥到服务器
	ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.2.201
	
# 测试是否能免密码登录
ssh root@172.16.23.129

# 格式化

启动前先执行格式化 HDFS 命令（只需格式化一次）：

./bin/hdfs namenode -format
1

看到这句话就代表格式化成功了：

namenode.FSImageFormatProtobuf: Storage directory /opt/data/hadoop/dfs/name 
has been successfully formatted.
1
2

若出现异常，请检查一下上面的配置是否正确。

注意

格式化成功之后请勿重复格式化，否则集群再启动时会出现问题。

若需要重新格式化，请删除临时目录（/opt/data/hadoop/）下的文件

# 启动集群

# 开启所有服务命令；如果只需要启动hdfs或yarn,可单独执行start-hdfs.sh或start-yarn.sh
./sbin/start-all.sh

# 关闭服务命令
./sbin/stop-all.sh

# 五、Hadoop 客户端节点

在实际工作中，不建议直接连接集群中的节点来操作集群，因为直接把集群中的节点暴露给普通开发人员是不安全的。

建议在业务机器上安装 Hadoop，然后通过业务机器来操作 Hadoop 集群，这台业务机器就被称作客户端节点。

配置客户端节点的 Hadoop 是非常简单的，只需要把集群中已经配置好的安装包直接复制一份到客户端的服务器上，就可以直接使用。

需要注意的是客户端节点不需要启动进程，因为一旦启动进程就会连接到集群中，从而影响集群。
