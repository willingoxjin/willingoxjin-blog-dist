# 二、常用函数

# 1.分组排序取 TopN

使用的函数：

  • ROW_NUMBER()
    • 对数据编号，编号从1开始
  • OVER()
    • 可以理解为把数据划分到一个窗口内
    • 里面可以加上 partition by，表示按照字段对数据进行分组，还可以加上 order by 表示对每个分组内的数据按照某个字段进行排序

# rank()

上面 SQL中 的 row_number() 可以替换为 rank() 或者 dense_rank()。我们先来看看 rank()。

rank()表示上下两条记录的 score 相等时，记录的行号是一样的，但下一个 score 值的行号递增 N（N是重复的次数）。

使用 rank() 示例如下：

select * from (
	select
	*,
	rank() over (partition by subject order by score desc) as num
	from student_score
) s where s.num <= 3;

  • rank() over()是跳跃排序，有两个第一名时接下来就是第三名（在各个分组内）
  • dense_rank() over()是连续排序，有两个第一名时仍然跟着第二名（在各个分组内）

# 2.行转列

行转列就是把多行数据转为一列数据，行转列主要需要使用到 CONCAT_WS()、COLLECT_SET() 、COLLECT_LIST() 函数。

  • CONCAT_WS()
    • CONCAT_WS 函数可以实现根据指定的分隔符拼接多个字段的值，最终转化为一个带有分隔符的字符串
    • 它可以接收多个参数，第一个参数是分隔符，后面的参数可以是字符串或者字符串数组，最终返回结果则是使用分隔符把后面的所有字符串拼接到一块
  • COLLECT_LIST()
    • COLLECT_LIST 函数返回一个 list 集合，集合中的元素会重复，一般和 group by 结合使用
  • COLLECT_SET()
    • COLLECT_SET 函数返回一个 set 集合，集合中的元素不重复，一般和 group by 结合使用

    stu_name string,

# 3.列转行

列转行主要用到如下函数：

  • SPLIT()
    • SPLIT 函数的参数接受一个字串符和切割规则，类似于 java 的 split 函数，使用切割规则对字符串中的数据进行切割，最终返回一个 array 数组
  • EXPLODE()
    • EXPLODE 函数的参数可以接受 array或者map
    • EXPLODE(ARRAY)：表示把数组中的每个元素转成一行
    • EXPLODE(MAP) ：表示把 map 中每个 key-value 对，转成一行，key 为一列，value 为一列
  • LATERAL VIEW
    • LATERAL VIEW 通常和 SPLIT， EXPLODE 等函数一起使用。
    • SPLIT 函数可以对表中的某一列进行切割，返回一个数组类型的字段
    • EXPLODE 函数可以对这个数组中的每一个元素转为一行，LATERAL VIEW 可以对这份数据产生一个支持别名的虚拟表

# 三、排序相关函数

# 1.ORDER BY

Hive 中的 ORDER BY 跟标准 sql 语法中的 ORDER BY 作用一样。使用 ORDER BY 会对查询的结果做一次全局排序，使用这个语句的时生成的 reduce 任务只有一个。

语法：

select id from t1 order by id;
1

# 2.SORT BY

使用 SORT BY 语法时，如果有多个 reduce，那么在每个 reducer 端都会做排序，也就是说保证了局部有序。即每个 reducer 出来的数据是有序的，但是不能保证所有的数据是全局有序的，除非只有一个 reducer。

语法：

select id from t1 sort by id;
1

# 3.DISTRIBUTE BY

DISTRIBUTE BY 是控制 map 输出到 reducer 时的划分规则。DISTRIBUTE BY 只会根据指定的 key 对数据进行分区，但是不会排序。

一般情况下可以和 SORT BY 结合使用，先对数据分区，再进行排序。两者一起使用时，DISTRIBUTE BY 必须要写在 SORT BY 之前。

示例

# 设置mapreduce的任务数2。任务数为1时，使用distribute by是没有意义的
set mapreduce.job.reduces = 2;
# 使用distribute by
select id from t2 distribute by id;

# 结合sort by实现分区内的排序（默认asc升序）
select id from t2 distribute by id sort by id;

# 4.CLUSTER BY

CLUSTER BY 是 DISTRIBUTE BY 和 SORT BY简写形式。

即 cluster by id 相当于 distribute by id sort by id

被 cluster by 指定的列只能是升序，不能设置排序规则为 asc 和 desc

set mapreduce.job.reduces = 2;

# 使用cluster by
select id from t2 cluster by id;

# 四、分组和去重函数

  • DISTINCT：对数据中指定字段的重复值进行去重
  • GROUP BY ：对数据按照指定字段进行分组

示例 DISTINCT：

select count(distinct user_id) from t_order
1

上面语句中，使用 distinct 会将所有的 user_id 都 shuffle 到一个 reducer 里面，性能较低。

示例 GROUP BY ：

select count(tmp.user_id) 
from (
    select user_id
    from t_order 
    group by user_id
) tmp
1
2
3
4
5
6

先对 user_id 分组，因为分组的同时其实就是去重，此时是可以并行计算的。然后再计算 count 即可，性能高。
