# 介绍

FFmpeg是一个自由软件，可以运行音频和视频多种格式的录影、转换、流功能，包含了libavcodec ─这是一个用于多个项目中音频和视频的解码器库，以及libavformat——一个音频与视频格式转换库

FFmpeg这个单词中的"FF"指的是Fast Forward。有些新手写信给"FFmpeg"的项目负责人，询问FF是不是代表“Fast Free”或者“Fast Fourier”等意思，"FFmpeg"的项目负责人回信说“Just for the record, the original meaning of "FF" in FFmpeg is "Fast Forward"...”

FFmpeg在Linux平台下开发，但它同样也可以在其它操作系统环境中编译运行，包括Windows、Mac OS X等。这个项目最初是由Fabrice Bellard发起的，而现在是由Michael Niedermayer在进行维护。许多FFmpeg的开发者同时也是MPlayer项目的成员，FFmpeg在MPlayer项目中是被设计为服务器版本进行开发。

2011年3月13日，FFmpeg部分开发人士决定另组libav，网址http://libav.org，同时制定了一套关于项目继续发展和维护的规则。

  • Fabrice Bellard （法国著名程序员 Born in 1972）于 2000 年发起创建的开源项目
  • 官网：https://www.ffmpeg.org/
  • 功能：视频采集功能、视频格式转换、视频抓图、给视频加水印
  • 媒体播放器三大底层框架：MPC、MPlayer 和 VLC

# FFmpeg 目录及作用

libavcodec： 提供了一系列编码器的实现。

libavfilter： 提供了各种音视频过滤器。

libswresample： 实现了混音和重采样。

# 人耳所能听到的声音的频段

20HZ-20KHZ，16 位-96dB 的信噪比，线性脉冲编码调制 PCM，采样速率 40~50KHZ

香农采样定理，又称奈奎斯特采样定理，为了不失真地恢复模拟信号，采样频率应该不小于模拟信号频谱中最高频率的 2 倍。 　fs ≥ 2fmax

# 文件格式

​ 文件的后缀名.mp4.mov的作用是为了方便操作系统识别文件类型，若我们将文件后缀名修改之后，文件的格式不会发生变化。

​ 如：将test.mp4文件名改为test.txt你可以继续使用视频播放器打开，并且正常播放。不过一般不会随意修改后缀名

# 封装

把编码器生成的多媒体内容，如视频、音频、字幕等按照一定的标准封装在一起。

# 容器

存放视频、音频、字幕的盒子，容器为多媒体内容提供索引，若没有容器，视频只能从头开始播放，并且没有声音。容器的标准有多种，由不同的公司和组织制定，不同的容器标准之间有些可以兼容，这些兼容的可以互相转换，对于不兼容的容器标准，需要改变编码格式才能互相转换，一般使用效率高，兼容性好的标准。

# 容器格式和编码格式

  • 文件是既包括视频又包括音频、甚至还带有脚本的一个集合，也可以叫容器；容器格式即 文件格式
  • 文件当中的视频和音频的压缩算法才是具体的编码格式
  • 例如：一个.avi 文件，当中的视频可能是编码 a，也可能是编码 b，音频可能是编码 5，也可能是编码 6，具 体的用那种编码的解码器，则由播放器按照 avi 文件格式读取信息去调用了。

# 容器标准

  • RMVB 过于封闭
  • MKV 几乎支持所有的音视频，但是体积过大
  • AVI 过时，不支持当下主流的媒体内容
  • MP4 主流 目前最主流的封装格式，软硬件厂商的支持都非常好，一般都是将其他格式文件转为MP4
  • MOV 主流的容器标准，由苹果公司制定，并支持透明视频

音视频方向两大标准制定组织

  • ISO
  • ITU

# 编码

​ 即压缩，像zip、rar一样，它们的目的都是为了压缩文件体积，便于传输和存储，ffmpeg支持丰富的编码方式，可以使用命令行ffmpeg -codecs来查看支持的编码方式。

​ 至于编码方式的选择，我们直接采用最主流的方式即可：

首选：MP4封装：H264视频编码 + AAC音频编码

常见的音频视频编码

  • YouTuBe使用的方式（效率不高）：WebM封装：VP8视频编码 + Vorbis音频编码

  • 完全开源的视频编码：OGG封装：Theota视频编码 + Vorbis音频编码

  • MPEG 系列：（由 ISO 下属的 MPEG[运动图象专家组]开发 ）

    • 视频编码方面主要是 Mpeg1（vcd 用的就是它）、Mpeg2（DVD 使用）、Mpeg4（现在的 DVDRIP 使用的都是它的变种，如：divx，xvid 等）、Mpeg4 AVC（现在正热门）；

# IBP 帧

# 基本概念
  • P frame: 前向预测编码帧 又称 predictive-frame，通过充分将低于图像序列中前面已编码帧的时间冗余信息来压缩传输数据量的编码图像，也叫预测帧；
  • B frame: 双向预测内插编码帧 又称 bi-directional interpolated prediction frame，既考虑与源图像序列前面已编码帧，也顾及源图像序列后面已编码帧之间的时间冗余信息来压缩传输数据量的编码图像，也叫双向预测帧；
# IBP 帧特点

​ I、B、P 各帧是根据压缩算法的需要，是人为定义 它们都是实实在在的物理帧，至于图像中的哪一帧是 I 帧,是随机的,一但确定了 I 帧,以后的各帧就严格按规定顺序排列

  • I 帧特点：

    • 它是一个全帧压缩编码帧。它将全帧图像信息进行 JPEG 压缩编码及传输;
    • 解码时仅用 I 帧的数据就可重构完整图像;
    • I 帧描述了图像背景和运动主体的详情;
    • I 帧不需要参考其他画面而生成;
    • I 帧是 P 帧和 B 帧的参考帧(其质量直接影响到同组中以后各帧的质量);
    • I 帧是帧组 GOP 的基础帧(第一帧),在一组中只有一个 I 帧;
    • I 帧不需要考虑运动矢量;
    • I 帧所占数据的信息量比较大。

  • P 帧:前向预测编码帧

    P 帧的预测与重构：P 帧是以 I 帧为参考帧,在 I 帧中找出 P 帧“某点”的预测值和运动矢量, 取预测差值和运动矢量一起传送。在接收端根据运动矢量从 I 帧中找出 P 帧“某点”的预测值并与差值相加以得到 P 帧“某点”样值,从而可得到完整的 P 帧

    • P 帧是 I 帧后面相隔 1~2 帧的编码帧;
    • P 帧采用运动补偿的方法传送它与前面的 I 或 P 帧的差值及运动矢量(预测误差);
    • 解码时必须将 I 帧中的预测值与预测误差求和后才能重构完整的 P 帧图像;
    • P 帧属于前向预测的帧间编码。它只参考前面最靠近它的 I 帧或 P 帧;
    • P 帧可以是其后面 P 帧的参考帧,也可以是其前后的 B 帧的参考帧;
    • 由于 P 帧是参考帧,它可能造成解码错误的扩散;
    • 由于是差值传送,P 帧的压缩比较高

  • B 帧:双向预测内插编码帧

    B 帧的预测与重构：B 帧以前面的 I 或 P 帧和后面的 P 帧为参考帧,“找出”B 帧“某点”的预测值和两个运动矢量,并取预测差值和运动矢量传送。接收端根据运动矢量在两个参考帧中“找出(算出)”预测值并与差值求和,得到 B 帧“某点”样值,从而可得到完整的B帧

    • B 帧是由前面的 I 或 P 帧和后面的 P 帧来进行预测的;
    • B 帧传送的是它与前面的 I 或 P 帧和后面的 P 帧之间的预测误差及运动矢量;
    • B 帧是双向预测编码帧;
    • B 帧压缩比最高,因为它只反映丙参考帧间运动主体的变化情况,预测比较准确;
    • B 帧不是参考帧,不会造成解码错误的扩散

# GOP（Group of Picture，画面组）

GOP（Group of Pictures）策略影响编码质量：所谓 GOP，意思是画面组，一个 GOP 就是一组连续的画面。MPEG 编码将画面（即帧）分为 I、P、B 三种，I 是内部编码帧，P 是前向预测帧，B 是双向内插帧；I 帧是一个完整的画面，而 P 帧和 B 帧记录的是相对于 I 帧的变化。

MPEG-2 压缩的帧结构有两个参数，一个是 GOP（Group Of Picture）图像组的长度，一般可按编码方式从 1－15；另一个是 I 帧和 P 帧之间 B 帧的数量，一般是 1－2 个。前者在理论上记录为 N，即多少帧里面出现一次 I 帧；后者描述为多少帧里出现一次 P 帧，记录为 M；

# 分辨率（指两个概念）

  • 物理分辨率, 即手机屏幕能显示的像素数，用 W x H 个像素表示。常见的手机屏幕分辨率为 320x240(QVGA), 随着大屏幕手机的普及， 更高的分辨率也开始出现. 例如: 480x320(iphone),640x360(nHD, 诺基亚触屏系列常见),640x480(VGA, 多普达系列常见), 甚至高达 852x480(夏普高端手机常见).
  • 视频文件的分辨率， 这个是指视频画面的实际分辨率, 如， 320x240, 480x272, 640x480 等等。一般来说，大部分手机的解码芯片不支持超过其屏幕物理分辨率的视频, 部分可以支持超过其屏幕物理分辨率的视频， 例如, 虽然 iphone 的屏幕物理分辨率为 480x320, 但它支持 640x480 的视频， 此时播放的画面实际是把原视频缩小的

(FPS, 帧/秒), 就是视频画面刷新的速度， 作为参考, 国内电视机一般是 25FPS, 电影标准为 24FPS. 手机芯片，最高支持 30FPS, 早期型号最大只能 15fps。

ffmpeg支持的码率控制模式：

# 恒定量化器模式

​ 利用特殊算法将画质转化为数字，每一帧画面的质量都可以通过量化的参数来判定，当然这些量化参数可以手动设置，但是不建议手动设置，最好使用软件自动的设置

恒定量化器模式下，画面被分为0—51个级别：

  • 0 无损画质，画质最好，但是体积最大。一般不使用，除非需要无损压缩视频
# 无损压缩的例子（快速编码）
ffmpeg -i input -vcodec libx264 -preset ultrafast -qp 0 output.mkv
# 无损压缩的例子（高压缩比）
ffmpeg -i input -vcodec libx264 -preset veryslow -qp 0 output.mkv

​ 浮动的恒定量化器模式，按照人体肉眼的特点，给予每帧画面不同的量化参数。

# 固定目标码率模式

​ 固定目标码率模式下，编码器的码率和视图会与最终文件的整体码率会跟我们给定的码率相等，如此一来，视频码率和体积是确定的。

固定目标码率模式适用于码率和体积有限制的情况。一般的视频网站会有此限制， 若需要用户在上传视频时，需要视频达到最高画质，可以使用此种码率控制模式。

一般情况下不使用此码率模式，除非需要制作极其小的视频，或者遇到对文件体积有严格限制的场景。

恒定量化器模式、浮动的恒定量化器模式、固定目标码率模式。这三种模式默认单遍编码

# RTMP

RTMP 是 Real Time Messaging Protocol（实时消息传输协议）的首字母缩写。是 Adobe 公司开发的一个基于 TCP 的应用层协议，也就是说，RTMP 是和 HTTP/HTTPS 一样，是应用层的一个协议族。RTMP 在 TCP 通道上一般传输的是 flv 格式流。请注意，RTMP 是网络传输协议，而 flv 则是视频的封装格式。flv 封装格式设计出来的目的是为了用于网络传输使用的，因此 RTMP+FLV 可以说是”黄金搭档“。

RTMP 协议包括：基本协议及 RTMPT/RTMPS/RTMPE 等多种变种。RTMP 协议家族有以下几个点挺有趣，读者们不妨看看：

  • RTMP 工作在 TCP 之上，默认使用端口 1935；
  • RTMPE 在 RTMP 的基础上增加了加密功能；
  • RTMPT 封装在 HTTP 请求之上，可穿透防火墙；
  • RTMPS 类似 RTMPT，增加了 TLS/SSL 的安全功能；
  • RTMFP 使用 UDP 进行传输的 RTMP；

RTMP 就是专门针对多媒体数据流的实时通信设计出来的一种网络数据传输协议，主要用来在 Flash/AIR 平台和支持 RTMP 协议的流媒体/交互服务器之间进行音视频和数据通信。现在 Adobe 公司已经不支持了，不过目前，该协议还在广泛使用。

    /***
     * 视频 + 音频的转换
     * 若音频过长，视频重播
     * 视频过长，静音
     * @param videoUrlStr 已经上传的网络地址
     * @param audioUrlStr 需要合并的录音地址
     * @param id 唯一id
     * @return 生成为文件路径
     */
    public static String doConvertWithVideoAndAudio(String videoUrlStr, String audioUrlStr, String id) {
        FFmpegFrameGrabber videoFrameGrabber = null;
        FFmpegFrameGrabber audioFrameGrabber = null;
        // 合成完成的视频
        String returnVideoFileName = TEMPORARY_FOLDER + "returnVideoFile_" + id + ".mp4";
        try {
            createTmpDir();
            System.out.println("===>> 加载视频文件中....");
            URL videoUrl = new URL(videoUrlStr);
            videoFrameGrabber = new FFmpegFrameGrabber(videoUrl.openStream());
            videoFrameGrabber.setOption(TimeoutOption.STIMEOUT.getKey(), String.valueOf(TIMEOUT * 1000000));
            videoFrameGrabber.start();

            // 先判断是否有音频 视频
            boolean hasVideo = videoFrameGrabber.hasVideo();
            boolean hasAudio = videoFrameGrabber.hasAudio();
            System.out.println("=== 是否存在视频" + hasVideo);
            System.out.println("=== 是否存在音频" + hasAudio);

            System.out.println("===>> 加载视频文件中....");
            URL audioUrl = new URL(audioUrlStr);
            audioFrameGrabber = new FFmpegFrameGrabber(audioUrl.openStream());
            audioFrameGrabber.setOption(TimeoutOption.STIMEOUT.getKey(), String.valueOf(TIMEOUT * 1000000));
            audioFrameGrabber.start();

            // 2-计算音频视频长度，判断是否重复播放视频
            // duration 总时长 单位：微秒
            long audioDuration = audioFrameGrabber.getFormatContext().duration();
            long videoDuration = videoFrameGrabber.getFormatContext().duration();
            long repeatCount = 1;
            long cutMicroseconds = 0;
            System.out.println("===>> 音频总时长：" + audioDuration);
            System.out.println("===>> 视频总时长：" + videoDuration);

            if (audioDuration > videoDuration) {
                // 计算截取的秒
                repeatCount = audioDuration / videoDuration;
                cutMicroseconds = audioDuration - (videoDuration * repeatCount);
                System.out.println("=====>> 截取的时长：" + cutMicroseconds);
                System.out.println("=====>> 需要重复：" + repeatCount + " 次=====");
            }

            // 3-合成音频视频
            System.out.println("== 合成视频和音频 ===================================");
            mergeWithAudioAndVideo(videoFrameGrabber, audioFrameGrabber, returnVideoFileName, repeatCount, cutMicroseconds);
            System.out.println("== 任务结束 ===================================");
        } catch (IOException e) {
            System.out.println("== 任务异常 ===================================");
            e.printStackTrace();
        } finally {
            try {
                if (videoFrameGrabber != null) {
                    videoFrameGrabber.close();
                }
                if (audioFrameGrabber != null) {
                    audioFrameGrabber.close();
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
        return returnVideoFileName;
    }

    public static boolean cutVideo(FFmpegFrameGrabber grabber, String videoFolder, long start, long end) {
        String videoFormat = "mp4";
        int videoCodec = avcodec.AV_CODEC_ID_MPEG4;
        double frameRate = grabber.getFrameRate();
        int pixelFormat = avutil.AV_PIX_FMT_YUV420P;
        try {
            FFmpegFrameRecorder recorder = new FFmpegFrameRecorder(videoFolder, grabber.getImageWidth(), grabber.getImageHeight());
            recorder.setVideoCodec(videoCodec);
            recorder.setFormat(videoFormat);
            recorder.setFrameRate(frameRate);
            recorder.setPixelFormat(pixelFormat);

            // 获取样本，并且用recorder记录
            recorder.start();
            Frame frame;
            grabber.restart();
            int i = 0;
            while ((frame = grabber.grabFrame()) != null) {
                if (i <= start && i <= end) {
                    recorder.record(frame);
                }
                i++;
            }

            recorder.close();
            return true;
        } catch (Exception e) {
            e.printStackTrace();
        }

        return false;
    }


    /**
     * 视频截取 + 合并
     * 结果：原视频 + 原视频 + 原视频 +... + 剪切的视频
     * @param videoGrabber 全部视频
     * @param count  全部视频需要重复的次数
     * @param videoFolder 生成文件的目录
     * @return
     */
    public static boolean cutAfterMergeVideo(FFmpegFrameGrabber videoGrabber, long start, long end, int count,
                                             String videoFolder) {
        String videoFormat = "mp4";
        int videoCodec = avcodec.AV_CODEC_ID_MPEG4;
        double frameRate = videoGrabber.getFrameRate();
        int pixelFormat = avutil.AV_PIX_FMT_YUV420P;
        try {
            // 结果
            FFmpegFrameRecorder recorder = new FFmpegFrameRecorder(videoFolder, videoGrabber.getImageWidth(), videoGrabber.getImageHeight());
            recorder.setVideoCodec(videoCodec);
            recorder.setFormat(videoFormat);
            recorder.setFrameRate(frameRate);
            recorder.setPixelFormat(pixelFormat);

            // 获取样本 重复播放
            recorder.start();
            Frame frame;
            for (int j = 0; j < count; j++) {
                videoGrabber.restart();
                while ((frame = videoGrabber.grabFrame()) != null) {
                    recorder.record(frame);
                }
            }

            // 获取样本 重复播放最后一点
            videoGrabber.restart();
            while ((frame = videoGrabber.grabFrame()) != null) {
                // 过滤掉开始帧之前的帧
                if (frame.timestamp < start) {
                    continue;
                }
                // 超过结束帧位置了及跳出while
                if (frame.timestamp >= end) {
                    break;
                }
                recorder.record(frame);
            }

            recorder.close();
            return true;
        } catch (Exception e) {
            e.printStackTrace();
        }

        return false;
    }

    /**
     * 音频视频合并
     */
    public static void mergeWithAudioAndVideo(FrameGrabber videoGrabber, FrameGrabber audioGrabber, String videoFolder,
                                              long repeatCount, long cutMicroseconds) {
        String videoFormat = "mp4";
        try {
            // 创建生成有声视频的 recorder
            FFmpegFrameRecorder recorder = new FFmpegFrameRecorder(videoFolder,
                    videoGrabber.getImageWidth(),
                    videoGrabber.getImageHeight(),
                    audioGrabber.getAudioChannels());
            // recorder.setVideoCodec(videoGrabber.getVideoCodec());
            recorder.setVideoCodec(avcodec.AV_CODEC_ID_H264);
            // recorder.setVideoCodec(avcodec.AV_CODEC_ID_MPEG4);

            // 设置格式 mp4
            recorder.setFormat(videoFormat);
            // 设置图像帧率
            recorder.setFrameRate(videoGrabber.getFrameRate());
            // 设置音频采样率
            recorder.setSampleRate(audioGrabber.getSampleRate());
            // 设置像素格式
            recorder.setPixelFormat(avutil.AV_PIX_FMT_YUV420P);
            // 采样率
            recorder.setSampleRate(44100);
            // recorder.setAudioBitrate(192000);
            recorder.setAudioCodec(avcodec.AV_CODEC_ID_AAC);
            recorder.setOption("preset", "veryslow");
            // 开始记录
            recorder.start();
            Frame frame;
            // 记录视频帧
            for (int i = 0; i < repeatCount; i++) {
                videoGrabber.restart();
                while ((frame = videoGrabber.grabFrame()) != null) {
                    if (frame.getTypes().contains(Frame.Type.VIDEO)) {
                        recorder.record(frame);
                    }
                }
            }
            // 获取样本 重复播放最后一点
            videoGrabber.restart();
            while ((frame = videoGrabber.grabFrame()) != null) {
                // 过滤掉开始帧之前的帧
                if (!frame.getTypes().contains(Frame.Type.VIDEO) || frame.timestamp < 0) {
                    continue;
                }
                // 超过结束帧位置了及跳出while
                if (frame.timestamp >= cutMicroseconds) {
                    break;
                }
                recorder.record(frame);
            }

            // 记录音频帧
            while ((frame = audioGrabber.grabFrame()) != null) {
                if (frame.getTypes().contains(Frame.Type.AUDIO)) {
                    // recorder.setTimestamp(frame.timestamp);
                    recorder.record(frame);
                }
            }

            // 完成录入
            recorder.stop();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            try {
                videoGrabber.stop();
                audioGrabber.stop();
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }
