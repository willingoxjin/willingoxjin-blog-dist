Java网络编程

# RMI远程调用

# 什么是RMI？

RMI（Rmote Method Invocation）指的是远程方法调用。即一个JVM中的代码可以通过网络实现远程调用另一个JVM的某个方法。通俗的说就是把编译好的class文件放到一个服务器中，我们的JAVA代码（客户端）运行时通过远程调用的方式来访问``class`文件，并将其实例化。

提供服务的一方我们称之为服务器，而实现远程调用的一方我们称之为客户端。

远程方法调用是一种计算机之间对象互相调用对方函数，启动对方进程的一种机制，使用这种机制，某一台计算机上的对象在调用另外一台计算机上的方法时，使用的程序语法规则和在本地机上对象间的方法调用的语法规则一样

# 调用原理

客户端（Client）的“实现类”（客户端代理）称为stub，而服务器端（Server）的网络服务类（服务端代理）称为skeleton

注册

服务端需要先启动RMI Registry，然后将远程对象注册到 RMI Register

调用

Client通过访问 RMI Register查找注册的远程对象。

当Client获取到对应的远程对象时，就会通过stub调用远程方法，但是此时不是直接调用，而是调用skeleton

skeleton接收到调用后，会调用Server端的具体实现，获取结果，然后把结果通过网络传递给Client端。整个过程由RMI底层负责实现序列化和反序列化；

RMI用到的类：

java.rmi.Remote 所有可以被远程调用的对象都必须实现该接口 java.rmi.server.UnicastRemoteObject 所有可以被远程调用的对象都必须扩展该类

如果看不明白就对了，还是结合代码示例来理解吧！

# 简单示例

# 共享接口

创建一个Maven项目rmi-api

要实现RMI，服务器和客户端必须共享同一个接口。我们定义一个WorldClock接口，继承java.rmi.Remote

代码如下：

public interface TimeService extends Remote {
	
    /**
     * 打印当前时间
     */
    void printCurrentTime() throws RemoteException;

    /**
     * 根据市区id获取当前时间
     */
    LocalDateTime getLocalDateTime(String zoneId) throws RemoteException;
}

下一步是编写服务器的实现类，因为客户端请求的调用方法getLocalDateTime()最终会通过这个实现类返回结果。实现类WorldClockService代码如下：

public class TimeServiceImpl implements TimeService{

    @Override
    public void printCurrentTime() throws RemoteException {
        System.out.println(LocalDateTime.now());
    }

    @Override
    public LocalDateTime getLocalDateTime(String zoneId) throws RemoteException {
        return LocalDateTime.now(ZoneId.of(zoneId)).withNano(0);
    }

}

我们需要把该接口以RMI的形式暴露在网络中，以便客户端能够访问的到该接口并调用

Java RMI提供的一系列底层支持接口：

public class Server {
    public static void main(String[] args) throws RemoteException {
        System.out.println("create remote service...");
        // 实例化一个TimeService:
        TimeService timeService = new TimeServiceImpl();
        // 将此服务转换为远程服务接口:
        TimeService skeleton = (TimeService) UnicastRemoteObject.exportObject(timeService, 0);
        // 将RMI服务注册到1099端口（RMI服务器默认1099端口）:
        Registry registry = LocateRegistry.createRegistry(1099);
        // 注册此服务，服务名为"time":
        registry.rebind("TimeService", skeleton);
        System.out.println("remote started port 1099");
    }
}

客户端调用远程RMI服务，来获取到对应的实例对象

public class Client {
    public static void main(String[] args) throws RemoteException, NotBoundException {
        // 连接到服务器localhost，端口1099:
        Registry registry = LocateRegistry.getRegistry("localhost", 1099);
        // 查找名称为"WorldClock"的服务并强制转型为WorldClock接口:
        TimeService timeService = (TimeService) registry.lookup("TimeService");
        // 正常调用接口方法:
        timeService.printCurrentTime();
        LocalDateTime now = timeService.getLocalDateTime("Asia/Shanghai");
        // 打印调用结果:
       System.out.println("Client getLocalDateTime: " + now);
    }
}

先运行服务器，再运行客户端。从运行结果可知，因为客户端只有接口，并没有实现类，因此，客户端获得的接口方法返回值实际上是通过网络从服务器端获取的。

客户端打印：

Client getLocalDateTime: 2021-12-14T09:59:54
1

服务端打印：

2021-12-14T09:59:54.371
1

根据上面的打印结果发现，printCurrentTime()中的打印日志是在服务端输出的，说明实例化运行对象，并且方法运行过程都是在服务端执行的，而客户端只接收服务端方法调用后的返回值。

整个过程实际上非常简单，

  1. Registry先启动，并监听一个端口，一般为1099（1099为默认端口）
  2. Server向Registry注册远程对象
  3. 发起调用
    • 客户端：客户端从Registry获得远程对象的代理（这个代理知道远程对象的在网络中的具体位置：ip、端口、标识符。然后Client通过这个代理调用远程方法
    • 服务端：服务端的代理会收到客户端的调用的方法、参数等，然后代理执行对应方法，并将结果通过网络返回给Client。

在编写代码时注意：

  1. 接口需要派生自Remote接口，且方法必须需要抛出RemoteException异常
  2. 接口的实现类需要继承UnicastRemoteObject，同样的方法需要抛出RemoteException异常
  3. 如果远程方法需要传参，需要保证参数是可序列化的，如果传参是自定义的对象，那么这个对象需要实现Serilizable接口

# 注意

Java的RMI严重依赖序列化和反序列化，而这种情况下可能会造成严重的安全漏洞，因为Java的序列化和反序列化不但涉及到数据，还涉及到二进制的字节码，即使使用白名单机制也很难保证100%排除恶意构造的字节码。因此，使用RMI时，双方必须是内网互相信任的机器，不要把1099端口暴露在公网上作为对外服务。

此外，Java的RMI调用机制决定了双方必须是Java程序，其他语言很难调用Java的RMI。如果要使用不同语言进行RPC调用，可以选择更通用的协议，例如gRPC (opens new window)。
