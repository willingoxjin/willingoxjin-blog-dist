Java Agent

# 一、Java Agent 介绍

Java Agent 是从 JDK1.5 及以后引入的，也可以叫做 java 代理。

代理 (agent) 是指在你的 main() 方法前的一个拦截器，即在 main 方法执行之前，执行 agent 的代码。

agent 的代码与 main 方法在同一个 JVM 中运行，并被同一个 system classloader 装载，被同一的安全策略 (security policy) 和上下文（context）所管理。

# 二、Java Agent 的实现与使用

# 第一步：编写 Java Agent 代码

在 JDK 1.5 和 JDK1.6 中使用 Java Agent 只需要实现 premain 方法即可：

public static void premain(String agentArgs, Instrumentation inst)

// 如果找不到上面的这种premain的定义，会尝试调用下面的这种premain定义：
public static void premain(String agentArgs)

package org.willingoxjin.monitor;

import java.lang.instrument.Instrumentation;
 
public class MyAgent {
	public static void premain(String agentOps, Instrumentation inst) {
		System.out.println(agentOps);
	}
}

# 第二步：将代理类打包

写完这个代理类后，需要打成 jar 包，在 jar 包里面的 META-INF/MAINIFEST.MF 的文件必须包含 Premain-Class 这个属性。如下：

Manifest-Version: 1.0
Premain-Class: org.willingoxjin.monitor.MyAgent
Can-Redefine-Classes: true

需要注意的是， META-INF/MAINIFEST.MF 文件的第四行必须是空行，这一行必须有，否则会报错。

# 第三步：使用代理

在一个 Java 应用中，-javaagent 这个参数的个数是不限的，我们可以指定多个 javaagent，所有的 javaagent 会按顺序执行，执行完各个 agent 之后，再执行 main 方法。

如：

java -javaagent:/opt/tools/myagent1.jar \
-javaagent:/opt/tools/myagent2.jar \
-javaagent:/opt/tools/myagent3.jar \
 -jar myprogram.jar

那么执行顺序就是：myagent1 > myagent2 > myagent3 > myprogram.main（当前运行 jar 的主程序）

注意：放在 main 方法之后的 premain 方法是不会被执行的

执行顺序：myagent1 > myprogram > myagent2 > myagent3

每一个 Java Agent 都可以接收一个字符串类型的参数，也就是premain() 方法中的 agentArgs 参数。 该 agentArgs 参数是通过 java option 定义的。

如：

java -javaagent:myagent.jar=thisIsAgentArgs -jar MyProgram.jar
1

myagent 中 premain()方法接收到的 agentArgs 参数的值将是 thisIsAgentArgs

Instrumentation 参数 ：通过参数中的 Instrumentation inst，添加自己定义的 ClassFileTransformer，来改变 class 文件。

通过 Java Agent 就可以不用修改原有的 Java 程序代码，通过 Agent 的形式来修改或者增强程序了，或者做热启动等等。
