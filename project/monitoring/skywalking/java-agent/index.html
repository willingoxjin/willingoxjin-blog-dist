Skywalking — Java Agent

# 一、Java Agent 配置

# 1.Java Agent 配置属性

Java Agent 支持的配置属性可在官方文档中查看：https://skywalking.apache.org/docs/skywalking-java/v8.13.0/en/setup/service-agent/java-agent/configurations/ (opens new window)（根据自己使用 Java Agent 的版本进行查看，这个链接是 8.13.0 的版本）

Java Agent 的属性配置可在 agent/config/agent.config 配置文件中配置

# 2.配置方式

Java Agent 配置方式有很多种：

# 配置方式一：系统属性(-D)

使用 -Dskywalking.[agent.config配置文件中的key]=[配置内容] 即可。

例如：agent.config 文件中有一个属性名为 agent.service_name ，那么如果使用系统属性的方式，则可以写成

java -javaagent:/opt/agent/skywalking-agent.jar \
-Dskywalking.agent.service_name=[你的服务名称] \
-Dskywalking.agent.namespace=[namespace] \
-jar skywalking-test-api.jar

常用配置：

  • skywalking.agent.service_name 追踪的服务名称
  • skywalking.agent.namespace 命名空间
  • skywalking.collector.backend_servic 服务端oap地址
  • 其他可以参考 skywalking-agent 包目录下的config/agent.config ，只需将配置前添加 skywalking 即可。如文件中的 agent.service_name 配置，在 -javaagent -D 参数中使用就是： skywalking.agent.service_name。

# 配置方式二：代理选项

在JVM参数中的代理路径之后添加属性即可。

格式如下：

agent.config 文件中默认的大写值，都可以作为环境变量引用。

如：

# 启动参数

代理选项 > 系统属性（-D） > 环境变量 > 配置文件

# 3.配置示例：设置采样并打印 SQL

SkyWalking 默认情况下是不采样的，所有的监控数据都会上报给 SkyWalking。如果在并发很高服务中，或并发很高的时间内，所有性能监控数据都上报到 SkyWalking，会影响服务器的性能，耗用服务器资源。

需求：

  • 设置采样（3 秒采样一次）
  • 打印 SQL 信息，包括 SQL 的参数（默认 SkyWalking 会收集执行的 SQL，但是不会收集其参数）

SkyWalking 中采样率若不设置就是100% 采样。

生产环境下一般不会设置采样率为 100%，除非， 100% 的采样率对服务没有什么影响。

这里使用 -D 的方式配置：

java -javaagent:/opt/agent/skywalking-agent.jar \
-Dskywalking.agent.service_name=skywalking-test-api \
-Dskywalking.agent.sample_n_per_3_secs=1 \
-Dskywalking.plugin.jdbc.trace_sql_parameters=true \
-Dskywalking.plugin.jdbc.sql_parameters_max_length=1024 \
-jar skywalking-test-api.jar

参数说明：

  • skywalking.agent.service_name 服务名称
  • skywalking.agent.sample_n_per_3_secs 默认情况下，负或零表示关闭。SAMPLE_N_PER_3_SECS 表示在3秒内对 N TraceSegment 进行采样。
  • skywalking.plugin.jdbc.trace_sql_parameters 如果设置为true，将收集 sql 的参数（通常是 java.sql.PreparedStatement）。
  • skywalking.plugin.jdbc.sql_parameters_max_length 如果设置为正数，db.sql.parameters 将被截断为该长度，否则将被完全保存，这可能会导致性能问题。

启动 jar 后，请求接口（连续请求），在 SkyWalking 中的 web 界面查看，发现数据时间隔了三秒才被收集到了，点击追踪中的 Mysql/JDBI/PreparedStatement/execute ，发现可以看到相关的sql 的参数

# 二、Java Agent 插件

Java Agent 插件 文档：https://skywalking.apache.org/docs/skywalking-java/v8.13.0/readme/ (opens new window)

# 1.Java Agent 插件介绍

Java Agent 是插件化、可插拔的。SkyWalking 的插件分为三种：

  • 内置插件：在 agent 的 plugins 目录下
  • 引导插件：在 agent 的 bootstrap-plugins 目录下
  • 可选插件：在 agent 的 optional-plugins 目录下

Java Agent 只会启用 plugins 目录下的所有插件，bootstrap-plugins 目录以及 optional-plugins 目录下的插件不会启用。

如需启用引导插件或可选插件，只需将 jar 包手动移到 plugins 目录下。如需禁用某款插件，只需从 plugins 目录中移除即可。

根据类型划分：

  • 跟踪插件（Tracing plugin (opens new window)）：遵循分布式跟踪概念，使用标记和日志收集跨。

  • 计数器插件（Meter plugin (opens new window)）：收集计数器、度量和直方图格式的数字指标。

# 2.插件生态

# 内置插件

内置插件主要用来为业界主流的技术与框架提供支持。所支持的技术&框架，详见 https://skywalking.apache.org/docs/skywalking-java/v8.13.0/en/setup/service-agent/java-agent/supported-list/ (opens new window)。

# 引导插件

目前只有两款引导插件：

  • apm-jdk-http-plugin 用来是监测 HttpURLConnection；
  • apm-jdk-threading-plugin 用来监测 Callable 以及 Runnable；
  • apm-jdk-threadpool-plugin 用来监测 ThreadPoolExecutor；

有关引导插件的功能描述，可详见：https://skywalking.apache.org/docs/skywalking-java/v8.13.0/en/setup/service-agent/java-agent/bootstrap-plugins/ (opens new window) 。

# 可选插件

关于支持的可选插件及功能描述，见 https://skywalking.apache.org/docs/skywalking-java/v8.13.0/en/setup/service-agent/java-agent/optional-plugins/ (opens new window) 。

# 插件扩展

SkyWalking 生态还有一些插件扩展，例如 Oracle、Resin 、Aliyun ONS 插件。这部分插件主要是由于开源许可证的限制，SkyWalking 无法将这部分插件直接打包到 SkyWalking 安装包内，于是托管在这个地址： https://github.com/SkyAPM/java-plugin-extensions (opens new window) 。

使用方式：

  • 前往 https://github.com/SkyAPM/java-plugin-extensions/releases (opens new window) ，下载插件JAR包
  • 将 jar 包挪到 plugins 目录即可启用。

# 2.相关概念

# Span

Span 是分布式追踪系统中一个重要且常用的概念. 可从 Google Dapper Paper (opens new window) 和 OpenTracing (opens new window) 学习更多与 Span 相关的知识.

SkyWalking 从 2017 年开始支持 OpenTracing 和 OpenTracing-Java API，SkyWalking 的 Span 在概念上与谷歌论文和 OpenTracing 里描述的类似。

Span 有三种类型：

  • EntrySpan：EntrySpan 代表服务提供者, 也是服务器端的端点. 作为一个 APM 系统, 我们的目标是应用服务器. 所以几乎所有的服务和 MQ-消费者 都是 EntrySpan。

  • LocalSpan：LocalSpan 表示普通的 Java 方法, 它与远程服务无关, 不是 MQ 生产者/消费者, 也不是服务（例如 HTTP 服务）提供者/消费者。

  • ExitSpan：ExitSpan 代表一个服务客户端或 MQ 的生产者，在 SkyWalking 的早期命名为 LeafSpan. 例如 通过 JDBC 访问 DB, 读取 Redis/Memcached 被归类为 ExitSpan。

Span 可理解为一次方法调用，一个程序块的调用，或一次RPC/数据库访问。只要是一个具有完整时间周期的程序访问，都可以被认为是一个 Span。

SkyWalking 中的 Span 对象的重要属性：

属性 名称 备注
component 组件 插件的组件名称，如：Lettuce，详见:ComponentsDefine.Class。
tag 标签 k-v结构，关键标签，key详见：Tags.Class。
peer 对端资源 用于拓扑图，若DB组件，需记录集群信息。
operationName 操作名称 若span=0，operationName将会搜索的下拉列表。
layer 显示 在链路页显示，详见SpanLayer.Class。

# Trace

调用链，通过归属于其的 Span 来隐性的定义。一条 Trace 可被认为是一个由多个 Span 组成的有向无环图（DAG图），在 SkyWalking 链路模块你可以看到，Trace 又由多个归属于其的 trace segment 组成。

# Trace segment

Segment 是 SkyWalking 中的一个概念，它应该包括单个 OS 进程中每个请求的所有范围，通常是基于语言的单线程。由多个归属于本线程操作的 Span 组成。

TIPS

Skywalking 的这些概念和 Spring Cloud Sleuth 类似，借鉴自谷歌的Dapper。

# 上下文载体（ContextCarrier）

为了实现分布式追踪，需要绑定跨进程的追踪，并且上下文应该在整个过程中随之传播. 这就是 ContextCarrier 的职责.

以下是有关如何在 A -> B 分布式调用中使用 ContextCarrier 的步骤.

  1. 在客户端, 创建一个新的空的 ContextCarrier.
  2. 通过 ContextManager#createExitSpan 创建一个 ExitSpan 或者使用 ContextManager#inject 来初始化 ContextCarrier.
  3. 将 ContextCarrier 所有信息放到请求头（如 HTTP HEAD）， 附件（如 Dubbo RPC 框架）或者消息（如 Kafka）中
  4. 通过服务调用，将 ContextCarrier 传递到服务端.
  5. 在服务端，在对应组件的头部, 附件或消息中获取 ContextCarrier 所有内容.
  6. 通过 ContestManager#createEntrySpan 创建 EntrySpan 或者使用 ContextManager#extract 来绑定服务端和客户端.

通过阅读 Apache HttpComponent Client 插件和 Tomcat 7 服务器插件的源码进行理解， 步骤如下:

  • 客户端 Apache HttpComponent Client 插件
span = ContextManager.createExitSpan("/span/operation/name", contextCarrier, "ip:port");
CarrierItem next = contextCarrier.items();
while (next.hasNext()) {
    next = next.next();
    httpRequest.setHeader(next.getHeadKey(), next.getHeadValue());
}

# 上下文快照（ContextSnapshot）

除了跨进程，跨线程也是需要支持的。例如异步线程（内存中的消息队列）和批处理在 Java 中很常见。跨进程和跨线程十分相似，因为都是需要传播上下文。他们唯一的区别就是是跨线程不需要序列化。

以下是有关跨线程传播的三个步骤：

  1. 使用 ContextManager#capture 方法获取 ContextSnapshot 对象.
  2. 让子线程以任何方式, 通过方法参数或由现有参数携带来访问 ContextSnapshot
  3. 在子线程中使用 ContextManager#continued.

# 3.核心 API

# 上下文管理器 (ContextManager)

ContextManager 提供所有主要 API。

  1. 创建 EntrySpan：根据操作名称（例如服务名称、uri）和 上下文载体 (ContextCarrier) 创建 EntrySpan。
public static AbstractSpan createEntrySpan(String endpointName, ContextCarrier carrier)
1
  1. 创建 LocalSpan：根据操作名称（例如完整的方法签名）创建 LocalSpan。
public static AbstractSpan createLocalSpan(String endpointName)
1
  1. 创建 ExitSpan：根据操作名称(例如服务名称、uri)、上下文载体 (ContextCarrier) 以及对等端 （peer）地址。例如 ip + port 或 hostname + port创建 ExitSpan.
public static AbstractSpan createExitSpan(String endpointName, ContextCarrier carrier, String remotePeer)
1

# AbstractSpan

除了设置操作名称，标签信息和日志外，还要设置两个属性，即 component（组件）和 layer（层），特别是对于 EntrySpan 和 ExitSpan。

/**
 * Set the component id, which defines in {@link ComponentsDefine}
 *
 * @param component
 * @return the span for chaining.
 */
AbstractSpan setComponent(Component component);

/**
 * Only use this method in explicit instrumentation, like opentracing-skywalking-bridge.
 * It it higher recommend don't use this for performance consideration.
 *
 * @param componentName
 * @return the span for chaining.
 */
AbstractSpan setComponent(String componentName);

AbstractSpan setLayer(SpanLayer layer);

/**
 * Set a key:value tag on the Span.
 *
 * @return this Span instance, for chaining
 */
AbstractSpan tag(String key, String value);

/**
 * Record an exception event of the current walltime timestamp.
 *
 * @param t any subclass of {@link Throwable}, which occurs in this span.
 * @return the Span, for chaining
 */
AbstractSpan log(Throwable t);

# 异步 Span API

Span 有一系列的高级 API，他们都是在异步场景下使用的。当 Span 的 tag、日志和属性（包括结束时间）需要在另一个线程中设置时，你就应该使用这些 API。

/**
 * The span finish at current tracing context, but the current span is still alive, until {@link #asyncFinish}
 * called.
 *
 * This method must be called<br/>
 * 1. In original thread(tracing context).
 * 2. Current span is active span.
 *
 * During alive, tags, logs and attributes of the span could be changed, in any thread.
 *
 * The execution times of {@link #prepareForAsync} and {@link #asyncFinish()} must match.
 *
 * @return the current span
 */
AbstractSpan prepareForAsync();
/**
 * Notify the span, it could be finished.
 *
 * The execution times of {@link #prepareForAsync} and {@link #asyncFinish()} must match.
 *
 * @return the current span
 */
AbstractSpan asyncFinish();

# 4.插件开发

官方文档：https://skywalking.apache.org/docs/skywalking-java/v8.13.0/en/setup/service-agent/java-agent/java-plugin-development-guide/#develop-a-plugin (opens new window)

# 4.1 概述

追踪的基本方法是拦截 Java 方法，使用字节码操作技术和 AOP，SkyWalking 包装了字节码操作技术，并追踪上下文的传播。所以我们只需要定义拦截点（即 Spring 的切面）。

# 4.2 拦截

SkyWalking 提供两类通用的定义去拦截构造方法, 实例方法和类方法.

  • ClassInstanceMethodsEnhancePluginDefine 定义了构造方法 Contructor 拦截点和 instance method 实例方法拦截点.
  • ClassStaticMethodsEnhancePluginDefine 定义了类方法 class method 拦截点.

当然，也可以继承 ClassEnhancePluginDefine 去设置所有的拦截点，但并不常用。

# 4.3 实现插件

第一步：引入依赖

第二步：扩展 ClassInstanceMethodsEnhancePluginDefine （继承该类）

1.定义目标类的名称

重新 enhanceClass() 方法，用来匹配要增强的类：

protected abstract ClassMatch enhanceClass();
1

ClassMatch 表示如何去匹配目标类，这里有四种方法:

  • byName 通过类的全限定名（Fully Qualified Class Name, 即 包名 + . + 类名）
  • byClassAnnotationMatch 根据目标类是否存在某些注解
  • byMethodAnnotationMatch 根据目标类的方法是否存在某些注解
  • byHierarchyMatch 根据目标类的父类或接口

WARNING

  • 禁止使用 *.class.getName() 去获取类名，建议使用文本字符串，来避免 ClassLoader 的问题
  • by*AnnotationMatch 不支持从父类继承来的注解
  • 除非必要的情况，否则不建议使用 byHierarchyMatch， 因为使用它可能会触发拦截许多预期之外的方法，会导致性能问题和不稳定

示例：

@Override
protected ClassMatch enhanceClassName() {
    return byName("org.apache.catalina.core.StandardEngineValve");
}

# 第三步：编写拦截器

package org.willingoxjin.skywalking.agent.interceptor.v2;

public class StringIsBlankInterceptorV2 implements StaticMethodsAroundInterceptorV2 {

    @Override
    public void beforeMethod(Class aClass, Method method, Object[] objects, Class<?>[] classes,
            MethodInvocationContext methodInvocationContext) {
        // 创建span(监控的开始)，本质上是往ThreadLocal对象里面设值
        AbstractSpan span = ContextManager.createLocalSpan("org.apache.commons.lang3.StringUtils.isBlank");

        /*
         * 可用ComponentsDefine工具类指定SkyWalking官方支持的组件
         * 也可自己new OfficialComponent或者Component，但是在SkyWalking的控制台上不会被识别，只会显示N/A
         */
        span.setComponent(ComponentsDefine.TOMCAT);
		 // 参数
        span.tag(new StringTag(1000, "params"), objects[0].toString());
        // 指定该调用的layer，layer是个枚举
        span.setLayer(SpanLayer.CACHE);
    }

    @Override
    public Object afterMethod(Class aClass, Method method, Object[] objects, Class<?>[] classes, Object o,
            MethodInvocationContext methodInvocationContext) {
        Boolean retString = (Boolean) o;
        // 激活span，本质上是读取ThreadLocal对象
        AbstractSpan span = ContextManager.activeSpan();
        // 状态码，任意写，Tags是SkyWalking的工具类，用来比较方便地操作tag
        // Tags.STATUS_CODE.set(span, "20000");
        // 自定义 tag
        StringTag tag = new StringTag("version");
        tag.set(span, "v2");
        // 停止span(监控的结束)，本质上是清理ThreadLocal对象
        ContextManager.stopSpan();
        return retString;
    }

    @Override
    public void handleMethodException(Class aClass, Method method, Object[] objects, Class<?>[] classes,
            Throwable throwable, MethodInvocationContext methodInvocationContext) {
        AbstractSpan activeSpan = ContextManager.activeSpan();
        // 记录日志
        activeSpan.log(throwable);
        activeSpan.errorOccurred();
    }
}
