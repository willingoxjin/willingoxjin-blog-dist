Tomcat Manager

# 一、Tomcat Manager 介绍

Tomcat Manager 是 Tomcat 默认内置的应用监控管理工具。启动 Tomcat 之后，进入的默认界面，上方左侧位置，就有一个 Manager App，点击就可以进入 Tomcat Manager 的页面。

# 二、Tomcat Manager 配置

# 1.用户与角色

在 tomcat 包文件的根目录下，修改 conf/tomcat-users.xml ，指定允许访问 Tomcat Manager 的用户与用户角色。

从Tomcat 7开始，Tomcat Manager 有四种管理角色，不同的角色可以访问不同路径。如下：

  • manager-gui ：允许访问 HTML GUI（ /manager/html ）和状态页（/manager/status ）
  • manager-script ：允许访问文本界面（ /manager/text/* ）和状态页（ /manager/status ）
  • manager-jmx ：允许访问JMX代理（ /manager/jmxproxy/* ）和状态页（/manager/status ）
  • manager-status ：仅允许访问状态页面（/manager/status ）

我们这里设置一个用户 admin，为该用户添加 Tomcat Manager 的所有角色。

conf/tomcat-users.xml 配置如下：

<?xml version="1.0" encoding="UTF-8"?>
<tomcat-users xmlns="http://tomcat.apache.org/xml"
              xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
              xsi:schemaLocation="http://tomcat.apache.org/xml tomcat-users.xsd"
              version="1.0">
  <!-- 用户 -->
  <user username="admin" 
        password="admin" 
        roles="manager-gui,manager-script,manager-jmx,manager-status"/>
 
  <!-- 角色 -->
  <role rolename="manager-gui" />
  <role rolename="manager-script" />
  <role rolename="manager-jmx" />
  <role rolename="manager-status" />

</tomcat-users>

# 2.设置远程访问

默认情况下，Tomcat Manager 只允许在本机访问 Tomcat Manager。

如果希望远程访问，那么需修改 webapps/manager/META-INF/context.xml ，将 Valve 标签的 allow 属性修改成你想要的地址正则。

这里修改为允许所有远程IP访问：

<!-- 默认: 仅允许127.0.0.1访问 -->
<!--
<Valve className="org.apache.catalina.valves.RemoteAddrValve"
       allow="127\.\d+\.\d+\.\d+|::1|0:0:0:0:0:0:0:1" />
-->

<!-- 允许所有ip访问； -->
<Valve className="org.apache.catalina.valves.RemoteAddrValve" 
       allow="^.*$" />

# 三、使用

启动 Tomcat，访问 http://localhost:8080 （默认是8080端口，可修改了 conf/server.xml 中的 Connector 标签的 port 属性修改访问的端口）。

# 1. Tomcat Manager 信息

  • 点击上方右侧的 Manager App ，即可访问 Tomcat Manager：

  • 在 Tomcat Manager 中，可以启动、停止、重新加载、卸载应用；也可以使 Session 过期：

  • 也可以部署 war 包：通过设置war包及配置的路径，或者上传war包来部署

  • 配置 TLS 配置文件

  • 诊断是否存在内存泄漏：点击发现泄露，即可进行诊断。（生产环境勿用）

  • 服务器信息：当前服务器、Tomcat、JDK 版本

# 2.服务状态信息

  • 服务器状态：点击管理器—服务器状态，进入服务器状态页面。

    • 在服务器状态界面，可查看服务器信息、JVM 信息、性能信息等

    • 点击“完整的服务器状态”按钮，可以查看更多其他详细的信息
