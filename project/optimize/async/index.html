01.异步化

# 一、同步和异步

同步：

  • 定义：程序按照定义的顺序执行，每一行都必须等待上一行程序完成后才能执行。

异步：

  • 定义：程序执行时，无需等待异步调用的语句执行完，即可执行后面的操作。
  • 适用场景：耗时操作中，将不影响主流程的操作异步执行，降低整体程序的执行时间

# 二、本地调用异步化

本地调用实现异步的方式：

  • 创建线程
    • 缺点：没有线程重用，并发控制的功能，应该选择使用线程池
  • 线程池
  • @Async
  • 发布—订阅模式

# 1.同步代码

以下代码模拟一段业务：查询数据、打印日志、修改数据。我们对每个方法都加了 Thread.sleep(1000) ，让程序增加一秒的运行时间，并且方法全部都是同步调用。

public class SyncDemo {

    public void queryData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 数据库查询");
    }

    public void printLog() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 打印日志");
    }

    public void updateData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 修改数据");
    }

    public static void main(String[] args) {
        long start = System.currentTimeMillis();
     	SyncDemo procedure = new SyncDemo();
        procedure.queryData();
        procedure.printLog();
        procedure.updateData();
        long end = System.currentTimeMillis();
        System.out.println("Total time (ms): "  + (end - start));
    }

}

# 2.创建线程

上面的打印日志的 printLog() 方法其实可以做异步处理，我们使用创建线程的方法优化代码：

public class AsyncByThreadDemo {
    public void queryData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 数据库查询");
    }

    public void printLog() {
        new Thread(() -> {
            try {
                Thread.sleep(1000L);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
            System.out.println(LocalTime.now() + " 打印日志");
        }).start();
    }

    public void updateData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 修改数据");
    }

    public static void main(String[] args) {
        AsyncByThreadDemo procedure = new AsyncByThreadDemo();
        long start = System.currentTimeMillis();
        procedure.queryData();
        procedure.printLog();
        procedure.updateData();
        long end = System.currentTimeMillis();
        System.out.println("Total time (ms): "  + (end - start));
    }
}

# 3.线程池

创建线程的方式其实不够好，因为当存在多个并发调用时，每次调用都创建一个线程，这并不是我们想要的。并且线程的创建也是耗费性能的。

我们可以采用线程池的方式来代替创建线程，以达到异步处理的目的：

public class AsyncByThreadPoolDemo {

    private static final ThreadPoolExecutor executor =
            new ThreadPoolExecutor(
                    5,
                    10,
                    10L,
                    TimeUnit.SECONDS,
                    new LinkedBlockingQueue<>(100),
                    Executors.defaultThreadFactory(),
                    new ThreadPoolExecutor.AbortPolicy()
            );

    public void queryData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 数据库查询");
    }

    public void printLog() {
        executor.execute(() -> {
            try {
                Thread.sleep(1000L);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
            System.out.println(LocalTime.now() + " 打印日志");
        });
    }

    public void updateData() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }

        System.out.println(LocalTime.now() + " 修改数据");
    }

    public static void main(String[] args) {
        long start = System.currentTimeMillis();
        AsyncByThreadPoolDemo procedure = new AsyncByThreadPoolDemo();
        procedure.queryData();
        procedure.printLog();
        procedure.updateData();
        long end = System.currentTimeMillis();
        System.out.println("Total time (ms): "  + (end - start));
    }
}

# 4.3 @Async 使用注意

注意1：@Async 声明的方法的返回值必须是 void 或 Future 类型的

  • 否则会造成异步方法失效

注意2：建议将 @Async 声明的方法放到独立的类中

  • 因为两个方法都在同一个类里面，使用 this 调用会导致 @Async 不生效；

  • @Async 是使用动态代理实现的，其会将原始的对象包装成代理对象，而使用 this 调用的是对象本身的方法，并不是代理对象的方法

如果想要将 @Async 声明的方法和调用的方法放到一起，可以使用 ApplicationContext 进行调用。但是并不推荐这样做，因为可读性，可维护性都会降低。

示例如下，在 procedure() 中调用异步方法：

@Component
public class AsyncByAsyncAnnotationDemo {

    @Autowired
    ApplicationContext applicationContext;

    @Async
    public void asyncPrintLogVoid() {
        try {
            Thread.sleep(1000L);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
        String result = LocalTime.now() + " asyncPrintLogVoid打印日志";
        System.out.println(result);
    }

   // 在同一个类中调用异步方法
    public void procedure() throws ExecutionException, InterruptedException {
        // 从applicationContext获取bean
        AsyncByAsyncAnnotationDemo bean = applicationContext.getBean(AsyncByAsyncAnnotationDemo.class);
        // 调用异步方法
        bean.asyncPrintLogVoid();
    }

}

注意3：自定义 BlockiingQueue 的容量

  • @Async 使用的线程池：ThreadPoolTaskExecutor 类。
  • 线程池 ThreadPoolTaskExecutor 是在 TaskExecutionAutoConfiguration 类中的 taskExecutorBuilder() 方法中进行初始化的。queueCapacity 属性就是等待队列的容量，默认 queueCapacity = Integer.MAX_VALUE。
  • ThreadPoolTaskExecutor 初始化时，如果 queueCapacity > 0 ，则会使用 LinkedBlockingQueue ，否则使用 SynchronousQueue。
  • 也就是说默认情况下 会使用一个 容量为 Integer.MAX_VALUE 的 LinkedBlockingQueue 作为排队队列。
  • 可以在 application.properties 配置文件中设置属性spring.task.execution.pool.queue-capacity ，以此设置 queueCapacity 的大小

# 5.发布订阅模式

可以使用 Spring 提供的 ApplicationListener 、ApplicationEvent、ApplicationEventPublisher、ApplicationEventPublisherAware 等类，实现发布订阅模式，进行异步处理。

这种方式用起来更加优雅，适合多个位置调用，处理逻辑类似的情况，也是比较推荐实现异步的方式。

也可以借助 RabbitMQ、RocketMQ 等消息中间件，实现发布—订阅模式。

# 三、远程调用异步化

远程调用实现异步的方式：

  • AsyncRestTemplate
  • WebClient
  • 基于 MQ （这里暂不介绍）

# 1.AsyncRestTemplate

# 1.1 介绍

RestTemplate 是 Spring 3.0 后内置的用于调用 rest 服务的类，它简化了与http服务的通信方式，统一了RESTFul的标准，封装了http连接。

而 AsyncRestTemplate 和 RestTemplate 的使用差不多，只不过 AsyncRestTemplate 是异步调用，RestTemplate 是同步调用。如果你使用或了解过 RestTemplate ，那你能够很快的使用 AsyncRestTemplate。

# 1.2 配置

@Configuration
public class AsyncRequestConfig {

    @Bean
    public AsyncRestTemplate asyncRestTemplate() {
        return new AsyncRestTemplate();
    }

}

# 1.3 使用

写一个接口，用来被请求：

@RestController
public class TestController {
    
	// 被调用的接口
    @RequestMapping(value = "/test", method = { RequestMethod.POST, RequestMethod.GET })
    public String test(
            @RequestParam(required = false) String param,
            @RequestBody(required = false) Map<String, Object> body,
            @RequestHeader(required = false) String token) {
        if (param != null) {
            System.out.println(param);
        }
        if (body != null) {
            body.forEach((k,v) -> {
                System.out.println(k + "=" + v);
            });
        }
        if (token != null) {
            System.out.println(token);
        }
        return "Hello AsyncRestTemplate !";
    }
}

使用 AsyncRestTemplate ，新建几个测试方法，如下：

@RestController
@RequestMapping("/test/asyncRestTemplate")
public class AsyncRequestTestController {

    @Autowired
    private AsyncRestTemplate asyncRestTemplate;

    // 阻塞方式
    @GetMapping("/get")
    public ResponseEntity<String> testGet() throws ExecutionException, InterruptedException {
        ListenableFuture<ResponseEntity<String>> future = asyncRestTemplate.getForEntity(
                "http://localhost:28000/test",
                String.class
        );
        // 阻塞方式
        ResponseEntity<String> response = future.get();
        System.out.println("[testAsyncRestTemplate1] response=" + response);
        return response;
    }

    // 非阻塞方式
    @GetMapping("/getWithCallback")
    public String testGetWithCallback()  {
        ListenableFuture<ResponseEntity<String>> future = asyncRestTemplate.getForEntity(
                "http://localhost:28000/test",
                String.class
        );
        // 非阻塞方式：回调
        future.addCallback(new ListenableFutureCallback<>() {
            @Override
            public void onFailure(Throwable ex) {
                System.out.println("请求失败" + ex);
            }

            @Override
            public void onSuccess(ResponseEntity<String> result) {
                System.out.println("[testAsyncRestTemplate2 onSuccess] result=" + result);
            }
        });
        return "OK";
    }

    // 测试post请求
    @GetMapping("/post")
    public ResponseEntity<String> testPost() throws ExecutionException, InterruptedException {
        // url
        String url = "http://localhost:28000/test?param={param}";
        // Headers
        HttpHeaders httpHeaders = new HttpHeaders();
        httpHeaders.set("token", "123456");
        // Body参数
        Map<String, Object> body = new HashMap<>();
        body.put("name", "张三");
        body.put("username", "zhangsan");
        body.put("age", 18);
        // Url参数替换
        Map<String, Object> uriVariables = new HashMap<>();
        uriVariables.put("param", "zhangsan");
        // 构造请求体
        HttpEntity<Map<String, Object>> request = new HttpEntity<>(body, httpHeaders);
        // 发起请求
        ListenableFuture<ResponseEntity<String>> future = asyncRestTemplate.postForEntity(
                url,
                request,
                String.class,
                uriVariables
        );
        
        return future.get();
    }

}

# 2.3 使用

@RestController
@RequestMapping("/test/webClient")
public class WebClientTestController {

    @Autowired
    private WebClient webClient;

    @GetMapping("/get")
    public String testGet() {
        Mono<String> mono = webClient.get()
                .uri("http://localhost:28000/test")
                .retrieve()
                .bodyToMono(String.class);

        return mono.block();
    }

    @GetMapping("/post")
    public String testPost() {
        // Url参数替换
        Map<String, Object> uriVariables = new HashMap<>();
        uriVariables.put("param", "zhangsan");

        // Headers
        Consumer<HttpHeaders> httpHeader = httpHeaders -> {
            httpHeaders.set("token", "666666");
        };

        // Body参数
        Map<String, Object> body = new HashMap<>();
        body.put("name", "李四");
        body.put("username", "lisi");
        body.put("age", 19);

        // 请求
        return this.webClient.post()
                .uri("http://localhost:28000/test?param={param}", uriVariables)
                // 指定发送请求时候的content-type
                .contentType(MediaType.APPLICATION_JSON)
                .headers(httpHeader)
                .body(BodyInserters.fromValue(body))
                .retrieve()
                .bodyToMono(String.class)
                .block();
    }

}

注意

如果你使用的是 Mac 的 CPU 为 Apple M1 系列的，会出现报错：

no netty_resolver_dns_native_macos_aarch_64 in java.library.path
1

解决此问题，只需要添加依赖：

    <dependency>
      <groupId>io.netty</groupId>
      <artifactId>netty-resolver-dns-native-macos</artifactId>
      <classifier>osx-aarch_64</classifier>
    </dependency>
	<!-- 排除原来的osx-x86_64版本依赖 -->
    <dependency>
      <groupId>io.projectreactor.netty</groupId>
      <artifactId>reactor-netty</artifactId>
      <exclusions>
        <exclusion>
          <artifactId>netty-resolver-dns-native-macos</artifactId>
          <groupId>io.netty</groupId>
        </exclusion>
      </exclusions>
    </dependency>
