02.使用 Common-Pool2 自定义连接池

# 一、Common-Pool2 介绍

Common-Pool2 是 Apache 开源的对象池框架。

官方网站：https://commons.apache.org/proper/commons-pool (opens new window)

Github 地址：https://github.com/apache/commons-pool (opens new window)

# 二、核心 API

Common-Pool2 提供了两类的对象池：ObjectPool 和 ObjectPool

# ObjectPool

ObjectPool 的实现类：

实现类 作用
BaseObjectPool 抽象类，用来扩展自己的对象池
ErodingObjectPool “腐蚀”对象池，代理一个对象池，并基于 factor 参数，为其添加“腐蚀”行为，归还的对象被腐蚀后，将会弃用，而不是放到空闲容器中。
GenericObjectPool 一个可配置的通用对象池实现。该实现类是最常用的。
ProxiedObjectPool - 代理一个其他的对象池，并基于动态代理（支持 JDK 的动态代理和 CFLIB 的动态代理），返回一个代理的对象。
- 该对象池主要用来增强对池化对象的控制，如防止在归还该对象后，还继续使用该对象等。
SoftReferenceObjectPool 基于软引用的对象池。
SynchronizedObjectPool 代理一个其他对象，并为其提供线程安全的能力。

ObjectPool 的核心API

方法 作用
void addObject() 增加一个空闲对象，该方法使用于使用空闲对象预加载对象池
T borrowObject() 从对象池中借对象
void clear() 清空空闲的所有对象，并释放相关资源
void close() 关闭对象池，并释放相关资源
int getNumActive() 获得被借出的对象数量
int getNumIdle() 获得空闲的对象数量
void invalidateObject(T obj) 失效一个对象
void returnObject(T obj) 将对象归还到对象池

# KeyedObjectPool

KeyedObjectPool 相对于 ObjectPool 做个一个 key，通过 key 去获取对象。

KeyedObjectPool 的实现类其实与 ObjectPool 相对应：

  • ErodingKeyedObjectPool
  • ErodingPerKeyKeyedObjectPoo
  • GenericKeyedObjectPool
  • ProxiedKeyedObjectPool
  • SynchronizedKeyedObjectPool

# PooledObjectFactory

PooledObjectFactory 是 ObjectFactory 的工厂类。用来创建管理 PooledObject。

子类 说明
BasePooledObjectFactory 抽象类，用于扩展自己的 PooledObjectFactory
SynchronizedPooledObjectFactory 内部类，代理一个其他的 PooledObjectFactory，实现线程同步。用 PoolUtils.synchronizedPooledFactory() 创建

PooledObjectFactory 一般需要自己去扩展实现。

# PooledObject

PooledObject 用来包装原有的对象，从而让对象池管理。

PooledObject 的实现类有DefaultPooledObject 和 PooledSoftReference。 一般使用 DefaultPooledObject 。

# PooledObjectState

状态 说明
IDLE 对象在队列中，并空闲
ALLOCATED 使用中，即出借中
EVICTION 对象当在队列中，正在进行驱逐测试
EVICTION_RETURN_TO_HEAD 对象驱逐测试通过后，放回队列头部
VALIDATION 对象在当前队列中，空闲校检中
VALIDATION_PREALLOCATED 对象当前不在队列中，出借前校检中
VALIDATION_RETURN_TO_HEAD 对象当前不在队列中，校检通过后放回头部
INVALID 对象失败。驱逐测试失败、校检失败、对象销毁，都会将对象置为 INVALID
ABANDONED 放逐中，如果对象上次使用时间超过 removeAbandonedTimeout 的配置，则将其标记为 ABANDONED。标记为 ABANDONED 的对象即将变成 INVALID
RETURNING 对象归还池中

public class User {

    private String username;

    private Integer age;

    private Sex sex;

    public User(String username, Integer age, Sex sex) {
        this.username = username;
        this.age = age;
        this.sex = sex;
        // 模拟创建对象较慢的情况
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }

    public Integer addAge() {
        return ++this.age;
    }
	
    // getter setter...

    public static enum Sex {
        UN_KNOW(0),
        MALE(1),
        FEMALE(2),
        ;

        private final Integer sex;

        // constructor、getter...
    }

}

  • UserPooledObjectFactory
public class UserPooledObjectFactory implements PooledObjectFactory<User> {

    public void activateObject(PooledObject p) throws Exception {
        // 获取对象时调用
        this.printState("activateObject", p.getState());
    }

    public void destroyObject(PooledObject p) throws Exception {
        // 销毁对象，销毁对象池时被调用，连接池调用invalidateObject(obj);时被调用。
        this.printState("destroyObject", p.getState());
    }

    public PooledObject<User> makeObject() throws Exception {
        // 创建对象实例
        // 返回DefaultPooledObject
        DefaultPooledObject<User> p = new DefaultPooledObject<User>(new User("Tony", 28, Sex.MALE));
        this.printState("makeObject", p.getState());
        return p;
    }

    public void passivateObject(PooledObject p) throws Exception {
        // 将对象还给对象池时被调用
        this.printState("passivateObject", p.getState());
    }

    public boolean validateObject(PooledObject p) {
        // 检查对象有效性，需要对象池设置setTestOnXxxx(true)
        this.printState("validateObject", p.getState());
        return true;
    }

    private void printState(String method, PooledObjectState state) {
        System.out.printf("[%s] [%s] state=%s\n",
                LocalTime.now(), method, state);
    }
}

# 1.回顾 JDBC 的使用

public class JdbcDemo {

    private final static String JDBC_DRIVER_CLASS_NAME = "com.mysql.cj.jdbc.Driver";
    private final static String JDBC_URL = "jdbc:mysql://localhost:3306/test_db?useUnicode=true&characterEncoding=UTF-8&autoReconnect=true&serverTimezone=Asia/Shanghai&useSSL=false";
    private final static String JDBC_USERNAME = "root";
    private final static String JDBC_PASSWORD = "123456";

    public static void main(String[] args) throws ClassNotFoundException, SQLException {
        Class.forName(JDBC_DRIVER_CLASS_NAME);
        // 使用 try () {} , 会自动关闭 connection
         try (Connection connection = DriverManager.getConnection(
                JDBC_URL,
                JDBC_USERNAME,
                JDBC_PASSWORD)
        ) {
            // 查询1
            String sql = "select * from user limit 10;";
            PreparedStatement preparedStatement = connection.prepareStatement(sql);
            ResultSet resultSet = preparedStatement.executeQuery();
            while (resultSet.next()) {
                System.out.println(resultSet.getString("id"));
                System.out.println(resultSet.getString("username"));
            }
            resultSet.close();
            preparedStatement.close();

            // 查询2
            String sql2 = "select * from user limit 10, 20;";
            PreparedStatement preparedStatement2 = connection.prepareStatement(sql2);
            ResultSet resultSet2 = preparedStatement2.executeQuery();
            while (resultSet2.next()) {
                System.out.println(resultSet2.getString(1));
                System.out.println(resultSet2.getString(2));
            }
            resultSet2.close();
            preparedStatement2.close();
        }
    }

}

我们想要在一个JDBC连接中查询多次，就需要在连接关闭之前执行 sql（即调用 connection.close()）。

而在使用连接池的时候。先从连接池中取出一个 connection 对象，然后执行 sql 语句，在 sql 执行完最后，不会关闭连接，而是把 connection 对象放回连接池中，以便后续进行复用。

  • ConnectionWrapper

ConnectionWrapper 用来包装 java.sql.Connection ，实现连接池的增强功能。

这里除了 createStatement、prepareStatement 、close 方法需要自定义，其他均直接调用 connection 的方法

@Getter
@Setter
public class ConnectionWrapper implements Connection {

    private Connection connection;

    private Statement statement;

    private ObjectPool<ConnectionWrapper> objectPool;
 
    @Override
    public Statement createStatement() throws SQLException {
        return this.connection.createStatement();
    }
    
    @Override
    public PreparedStatement prepareStatement(String sql) throws SQLException {
        PreparedStatement prepareStatement = this.connection.prepareStatement(sql);
        this.statement = prepareStatement;
        return prepareStatement;
    }
    
    @Override
    public void close() throws SQLException {
        // 如果底层的Connection已经关闭
        if (this.isClosed()) {
            try {
                objectPool.invalidateObject(this);
            } catch (Exception e) {
                throw new SQLException(e);
            }
        }
        // 底层Connection没有关闭，可以继续复用
        else {
            try {
                objectPool.returnObject(this);
            } catch (Exception e) {
                this.connection.close();
                throw new SQLException(e);
            }
        }
    }
    
    @Override
    public Statement createStatement(int resultSetType, int resultSetConcurrency) throws SQLException {
        Statement statement = this.connection.createStatement(resultSetType, resultSetConcurrency);
        this.statement = statement;
        return statement;
    }
    
    @Override
    public PreparedStatement prepareStatement(String sql, int resultSetType, int resultSetConcurrency) throws SQLException {
        PreparedStatement preparedStatement = this.connection.prepareStatement(sql, resultSetType, resultSetConcurrency);
        this.statement = preparedStatement;
        return preparedStatement;
    }
    
    
    @Override
    public Statement createStatement(int resultSetType, int resultSetConcurrency, int resultSetHoldability) throws SQLException {
        Statement statement = this.connection.createStatement(resultSetType, resultSetConcurrency, resultSetHoldability);
        this.statement = statement;
        return statement;
    }
    
    @Override
    public PreparedStatement prepareStatement(String sql, int resultSetType, int resultSetConcurrency, int resultSetHoldability) throws SQLException {
        PreparedStatement statement = this.connection.prepareStatement(sql, resultSetType, resultSetConcurrency, resultSetHoldability);
        this.statement = statement;
        return statement;
    }
    
    @Override
    public PreparedStatement prepareStatement(String sql, int autoGeneratedKeys) throws SQLException {
        PreparedStatement statement = this.connection.prepareStatement(sql, autoGeneratedKeys);
        this.statement = statement;
        return statement;
    }

    @Override
    public PreparedStatement prepareStatement(String sql, int[] columnIndexes) throws SQLException {
        PreparedStatement statement = this.connection.prepareStatement(sql, columnIndexes);
        this.statement = statement;
        return statement;
    }
    
    @Override
    public PreparedStatement prepareStatement(String sql, String[] columnNames) throws SQLException {
        PreparedStatement statement = this.connection.prepareStatement(sql, columnNames);
        this.statement = statement;
        return statement;
    }
    
    // other code...
    
    // 其他方法均调用 connection 属性的方法
    // 如下方的setAutoCommit、getAutoCommit方法
    @Override
    public void setAutoCommit(boolean autoCommit) throws SQLException {
        this.connection.setAutoCommit(autoCommit);
    }

    @Override
    public boolean getAutoCommit() throws SQLException {
        return this.connection.getAutoCommit();
    }
    
}

  • ConnectionPooledObjectFactory

即在使用 common-pool2的时候，需要实现 PooledObjectFactory 。

@Getter
@Setter
public class ConnectionPooledObjectFactory implements PooledObjectFactory<ConnectionWrapper> {

    private final static String TEST_QUERY_SQL = "SELECT 1";

    private ObjectPool<ConnectionWrapper> objectPool;

    private DataSourceProperties dataSourceProperties;

    @Override
    public PooledObject<ConnectionWrapper> makeObject() throws Exception {
        // 创建连接
        Class.forName(dataSourceProperties.getDriverClassName());
        Connection connection = DriverManager.getConnection(
                dataSourceProperties.getUrl(),
                dataSourceProperties.getUsername(),
                dataSourceProperties.getPassword()
        );
        ConnectionWrapper conn = new ConnectionWrapper();
        conn.setConnection(connection);
        conn.setObjectPool(objectPool);
        return new DefaultPooledObject<>(conn);
    }

    @Override
    public void activateObject(PooledObject<ConnectionWrapper> p) throws Exception {
        // TODO 可在此添加connection的额外配置
    }

    @Override
    public void passivateObject(PooledObject<ConnectionWrapper> p) throws Exception {
        // 归还对象前，校检连接
        // 钝化
        ConnectionWrapper conn = p.getObject();
        Statement statement = conn.getStatement();
        if (statement != null) {
            statement.close();
        }
    }

    @Override
    public boolean validateObject(PooledObject<ConnectionWrapper> p) {
        // 校检连接
        Connection connection = p.getObject();
        try {
            // 可以将test sql的配置放到外部
            PreparedStatement statement = connection.prepareStatement(TEST_QUERY_SQL);
            ResultSet resultSet = statement.executeQuery();
            resultSet.next();
            return resultSet.getInt(1) == 1;
        } catch (SQLException e) {
            e.printStackTrace();
            return false;
        }
    }

    @Override
    public void destroyObject(PooledObject<ConnectionWrapper> p) throws Exception {
        // 销毁对象时，关闭连接
        p.getObject().close();
    }
}

实现 DataSource ，通过 getConnection() 方法以获取数据库连接。

@Slf4j
@Getter
@Setter
public class WillingDataSource implements DataSource {

    private GenericObjectPool<ConnectionWrapper> pool;

    public WillingDataSource(DataSourceProperties dataSourceProperties) {
        ConnectionPooledObjectFactory factory = new ConnectionPooledObjectFactory();
        factory.setDataSourceProperties(dataSourceProperties);
        // TODO 可在这里增加相关的配置参数, 详情见GenericObjectPool的构造方法
        this.pool = new GenericObjectPool<>(factory);
         // 创建时检查有效性
        this.pool.setTestOnCreate(true);
        // TODO ConnectionPooledObjectFactory和GenericObjectPool存在循环依赖问题
        factory.setObjectPool(this.pool);
    }

    @Override

# 五、Abandon 与 Evict 的区别

# 1.区别

  • Abandon（废弃）

ABANDONED 是对象池中，对象的状态，此时对象还是会在对象池中。

  • Evict（清理对象的过程）

清理的不一定是状态为 ABANDONED 的对象，清除的可能是对象池中状态为 INVALID、ABANDONED 等状态的对象。

# 2.源码

进入 BaseGenericObjectPool#setTimeBetweenEvictionRuns 方法，该方法会根据 delay 参数启动一个定时任务（delay > 0 的时候才会启动），如果已经启动了定时任务会进行重启操作。

使用 EvictionTimer.schedule() 方法启动定时任务 ，其第一个参数是一个 Evictor 类型的对象。

源码如下：

private Evictor evictor; // @GuardedBy("evictionLock")

public final void setTimeBetweenEvictionRuns(final Duration timeBetweenEvictionRuns) {
    this.durationBetweenEvictionRuns = PoolImplUtils.nonNull(timeBetweenEvictionRuns, BaseObjectPoolConfig.DEFAULT_TIME_BETWEEN_EVICTION_RUNS);
    startEvictor(this.durationBetweenEvictionRuns);
}


final void startEvictor(final Duration delay) {
    synchronized (evictionLock) {
        // isPositiverDelay: delay的时间是否大于0
        // 仅在大于0的时候才会启动定时任务
        final boolean isPositiverDelay = PoolImplUtils.isPositive(delay);
        if (evictor == null) {
            // 第一次或在取消后启动定时驱逐器
            if (isPositiverDelay) {
                evictor = new Evictor();
                EvictionTimer.schedule(evictor, delay, delay);
            }
        } else if (isPositiverDelay) {
            // 停止或重新启动现有定时驱逐器:即重新启动
            synchronized (EvictionTimer.class) { // 确保在取消/定时任务执行之间不会发生取消
                EvictionTimer.cancel(evictor, evictorShutdownTimeoutDuration, true);
                evictor = null;
                evictionIterator = null;
                evictor = new Evictor();
                EvictionTimer.schedule(evictor, delay, delay);
            }
        } else {
            // 停止定时驱逐器
            EvictionTimer.cancel(evictor, evictorShutdownTimeoutDuration, false);
        }
    }
}

Evictor 是 BaseGenericObjectPool 的内部类，其实现了 Runnable 接口，在 run() 方法中的核心逻辑就是调用 evict() 方法，进行驱逐操作。

其源码如下：

public abstract void evict() throws Exception;

class Evictor implements Runnable {

    private ScheduledFuture<?> scheduledFuture;

    /**
    * Cancels the scheduled future.
    */
    void cancel() {
        scheduledFuture.cancel(false);
    }

    /**
     * Run pool maintenance.  Evict objects qualifying for eviction and then
     * ensure that the minimum number of idle instances are available.
     * Since the Timer that invokes Evictors is shared for all Pools but
     * pools may exist in different class loaders, the Evictor ensures that
     * any actions taken are under the class loader of the factory
     * associated with the pool.
     */
    @Override
    public void run() {
        final ClassLoader savedClassLoader =
            Thread.currentThread().getContextClassLoader();
        try {
            // 校检ClassLoader
            if (factoryClassLoader != null) {
                // Set the class loader for the factory
                final ClassLoader cl = factoryClassLoader.get();
                if (cl == null) {
                    // The pool has been dereferenced and the class loader
                    // GC'd. Cancel this timer so the pool can be GC'd as
                    // well.
                    cancel();
                    return;
                }
                Thread.currentThread().setContextClassLoader(cl);
            }

            // Evict from the pool
            try {
                evict();
            } catch(final Exception e) {
                swallowException(e);
            } catch(final OutOfMemoryError oome) {
                // Log problem but give evictor thread a chance to continue
                // in case error is recoverable
                oome.printStackTrace(System.err);
            }
            // Re-create idle instances.
            try {
                ensureMinIdle();
            } catch (final Exception e) {
                swallowException(e);
            }
        } finally {
            // Restore the previous CCL
            Thread.currentThread().setContextClassLoader(savedClassLoader);
        }
    }


    /**
     * Sets the scheduled future.
     *
     * @param scheduledFuture the scheduled future.
     */
    void setScheduledFuture(final ScheduledFuture<?> scheduledFuture) {
        this.scheduledFuture = scheduledFuture;
    }

}

evict() 方法主要用来清理空闲的对象，以及进行空闲对象的检查，如果检查出错或失败，则进行清理操作。

源码如下：

public class GenericObjectPool<T> extends BaseGenericObjectPool<T>
        implements ObjectPool<T>, GenericObjectPoolMXBean, UsageTracking<T> {

    // other code...
    
	@Override
    public void evict() throws Exception {
        // 判断对象池是否关闭
        assertOpen();

        // 存在空闲的对象
        if (!idleObjects.isEmpty()) {

            PooledObject<T> underTest = null;
            // 获取驱逐策略
            final EvictionPolicy<T> evictionPolicy = getEvictionPolicy();

            synchronized (evictionLock) {
                // 配置
                final EvictionConfig evictionConfig = new EvictionConfig(
                        getMinEvictableIdleDuration(),
                        getSoftMinEvictableIdleDuration(),
                        getMinIdle());

                // 是否进行空闲有效性测试
                final boolean testWhileIdle = getTestWhileIdle();

                // 遍历空闲的对象
                // 根据单次清理的对象个数及对象总数来决定遍历的次数
                for (int i = 0, m = getNumTests(); i < m; i++) {
                    if (evictionIterator == null || !evictionIterator.hasNext()) {
                        evictionIterator = new EvictionIterator(idleObjects);
                    }
                    // 遍历逻辑：1-校检是否进行清理
                    if (!evictionIterator.hasNext()) {
                        // Pool exhausted, nothing to do here
                        return;
                    }

                    try {
                        underTest = evictionIterator.next();
                    } catch (final NoSuchElementException nsee) {
                        // Object was borrowed in another thread
                        // Don't count this as an eviction test so reduce i;
                        i--;
                        evictionIterator = null;
                        continue;
                    }

                    if (!underTest.startEvictionTest()) {
                        // Object was borrowed in another thread
                        // Don't count this as an eviction test so reduce i;
                        i--;
                        continue;
                    }

                	// 遍历逻辑：2-执行清理操作
                    // User provided eviction policy could throw all sorts of
                    // crazy exceptions. Protect against such an exception
                    // killing the eviction thread.
                    boolean evict;
                    try {
                        // 对象是否需要清理
                        evict = evictionPolicy.evict(evictionConfig, underTest,
                                idleObjects.size());
                    } catch (final Throwable t) {
                        // Slightly convoluted as SwallowedExceptionListener
                        // uses Exception rather than Throwable
                        PoolUtils.checkRethrow(t);
                        swallowException(new Exception(t));
                        // Don't evict on error conditions
                        evict = false;
                    }

                    if (evict) {
                        // 调用destroy()清理对象
                        destroy(underTest, DestroyMode.NORMAL);
                        destroyedByEvictorCount.incrementAndGet();
                    } else {
                        // 如果配置了空闲时间校检，则进行校检逻辑
                        if (testWhileIdle) {
                            // 调用activateObject借出对象
                            boolean active = false;
                            try {
                                factory.activateObject(underTest);
                                active = true;
                            } catch (final Exception e) {
                                destroy(underTest, DestroyMode.NORMAL);
                                destroyedByEvictorCount.incrementAndGet();
                            }
                            // 调用validateObject检查对象
                            if (active) {
                                boolean validate = false;
                                Throwable validationThrowable = null;
                                try {
                                    validate = factory.validateObject(underTest);
                                } catch (final Throwable t) {
                                    PoolUtils.checkRethrow(t);
                                    validationThrowable = t;
                                }
                                // 校检失败，清理对象
                                if (!validate) {
                                    destroy(underTest, DestroyMode.NORMAL);
                                    destroyedByEvictorCount.incrementAndGet();
                                    if (validationThrowable != null) {
                                        if (validationThrowable instanceof RuntimeException) {
                                            throw (RuntimeException) validationThrowable;
                                        }
                                        throw (Error) validationThrowable;
                                    }
                                } else {
                                    // 校检成功，调用passivateObject还回对象
                                    try {
                                        factory.passivateObject(underTest);
                                    } catch (final Exception e) {
                                        // 还回出现异常，则进行清理
                                        destroy(underTest, DestroyMode.NORMAL);
                                        destroyedByEvictorCount.incrementAndGet();
                                    }
                                }
                            }
                        }

                        if (!underTest.endEvictionTest(idleObjects)) {
                            // TODO - May need to add code here once additional
                            // states are used
                        }
                    }
                }
            }
        }
        // 根据AbandonedConfig配置，清理掉AbandonedConfig的对象
        final AbandonedConfig ac = this.abandonedConfig;
        if (ac != null && ac.getRemoveAbandonedOnMaintenance()) {
            removeAbandoned(ac);
        }
    }

}

class EvictionTimer {
    
    // other code...
    
	static synchronized void schedule(
            final BaseGenericObjectPool<?>.Evictor task, final Duration delay, final Duration period) {
        if (null == executor) {
            executor = new ScheduledThreadPoolExecutor(1, new EvictorThreadFactory());
            executor.setRemoveOnCancelPolicy(true);
            executor.scheduleAtFixedRate(new Reaper(), delay.toMillis(), period.toMillis(), TimeUnit.MILLISECONDS);
        }
        final WeakReference<Runnable> ref = new WeakReference<>(task);
        final WeakRunner runner = new WeakRunner(ref);
        final ScheduledFuture<?> scheduledFuture = executor.scheduleWithFixedDelay(runner, delay.toMillis(),
                period.toMillis(), TimeUnit.MILLISECONDS);
        task.setScheduledFuture(scheduledFuture);
        taskMap.put(ref, runner);
    }

}
