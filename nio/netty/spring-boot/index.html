06.Netty 与 SpringBoot 整合

# 一、需求分析与设计

数据通信需求场景分析：

  • 数据通信要求实时性高，性能好，异构系统（不同开发语言的系统，如Java、Go、C#等）
    • 使用 Google 的 Protobuf 来序列化，支持多语言的序列化
  • 需要保障不同业务对应不同的实现
  • 支持负载均衡策略（TCP级别）、故障切换
  • 需要可靠性保障的支持，数据不允许丢失

可靠性数据架构设计：

  • Client 端实时推送消息
  • Server 接收到消息后对数据进行幂等性校检，判断消息是否重复等。校检成功，则进行业务处理。业务处理完成，向 Client 端进行ACK消息完成确认
  • 后续使用定时任务查询出失败消息，执行补偿机制或发送异常消息进行人工干预

这里的通信可靠性设计与 MQ 的消息处理非常相似

# 2.自动扫描

在 netty-common 模块下：反射对象

package org.willingoxjin.netty.common.scanner;

/**
 * 反射
 */
@Data
public class Invoker {

	private Method method;
	
	private Object target;
	
	/**
	 * 创建invoker对象
	 */
	public static Invoker createInvoker(Method method, Object target) {
		Invoker invoker = new Invoker();
		invoker.setMethod(method);
		invoker.setTarget(target);
		return invoker;
	}
	
	/**
	 * 反射调用
	 */
	public Object invoke(Object... params) {
		try {
			return method.invoke(target, params);
		} catch (IllegalAccessException e) {
			e.printStackTrace();
		} catch (IllegalArgumentException e) {
			e.printStackTrace();
		} catch (InvocationTargetException e) {
			e.printStackTrace();
		}
		return null;
	}
	
}

在 netty-common 模块下：反射缓存类

package org.willingoxjin.netty.common.scanner;

/**
 * 缓存反射对象
 */
public class InvokerTable {

	/**
	 * key-module
	 * value-[key-execute, value-Invoker]
	 */
	private final static ConcurrentHashMap<String, Map<String, Invoker>> invokerTable = new ConcurrentHashMap<>();
	
	/**
	 * 添加Invoker象到缓存
	 */
	public static void addInvoker(String module, String execute, Invoker invoker) {
		Map<String, Invoker> map = invokerTable.get(module);
		if(map == null) {
			map = new HashMap<>();
			invokerTable.put(module, map);
		}
		map.put(execute, invoker);
	}
	
	/**
	 * 获取缓存的Invoker
	 */
	public static Invoker getInvoker(String module, String execute) {
		Map<String, Invoker> map = invokerTable.get(module);
		if(map != null) {
			return map.get(execute);
		}
		return null;
	}
	
}

在 netty-common 模块下，添加自动扫描类，并将反射对象放到缓存中（由于 client、server都需要扫描，因此放在 netty-common 下）

具体逻辑如下：

  • 在 BeanPostProcessor 的 postProcessAfterInitialization 方法中，在 bean 初始化之后加载所有的 bean 对象
  • 然后找到被 @Module 修饰的 bean 对象，对 bean 对象下的方法中被 @Execute 修饰的方法。
  • 根据 @Module 和 @Execute 的参数创建 Invoker 对象，并缓存到 InvokerTable 中。
package org.willingoxjin.netty.common.scanner;

@Component
public class NettyProcessBeanScanner implements BeanPostProcessor {

	@Override
	public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
		return bean;
	}

	@Override
	public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
		//	1.首先获取当前bean的class类型
		Class<?> clazz = bean.getClass();
		boolean isPresent = clazz.isAnnotationPresent(Module.class);
		boolean isPresent = clazz.isAnnotationPresent(Module.class);
		if(!isPresent) {
			return bean;
		}

        Method[] methods = clazz.getMethods();
        if(methods.length > 0) {
            Module module = clazz.getAnnotation(Module.class);
            String moduleValue = module.value();
            if(ObjectUtils.isEmpty(moduleValue)) {
                // 转为小写
                char[] chars = clazz.getSimpleName().toCharArray();
                chars[0] += 32;
                moduleValue = String.valueOf(chars);
            }

            for(Method m : methods) {
                Execute execute = m.getAnnotation(Execute.class);
                if(execute == null) {
                    continue;
                }

                String executeValue = execute.value();
                if (ObjectUtils.isEmpty(executeValue)) {
                    executeValue = m.getName();
                }

                // 把反射对象(invoker) 放到InvokerTable类中进行管理
                if(InvokerTable.getInvoker(moduleValue, executeValue) == null) {
                    InvokerTable.addInvoker(moduleValue, executeValue, Invoker.createInvoker(m, bean));
                } else {
                    System.err.println("模块下的命令对应的程序缓存已经存在, module: " + moduleValue + " ,cmd: " + executeValue);
                }
            }
        }

		return bean;
	}
}

# 2. ServerHandler

在 netty-common 模块下：添加请求处理的具体实现类（异步处理）：通过反射调用业务处理类（UserService）进行处理

package org.willingoxjin.netty.common.execute;

public class MessageTask4Request implements Runnable {

    private MessageModule.Message message;

    private ChannelHandlerContext ctx;

    private final static String RETURN = "-return";

    public MessageTask4Request(MessageModule.Message message, ChannelHandlerContext ctx) {
        this.message = message;
        this.ctx = ctx;
    }

    @Override
    public void run() {
        String module = message.getModule();
        String execute = message.getExecute();
        byte[] data = message.getBody().toByteArray();
        // 获取缓存的Invoker，然后执行
        Invoker invoker = InvokerTable.getInvoker(module, execute);
        Result<?> result = (Result<?>) invoker.invoke(data);

        // 返回数据
        ctx.writeAndFlush(MessageBuilder
        		.getResponseMessage(module + RETURN,
                                    execute + RETURN, 
                                    result.getResultType(),
                                    result.getContent()));
    }

}

package org.willingoxjin.netty.server;

public class ServerHandler extends ChannelInboundHandlerAdapter {

    private final ThreadPoolExecutor workerPool = new ThreadPoolExecutor(5,
            10,
            60L,
            TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(4000),
            new ThreadPoolExecutor.DiscardPolicy());

    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        MessageModule.Message request = (MessageModule.Message) msg;
        workerPool.submit(new MessageTask4Request(request, ctx));
    }

}

ChannelInboundHandlerAdapter 与 SimpleChannelInboundHandler

  • ChannelInboundHandlerAdapter 是 SimpleChannelInboundHandler 的父类。
  • ChannelInboundHandlerAdapter 在处理完成后，不会回收传输的数据（channelRead() 方法中的 msg 参数）
  • SimpleChannelInboundHandler 在处理完后，会释放传输的数据（调用 ReferenceCountUtil.release(msg) 释放资源）

我们在这里使用 ChannelInboundHandlerAdapter 的原因：

  • 如果使用 SimpleChannelInboundHandler ，其会在 channelRead0 函数执行完后，释放资源。
  • 由于这里是异步处理 msg，msg 在 MessageTask4Request 处理时，可能已经被释放了，所以不能使用 SimpleChannelInboundHandler，而是需要使用 ChannelInboundHandlerAdapter。

package org.willingoxjin.netty.server;

@Slf4j
public class Server {

    public Server() {
        // 配置服务端的NIO线程组
        EventLoopGroup bossGroup = new NioEventLoopGroup();
        EventLoopGroup workerGroup = new NioEventLoopGroup();
        ServerBootstrap b = new ServerBootstrap();
        try {
            b.group(bossGroup, workerGroup)
                    .channel(NioServerSocketChannel.class)
                    .option(ChannelOption.SO_BACKLOG, 1024)
                    .handler(new LoggingHandler(LogLevel.INFO))
                    .childHandler(new ChannelInitializer<SocketChannel>() {
                        @Override
                        public void initChannel(SocketChannel ch) {
                            ch.pipeline()
                                    .addLast(new ProtobufVarint32FrameDecoder())
                                    .addLast(new ProtobufDecoder(MessageModule.Message.getDefaultInstance()))
                                    .addLast(new ProtobufVarint32LengthFieldPrepender())
                                    .addLast(new ProtobufEncoder())
                                    .addLast(new ServerHandler());
                        }
                    });

            int port = 8888;
            // 	绑定端口，同步等待成功
            ChannelFuture cf = b.bind(port).sync();

            log.info("Server startup... port: [{}]", port);
            cf.channel().closeFuture().sync();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            bossGroup.shutdownGracefully();
            workerGroup.shutdownGracefully();
            log.info("Server shutdown... ");
        }
    }

}

# 2.ClientHandler

在 netty-common 模块下：添加返回结果处理的具体实现类（异步处理）：通过反射调用返回结果处理类（UserService）进行处理

public class MessageTask4Response implements Runnable {

	private MessageModule.Message message;
	private ChannelHandlerContext ctx;
	
	public MessageTask4Response(MessageModule.Message message, ChannelHandlerContext ctx) {
		this.message = message;
		this.ctx = ctx;
	}
	
	@Override
	public void run() {
        try {
            // user-return
            String module = this.message.getModule();
            // save-return
            String execute = this.message.getExecute();
            // 响应的结果
            ResultType resultType = this.message.getResultType();
            // 响应的内容
            byte[] data = this.message.getBody().toByteArray();

            // 通过反射，执行返回结果处理的函数
            Invoker invoker = InvokerTable.getInvoker(module, execute);
            invoker.invoke(resultType, data);	
        } finally {
            // 手动释放message
            ReferenceCountUtil.release(message);
        }
    }
    
}

# 3.Client

在 netty-client 模块下：

如果 Client 交给 Spring 管理，在 Client 被实例化的时候，初始化调用 connect 方法，由于 connect 方法是阻塞的，就会导致主线程一直处于阻塞状态，Spring 不能继续实例化其他 Bean。因此 Client 不能交给 Spring 来管理。

解决方式：

方式一：交给 Spring 管理，创建一个线程，异步调用 connect 方法，使得主线程不会阻塞

方式二（推荐）：不交给 Spring 管理，通过单例类 SingletonHolder 来获取 Client。如下：

@Slf4j
public class Client {

    private final ScheduledExecutorService executor = Executors.newScheduledThreadPool(1);

    private final EventLoopGroup group = new NioEventLoopGroup(2);

    public static final String HOST = "127.0.0.1";

    public static final int PORT = 18001;

    private Channel channel;

    /**
     * 校检是否已经连接到服务端
     */
    private final AtomicBoolean isConnect = new AtomicBoolean(false);

    /**
     * 单例类
     */
    private static class SingletonHolder {
        private static final Client INSTANCE = new Client();
    }

    /**
     * 获取单例对象
     */
    public static Client getInstance() {
        return SingletonHolder.INSTANCE;
    }

    private Client()  {
    }

    public synchronized void init() {
        if(!isConnect.get()) {
            try {
                this.connect(HOST, PORT);
                isConnect.set(true);
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }

    private void connect(String host, int port) throws Exception {

        // 配置客户端NIO线程组
        try {
            Bootstrap b = new Bootstrap();
            b.group(group).channel(NioSocketChannel.class)
                    .option(ChannelOption.TCP_NODELAY, true)
                    .handler(new ChannelInitializer<SocketChannel>() {
                        @Override
                        public void initChannel(SocketChannel ch) {
                            ch.pipeline()
                                    .addLast(new ProtobufVarint32FrameDecoder())
                                    .addLast(new ProtobufDecoder(MessageModule.Message.getDefaultInstance()))
                                    .addLast(new ProtobufVarint32LengthFieldPrepender())
                                    .addLast(new ProtobufEncoder())
                                    .addLast(new ClientHandler());
                        }
                    });
            // 发起异步连接操作
            ChannelFuture future = b.connect(host, port).sync();
            this.channel = future.channel();
            log.info("Client Start...");

            this.channel.closeFuture().sync();
        } finally {
            // 	所有资源释放完成之后，清空资源，再次发起重连操作
            executor.execute(new Runnable() {
                @Override
                public void run() {
                    try {
                        TimeUnit.SECONDS.sleep(1);
                        try {
                            connect(host, port);// 发起重连操作
                        } catch (Exception e) {
                            e.printStackTrace();
                        }
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            });
        }
    }

    /**
     * 发送数据
     * @param module 模块
     * @param execute 指令
     * @param messageData 数据内容
     */
    public void sendMessage(String module, String execute , GeneratedMessageV3 messageData) {
        this.channel.writeAndFlush(MessageBuilder.getRequestMessage(module, execute, messageData));
    }

}

# 八、负载均衡高可用

# 1.Netty 负载均衡架构

Netty Cluent 和 Netty Server 进行长连接的负载均衡，使得 Client 总是与同一台 Server 通信。

这里使用 HA-Proxy + Keeplived 作为负载均衡组件来实现。

这里可选择 HA-Proxy 或者 Nginx 作为负载均衡请求代理组件。

  • 使用自建机房，使用 HA-Proxy 或 Nginx 均可
  • 使用云服务器，只能使用 Nginx

长连接的负载均衡一般选择最小连接数（leastconn）的算法进行负载分配。

# 2.配置示例

服务器：

    • ip 172.16.23.131
  • 服务器02
