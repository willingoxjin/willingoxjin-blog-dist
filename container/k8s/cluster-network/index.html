# 一、集群网络初始化

在 docker 中的集群网络部署非常复杂，但是在 kubernates 中我们部署时网络只需要一条命令即可，

需要注意的是，在配置网络之前，我们使用 kubeadm init 命令初始化时，必须添加对应的网络参数。

如使用 flannel 网络，就需要在 kubeadm init 时，添加参数：--pod-network-cidr=10.244.0.0/16

# 初始化集群
kubeadm init \
--apiserver-advertise-address=192.168.31.85 \
--pod-network-cidr=10.244.0.0/16 \

TIPS

需要注意的是：如果你想修改已经配置好的网络，即使两个网络在 kubeadm init 中的参数一致，你也需要重新运行 kubeadm init 命令。

# 二、重置网络

演示重置到 Canal 网络

对所有的节点执行以下命令：

# 重置kubernates
kubeadm reset

# 初始化集群
kubeadm init \
--apiserver-advertise-address=192.168.31.85 \
--pod-network-cidr=10.244.0.0/16 \

# 配置文件操作
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernates/admin.conf/$HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# 配置Canal网络
kubectl apply -f https://projectcalico.docs.tigera.io/manifests/canal.yaml

# 三、服务发现与负载均衡原理

Pod 访问方式

  • ClusterIP Service （集群内部）

  • NodePort Service （由内而外）

    • 基于 ClusterIP，将网络服务绑定到每一个node节点对应的端口上

  • LoadBalancer Service（外部）

    • 跨服务节点，形成负载均衡

  • Ingress（外部）

    • 提供一个统一的URL或Path对外提供服务

# 1.负载均衡

# ClusterIP

  • 若外部用户想访问，可以使用 proxy service 的方式，通过代理对外提供服务

  • 此种场景一般用于测试环境，但应用不多

# NodePort

  • 调用服务时，先访问node中指定的端口（虚拟的IP和端口），然后会自动进行转换，转换到对应的service

# LoadBalancer

  • 需要在云服务平台上

  • 云供应商提供LoadBalance，自动的转发到对应的service

  • 支持多种协议

# Ingress

  • 通过不同的url和content path映射到不同的ip和端口

# 2.服务发现

  • Pod 内的服务调用：loadhost:容器内应用端口

  • Pod 之间的服务调用：服务名.namespace名称:服务端口

  • 外部服务调用：负载均衡IP地址:负载均衡内映射端口或Ingress Url

# 3.1 示例—ClusterIP

1.新建 nginx-service.yaml

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  # 选择的类型 默认为ClusterIP
  type: ClusterIP
  selector:
    app: web_server
  ports:
    - protocol: TCP  # 协议类型
      port: 8080     # 端口
      targetPort: 80  # 转发到目标端口

# 3.2 示例：NodePort

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 3
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.20.1
          ports:
            - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
	type: NodePort
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
      nodePort: 30080 # 外网可以访问的端口

# 四、访问策略

支持的网络协议：Calico、Canal 等

配置的属性：

  • podSelector
  • ingress/egress
    • 这里的 ingress 与上面的负载均衡ingress不同，而是用来做访问限制的

配置示例：

apiVersion: networking.k8s.io.v1
kind: NetworkPolicy
metadata:
	name: access-nginx
spec:
	podSelector:
	matchLabels:
		app: web_server
	ingress:
    - from:
      - podSelector:
          matchLabels:
            access: "true" # 必须存在label：access=true 才可以访问
      - ipBlock:
          cidr: 10.244.0.0/16	# 允许内网其他容器可以访问
		ports:
      - protocol: TCP
        port: 80
