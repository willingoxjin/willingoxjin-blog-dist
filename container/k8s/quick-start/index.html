02.集群搭建与快速使用

# 一、Kubernates 集群环境搭建

Kubernates环境搭建官方文档：Installing kubeadm | Kubernetes (opens new window)

关于 kubectl 的文档：

  • Kubectl Reference Docs (opens new window)

  • Command line tool (kubectl) | Kubernetes (opens new window)

这里使用 CentOS安装

  • 软件安装：Docker、Kubelet（Kubernates进程）、Kubeadm（工具）、Kubectl（工具）
  • Master 上创建集群：kubeadm init && kubeadm apply
  • Node 加入集群：kubeadm join

# 1.软件安装

在所有的节点安装 Docker

# 下载安装脚本并运行，使用阿里云镜像
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun

# 使用国内 daocloud 一键安装
curl -sSL https://get.daocloud.io/docker | sh

# 重新加载daemon
systemctl daemon-reload

# 重新启动docker
systemctl restart docker

# 查看docker版本
docker version

在所有的节点安装 Kubelet、Kubeadm、Kubectl，命令如下：

# 配置kubernetesy的um镜像源
cat <<EOF | sudo tee /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=https://packages.cloud.google.com/yum/repos/kubernetes-el7-\$basearch
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://packages.cloud.google.com/yum/doc/yum-key.gpg https://packages.cloud.google.com/yum/doc/rpm-package-key.gpg
exclude=kubelet kubeadm kubectl
EOF


# Set SELinux in permissive mode (effectively disabling it)
sudo setenforce 0
# 更改selinux级别：用来最大限度地减小系统中服务进程可访问的资源
sudo sed -i 's/^SELINUX=enforcing$/SELINUX=permissive/' /etc/selinux/config

# 安装 install -y kubelet kubeadm kubectl
sudo yum install -y kubelet kubeadm kubectl --disableexcludes=kubernetes

# 设置为开机自动激活单元并现在立刻启动
sudo systemctl enable --now kubelet
# 或
systemctl enable kubelet
systemctl restart kubelet

# 2.Master 上创建集群

参考文档：Creating a cluster with kubeadm | Kubernetes (opens new window)

Master 节点中执行：

# 初始化集群
kubeadm init \
--apiserver-advertise-address=192.168.31.85 \
--pod-network-cidr=10.244.0.0/16 \

# 配置文件操作
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernates/admin.conf/$HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# 配置网络
kubectl apply -f [podnetwork].yaml
# 这里使用flannel网络，在使用时根据实际情况更改所选网络
kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
# or
kubectl apply -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml

# 到这里就安装好了，可以使用以下命令查看节点
kubectl get node

# 3.Node 加入集群

其他 Node 节点，加入到 master，以下命令会在master节点执行完kubeadm init之后进行打印：

kubeadm join 192.168.31.85 \
--token xxxxx \
--discovery-token-ca-cert-hash sha256:xxxxxxxxxx

# 二、使用 Kubernates 部署应用

# 1.Pod 方式部署 Nginx

新增配置文件 nginx-pod.yaml

# vi nginx-pod.yaml

# 调用的api版本
apiVersion: v1
# 类型
kind: Pod
# 元信息
metadata:
  name: my-nginx
# 配置
spec:
  containers:
    - name: nginx-container # 容器名
      image: nginx:1.20.1 # 容器镜像
      ports:
      	- containerPort: 80

kubectl exec -it my-nginx /bin/bash

# 2.Deployment 方式部署 Nginx

# 2.1 创建deployment

新增配置文件 nginx-deployment.yaml

# vi nginx-deployment.yaml

# api版本
apiVersion: apps/v1
kind: Deployment
# deployment的metadata
metadata:
  name: my-nginx-deployment
# 配置
spec:
  selector:
    matchLabels:
      app: web_server
  replicas: 3 # 部署三个节点
  template:
    # pod的metadata
    metadata:
      labels:
        app: web_server
    # pod的spec
    spec:
      containers:
        - name: nginx-container # 容器名
          image: nginx:1.20.1 # 容器镜像
          ports:
          	- containerPort: 80

# 2.2 修改 deployment

执行以下命令进入修改配置界面：

kubectl edit deployment my-nginx-deployment
1

然后我们修改配置：

spec:
  replicas: 5 # 修改节点个数为5个
1
2

修改后，按 Esc 键，输入 wq 保存

保存配置成功之后，配置没有问题的话，就会看到打印出成功的信息
