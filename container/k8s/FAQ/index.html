# 2.证书错误

## 执行的命令
kubectl apply

## 报错信息
Unable to connect to th server:x509: 
certificate signed by unknown authority  

## 解决方法：执行以下命令
mkdir -p $HOME/.kube
cp -i /ect/kubernates/admin.conf $HOME/.kube/config
chown $(id -u):$(id -g) $HOME/.kube/config

  • 常见错误：大小写、空格、对齐缩进、拼写、apiVersion、属性名、列表、args（/bin/sh -c 等）、英文符号

# 5.Pod状态异常

Pod 正常的状态：Running

Pod 异常状态：Init:CrashLoopBackOff

关闭SELinux

# 修改文件： /etc/selinux/config: 
SETLINUX=disabled

# 或执行命令
setenforce=0

# 6.Pod状态正常，但是无法解析DNS

6.1 iptable的问题：

# 修改iptable配置：转发策略开启
iptables -P FORWARD ACCEPT
1
2

6.2 检查service是否正常

kubectl get svc kube-dns --namespace=kube-system
1

6.3 检查endpoint是否正常

kubectl get ep kube-dns --namespace=kube-system
1

或使用 kubectl apply 命令刷新
