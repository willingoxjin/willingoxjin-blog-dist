项目管理基础

# 一、项目

# 1.什么是项目？

  • 为了创造一个唯一的独特的产品或服务，而进行的临时性的动作，而且随着项目的开展，最终目标会逐渐清晰明确。

  • 虽然项目是临时性的，但是项目的最终产物是一直存在的。这个最终产物叫做可交付成果。

如下属于项目：

  • 某次旅游
  • 某次团建
  • 长城建造
  • 研发新冠疫苗
  • 学习 Java 基础
  • 装修房子
  • 减肥

项目总结：

  • 明确的目标
  • 时间、成本、人
  • 不确定性
  • 一次性、临时性

# 2.项目与运营

需要注意的是，项目是一次性的，如果重复的就是属于日常活动，或叫运营，属于连续不断地动作。

如下例子属于运营：

  • 工厂生产汽车（每天都在生产）
  • 喝水吃饭（日常活动）
  • 上厕所（日常活动）
  • 淘宝双11/618 （每年的周期性活动）

运营总结：

  • 周而复始
  • 重复性
  • 日常活动
  • 运营活动

# 二、项目管理的五大过程

# 1.项目的过程

  • 输入
    1. 项目章程
    2. 商业文件
    3. 合同协议
    4. ……
  • 使用的方法、工具、技术
    1. 专家判断
    2. 举行会议
    3. 数据分析
    4. 问卷调查
    5. 头脑风暴
    6. ……
  • 输出
    1. 干系人登记册
    2. 变更请求
    3. ……

# 2.项目管理的五大过程组

  • 启动过程
  • 规划过程：项目的执行计划
  • 执行过程：根据规划，对项目进行实施或执行
  • 监控过程：贯穿于整个项目的生命周期，对项目进行审查，是否有偏差，并进行纠正
  • 收尾过程：项目的完成和失败，都需要进行一个收尾的过程。

规划过程和执行过程是一个循环，彼此相互依赖。

# 三、明确目标与过程

# 1.明确目标与过程

  • 项目确定
  • 启动项目
  • 明确目标
  • 驱动变更
  • 项目完成收尾

项目确定后，首先要做好基本工作与前期的规划，进行项目的启动工作，并告知所有的相关干系人，让相关人员了解项目的目标，相互之间的配合等工作。

驱动变更：项目的执行有时候会推动业务的发展

项目完成后，需要对项目进行总结和收尾工作，也可以开一个发布会议，对相关干系人进行项目的展示。

# 2.项目的收益

  • 增加金钱、股票等物质收益
  • 公共设施：部门项目属于公路、铁路等公共设施类型建设的项目，能够造福大众
  • 增加市场占有率
  • 提高品牌效应
  • 对产品申请知识产权等专利类的，后续专利可能会带来收益

# 3.项目的终止

项目终止原因：

  • 绩效达标
  • 项目烂尾
  • 缺少资金
  • 被动取消
    • 如需求消失、甲方终止、战略变更、环境因素（疫情）等
  • 缺乏资源
    • 如特殊的人力资源
  • 市场政策

# 四、项目与产品的生命周期

# 1.项目的生命周期

项目生命周期的特点：

  • 成本投入

  • 风险与影响

红-风险不确定性、干系人的影响、需求的变更数

绿-风险的影响力及代价

前期会有领导或甲方等干系人会对需求变更、项目范围变化等有较的0影响，随着时间的推移，项目距离收尾越来越近，由于项目已经完成了部分工作，这些影响就会越来越小。

风险也会随着项目的进度逐渐降低。

随着项目的执行，项目的功能越来越多，一旦存在需求变更，就会影响已经做好的一些功能；同样的投入的成本也会越来越多，所以风险的影响也会更大。

# 2.产品的生命周期

  • 概念：提出概念，构思
  • 计划：计划规划产品
  • 开发：研发产品
  • 验证：投入市场反馈
  • 发布：产品上线
  • 运维：运行维护产品、运营
  • 下线：产品的下线、消亡

产品中的计划、开发、验证、发布的过程就相当于项目的生命周期。

# 五、项目生命周期的类型

预测型

  • 一次计划、一次交付、交付即完工
  • 此类型需要公司或产品经理对该类型产品有丰富的经验
  • 一般为传统型项目，如 ERP、OA系统

迭代型

  • 随着客户的反馈，逐步迭代
  • 产品从模糊到越来越清晰
  • 大部分的互联网产品都是迭代性质的

增量型

  • 基于之前的项目，添加新的功能。从少到多
  • 如，微信基于聊天朋友圈，后续逐渐增加了微信支付、公众号、短视频等功能

适应型

  • 迭代和型增量型组合
  • 功能变更，并增加新的功能

混合型

  • 预测型与适应型的组合
  • 在硬件上的软件进行更新。硬件一次性开发，软件逐渐迭代
  • 常在硬件类型的开发中

# 六、项目管理的误区

  1. 把资质认证（PMP）作业项目管理的前提
    • 没有认证证书也能做项目经理
  2. 光做计划、写文档、发邮件
    • 最主要的工作是协调各方，保证项目按照计划进行
  3. 不确定项目范围（需求）
    • 没有确定项目范围或需求时，会造成很多成本
  4. 随意变更范围（需求）
    • 需求或其他发生变化，会影响进度、造成成本超支。一旦出现变更一定要走变更流程
  5. 不考虑风险
  6. 把自己与团队隔离
    • 多增加团队的凝聚力、打造良好的工作氛围，提交成员的归属感。如组织活动、团聚等活动
  7. 不进行有效沟通
    • 提高沟通效率、最好选择当面沟通或电话沟通
  8. 亲自上阵
    • 管理者应该做好资源调度，做好人力管理

# 七、什么是成功的项目？

  • 基本要素

    • 多：产能、产出多
    • 快：项目研发进度快
    • 好：产品质量高
    • 省：控制好成本

  • 干系人满意度

    • 领导或同级别同事的想法、用户满意度等

  • 公司的战略规划

# 八、项目管理与项目经理

# 1.项目经理

项目经理（Project Manager），简称 PM。是由组织委派的，去领导团队调动资源实现项目目标的个人。

若项目存在多个模块，需要共同执行，每个模块可以由一个项目经理领导，所有的项目经理由项目总监来领导与协调。

项目经理应该协调好其他领域的同事，使其帮助完成工作。

其他职能领域：

  • 技术领域：技术经理，或架构师

  • 业务领域：业务经理，业务架构师

  • 产品领域：产品经理

  • 设计领域：设计经理，设计总监

  • 网络领域：网络安全经理

项目经理最好对其他领域有些了解（无需精通）。面对客户时，对需求研发方面能够有所回应（显得更专业）。对下属管理也能更容易。

项目经理的基本能力

  • 其他领域的基础理论知识（设计、开发、产品、财务等）
