# 1.2 Home.vue 加入组件

  1. 先在<template>中加入轮播图组件
  2. 然后在 <script> 中配置组件的数据
  3. 最后在函数 created()函数中使用axios请求后端，得到后端返回的数据。并将数据赋值给变量imgs
<template>
 <div class="home">
    <!-- banner图片 -->
    <mt-swipe :auto="4000">
      <!-- v-for 组件的时候需要给key，避免vue计算key，来提升性能 -->
      <!-- key就是位置的标识 -->
      <mt-swipe-item v-for="(item,index) in imgs" :key="index">
        <img :src="item.bannerUrl" :alt="item.bannerName">
      </mt-swipe-item>
    </mt-swipe>
  </div>
</template>

<script>
export default {
  name: 'Home',
  data () {
    return {
      imgs : [],// 轮播图数据
    }
  },
  // 生命周期 created 
  created () {
    // request 请求 （基础请求配置详见main.js）
    this.$axios.get('home/banner')
    .then(res => {
      //console.log(res)
      this.imgs = res.data.data
    })
    .catch(err => console.log(err))
  }
}
</script>

        next(); // 放行
    },
}

这里使用mint-ui的组件：加载提示框 | Indicator

由于所有的请求时间过长的时候，我们都需要一个加载提示框，因此我们定义一个拦截器，在触发事件/请求发起前显示加载提示，到响应结束后关闭加载提示。

在src/main.js中配置拦截器

// 定义拦截器——加载提示>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>START
// 1.发起请求前显示 Loading open()
Axios.interceptors.request.use(function (config) {
  // 配置不变，配置改变：可以设置公共的请求头操作
  // 显示加载图标
  MintUI.Indicator.open({
    text: "正在玩命加载中...",
    spinnerType: 'fading-circle'
  });
  //console.log(config);
  return config;
})

// 2.响应收到之后关闭 Loading close()
Axios.interceptors.response.use((response)=>{
    	// response: {config:{header:{}},data()}

    	// 接收响应头或者响应体中的数据，保存起来供请求的拦截器中使用头信息操作
    	MintUI.Indicator.close(); // 关闭加载图标
    	//console.log(response);
   		return response;
	},function (error) {
       // 对请求错误做些什么
       MintUI.Indicator.close(); // 关闭加载图标
       // 提示
       MintUI.Toast({
            message: '请求超时',
            position: 'middle',
            duration: 1*1000
       });
    	return Promise.reject(error);
	}
)

​ 由于首页和商品详情页都用到了轮播图，并且两个地方的效果差不多，我们就考虑封装一个组件，以便我们多个地方使用

  • 根据请求地址传入子组件
  • 根据对象(轮播图对象)传入子组件

我们在这里绑定了父组件传来的属性url和属性imgs:

​ url属性是用来传递父组件的请求地址的，由于轮播图是只用来展示的，大概率是使用get请求，因此我们在这里选择传入url地址，我们在created()中根据传过来的请求地址向后端服务区发送请求，然后获得返回数据渲染到页面。

​ imgs属性用来接收父组件传过来的的轮播图对象。

​ 在这里我们使用了计算属性来对两个数据进行判断，哪个对象有值就是使用那个对象渲染到页面上

	// 不过，你可以通过传一个回调给 next来访问组件实例。在导航被确认的时候执行回调，并且把组件实例作为回调方法的参数。
        next(vm => {
