# kali的apt源说明

​ kali-rolling是Kali的即时更新版，只要kali中有更新，更新包就会放入kali-rolling中，供下载和使用，它为用户提供稳定更新的版本，同时带有新的安装包，是我们常用的源

kali-rolling下有3类软件包：main 、non-free、 contrib

dists区域 软件包组件标准
main 遵从Debian自由软件指导方针（DFSG），并不依赖于 non-free
contrib 遵从Debian自由软件指导方针（DFSG），但依赖于non-free
non-free 不遵从Debian自由软件指导方针（DFSG）

# DNS域名解析

# 1.DNS服务器

​ 运行DNS服务器程序的计算机，储存DNS数据库信息，DNS服务器分为根域DNS服务器，顶级域名DNS服务器。

​ 顶级DNS服务器有13个，都存储了全部的顶级 域名服务器所在的地址；顶级域名服务器存储了每位客户注册的主机地址。

# 2.域名记录：A、Cname、NS、MX、PTR

  • A记录（Address）正向解析

    A记录是将一个主机名（全称域名FQDN）和一个IP地址关联起来，这也是大多数客户端程序默认的查询类型，例：www.baidu.com -> 14.215.177.39

  • PTX记录(Point)反向解析 PTX记录将一个ip地址对应到主机名（全称 FQDN），这些记录保存在in-addr.arpa域中。

  • CNAME记录（Canonical Name）别名 别名记录，也称为规范名字。这种记录允许您将多个名字映射到同一台计算机。 例如：www.baidu.com 对应ip8.8.8.8 ，web.baidu.com 对应ip8.8.8.8

  • MX记录（Mall eXchange） MX记录是邮件交换记录，它指向一个邮件服务器，用于电子邮件系统时根据 收件人的地址后缀定位邮件服务器 例如： aaa@163.com 当有多个MX记录（即有多个邮件服务器）时，则需要设置数值来确定其优先级。通过设置优先级数字来指明首选服务器，数字越小表示优先级越高

  • NS记录（Name Server） NS记录是域名服务器记录，也成为授权服务器，用来指定该域名由哪个DNS服务器来进行解析 即域名和服务器ip地址绑定 可在https:wanwang.aliyun.com/ 购买域名绑定

  • DNS缓存服务器 不负责解析域名，只是用来缓存域名解析结果

DNS查询方式：递归查询和迭代查询 一个DNS查询过程，通过8个步骤的解析过程就使得客户端可以顺利访问www.163.com这个域名，但实际应用中，通常这个过程是非常迅速的

DNS查询过程 1.浏览器缓存：当用户通过浏览器访问某域名时，浏览器首先会在自己的缓存中查找是否有该域名对应的ip地址 2.系统缓存：当浏览器缓存中无域名对应的ip则会自动检查用户计算机系统hosts文件DNS缓存是否有该域名对应ip 3.路由器缓存：当浏览器和系统缓存中都没有域名对应的ip，则会进入路由器缓存中检查 ，以上三步为客户端的DNS缓存 3.ISP（互联网服务提供商）DNS缓存（一般就是本地DNS服务器） 当在用户客户端查找不到域名对应ip地址，则将进入ISP DNS缓存中进行查询，比如你用的是电信的网络，则会进入电信的DNS缓存服务器中进行查找 5.根域名服务器 当以上步骤均为查询到域名对应的ip，则进入根服务器进行查询。 全球仅有13台根服务器，1个主根服务器，其余12为辅根域名服务器。 根域名收到请求后会查看区域文件记录，若无则将其管辖范围内顶级域名（如.com）服务器IP告诉本地DNS服务器 6.顶级域名服务器 顶级域名服务器受到请求后查看区域文件记录，若无则将其管辖范围内主域名服务器的IP地址告诉本地DNS服务器 7.主域名服务器 主域名服务器接受到请求后查询自己的缓存，如果没有则进入下一级域名服务器进行查找，并重复该步骤直到找到正确记录 8.保存结果至缓存 本地域名服务器把返回的结果保存到缓存，以备下一次使用，同时将该结果反馈给客户端，客户端通过这个IP地址与web服务器建立连接

# 主动信息收集原理

特点

  • 直接与目标系统交互通信
  • 无法避免留下访问痕迹
  • 使用受控的第三方电脑进行探测（跳板机器，肉机），使用代理或已经被控制的机器，做好被封杀的准备
  • 扫描发送不同的探测，根据返回结果判断目标状态

发现目标主机

  1. 识别存活主机，发现潜在的被攻击目标
  2. 输出一个IP地址列表，比如IP地址段、IP地址范围
  3. 使用二三四层()进行探测发现

OSI七层模型和TCP/IP五层模型

OSI参考模型 解释
应用层 为应用程序提供服务
表示层 数据格式转化，数据加密
会话层 建立、管理、维护会话
传输层 建立、管理、维护端到端的连接
网络层 IP选址及路由选择
数据链路层 提供介质访问和链路管理
物理层 物理层

第七层：应用层

​ OSI中的最高层，为特定类型的网络应用提供了访问OSI环境的手段，应用层确定进程之间通信的性质，以满足用户的需要。应用层不仅要提供进程所需要的信息交换和远程操作，而且还要作为应用进程的用户代理，来完成医学为进行信息交换所必须的功能，它包括：文件传送访问和管理FTAM、虚拟终端VT、事务处理TP、远程数据库访问RDA、制造报文规范MMS、目录服务DS等协议：应用层能与应用程序界面沟通，已达到展示给用户的目的。在此参考的常见协议：HTTP、HTTPS、FTP、TELNET、SSH、SMTP、POP3等。

第六层：表示层

​ 主要是用于处理两个通信系统中交换信息的表示方式，为上层用户解决用户信息的语法问题。它包括数据格式交换、数据加密和解密、数据压缩与终端类型的转换

第五层：会话层

​ 在两个节点中建立端连接，为端系统的应用程序之间提供了对话控制机制。此服务包括建立连接是以全双工还是以半双工的方式进行设置，尽管可以在层4中处理双工方式；会话层管理登入和注销过程，它具体管理两个用户和进程之间的对话。如果在某一时刻只允许一个用户执行一项特定的操作，会话层协议就会管理这些操作，如阻止两个用户同时更新数据库中的同一组数据。

第四层：传输层

​ 传输层建立了主机端口到端口的链接，传输层的作用是为上层协议（上层会话层）提供端口到端口的可靠和透明的数据传输服务，包括处理差错控制和流量控制等问题。该层向高层屏蔽了下层数据通信的细节，使高层用户看到的只是在两个传输实体间的一条主机到主机的、可由用户控制和设定的、可靠的数据通路。我们通常说的，TCP UDP就是在这一层。

第三层：网络层

​ 本层通过IP寻址来建立两个节点之间的连接，为源端的运输层送过来的分组，选择合适的路由和焦化节点，正确无误的按照地址传送给目的端的运输层。就是通常说的IP层。这一层就是我们经常说的IP协议层。IP协议是Internet的基础

第二层：数据链路层

​ 将比特组合或字节，再将字节组合成帧，使用链路层地址（以太网使用MAC地址）来访问介质，并进行差错检测

第一层：物理层

​ 处于OSI参考模型的最底层。物理层的主要功能是利用物理传输介质为数据链路层提供物理连接，以便透明的传送比特流，常用设备有（各种物理设备）网卡、集线器、中继器、调制调解器、网线、双绞线、同轴电缆。

基于OSI模型进行扫描的优缺点

  • 二层：数据链路层

    • 优点：扫描速度快，可靠
    • 缺点：不可路由

  • 三层：网络层

    • 优点：可路由，速度较快
    • 缺点：速度比二层慢，经常被边界防火墙过滤
    • 如ping命令基于IP ICMP

  • 四层：传输层

    • 优点
      • 可路由且结果可靠
      • 不太可能被防火墙过滤
      • 可以发现所有端口都被过滤的主机
    • 缺点
      • 基于状态过滤的防火墙可能被过滤
      • 全端口扫描速度慢
    • 如Namp

# nmap介绍

Namp，也就是Network Mapper，最早是Linux下的网络扫描和嗅探工具包

namp可用于：

  • 发现主机
  • 校测主机的开放端口
  • 检测到相应的端口（服务发现）的软件和版本
  • 检测操作系统、硬件地址、以及软件版本
  • 检测脆弱的漏洞（Namp的脚本）

namp端口状态解析

  • open：应用程序在该端口接收TCP连接或UDP报文
  • closed：关闭的端口对于nmap也是可访问的，它接收nmap探测报文并作出响应。但没有应用程序在其上监听。
  • filtered：由于包过滤阻止探测报文到达端口，nmap无法确定该端口是否开放过。过滤可能来自专业的防火墙设备，路由规则或者主机上的软件防火墙。filtered状态的端口可能是开放的也可能是关闭的
  • unfiltered：未被过滤状态意味着端口可访问，但是nmap无法确定它是开放还是关闭。只有用于映射防火墙规则集的ACK扫描才会把端口分类到这个状态。
  • open|filtered：无法确定端口是开放还是被过滤，如开放的端口不响应，可能意味着报文过滤器丢弃了探测报文或者它引发的任何反应，UDP，IP，FIN，Null等扫描会引起
  • close|filtered：无法确定端口是关闭或被过滤

nmap扫描主要有TCP的全连接扫描（会被扫描机器留下记录），半连接扫描（不会留下记录）

TCP三次握手：

四次挥手：

为什么“握手”是三次，“挥手”却要四次：在第二次"握手"过程中，服务器端发送给客户端的TCP报文是以SYN与ACK作为标志位的。SYN是请求连接标志，表示服务器端同意建立连接；ACK是确认报文，表示告诉客户端，服务器端收到了它的请求报文。

  • URG：紧急指针（urgent pointer）有效。

  • ACK：确认序号有效。

  • PSH：接收方应该尽快将这个报文交给应用层。

  • RST：重置连接。

  • SYN：发起一个新连接。

  • FIN：释放一个连接。

扫描方式：

  • Intense scan
    • 命令：nmap -T4 -A -v
      • -A 对操作系统和软件版本进行检测，并对目标主机traceroute路由探测，-O 参数仅识别目标操作系统，并不做软件版本检测和路由探测
      • -T4 指定扫描过程使用的时序（Timing），总有6个级别（0-5），级别越高，扫描速度越快，但也容易被防火墙或IDS检测并屏蔽掉，在网络通讯状况良好的情况下推荐使用T4
      • -v 表示显示冗余（verbosity）信息，在扫描过程中显示扫描的细节，从而让用户了解当前的扫描状态。
  • Intense scan plus UDP
    • 命令：nmap -sS -sU -T4 -A -v
    • UDP扫描
      • -sS TCP SYN扫描（半连接扫描）
      • -sU UDP 扫描
  • Intense scan,all TCP ports
    • 命令：nmap -p 1-65535 -T4 -A -v
      • 扫描所有TCP端口，范围在1-65535，试着扫描所有端口的开放情况，速度较慢
      • -p指定端口扫描范围
  • Intense scan,no ping
    • 命令：nmap -T4 -A -v -Pn
      • -Pn 非ping扫描
  • Ping san
    • 命令：nmap -sn
      • -sn ping扫描，速度快，容易被防火墙屏蔽，导致扫描结果无效
  • Quick scan
    • 命令：nmap -T4 -F
    • 快速扫描
    • -F快速扫描
  • Quick scan plus
    • 命令：nmap -sV -T4 -O -F --version-light
    • 快速扫描加强模式
      • -sV 探测端口及版本服务信息
      • -O开启OS检测，操作系统检测
      • --version-light 设定侦测等级为2
  • Quick traceroute
    • 命令：nmap -sn -traceroute
    • 路由跟踪
    • -sn ping扫描，关闭端口扫描
    • -traceroute 显示本机到目标的路由跃点
  • Regular scan
    • 命令：namp ip地址
    • 常规扫描
  • Slow comprehensive scan
    • 命令： nmap -sS -Su -T4 -A -v -PE -PP -PS80,443,-PA3389,PU40125 -PY -g 53 --script all
    • 慢速全面扫描

# 僵尸扫描

做渗透最重要的是思维，僵尸扫描的实用性不大，但是思维重要

僵尸主机，僵尸主机是指感染僵尸程序病毒，从而被黑客程序控制的计算机设备，但是僵尸扫描中的僵尸主机指的是一个闲置的操作系统（这里的闲置是指主机不会主动和任何人通信），且此系统中IP数据包中ID是递增的

僵尸扫描拥有极高的隐蔽特性，但是实施条件苛刻：

​ 1.目标网络可伪造源地址进行访问

​ 2.选择僵尸机，僵尸机需要在互联网上是一个闲置的操作系统，需要系统使用递增的IPID（如XP系统）

我们可以使用一台/多台其他的代理、肉机、作为跳板机扫描探测目标主机。

僵尸扫描可以不拿到肉机权限，只要对方的IPID是自增长的就可以。

僵尸扫描原理：

1.端口开放状态扫描

​ TCP三次握手发包过程中，SYN/ACK是第二次包。若没有收到数据包则端口是关闭的

  • 第一步，我们向僵尸机发送SYN/ACK数据包，僵尸主机会拒绝连接，并回复RST数据，其中带着IPID
  • 第二步，我们模拟僵尸机(IPSRC=僵尸机)向目标主机发送SYN数据包，目标主机就会向僵尸机发送SYN/ACK数据包，回复RST数据，其中带着IPID（此时IPID加一）
  • 第三步骤，我们重复第一步，
    • 若最后得到的僵尸机的IPID比我们第一步得到的IPID增长了2，就说明目标主机的端口是开放的
    • 若最后得到的僵尸机的IPID比我们第一步得到的IPID增长了1，就说明目标主机的端口是开放的
