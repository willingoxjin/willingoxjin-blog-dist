# 一、绘制实时流程图片

# 1.绘制普通流程图片

@ParameterizedTest
@CsvSource({
        "draw_workflow_test, /tmp/draw-diagram/model--flow-diagram.png",
})
@DisplayName("绘制普通流程图")
void drawDiagram(String processDefinitionKey, String filepath) throws IOException {
    ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
            .processDefinitionKey(processDefinitionKey)
            .latestVersion()
            .singleResult();
    Assertions.assertNotNull(processDefinition);

    // 查询流程定义对象
    BpmnModel bpmnModel = repositoryService.getBpmnModel(processDefinition.getId());
    // 获取流程图生成器
    ProcessDiagramGenerator diagramGenerator = new DefaultProcessDiagramGenerator();
    // 调用方法生成流程图
    InputStream inputStream = diagramGenerator.generatePngDiagram(bpmnModel, 1.0, true);
    // 保存图片
    org.apache.commons.io.FileUtils.copyInputStreamToFile(inputStream, new File(filepath));
}

@ParameterizedTest
@CsvSource({
        "draw_workflow_test, e3766d9c-0345-11ee-be4e-b6215b2c8d0e, /tmp/draw-diagram/ru-activity-flow-diagram.png",

@DisplayName("绘制实时流程图")
void drawRuntimeDiagram(String processDefinitionKey, String processInstanceId, String filepath) throws IOException {
    ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
            .processDefinitionKey(processDefinitionKey)
            .latestVersion()
            .singleResult();

    // 查询实时完成的流程
    List<String> highLightedActivities = new ArrayList<>();
    List<String> highLightedFlows = new ArrayList<>();
    List<ActivityInstance> activityInstanceList = runtimeService.createActivityInstanceQuery()
            .processDefinitionId(processDefinition.getId())
            .processInstanceId(processInstanceId)
            .list();
    for (ActivityInstance instance : activityInstanceList) {
        if (instance.getActivityType().equals("sequenceFlow")) {
            highLightedFlows.add(instance.getActivityId());
        } else {
            highLightedActivities.add(instance.getActivityId());
        }
    }

    // 查询流程定义对象
    BpmnModel bpmnModel = repositoryService.getBpmnModel(processDefinition.getId());
    // 获取流程图生成器
    ProcessDiagramGenerator diagramGenerator = new DefaultProcessDiagramGenerator();
    // 调用方法生成流程图
    InputStream inputStream = diagramGenerator.generateDiagram(bpmnModel, "PNG", highLightedActivities,
            highLightedFlows, 1.0, true);
    // 保存图片
    org.apache.commons.io.FileUtils.copyInputStreamToFile(inputStream, new File(filepath));
}

# 3.绘制已结束的流程图片

已经结束的流程，我们需要使用 HistoryService 中的方法查询出激活的实例。

如下代码，将绘制流程已经结束的流程图：

@ParameterizedTest
@CsvSource({
        "draw_workflow_test, e3766d9c-0345-11ee-be4e-b6215b2c8d0e, /tmp/draw-diagram/hi-activity-flow-diagram.png",
})
@DisplayName("绘制已结束的历史流程图")
void drawHiDiagram(String processDefinitionKey, String processInstanceId, String filepath) throws IOException {
    ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
            .processDefinitionKey(processDefinitionKey)
            .latestVersion()
            .singleResult();
    // 查询历史完成的流程
    List<String> highLightedActivities = new ArrayList<>();
    List<String> highLightedFlows = new ArrayList<>();
    List<HistoricActivityInstance> activityInstanceList = historyService.createHistoricActivityInstanceQuery()
            .processDefinitionId(processDefinition.getId())
            .processInstanceId(processInstanceId)
            .list();
    for (HistoricActivityInstance instance : activityInstanceList) {
        if (Objects.equals(instance.getActivityType(), "sequenceFlow")) {
            highLightedFlows.add(instance.getActivityId());
        } else {
            highLightedActivities.add(instance.getActivityId());
        }
    }

    // 查询流程定义对象
    BpmnModel bpmnModel = repositoryService.getBpmnModel(processDefinition.getId());
    // 获取流程图生成器
    ProcessDiagramGenerator diagramGenerator = new DefaultProcessDiagramGenerator();
    // 调用方法生成流程图
    InputStream inputStream = diagramGenerator.generateDiagram(bpmnModel, "PNG", highLightedActivities,
            highLightedFlows, 1.0, true);
    // 保存图片
    org.apache.commons.io.FileUtils.copyInputStreamToFile(inputStream, new File(filepath));
}

# 4.自定义流程绘制类

如果我们想要修改绘制图片中，高亮的颜色，字体等样式，可以进行如下操作：

  1. 首先，新建一个自定义的 ProcessDiagramCanvas 类，继承默认的 DefaultProcessDiagramCanvas 类，重写相关的方法，如 drawHighLight 方法 。
  2. 然后，创建一个自定义的 ProcessDiagramGenerator 类，实现 ProcessDiagramGenerator 接口或者继承自 DefaultProcessDiagramGenerator 类，并将 DefaultProcessDiagramGenerator 中原有使用 DefaultProcessDiagramCanvas 类的地方修改为你自己实现的的 DefaultProcessDiagramCanvas 子类。
  3. 最后，在使用时，直接创建你自定义的 ProcessDiagramGenerator 类即可。

# 2.使用示例

下面的代码是一个使用了 Camunda 的示例：

<template>
  <div style="width: 100%;">
    <div class="page-container">
      <div class="bpmn-container" id="bpmn-container"></div>
      <div id="bpmn-properties"></div>
    </div>

    <!-- 下载xml文件按钮 -->
    <div>
      <button @click="downloadXML">下载xml</button>
    </div>
  </div>
</template>

<script setup>

import { onMounted } from 'vue'
import BpmnModeler from 'bpmn-js/lib/Modeler'
import {
  BpmnPropertiesPanelModule,
  BpmnPropertiesProviderModule,
  CamundaPlatformPropertiesProviderModule
} from 'bpmn-js-properties-panel'
import camundaModdleDescriptors from 'camunda-bpmn-moddle/resources/camunda'
// 引入样式文件
import 'bpmn-js/dist/assets/diagram-js.css'
import 'bpmn-js/dist/assets/bpmn-font/css/bpmn.css'
import 'bpmn-js/dist/assets/bpmn-font/css/bpmn-codes.css'
import 'bpmn-js/dist/assets/bpmn-font/css/bpmn-embedded.css'
import 'bpmn-js-properties-panel/dist/assets/properties-panel.css'

let bpmnModeler;

onMounted(() => {
  // 初始化画布
  bpmnModeler = new BpmnModeler({
    // 左边画布工具
    container: '#bpmn-container',
    // 右边属性面板栏
    propertiesPanel: {
      parent: '#bpmn-properties'
    },
    additionalModules: [
      BpmnPropertiesPanelModule,
      BpmnPropertiesProviderModule,
      CamundaPlatformPropertiesProviderModule,
    ],
    moddleExtensions: {
      camunda: camundaModdleDescriptors
    }
  })
  // 开始绘制
  bpmnModeler.createDiagram()
})

function downloadXML() {
  bpmnModeler.saveXML({
    format: true
  }, (err, xml) => {
    if (err) {
      console.error(err)
      return
    }
    if (!xml) return

    // 获取文件名
    const name = getFileName(xml);
    // xml内容转换为下载的为URI
    const encodedData = encodeURIComponent(xml);
    const downloadLink = document.createElement('a');
    // 将数据给到链接
    downloadLink.href = 'data:application/bpmn20-xml;charset=UTF-8,' + encodedData
    // 设置文件名
    downloadLink.download = name
    // 触发点击事件开始下载
    downloadLink.click()
  })
}

function getFileName(xml) {
  let strs = xml.split('process id="')
  return strs[1].split('" ')[0] + ".bpmn20.xml";
}

</script>

# 三、workflow-bpmn-modeler

Github 地址：https://github.com/GoldSubmarine/workflow-bpmn-modeler (opens new window)

在线体验地址：https://goldsubmarine.github.io/workflow-bpmn-modeler/demo/ (opens new window)

workflow-bpmn-modeler 是基于 Vue 和 bpmn.io@7.0 实现的 Flowable 工作流绘制工具包。

该工具建议使用 Flowable 6.4.1 版本。

并且该工具适用于 Vue 2.x 的版本。

安装命令：

npm i workflow-bpmn-modeler
