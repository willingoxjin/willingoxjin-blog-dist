6.Flowable 流程变量

# 一、流程变量介绍

# 1.流程变量的作用

在 Flowable 中，流程变量是在流程实例执行过程中存储和传递数据的一种机制。

流程变量可以包含任何类型的数据，如字符串、数字、日期、布尔值等，它们在流程实例中被使用，例如用于控制流程执行路径、在任务中传递数据、实现分支条件等。

通过设置流程变量，我们可以将数据从一个流程步骤传递到另一个流程步骤。例如，可以在启动流程时设置流程变量，在整个流程实例执行期间使用该变量，然后在最后一个流程步骤中检索和使用该变量。流程变量还可以用于存储和检索业务规则和决策所需的数据，以及在流程实例中实现动态计算和逻辑。

# 2.流程变量的分类

流程变量按照范围，可以分为以下类型：

  1. 全局流程变量
    • 全局流程变量是在流程实例启动时创建的，并在整个流程执行期间一直存在。
    • 全局流程变量可以被所有的任务和子流程访问，并且可以被用于在流程中共享数据。
  2. 本地流程变量
    • 本地流程变量是在特定的执行环境中创建的，通常是与任务或子流程相关联。
    • 本地流程变量只能在创建它们的任务或子流程中访问，并且在这些任务或子流程生命周期内一直存在。
  3. 临时流程变量
    • 临时流程变量是在流程执行期间临时创建的，通常是在脚本或表达式中定义的。
    • 临时流程变量只在定义它们的表达式或脚本的上下文中存在，并且一旦脚本或表达式完成执行，它们就会被销毁。

# 1.流程启动时设置

可以通过如下方式在流程启动时，设置全局流程变量。

@ParameterizedTest
@CsvSource({
        "variable_test, zhangsan, willingoxjin"
})
@DisplayName("启动时设置全局流程变量")
void testSetVarByStartProcess(String processDefinitionKey, String userId, String tenantId) throws ParseException {
    Authentication.setAuthenticatedUserId(userId);
    Map<String, Object> var = new HashMap<>();
    var.put("name", "张三");
    var.put("reason", "身体不舒服，请假一天");
    Date startTime = DateUtils.parseDate("2023-05-24 09:00:00", "yyyy-MM-dd HH:mm:ss");
    var.put("startTime", startTime);
    Date endTime = DateUtils.parseDate("2023-05-24 18:00:00", "yyyy-MM-dd HH:mm:ss");
    var.put("endTime", endTime);
    ProcessInstance processInstance = runtimeService.startProcessInstanceByKeyAndTenantId(processDefinitionKey, var,
            tenantId);
}

    taskService.setVariables(task.getId(), var);
}

# 三、本地流程变量

本地流程变量和某一个执行的 Task 或子流程绑定，因此我们需要使用 TaskService 接口调用设置本地流程变量的方法。

我们新启动一个流程，然后在任务中设置一些本地流程变量。代码如下：

@ParameterizedTest
@CsvSource({
        "variable_test, willingoxjin"
})
@DisplayName("设置本地流程变量")
void testSetLocalVarByTask(String processDefinitionKey, String tenantId) {
    Task task = taskService.createTaskQuery()
            .processDefinitionKey(processDefinitionKey)
            .taskTenantId(tenantId)
            .singleResult();
    // 设置单个本地变量
    taskService.setVariableLocal(task.getId(), "name", "李四");
    // 批量设置本地变量
    Map<String, Object> var = new HashMap<>();
    var.put("age", 18);
    var.put("gender", "女");
    taskService.setVariablesLocal(task.getId(), var);
}

@ParameterizedTest

public void testHiVariable(String processDefinitionKey, String tenantId) {

            .processDefinitionKey(processDefinitionKey)
            .processInstanceTenantId(tenantId)
            .singleResult();
    Assertions.assertNotNull(processInstance);
    List<HistoricVariableInstance> list = historyService.createHistoricVariableInstanceQuery()
            .processInstanceId(processInstance.getId())
            .list();
    list.forEach(hiVar -> {
        log.info("name={}, value={}, type={}", 
        				hiVar.getVariableName(), hiVar.getValue(), hiVar.getVariableTypeName());
    });
}

# 四、临时流程变量

临时流程变量不会存储在数据库中，一般可以在启动流程或完成任务的时候使用。

在启动流程时设置临时流程变量，代码如下：

@ParameterizedTest
@CsvSource({
        "variable_test, willingoxjin"
})
@DisplayName("启动时设置临时变量")
void testSetTransientByStart(String processDefinitionKey, String tenantId) throws ParseException {
    Map<String, Object> vars = new HashMap<>();
    vars.put("reason", "身体不舒服，请假一天");
    Date startTime = DateUtils.parseDate("2023-05-24 09:00:00", "yyyy-MM-dd HH:mm:ss");
    vars.put("startTime", startTime);
    Date endTime = DateUtils.parseDate("2023-05-24 18:00:00", "yyyy-MM-dd HH:mm:ss");
    vars.put("endTime", endTime);
    ProcessInstance instance = runtimeService.createProcessInstanceBuilder()
            .processDefinitionKey(processDefinitionKey)
            .tenantId(tenantId)
            .transientVariable("name", "张三")
            .transientVariables(vars)
            .start();
}

# 五、查询流程变量

# 1.通过执行实例查询

我们可以通过以下方式查询流程变量：

@ParameterizedTest
@CsvSource({
        "variable_test, zhangsan, willingoxjin"
})
@DisplayName("启动时设置全局流程变量")
void testQueryVar(String processDefinitionKey, String tenantId) {
    List<Execution> list = runtimeService.createExecutionQuery()
            .processDefinitionKey(processDefinitionKey)
            .executionTenantId(tenantId)
            .list();
    list.forEach(execution -> {
        // 获取所有流程变量
        Map<String, Object> var = runtimeService.getVariables(execution.getId());
        log.info("执行实例ID：{}，流程变量：{}", execution.getId(), var);
        // 获取单个流程变量
        String name = runtimeService.getVariable(execution.getId(), "name", String.class);
       	log.info("name：{}", name);
    });
}

# 2.通过任务查询

我们也可以通过任务查询全局流程变量：

@CsvSource({
        "variable_test, willingoxjin"
})
@DisplayName("查询流程变量")
public void testSetVarByTask(String processDefinitionKey, String tenantId) {
    Task task = taskService.createTaskQuery()
            .processDefinitionKey(processDefinitionKey)
            .taskTenantId(tenantId)
            .singleResult();

  	// 先根据TASK_ID_查询流程变量再根据EXECUTION_ID_查询流程变量
  	// 获取所有流程变量
  	Map<String, Object> variables = taskService.getVariables(task.getId());
    variables.forEach((k, v) -> {
        System.out.println(k + "=" + v);
    });
		// 获取单个流程变量
    String name = taskService.getVariable(task.getId(), "name", String.class);
}
