# 一、租户

# 1.什么是租户？

租户是指在一个共享的系统或资源中，使用这些系统或资源的不同组织或个人。在软件开发中，租户可以是指在一个应用程序中同时使用的不同客户，或者在一个云服务平台中使用的不同用户。通常，每个租户都有自己的数据、设置和权限，以确保它们之间的隔离和安全性。

举个例子，现在公司有个校园系统的产品，有很多个学校在使用，所有的学校的数据是需要隔离的，每个学校的管理员只能看到自己学校的数据。 对这种情况我们就可以用租户实现，如在每个表中添加一个租户 id，每个学校通过自己学校的域名访问系统，然后我们通过域名找对应的租户 id。当在数据库中查询、新增、修改、删除数据时，都加上租户 id 的过滤条件，就可以实现在同一个系统中支持多个学校使用，并使得数据隔离。

实现租户的方式有很多种，上面示例中只是将数据隔离。

实现租户的其他方式如下：

  1. 数据库隔离：使用不同的数据库或者使用数据库表的前缀或者后缀来区分不同的租户，每个租户只能访问自己的数据库或表。
  2. 共享数据模型：使用共享的数据模型，但通过实现不同的数据访问策略或者访问控制规则来实现租户数据的隔离。（上面就是这种方式）
  3. 多租户架构：使用专门的多租户架构，对每个租户分配独立的资源和隔离机制，确保每个租户之间的数据、资源和安全性隔离。 （这里的资源表示服务器、带宽、短信服务、Redis 缓存等资源）
  4. 虚拟化：使用虚拟化技术来实现租户隔离，使用不同的虚拟机或者容器来隔离不同的租户。

# 2.Flowable 的租户

Flowable 的租户实现非常简单，就是跟我上面示例中说的一样，通过租户 id（tenantId）加以区分每一个流程属于哪个租户。

如下，添加、查询用户时，使用租户：

@ParameterizedTest
@CsvSource({

        "zhangyi, 张, 一, 张一, zhangyi@willingoxjin.com, 123456, willingoxjin.cn",
        "zhanger, 张, 二, 张二, zhanger@willingoxjin.com, 123456, willingoxjin.cn",
        "zhangsan, 张, 三, 张三, zhangsan@willingoxjin.com, 123456, willingoxjin.org",
})
@Order(1)
@DisplayName("保存用户")
void testSaveUser(String userId, String firstName, String lastName, String displayName, String email,
        String password, String tenantId) {
    User user = identityService.newUser(userId);
    user.setFirstName(firstName);
    user.setLastName(lastName);
    user.setDisplayName(displayName);
    user.setEmail(email);
    user.setPassword(password);
    user.setTenantId(tenantId);
    identityService.saveUser(user);
}

@ParameterizedTest
@CsvSource({
        "willingoxjin.cn",
        "willingoxjin.org",
})
@DisplayName("获取用户列表")
void testUserList(String tenantId) {
    List<User> list = identityService.createUserQuery()
            .tenantId(tenantId)
            .list();
    list.forEach(user -> System.out.println(user.getDisplayName()));
}

# 二、dataObject 数据对象

在 Flowable 中，dataObject 用于在流程实例生命周期内存储和传递与流程实例相关的数据。它可以是简单的数据类型（如字符串或整数）或复杂的自定义对象。

  • 数据对象可以在整个流程实例中共享，并且可以在流程中不同的任务和网关之间传递。

  • 它还可以在业务处理逻辑中使用，以便在流程实例中存储和检索与业务相关的数据。

通过使用数据对象，流程设计人员可以使业务逻辑更加灵活和可维护。

在流程图绘制的时候，我们可以设置一个为 dataObject（数据对象的）配置，这个 dataObject 可以指定变量 id、名称、数据类型和默认值。并且，在流程启动的时候，会自动的将 dataObject 的信息转换成流程实例变量。

@ParameterizedTest
@CsvSource({
        "data_object_test, willingoxjin"
})
@DisplayName("查询数据对象")
void testQueryObject(String processDefinitionKey, String tenantId) {
    List<Execution> list = runtimeService.createExecutionQuery()
            .processDefinitionKey(processDefinitionKey)
            .executionTenantId(tenantId)
            .list();
    list.forEach(execution -> {
        DataObject name = runtimeService.getDataObject(execution.getId(), "姓名");
        DataObject reason = runtimeService.getDataObject(execution.getId(), "事由");
        DataObject startTime = runtimeService.getDataObject(execution.getId(), "开始时间");
        DataObject endTime = runtimeService.getDataObject(execution.getId(), "结束时间");
        DataObject days = runtimeService.getDataObject(execution.getId(), "时长");
        System.out.println(name.getDataObjectDefinitionKey() + "=" + name.getValue());
        System.out.println(reason.getDataObjectDefinitionKey() + "=" + reason.getValue());
        System.out.println(startTime.getDataObjectDefinitionKey() + "=" + startTime.getValue());
        System.out.println(endTime.getDataObjectDefinitionKey() + "=" + endTime.getValue());
        System.out.println(days.getDataObjectDefinitionKey() + "=" + days.getValue());
    });
}

# 3.dataObject 与全局变量的区别

数据对象（dataObject）和全局变量（globalVariable）都用于在流程实例生命周期内存储和传递数据。但是，它们之间存在一些差异：

  1. 数据对象是流程实例级别的，而全局变量是引擎级别的。这意味着全局变量可以在引擎范围内共享，而数据对象只能在流程实例内共享。

  2. 数据对象可以在流程实例中的不同任务和网关之间传递，而全局变量不能。全局变量只能在流程实例启动时设置，在整个实例生命周期中都是不变的。

  3. 数据对象可以使用流程任务表单中的表单域直接访问，而全局变量不行。全局变量只能在 Java 代码中使用。

  4. 数据对象可以通过 Java API 进行动态访问和修改，而全局变量必须在引擎配置中进行静态设置。

综上所述，数据对象和全局变量都可以用于在流程实例中存储和传递数据，但它们在可用性和范围方面略有不同。
