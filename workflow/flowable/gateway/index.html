4.Flowable 网关

# 一、网关

网关（gateway）用于控制执行的流向。即根据某些条件来判断流程向哪个连线走。

# 二、排他网关

# 1.介绍

排他网关（exclusive gateway）也叫异或网关（XOR gateway），或者叫基于数据的排他网关（exclusive data-based gateway），用于对流程中的决策建模。

当执行到达这个网关时，会按照所有出口顺序流定义的顺序对它们进行计算。选择第一个条件计算为 true 的顺序流（如果没有设置条件，则认为顺序流为 true）继续流程。

这里出口顺序流的含义与BPMN 2.0中的一般情况不一样。

  • 一般情况下，会选择所有条件计算为 true 的顺序流，并行执行。
  • 而使用排他网关时，只会选择一条顺序流。当多条顺序流的条件都计算为 true 时，会且仅会选择在XML中最先定义的顺序流继续流程。如果没有可选的顺序流，会抛出异常。

# 2.使用

我们定义如下请假审批流程图：

排他网关后面的三条连线的流条件中设置的内容如下：

  • 小于等于三天：${days <= 3}
  • 大于三天，小于等于七天：${days>3 && days<=7}
  • 大于七天：${days > 7}

# 三、并行网关

# 1.介绍

并行网关（parallel gateway）可以执行一些并行任务。

并行网关一般都是成对出现的。其中一个并行网关用来分流，另外一个并行网关用来聚合：

  • 分流：所有的出口顺序流都并行执行，为每一条顺序流创建一个并行执行。
  • 聚合：所有到达并行网关的并行执行都会在网关处等待，直到每一条入口顺序流都到达聚合的并行网关，流程才会继续执行。

并行网关不会计算条件。如果连接到并行网关的顺序流上定义了条件，会直接忽略该条件

# 2.使用

我们定义如下部门办公用品申请的流程图：

# 三、包容网关

# 1.介绍

包容网关（inclusive gateway）可以看做排他网关与并行网关的组合。与排他网关一样，可以在包容网关的出口顺序流上定义条件，包容网关会计算条件。然而主要的区别是，包容网关与并行网关一样，可以同时选择多于一条出口顺序流。

# 2.使用

# 2.1 流程图

这里包容网关后面顺序流的条件配置如下：

  • 大于一千元：${amount > 1000}
  • 大于一万元：${amount > 10000}
