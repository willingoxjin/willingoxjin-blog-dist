7.Flowable 表单

# 一、表单的分类

Flowable 中的表单可以分成如下类型：

  1. 内置表单：我们在之前的 Flowable UI 中定义的表单就是内置表单
  2. 动态表单：动态表单可以根据不同的流程实例和任务自定义显示不同的字段和属性。
    • 动态表单可以通过使用 Flowable 的表单引擎来实现，可以使用表单设计器来创建和编辑表单，也可以使用表单 API 来动态生成表单。
    • 通过使用动态表单，可以实现更灵活的流程管理，提高流程的可扩展性和可维护性。
  3. 外置表单：外置表单只需要定义表单的 key，key 对应的内容有开发者维护。

上面代码执行后，在 ACT_RU_VARIABLE 表的数据如下：

看起来表单也是在 ACT_RU_VARIABLE 表中保存数据，好像和变量差不多，那表单和变量的区别又是什么呢？我为什么不直接用变量呢？

  • 表单
    • 用途：表单是用于收集用户输入的界面。表单可以与流程任务相关联，以便在流程执行期间收集必要的数据。
    • 相当于提交定义好的字段，打包成一个整体，这个整体内的属性是相关关联的。（可以理解为 Java 中的 一个 实体类，类中有很多对象）
  • 变量
    • 用途：流程变量是在流程执行期间使用的数据，用于控制流程的行为或为其他节点使用，例如决定流程的下一个步骤或条件。
    • 是零散的，变量之间可能不存在关系，也可能存在关系。

实际上，如果你用于收集用户的输入（如填写请假申请信息、报销信息等）这种情况，最好使用表单。

这就相当于你在 http 的接口中定义的 Body 参数（JSON 格式），最好使用一个定义好的实体对象来接受参数，而不是一个 Map。

# 1.3 编写相关接口

然后我们编写相关接口类：

@RestController
@RequestMapping("/htmlForm")
public class HtmlFormController {

    private final static Logger log = LoggerFactory.getLogger(HtmlFormController.class);

    @Autowired
    private RepositoryService repositoryService;

    @Autowired
    private TaskService taskService;

    @Autowired
    private FormService formService;

    @GetMapping("/submitHolidayRequest.html")
    public ResponseEntity<String> submitHolidayRequest() {
        ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
                .processDefinitionKey("html_form_test")
                .latestVersion()
                .singleResult();
        String startFormKey = formService.getStartFormKey(processDefinition.getId());
        String renderedStartForm = (String) formService.getRenderedStartForm(processDefinition.getId());

        log.info("startFormKey: {}\nrenderedStartForm: \n{}", startFormKey, renderedStartForm);
        return ResponseEntity.ok(renderedStartForm);
    }


    @PostMapping("/submitHolidayRequest")
    public ResponseEntity<Object> submitHolidayRequest(String name, String reason, String startTime, String endTime) {
        log.info("name: {}, reason: {}, startTime: {}, endTime: {}", name, reason, startTime, endTime);
        ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
                .processDefinitionKey("html_form_test")
                .latestVersion()
                .singleResult();
        Assert.notNull(processDefinition, "processDefinition is null");

        Map<String, String> form = new HashMap<>();
        form.put("name", name);
        form.put("reason", reason);
        form.put("startTime", startTime);
        form.put("endTime", endTime);
        ProcessInstance processInstance = formService.submitStartFormData(processDefinition.getId(), form);

        return ResponseEntity.ok("success");
    }

    @GetMapping("/checkHolidayRequest.html")
    public ResponseEntity<String> checkHolidayRequest() {
        ProcessDefinition processDefinition = repositoryService.createProcessDefinitionQuery()
                .processDefinitionKey("html_form_test")
                .latestVersion()
                .singleResult();

        Task task = taskService.createTaskQuery()
                .processDefinitionId(processDefinition.getId())
                .taskAssignee("zhangsan")
                .singleResult();
        Assert.notNull(task, "task is null");

        // String taskFormKey = formService.getTaskFormKey(processDefinition.getId(), task.getTaskDefinitionKey());
        String taskFormKey = task.getFormKey();
        String renderedTaskForm = (String) formService.getRenderedTaskForm(task.getId());
        log.info("startFormKey: {}\nrenderedStartForm: \n{}", taskFormKey, renderedTaskForm);
        return ResponseEntity.ok(renderedTaskForm);
    }

    @PostMapping("/checkHolidayRequest")
    public ResponseEntity<String> checkHolidayRequest(String checkResult) {
        log.info("checkResult: {}", checkResult);
        Task task = taskService.createTaskQuery()
                .processDefinitionKey("html_form_test")
                .taskAssignee("zhangsan")
                .singleResult();
        Assert.notNull(task, "task is null");

        Map<String, String> form = new HashMap<>();
        form.put("checkResult", checkResult);
        formService.submitTaskFormData(task.getId(), form);
        return ResponseEntity.ok("success");
    }

}

# 2.使用 HTML 表单

# 2.1 部署

我们把流程定义文件放到代码中的 resources/processes 目录下，把 HTML 表单文件放到 resources/forms 目录下，然后执行如下代码部署上面的流程定义文件和 HTML 表单：

@ParameterizedTest
@CsvSource({
        "html_form_test, html表单测试, html表单测试.bpmn20.xml, 'submitHolidayRequest.html,checkHolidayRequest.html'"
})
@DisplayName("部署流程和HTML表单")
void deployProcessAndHTMLForm(String key, String name, String bpmn, String forms) throws FileNotFoundException {
    DeploymentBuilder deploymentBuilder = repositoryService.createDeployment()
            .key(key)
            .name(name)
            .addClasspathResource("processes/" + bpmn);
    String[] formArray = forms.split(",");
    for (String form : formArray) {
        File file = ResourceUtils.getFile("classpath:forms/" + form);
        // 注意：这里不能使用<code>addClasspathResource("forms/submitHolidayRequest.html")</code>的方式添加文件
        // 因为使用addClasspathResource后，文件的key会为'forms/submitHolidayRequest.html',
        // 但是我们流程文件中配置的flowable:formKey="showHolidayRequest.html"，所以会导致找不到表单
        // 或者也可以使用addClasspathResource添加文件，并把bpmn文件改为flowable:formKey="forms/showHolidayRequest.html"
        deploymentBuilder.addInputStream(form, new FileInputStream(file));
    }

    Deployment deploy = deploymentBuilder.deploy();
    log.info("部署成功 deploy={}", deploy);
}

Flowable 在 Spring Boot 中使用时，外置表单默认放在 resources/forms 目录下。并且会自动部署该目录下后缀为 .form 的外置表单文件。

# 若为true，则在启动时，将检查文件目录下是否有表单定义文件，并将表单定义文件部署到数据库中
flowable.form.deploy-resources=false
# 表单文件的位置，默认为classpath*:/forms/
flowable.form.resource-location=classpath*:/forms/
# 表单文件的后缀名，默认为**.form
flowable.form.resource-suffixes=**.form
