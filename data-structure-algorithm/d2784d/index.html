# 复杂度分析

# 复杂度分析的意义

意义：为了分析代码的执行效率和资源消耗

由于代码的执行时间和占用cpu资源率受测试环境和测试数据的影响很大，通过运行代码测试代码的执行效率和资源消耗的结果有很大的局限性，因此需要使用复杂度分析。

# 复杂度分析的表示方法

对与代码执行时，假设每行代码执行的时间是相同的（定义为unit_time）,代码从开始运行到完成，代码总的执行次数 x 每行代码执行的时间 = 代码的执行总时间 。

但是收到数据量的影响，数据量为1或+∞的最终执行时间会有非常大的差别，当数据量过小的时候没有意义，因此只考虑数据量无穷大的情况。

大O复杂度表示法：大O复杂度表示法并不会表示代码执行的具体时间，而是表示代码执行时间随着数据规模增大的变化趋势，也称为渐进时间复杂度（asymmetric time complexity），简称时间复杂度。

int cal(int n) {
    int sum = 0;
    int i = 1;
    for(; i <= n; i++) {
        sum = sum + 1;
    }
    return sum;
}

# 时间复杂度分析方法

  • 加法法则：代码的时间复杂度等于量级最大的那段代码的复杂度
    • 大O时间复杂度表示方法，只表示一种变化趋势，我们通常会忽略公式中的低阶、常量、系数，只考虑循执行次数最多，并受到数据量影响的的那段代码
  • 乘法法则:： 嵌套代码的复杂度等于嵌套内外的代码复杂度的乘积

# 常见的时间复杂度量级

  • O(1) 常量阶
    • 代码的执行时间不随着数据量n变化，则代码就是常量级别时间复杂度，统一记为O(1)
  • O(n)、O(m+n) 线性阶
  • O(logn) 对数阶
  • O(nlogn) 线性对数阶
  • O(n2) 平方阶; O(n3) 立方阶; O(nk) k次方阶
  • O(2n) 指数阶
  • O(n!) 阶乘阶

O(m+n)

# 空间复杂度分析方法

空间复杂度：空间复杂度表示算法的存储空间与数据规模之间的增长关系，全称为渐进空间复杂度（asymmetric space complexity）

示例：

public void reverse(int a[], int n) {
    int tmp[] = new int[n];
    for (int i = 0; i < n; ++i) {
        tmp[i] = a[n-i-1];
    }
       for (int i = 0; i < n; ++i) {
        a[i] = tmp[i];
    }
}

常见的空间复杂度：O(1)、O(n)、O(n2)、O(logn)、O(nlogn)，其中*O(logn)、O(nlogn)*这样的对数阶复杂度常见于递归中

# 最好、最坏时间复杂度

示例代码

int find (int[] array, int n, int x) {
    int i = 0;
    int pos = -1;
    for (; i < n; ++i) {
        if (array[i] == x) {
            pos = i;
            break;
        }
    }
    return 0;
}

由于等于变量x的值不确定在第几次会结束循环，其时间复杂度也不确定。

最好时间复杂度：代码执行最好的情况，等于变量x的值在索引0位置，即循环一次就能找到对应的值的下标，则此时时间复杂度为O(1)

最坏时间复杂度：代码循环了n次仍未找到对应的变量，或者代码循环了n次才找到对应的变量，此时时间复杂度为O(n)

# 平均时间复杂度

平均时间复杂度：代码被执行无数次、对应的时间复杂度的平均值

继续上述代码

分析方法一：

代码执行的所有情况的执行次数相加所有的情况数

对于上述代码则为

(1+2+3+...+n+n)(n+1)=n(n+3)2(n+1)

省略系数、低阶、常量，最终得到的时间复杂度为 O(n)​

分析方法二：

每次的循环次数与当次的循环概率乘积之和：

1×12n+2×12n+3×12n+…+n×12n+n×12

共有n+1中情况：

  • x不存在数组array中，概率为12
  • 存在数组array中，概率为12
    • 存在数组array中又有n种情况：x存在数组array中的第i个位置
    • n种情况的概率均相等，则每种情况的概率为12×n

以上代码分为两种情况，count == array.length 和 count < array.length，两种情况对应的时间复杂度分别为O(1)、O(n)​

因为insert()函数在大多数情况下时间复杂度为O(1)，仅在个别情况下时间复杂度才会为O(n)

因此可使用摊还分析法，我们将耗时多的操作的耗费时间耗时少的操作中，最终的均摊时间复杂度则为 O(1)​​

对一个数据结构进行一组连续操作、大多数情况下时间复杂度很低，只有个别情况时间复杂度比较高，而且这些操作前后存在连贯的时序关系，此时就可以将这一组操作放在一起分析，观察能否使用摊还分析法。

一般均摊时间复杂度等于最好时间复杂度。
