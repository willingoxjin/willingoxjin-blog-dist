03.Java 中使用 MQTT

EMQX—MQTT 客户端编程 (opens new window)

# 一、Java 中使用 MQTT

我们可以使用 Eclipse Paho Java Client (opens new window) 作为客户端，该客户端是 Java 语言中使用最为广泛的 MQTT 客户端库。

# 1.相关 API

  • MqttClient: 同步调用客户端，使用阻塞方法通信。
  • MqttClientPersistence: 代表一个持久的数据存储，用于在传输过程中存储出站和入站的信息，使其能够传递到指定的 QoS。
  • MqttConnectOptions: 连接选项，用于指定连接的参数，下面列举一些常见的方法。
    • setUserName: 设置用户名
    • setPassword: 设置密码
    • setCleanSession: 设置是否清除会话
    • setKeepAliveInterval: 设置心跳间隔
    • setConnectionTimeout: 设置连接超时时间
    • setAutomaticReconnect: 设置是否自动重连

使用 Java SDK 连接的官方文档 (opens new window)

订阅端代码

public class SubscribeSample {

    public static void main(String[] args) {
        String broker = "tcp://localhost:1883";
        String topic = "mqtt/test";
        String username = "willingoxjin";
        String password = "public";
        String clientid = "subscribe_client";
        int qos = 0;

        try {
            MqttClient client = new MqttClient(broker, clientid, new MemoryPersistence());
            // 连接参数
            MqttConnectOptions options = new MqttConnectOptions();
            options.setUserName(username);
            options.setPassword(password.toCharArray());
            options.setConnectionTimeout(60);
            options.setKeepAliveInterval(60);
            // 设置回调
            client.setCallback(new MqttCallback() {

                public void connectionLost(Throwable cause) {
                    System.out.println("connectionLost: " + cause.getMessage());
                }

                public void messageArrived(String topic, MqttMessage message) {
                    System.out.println("topic: " + topic);
                    System.out.println("Qos: " + message.getQos());
                    System.out.println("message content: " + new String(message.getPayload()));

                }

                public void deliveryComplete(IMqttDeliveryToken token) {
                    System.out.println("deliveryComplete---------" + token.isComplete());
                }

            });
            // 建立连接
            client.connect(options);
            // 订阅消息
            client.subscribe(topic, qos);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

发布消息端代码：

public class PublishSample {

    public static void main(String[] args) {
        String broker = "tcp://localhost:1883";
        String topic = "mqtt/test";
        String username = "willingoxjin";
        String password = "public";
        String clientid = "publish_client";
        String content = "Hello MQTT";
        int qos = 0;

        try {
            MqttClient client = new MqttClient(broker, clientid, new MemoryPersistence());
            // 连接参数
            MqttConnectOptions options = new MqttConnectOptions();
            // 设置用户名和密码
            options.setUserName(username);
            options.setPassword(password.toCharArray());
            options.setConnectionTimeout(60);
            options.setKeepAliveInterval(60);
            // 连接
            client.connect(options);
            // 创建消息并设置 QoS
            MqttMessage message = new MqttMessage(content.getBytes());
            message.setQos(qos);
            // 发布消息
            client.publish(topic, message);
            System.out.println("Message published");
            System.out.println("topic: " + topic);
            System.out.println("message content: " + content);
            System.out.println();
            // 关闭连接
            client.disconnect();
            // 关闭客户端
            client.close();
        } catch (MqttException e) {
            throw new RuntimeException(e);
        }
    }
}

@Slf4j
public class DefaultMqttCallback implements MqttCallback {

    @Override
    public void connectionLost(Throwable throwable) {
        log.info("mqtt与服务器断开连接");
    }

    @Override
    public void messageArrived(String topic, MqttMessage mqttMessage) throws Exception {
        log.info("接收消息主题:" + topic);
        log.info("接收消息Qos:" + mqttMessage.getQos());
        log.info("接收消息内容:" + new String(mqttMessage.getPayload()));
        log.info("接收消息内容:" + mqttMessage);
    }

    @Override
    public void deliveryComplete(IMqttDeliveryToken iMqttDeliveryToken) {
        IMqttAsyncClient client = iMqttDeliveryToken.getClient();
        log.info(client.getClientId() + "发布消息成功！");
    }
}

  • MqttPublisher：封装消息发布方法
public class MqttPublisher {

    private final MqttClient mqttClient;

    public MqttPublisher(MqttClient mqttClient) {
        this.mqttClient = mqttClient;
    }

    public void publish(int qos, boolean retained, String topic, String message) {
        MqttMessage mqttMessage = new MqttMessage();
        mqttMessage.setQos(qos);
        mqttMessage.setRetained(retained);
        mqttMessage.setPayload(message.getBytes());
        // 主题目的地，用于发布/订阅消息
        MqttTopic mqttTopic = mqttClient.getTopic(topic);
        // 提供一种机制来跟踪消息的传递进度。
        // 用于在以非阻塞方式（在后台运行）执行发布时跟踪消息的传递进度
        MqttDeliveryToken token;
        try {
            // 将指定消息发布到主题，但不等待消息传递完成。返回的token可用于跟踪消息的传递状态。
            // 一旦此方法干净地返回，消息就已被客户端接受发布。当连接可用时，将在后台完成消息传递。
            token = mqttTopic.publish(mqttMessage);
            token.waitForCompletion();
        } catch (MqttException e) {
            e.printStackTrace();
        }
    }

}

  • MqttConfiguration：配置 MqttClient、MqttPublisher 实例。
@Slf4j
@Configuration
@EnableConfigurationProperties(MqttProperties.class)
public class MqttConfiguration {

    @Autowired
    private MqttProperties mqttProperties;

    @Bean
    @ConditionalOnMissingBean(MqttClient.class)
    public MqttClient mqttClient() throws MqttException {
        return this.getConnect(new DefaultMqttCallback());
    }

    @Bean
    @ConditionalOnProperty(prefix = "spring.mqtt.publisher", name = "enabled", havingValue = "true")
    @ConditionalOnMissingBean(MqttPublisher.class)
    public MqttPublisher mqttPublisher(MqttClient mqttClient) {
        return new MqttPublisher(mqttClient);
    }

    private MqttClient getConnect(MqttCallback callback) throws MqttException {
        // 创建MQTT客户端对象
        String clientId = mqttProperties.getClientId();
        MqttClient client = new MqttClient(mqttProperties.getEndpoint(), clientId, new MemoryPersistence());
        // 连接设置
        MqttConnectOptions options = new MqttConnectOptions();
        // 是否清空session，设置为false表示服务器会保留客户端的连接记录（订阅主题，qos），客户端重连之后能获取到服务器在客户端断开连接期间推送的消息
        // 设置为true表示每次连接到服务端都是以新的身份
        options.setCleanSession(true);
        // 设置连接用户名
        options.setUserName(mqttProperties.getUsername());
        // 设置连接密码
        options.setPassword(mqttProperties.getPassword().toCharArray());
        // 设置超时时间，单位为秒
        options.setConnectionTimeout(100);
        // 设置心跳时间 单位为秒，表示服务器每隔1.5*20秒的时间向客户端发送心跳判断客户端是否在线
        options.setKeepAliveInterval(20);
        // 设置遗嘱消息的话题，若客户端和服务器之间的连接意外断开，服务器将发布客户端的遗嘱信息
        options.setWill("willTopic", (clientId + "与服务器断开连接").getBytes(), 0, false);
        // 设置回调
        client.setCallback(callback);
        client.connect(options);
        log.info("Mqtt connect [{}] clientId = {}", mqttProperties.getEndpoint(), clientId);
        return client;
    }

}

# 三、使用 spring-integration-mqtt

# 1.spring-integration 介绍

spring-integration 是 Spring 系列的一个框架，专门用于构建企业集成解决方案。其基于 Spring 的核心理念和依赖注入机制，提供了一套声明式的编程模型来实现系统之间的异步消息传递、数据转换、路由、适配器连接等集成需求。

官方文档：Spring Integration MQTT Support (opens new window)

spring-integration 就像工厂的生产线，而数据就是原材料，而相关的处理器就是用于加工处理器的流程。

相关的术语如下：

  • 通道（Channel）：可以看作是数据流动的路径或管道
    • InChannel：是数据进入系统的通道，用于接收输入的数据。
    • OutChannel：数据流出系统的通道，经过处理后的数据通过它传递到后续的环节或目的地。
  • 处理器（Handler）：它是执行具体数据处理操作的组件。处理器定义了对消息进行何种处理动作，比如转换数据格式、进行计算、执行验证等，类似于工厂生产线上的特定加工工序。
  • 消息：是在系统中传输和处理的数据单元。它包含了要处理的数据以及相关的元数据，如消息头信息等。消息在通道中流动，由各个处理器进行处理。
  • 网关**（Gateway）**：系统与外部进行交互的接口。

相关 API 说明：

  • MqttPahoClientFactory：用于创建和管理 MQTT 客户端（如 Paho 客户端）的工厂类。它负责配置和初始化与 MQTT 代理的连接等相关设置。
  • MessageChannel：消息通道，用于在不同组件之间传递消息。它定义了消息流动的路径和方向。
  • MessageHandler：消息处理器，负责对通过特定通道传递过来的消息进行具体的处理操作。
  • MessageProducer：消息生产者，用于产生并发送消息到指定的通道或目的地。
  • @ServiceActivator：用于将一个方法标记为服务激活器。当有消息到达相关联的通道时，会触发该方法执行来处理消息。
  • @MessagingGateway：用于创建消息网关接口。通过这个注解标记的接口方法，可以方便地发送和接收消息，起到了简化消息交互的作用。

@Configuration
public class MqttIntegrationInputConfig {

    @Resource
    private MqttProperties mqttProperties;

    @Bean
    public MessageChannel mqttInputChannel() {
        return new DirectChannel();
    }

    @Bean
    public MessageProducer inbound(MqttPahoClientFactory mqttClientFactory) {
        MqttPahoMessageDrivenChannelAdapter adapter = new MqttPahoMessageDrivenChannelAdapter(
                        mqttProperties.getClientId() + "_sub",
                        mqttClientFactory,
                        "/test/topic",
                        "/demo/topic");
        adapter.setCompletionTimeout(5000);
        adapter.setConverter(new DefaultPahoMessageConverter());
        adapter.setQos(1);
        adapter.setOutputChannel(mqttInputChannel());
        return adapter;
    }

    @Bean
    @ServiceActivator(inputChannel = "mqttInputChannel")
    public MessageHandler handler() {
        return new MessageHandler() {
            @Override
            public void handleMessage(Message<?> message) throws MessagingException {
                System.out.println("mqttInputChannel - " + message.getPayload());
            }
        };
    }

}
