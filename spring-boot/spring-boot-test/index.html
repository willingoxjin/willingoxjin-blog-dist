Spring Boot 单元测试

# 单元测试

单元测试能让我们在项目开发中提高代码质量，单元测试在开发中是非常重要的，很多程序员很少写测试，或者只是简单的测试结果，这是非常不好的习惯。我们一起来学习一下 Spring Boot 提供的单元测试。

# JUit5

Java 开发人员对 JUit5一定不陌生，关于介绍或详细的文档，请移步官方：

JUit5 官方文档：https://junit.org/junit5/docs/current/user-guide/#writing-tests

JUit5 官方示例：对于 Maven，请查看junit5-jupiter-starter-maven (opens new window)项目。

这里，我们只介绍常用的一些注解、方法，以及示例：

# 常用的注解

测试方法 说明
@Test 表示方法是测试方法。与 JUnit 4 的@Test注解不同，此注解不声明任何属性，因为 JUnit Jupiter 中的测试扩展基于它们自己的专用注解进行操作。除非它们被覆盖，否则这些方法会被继承。
@ParameterizedTest 参数化测试 (opens new window)：该注解可以设置测试方法的参数
@RepeatedTest 重复测试 (opens new window)：
@TestFactory 动态测试 (opens new window)
@TestTemplate 测试用例的模板 (opens new window)：
@Timeout 超时测试：若测试、测试工厂、测试模板或生命周期方法执行超过给定持续时间，则方法测试失败
生命周期方法 说明
@BeforeAll 注解的方法应该在当前类的所有方法之前执行
@AfterAll 注解的方法应该在当前类中的所有方法之后执行
@BeforeEach 表示被注解的方法应该在当前类中的每个方法之前执行，即每个方法执行之前都会执行该方法
@AfterEach 表示被注解的方法应该在当前类中的每个方法之后执行，即每个方法执行之后都会执行该方法
方法 说明
@Disabled 进行测试类或测试方法
@DisplayName 为测试类、测试方法命名
@Order 可以设置测试方法的执行顺序，数字越小越先执行

# 断言

Jupiter 断言都是类中的static方法org.junit.jupiter.api.Assertions

常用的断言：

  • assertEquals 比较预期值和实际值是否相等

  • assertNotEquals 比较预期值和实际值是否不相等

  • assertTrue 判断预期值是否为true

  • assertFalse 判断预期值是否为false

  • assertNotNull 预期值是否为空

  • assertNotNull 预期值不为空

  • assertTimeout 执行函数是否超时

  • assertAll 使用分组断言

  • assertThrows 是否抛出了指定异常

  • assertNotThrows 是否未抛出了指定异常

  • ......

使用断言的相关Demo

import static java.time.Duration.ofMillis;
import static java.time.Duration.ofMinutes;
import static org.junit.jupiter.api.Assertions.assertAll;
import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.junit.jupiter.api.Assertions.assertNotNull;
import static org.junit.jupiter.api.Assertions.assertThrows;
import static org.junit.jupiter.api.Assertions.assertTimeout;
import static org.junit.jupiter.api.Assertions.assertTimeoutPreemptively;
import static org.junit.jupiter.api.Assertions.assertTrue;

import java.util.concurrent.CountDownLatch;

import example.domain.Person;
import example.util.Calculator;

import org.junit.jupiter.api.Test;

class AssertionsDemo {

    // 标准断言
    // 在标准断言中，若断言失败会抛出异常，不再继续执行该测试方法
    @Test
    void standardAssertions() {
        // var1-预期值; var2-实际值
        assertEquals(2, 1+1);
        // var1-预期值; var2-实际值; var3-异常时的消息
        assertEquals(4, 2 * 2,
                "The optional failure message is now the last parameter");
        // var1-预期值; var2-实际值; var3-异常时的消息，接收函数式接口 Supplier<String>
        assertTrue('a' < 'b', () -> "Assertion messages can be lazily evaluated -- "
                + "to avoid constructing complex messages unnecessarily.");
    }

    // 分组断言
    // 在分组的断言中，所有的断言都被执行，所有的失败将一起报告。
    @Test
    void groupedAssertions() {
        // var1-分组标题; var2...n-测试的断言逻辑
        assertAll("person",
                () -> assertEquals("Jane", "a"),
                () -> assertEquals("Doe", "b")
        );
    }

    // 分组中使用代码块断言
    @Test
    void dependentAssertions() {
        // 在一个代码块中，如果断言失败，则该代码块中的后续代码将被跳过。
        assertAll("properties",
                () -> {
                    String firstName = "Joe";
                    assertNotNull(firstName);

                    // 仅在前面的断言有效时执行。
                    assertAll("first name",
                            () -> assertTrue(firstName.startsWith("J")),
                            () -> assertTrue(firstName.endsWith("e"))
                    );
                },
                () -> {
                    // 分组断言，因此处理时独立于名称断言的结果。
                    String lastName = "Dave";
                    assertNotNull(lastName);

                    // 仅在前面的断言有效时执行。
                    assertAll("last name",
                            () -> assertTrue(lastName.startsWith("D")),
                            () -> assertTrue(lastName.endsWith("e"))
                    );
                }
        );
    }

    // 异常断言
    @Test
    void exceptionTesting() {
        Exception exception = assertThrows(ArithmeticException.class, () -> {
            int i = 1 / 0;  // 模拟算数异常
        });
        assertEquals("/ by zero", exception.getMessage());
    }

    // 超时断言-无返回值
    @Test
    void timeoutNotExceeded() {
        // 下面的断言成功。
        assertTimeout(ofMinutes(1), () -> {
            // 执行耗时小于1分钟的任务
            Thread.sleep(1000);
        });
    }

    // 超时断言-有返回值
    @Test
    void timeoutNotExceededWithResult() {
        // 下面的断言成功，并返回提供的对象。
        String actualResult = assertTimeout(ofMinutes(1), () -> {
            Thread.sleep(1000);
            return "a result";
        });
        assertEquals("a result", actualResult);
    }

    // 注意:可执行文件将在与调用代码不同的线程中执行。
    // 此外，如果超过超时，可执行文件的执行将被优先中止。
    // 有关可能的不良副作用的讨论，请参阅类级Javadoc的抢占超时部分。
    @Test
    void timeoutExceededWithPreemptiveTermination() {
        // 以下断言失败，并给出类似的错误消息: 10毫秒后执行超时
        assertTimeoutPreemptively(ofMillis(10), () -> {
            // 模拟耗时超过10毫秒的任务。
            new CountDownLatch(1).await();
        });
    }

}

# 假设

假设的使用跟断言相似，区别点在于：

  • 假设的结果与预期不符的话不会中断测试方法继续执行；

  • 而断言则会中断测试方法

使用假设的相关Demo

@Test
void testOnlyOnCiServer() {
    assumeTrue("CI".equals(System.getenv("ENV")));
    // remainder of test
    System.out.println("hello");
}

@Test
void testOnlyOnDeveloperWorkstation() {
    assumeTrue("DEV".equals(System.getenv("ENV")),
               () -> "Aborting test: not on developer workstation");
    // remainder of test
    System.out.println("hello");
}

@Test
void testInAllEnvironments() {
    // var1为true时，才会执行var2
    assumingThat("CI".equals(System.getenv("ENV")),
                 () -> {
                     assertEquals(2, 4 / 2);
                 });

    // 在所有环境中执行这些断言
    assertEquals(42, 6 * 7);
}

# 生命周期

生命周期相关的测试，会在测试方法执行前后自动执行

  • @BeforeAll 该注解声明的方法，会在全部测试方法执行之前执行
    • 该注解声明的方法必须是静态无返回值方法
  • @AfterAll 该注解声明的方法，会在全部测试方法执行之后执行
    • 该注解声明的方法必须是静态无返回值方法
  • @BeforeEach 该注解声明的方法，会在每个的测试方法执行之前执行
  • @AfterEach 该注解声明的方法，会在每个的测试方法执行之后执行

以下是Demo：

import org.junit.jupiter.api.*;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class LifeCycleTest {

    @BeforeAll
    static void beforeAll() {
        System.out.println("beforeAll：我会在所有方法执行之前执行...");
    }

    @BeforeEach
    void beforeEach(TestInfo testInfo) {
        System.out.println("beforeEach：我会在每个方法调用之前执行..." + testInfo.getTestMethod());
    }

    @ParameterizedTest
    @ValueSource(strings = "apple")
    void testWithRegularParameterResolver(String argument, TestReporter testReporter) {
        // 向测试报告添加 argument
        testReporter.publishEntry("argument", argument);
    }

    @ParameterizedTest
    @ValueSource(strings = "banana")
    void testWithRegularParameterResolver2(String argument, TestReporter testReporter) {
        // 向测试报告添加 argument
        testReporter.publishEntry("argument", argument);
    }

    @AfterEach
    void afterEach(TestInfo testInfo) {
        System.out.println("beforeEach：我会在每个方法调用之后执行..." + testInfo.getTestMethod());
    }

    @AfterAll
    static void afterAll() {
        System.out.println("afterAll：我会在所有方法执行之后执行...");
    }

}

# 重复测试

重复测试会将某个方法执行多次

使用注解 @RepeatedTest 声明测试方法，即代表该测试方法为重复测试

以下是重复测试的Demo：

	// 重复测试
    @DisplayName("重复测试")
    @RepeatedTest(2)
    void repeatedTest() {
        System.out.println(1);
    }

    @DisplayName("重复测试，获取当前重复索引")
    @RepeatedTest(3)
    void repeatedTestWithRepetitionInfo(RepetitionInfo repetitionInfo) {
        assertEquals(3, repetitionInfo.getTotalRepetitions());
    }

    // 手动定义 displayName，使用 LongPattern
    @DisplayName("重复测试，使用expected模板")
    @RepeatedTest(value = 1, name = RepeatedTest.LONG_DISPLAY_NAME)
    void customDisplayNameWithLongPattern(TestInfo testInfo) {
        assertEquals("重复测试，使用RepeatedTest模板 :: repetition 1 of 1", testInfo.getDisplayName());
    }

    // 手动定义 displayName
    @DisplayName("重复测试，自定义expected")
    @RepeatedTest(value = 1, name = "{displayName} - {currentRepetition}/{totalRepetitions}")
    void customDisplayName(TestInfo testInfo) {
        assertEquals("重复测试，自定义expected - 1/1", testInfo.getDisplayName());
    }

    // 重复测试
    @RepeatedTest(value = 5, name = "Wiederholung {currentRepetition} von {totalRepetitions}")
    void repeatedTestInGerman() {
        // ...
    }

# 参数测试

参数测试需要添加 @ParameterizedTest 注解，即代表所声明的方法属于参数测试

# @ValueSource 单个参数

设置单个参数，可以使用 @ValueSource ，该注解支持多种类型参数传递。

若需要传递字符串，则用 strings ，传递 Integer/int 类型，则设置 ints 属性，其他类型都有与之对应的类型的属性，如booleans、doubles、classes等等

以下是Demo

// 设置单个参数
@ParameterizedTest
@ValueSource(strings = { "谢广坤", "谢腾飞" ,"谢大脚"})
void testValueSource(String name) {
    JUit5Demo.println(name);
}

# 参数为空值

若想要在 参数中添加空值，来测试方法，则使用以下注解

  • @NullSource 添加空对象到参数中
  • @EmptySource 添加空字符到对象中
  • @NullAndEmptySource 添加空对象和空字符到参数中。该注解可代替 @NullSource 和 @EmptySource ，且不能与 @NullSource 和 @EmptySource 同时使用

以下是Demo

// 设置空值、空字符参数
@ParameterizedTest
@ValueSource(strings = { "谢广坤"})
@NullSource
@EmptySource
// @NullAndEmptySource
void testNullAndEmpty(String name) {
    JUit5Demo.println(name);
}

# @EnumSource 枚举参数

@EnumSource 可以根据枚举类，来设置参数，但是要注意参数必须为枚举类型

以下是Demo

// 使用枚举参数

// 定义枚举
@Getter
@AllArgsConstructor
enum UserEnums {
    GUANG_KUN("谢广坤", 65),
    TENG_FEI("谢腾飞", 10),
    DA_JIAO("谢大脚", 18)
        ;
    private final String name;
    private final Integer age;
}

// 指定测试枚举参数的类型
@ParameterizedTest
@EnumSource(UserEnums.class)
void testEnumSource(UserEnums user) {
    System.out.println(user.getName() + ":" + user.getAge());
}

// 无须指定类型，会自动将 UserEnums.class 注入到 user 参数中
@ParameterizedTest
@EnumSource
void testEnumSource2(UserEnums user) {
    System.out.println(user.getName() + ":" + user.getAge());
}

// 包含 枚举中的某些参数
@ParameterizedTest
@EnumSource(mode = INCLUDE, names = { "ERAS", "FOREVER" })
void testWithEnumSourceInclude(ChronoUnit unit) {
    assertTrue(EnumSet.of(ChronoUnit.ERAS, ChronoUnit.FOREVER).contains(unit));
}

// 排除 枚举中的某些参数
@ParameterizedTest
@EnumSource(mode = EXCLUDE, names = { "ERAS", "FOREVER" })
void testWithEnumSourceExclude(ChronoUnit unit) {
    assertFalse(EnumSet.of(ChronoUnit.ERAS, ChronoUnit.FOREVER).contains(unit));
}

// 正则匹配
@ParameterizedTest
@EnumSource(mode = MATCH_ALL, names = "^.*DAYS$")
void testWithEnumSourceMatchAll(ChronoUnit unit) {
    // 结尾为 DAYS
    assertTrue(unit.name().endsWith("DAYS"));;
}

# @MethodSource 方法构造参数

@MethodSource 的 value 属性配置所调用的方法，来获取测试的参数。

若 @MethodSource 不设置 value 属性，则value默认会等于当前声明的方法名

所调用的用来获取参数的方法必须满足：

  • 方法必须为static
  • 返回值类型为 Stream<T>

以下是Demo

/**
 * 构造字符型参数
 * 必须为静态方法
 * 返回类型支持 Stream、DoubleStream、IntStream和LongStream
 * @return 返回流式对象
 */
static Stream<String> stringProvider() {
    return Stream.of("apple", "banana");
}

// 通过方法，定义测试参数
@ParameterizedTest
@MethodSource("stringProvider")
void testWithExplicitLocalMethodSource(String argument) {
    assertNotNull(argument);
}

// ------------------------------------------------------------

static Stream<String> testWithDefaultLocalMethodSource() {
    return Stream.of("apple", "banana");
}
// 按照约定搜索与当前方法同名的参数方法
@ParameterizedTest
@MethodSource   // value 属性默认等于当前方法：testWithDefaultLocalMethodSource
void testWithDefaultLocalMethodSource(String argument) {
    assertNotNull(argument);
}

// ------------------------------------------------------------

// 构造多个参数的情况
static Stream<Arguments> stringIntAndListProvider() {
    return Stream.of(
        arguments("apple", 1, Arrays.asList("a", "b")),
        arguments("lemon", 2, Arrays.asList("x", "y"))
    );
}
@ParameterizedTest
@MethodSource("stringIntAndListProvider")
void testWithMultiArgMethodSource(String str, int num, List<String> list) {
    assertEquals(5, str.length());
    assertTrue(num >=1 && num <=2);
    assertEquals(2, list.size());
}

// ------------------------------------------------------------

// 引用类的静态方法
static class StringsProviders {

# 动态测试

@TestFactory是动态测试的核心注解，它用于标志一个方法为测试用例的工厂方法，而不是普通的测试方法，@TestFactory方法本身不是测试用例，而是测试用例的工厂，它用来生产测试方法，并且会自动执行生产的所有测试方法。

动态测试的要求：

  • 这个方法不能被private和static修饰，并且它必须有返回值，这也是它区别于普通测试方法的一个重要标志。

  • 对于方法的返回值也是有要求的，它必须返回：Stream、Collection、Iterable、Iterator或是数组形式的DynamicNode类型，对于方法返回的Stream流（例如：Files.lines()，Files.list()等）会被自动关闭，因而你无需手动关闭。

注意：动态测试不会执行 @BeforeEach 和 @AfterEach 的回调方法

以下是Demo：

    // 返回单个 DynamicTest，并且DynamicTest 会被自动执行测试用例
    @TestFactory
    DynamicNode dynamicNodeSingleTest() {
        return dynamicTest("(10 / 2) > 5 ?", () -> assertFalse(10 / 2 > 5));
    }

    // 返回 DynamicTest 集合，并且DynamicTest 会被自动执行测试用例
    @TestFactory
    Collection<DynamicTest> dynamicTestsFromCollection() {
        return List.of(
                dynamicTest("1st dynamic test", () -> assertEquals(3 % 2, 1)),
                dynamicTest("2nd dynamic test", () -> assertEquals(4, 2 * 2))
        );
    }

    // 返回 Iterable<DynamicTest>，并且DynamicTest 会被自动执行测试用例
    @TestFactory
    Iterable<DynamicTest> dynamicTestsFromIterable() {
        return List.of(
                dynamicTest("1st dynamic test", () -> assertNotEquals(4 % 2, 1)),
                dynamicTest("2nd dynamic test", () -> assertEquals(8, 2 * 4))
        );
    }

    // 返回 Iterator<DynamicTest>，并且DynamicTest 会被自动执行测试用例
    @TestFactory
    Iterator<DynamicTest> dynamicTestsFromIterator() {
        return List.of(
                dynamicTest("1st dynamic test", () -> assertNotEquals(4 % 2, 1)),
                dynamicTest("2nd dynamic test", () -> assertEquals(8, 2 * 4))
        ).iterator();
    }

    // 返回 DynamicTest 类型的数组，并且DynamicTest 会被自动执行测试用例
    @TestFactory
    DynamicTest[] dynamicTestsFromArray() {
        return new DynamicTest[] {
                dynamicTest("1st dynamic test", () -> assertNotEquals(4 % 2, 1)),
                dynamicTest("2nd dynamic test", () -> assertEquals(8, 2 * 4))
        };
    }

    // 返回 DynamicTest 流，并且DynamicTest 会被自动执行测试用例
    // 该方法会随机生成参数，这才叫动态测试
    @TestFactory
    Stream<DynamicTest> dynamicTestsFromIntStream() {
        // 生成10个偶数测试用例
        return IntStream.iterate(0, n -> n + 2).limit(10)
                .mapToObj(n -> dynamicTest("test" + n, () -> assertEquals(0, n % 2)));
    }

    // 使用 DynamicTest.stream 构造测试用例1
    @TestFactory
    Stream<DynamicTest> generateRandomNumberOfTestsFromIterator() {
        // 1.参数
        // 生成0到100之间的随机正整数，直到遇到一个能被7整除的数
        Iterator<Integer> inputGenerator = new Iterator<>() {
            final Random random = new Random();
            int current;

            @Override
            public boolean hasNext() {
                current = random.nextInt(100);
                return current % 7 != 0;
            }

            @Override
            public Integer next() {
                return current;
            }
        };

        // 2.displayName 生成器。生成如下显示名称:input:5, input:37, input:85，等等。
        Function<Integer, String> displayNameGenerator = (input) -> "input:" + input;

        // 3.执行测试的逻辑，即根据当前输入值执行测试。
        ThrowingConsumer<Integer> testExecutor = (input) -> assertTrue(input % 7 != 0);

        // 返回动态测试流
        return DynamicTest.stream(inputGenerator, displayNameGenerator, testExecutor);
    }

    // 使用 DynamicTest.stream 构造测试用例2
    @TestFactory
    Stream<DynamicTest> dynamicTestsFromStreamFactoryMethod() {
        // 要检查的流
        Stream<Integer> inputStream = Stream.of(10, 20, 30, 40);

        // 生成显示名称-displayName
        Function<Integer, String> displayNameGenerator = text -> text + " is a palindrome";

        // 根据当前输入值执行测试
        ThrowingConsumer<Integer> testExecutor = text -> assertEquals(0, text % 10);

        // Returns a stream of dynamic tests.
        return DynamicTest.stream(inputStream, displayNameGenerator, testExecutor);
    }

    // 使用 DynamicTest.stream 构造测试用例3
    @TestFactory
    Stream<DynamicTest> dynamicTestsFromStreamFactoryMethodWithNames() {
        // 要检查的流即 displayName
        Stream<Named<Integer>> inputStream = Stream.of(
                named("one", 1),
                named("two", 2),
                named("three", 3),
                named("four", 4)
        );

        // Returns a stream of dynamic tests.
        return DynamicTest.stream(inputStream,
                text -> assertTrue(text > 0));
    }

    // 使用 dynamicContainer 测试，即使用嵌套/容器测试
    @TestFactory
    Stream<DynamicNode> dynamicTestsWithContainers() {
        return Stream.of("A", "B", "C")
                .map(input -> dynamicContainer("Container " + input, Stream.of(
                        dynamicTest("not null", () -> assertNotNull(input)),
                        dynamicContainer("properties", Stream.of(
                                dynamicTest("length > 0", () -> assertTrue(input.length() > 0)),
                                dynamicTest("not empty", () -> assertFalse(input.isEmpty()))
                        ))
                )));
    }

    // 单个容器
    @TestFactory
    DynamicNode dynamicNodeSingleContainer() {
        return dynamicContainer("palindromes",
               Stream.of("racecar", "radar", "mom", "dad")
               .map(text ->
                    dynamicTest("length > 0", () -> assertTrue(text.length() > 0)))
               );
    }

# @SpringBootTest

使用过Spring Boot 的同学应该对注解 @SpringBootTest 都不陌生，@SpringBootTest 注解声明在测试类上，代表该类为 Spring Boot 的测试类，就可以快速的搭建一个测试类，并引入Spring 的上下文，启动 Spring Boot 项目来测试。

该注解的 webEnvironment 属性可以

  • MOCK
    • 代表启用一个模拟的web服务器
    • webEnvironment 默认为 MOCK
  • RANDOM_PORT
    • 启动web服务器，且端口随机
    • 通常触发对随机端口的监听。通常与测试中的@LocalServerPort注入字段一起使用。
  • DEFINED_PORT
    • 启动web服务器，且端口使用默认端口
  • NONE
    • 不会启动web服务器

@RestController
public class RestfulTestController {

    private Map<Long, User> userMap = new HashMap<>();

    @PostMapping("user")
    public JsonResult<Void> addUser(@RequestBody User user) {
        if (user == null || user.getId() == null) {
            return JsonResult.fail();
        }
        userMap.put(user.getId(), user);
        return JsonResult.success();
    }

    @PutMapping("user")
    public JsonResult<Void> modifyUser(@RequestBody User user) {
        if (user == null || user.getId() == null) {
            return JsonResult.fail();
        }
        userMap.put(user.getId(), user);
        return JsonResult.success();
    }

    @GetMapping("user/{userId}")
    public User user(@PathVariable Long userId) {
        return userMap.get(userId);
    }

    @GetMapping("users")
    public List<User> users() {
        return new ArrayList<>(userMap.values());
    }

    @DeleteMapping("user/{userId}")
    public JsonResult<Void> deleteUser(@PathVariable Long userId) {
        var remove = userMap.remove(userId);
        if (remove==null) {
            return JsonResult.fail();
        }
        return JsonResult.success();
    }

}

@Slf4j
@SpringBootTest(
    webEnvironment = SpringBootTest.WebEnvironment.MOCK
)	// 启动应用使用 MOCK 
@AutoConfigureMockMvc	// 自动装配MockMvc
public class MockMvcTest {

    @Autowired
    private MockMvc mockMvc;

	// get请求    
    @Test
    void testIndex() throws Exception {
        // 发送请求
        RequestBuilder requestBuilder = MockMvcRequestBuilders.get(new URI("/"));
        ResultActions resultActions = mockMvc.perform(requestBuilder);
        // 返回结果
        ModelAndView modelAndView = resultActions.andReturn()
                .getModelAndView();
		// assert 
        Assertions.assertNotNull(modelAndView);
        Assertions.assertEquals(
                Map.of("message", "hello! this is my spring test demo index"),
                modelAndView.getModel());
        // viewName
        Assertions.assertEquals("index", modelAndView.getViewName());
    }

    // 为post请求
    @ParameterizedTest
    @CsvSource({
            "1, 谢广坤, 65",
            "2, 谢腾飞, 10"
    })
    void testAddUser(String id, String username, String age) throws Exception {
        // 发送请求
        RequestBuilder requestBuilder = MockMvcRequestBuilders
                .post(new URI("/addUser"))
                .param("id", id)
                .param("username", username)
                .param("age", age)
                ;
        ResultActions resultActions = mockMvc.perform(requestBuilder);
        // 获取模型视图
        ModelAndView modelAndView = resultActions.andReturn()
                .getModelAndView();
        // assert
        Assertions.assertNotNull(modelAndView);
        Assertions.assertEquals("redirect:/", modelAndView.getViewName());
        // FlashMap 对应 RedirectAttributes
        FlashMap flashMap = resultActions.andReturn()
                .getFlashMap();
        log.info("model={}", flashMap);
    }

}

# 测试业务组件

业务组件即 Dao 层（Mapper层） 和 Service 层的组件。

测试业务组件不需要启动 web 服务器，因此设置 @SpringBootTest 的 webEnvironment 属性为 NONE 即可。

demo：

@ParameterizedTest
@CsvSource({
    "1, 谢广坤, 65",
    "2, 谢腾飞, 10"
})
void testServiceAddUser(Long id, String username, Integer age) throws Exception {
    var user = User.builder()
        .id(id)
        .username(username)
        .age(age)
        .build();
    Integer result = userService.addUser(user);
    Assertions.assertNotEquals(0, result);
}

# 模拟数据

模拟数据：

  • @MockBean 会将 MockBean 注入到被测试的 userService 中
  • BDDMockito.given 设置 MockBean（测试的模型数据）

demo：

定义 UserDao 接口（无须实现）

public interface UserDao {

    List<User> findAll();

编写测试类：

@Slf4j
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.NONE)
public class MockTest {

    // @MockBean：会将 MockBean 注入到被测试的 userService 中
    @MockBean
    private UserDao userDao;

    @Autowired
    private UserService userService;

    @Test
    void testUserService() {
        // set Mock data
        BDDMockito.given(this.userDao.findAll())
                .willReturn(List.of(
                        User.builder().id(1L).username("谢广坤").age(65).build(),
                        User.builder().id(2L).username("谢腾飞").age(10).build(),
                        User.builder().id(3L).username("谢大脚").age(18).build()
                ));
        // test getAllUser
        List<User> userList = userService.getAllUser();
        log.info("userList={}", userList);
        Assertions.assertNotNull(userList);
    }
    
}
